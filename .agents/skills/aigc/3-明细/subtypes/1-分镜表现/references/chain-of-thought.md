# aigc 3-明细 / 1-分镜表现 / Chain Of Thought

本文件承载 `aigc 3-明细 / 1-分镜表现` 的字段主表、思维链与返工入口真源。

## 模块定位

- 本文件按最新 `think-think` 规范升级为 `运行模式 + 启发式工作链 + 三轴三重 + 可见快照 + Gate Summary` 的稳定合同。
- 主 `SKILL.md` 继续只保留父级边界、门禁、回链和 Mermaid 摘要；本文件负责把“如何把分镜表现 patch 回统一根文件”压成可执行判断链。
- 本模块默认只暴露可审计的决策快照，不要求外显完整内部推理全文。
- 本层真正负责的是 `分镜密度 -> 分镜构图 -> 分镜插入` 的父级编排，不是把镜头词写得更华丽。

## 设计立场与运行模式

### 设计立场

1. `隐藏推理层`：允许模型在内部比较 `panel_count`、锚点、候选插入位与行文顺序，但不把完整内部推理当成交付物。
2. `可见快照层`：对外只保留能支撑审计、回写和返工的决定性快照，例如子路径路由、锚点裁决、单行内联合同与 Gate 结论。
3. `字段落盘层`：所有关键判断都要回到 `FIELD-SBP-*`、`第N集.json`、sidecar 或 `validation-report.md`。
4. `Gate Summary`：最终必须能把 PASS/FAIL、失败码和返工入口压成可续跑结论。

### 运行模式判定

| 模式 | 触发条件 | 外显策略 | 禁止误用 |
| --- | --- | --- | --- |
| `推理优先模式` | 使用原生 reasoning / thinking 模型执行 `1-分镜表现` | 先用启发式工作链收窄 `panel_count -> 锚点 -> 单行插回`，再输出可见快照与字段落点 | 把分镜插入写成僵硬的逐字脚本，或把完整 CoT 外显成正文 |
| `支架回退模式` | 普通模型、thinking 关闭或上下文嘈杂 | 保留 `S1-S6` 明确支架，但每步只保留高价值判断 | 把流程描述写得比分镜任务本身更长 |
| `工具反思增强模式` | 依赖 `分镜密度`、`分镜构图` 或 `validation-report` 的中间结果 | 每次关键结果返回后先做二次判断，再继续插回主文件 | 工具或中间产物回来后机械进入下一步，不复核锚点与写位 |

硬规则：

1. 默认优先 `推理优先模式`，`S1-S6` 是可见支架，不是对内部推理的全文脚本化替代。
2. 若进入 `工具反思增强模式`，至少在 `S3` 和 `S5` 执行一次显式二次判断。
3. 任一模式都不得把完整 CoT 当成主产物。

## Think-Think Design Snapshot

### 启发式工作链

- `不可删性`：删掉这轮判断，下游会在哪一处重新猜 `panel_count` 是否可信、锚点落到哪一镜、`分镜表现` 应该长什么样，以及到底 patch 哪份主文件？
- `最先推动`：当前最该推进的是 `FIELD-SBP-INPUT-02` 与 `FIELD-SBP-ANCHOR-03`，还是 `FIELD-SBP-INLINE-04`；没有前置锚点时，禁止提前美化分镜行。
- `先砍什么`：先删掉无锚点的镜头句、偷写 `运镜/转场/光影/色彩` 的越权字段、以及只顾“好看”却解释不了为何插在这里的伪分镜。
- `真正比较尺`：本轮比较尺不是“哪句更像分镜术语”，而是“哪种插法最能解释这镜为什么存在、该挂到哪句之前、并且不改坏原文”。
- `立场纠偏`：当前判断必须站在主文件消费者和后续 sibling 层的立场，优先降低返工，而不是站在文档作者自嗨的立场堆镜头词。
- `工具后反思`：若 `分镜密度` 返回的新证据改变了 `panel_count` 或锚点，必须先重判 `FIELD-SBP-ANCHOR-03`；若 `分镜构图` 返回的字段越权或缺项，必须先重判 `FIELD-SBP-INLINE-04`。
- `落盘门`：这轮判断最终必须落到某个 `FIELD-SBP-*`、`第N集.json`、sidecar 或 `validation-report.md`；若落不到，就不应保留为显式合同。

### 三向三重裁决轴

| 裁决轴/层 | 本层先裁什么 | 绝不带进下一层什么 | 服务字段 | 正确产物形态 |
| --- | --- | --- | --- | --- |
| `粗裁决 / 方向轴` | 当前任务是否真的属于 `1-分镜表现`，以及是否必须沿 `分镜密度 -> 分镜构图 -> 分镜插入` 唯一路径推进 | 还没判父级边界就直接写分镜行；把两个子技能误当并行 | `FIELD-SBP-ROOT-01`、`FIELD-SBP-INPUT-02` | 父级职责、输入范围、唯一主路径 |
| `细裁决 / 成立轴` | 哪些 `panel_count`、句段锚点和静态字段真的成立，且能共同支撑插回动作 | 无锚点的好看句子；只看构图字段不看句段位置 | `FIELD-SBP-ANCHOR-03`、`FIELD-SBP-INLINE-04` | 锚点计划、候选排除理由、单行字段合同 |
| `离散裁决 / 优选轴` | 在成立候选里，哪种插法最稳、最省返工、最利于后续 sibling 继续发酵 | “都可以”式并列答案；写得聪明但落不到主文件的说明 | `FIELD-SBP-INLINE-04`、`FIELD-SBP-LAND-05`、`FIELD-SBP-HANDOFF-06` | 内联结果、主文件回写、验收与留口 |
| `全层 veto / 硬门禁轴` | 是否命中 `原文被改写 / 编号断裂 / 锚点漂移 / 越权写 sibling 字段 / 缺返工入口` | 带着已命中 veto 的结果继续润色 | `FIELD-SBP-LAND-05`、`FIELD-SBP-HANDOFF-06` | Fail code、返工入口、暂停/回退结论 |

### 层内自省

每次完成 `S2-S5` 后，至少要回答：

1. 为什么是这个锚点和这条分镜行，而不是相邻句段或另一种插法？
2. 如果不是这个结果，会不会得到更稳的原文守恒性或更低的下游返工成本？
3. 删掉当前保留项，下游会失去哪一个明确判断能力？
4. 如果把这一步改写成普通流程脚本，它为什么会比当前启发式合同更差？

若第 2 问答案为“会”，必须返回当前层重选，不得直接把旧结果带进下一层。

## 可见快照与隐藏推理分层

### 可见快照必须保留

| 可见块 | 最低内容 |
| --- | --- |
| `父级边界与输入归属` | 任务是否属于 `1-分镜表现`、输入链顺序、唯一主路径 |
| `锚点裁决与排除理由` | 每组 `panel_count`、句段锚点、放弃其他插法的原因 |
| `单行内联合同` | `分镜明细[]` 节点字段块、组内编号、禁止越权字段 |
| `主文件回写结论` | patch 到哪份 `第N集.json`、是否保持原文守恒、是否通过编号检查 |
| `验收与返工入口` | `validation-report.md` 中的 PASS/FAIL、失败码、返工入口、unknowns |

### 不进入可见合同的内容

- 完整内部比较全文
- 所有被淘汰锚点的逐条推理过程
- 与 `FIELD-SBP-*`、主文件写位或失败码无关的长篇感想

## 工具后反思与 Gate Summary

### 工具后反思

1. 若 `分镜密度` 返回的 `panel_count`、锚点或分组边界不稳，先回到 `S2-S3` 重判输入链与锚点，不得在 `S4-S5` 局部打补丁。
2. 若 `分镜构图` 结果缺字段、字段越权或与锚点不对齐，先回到 `S4` 重判单行合同，不得直接插回主文件。
3. 若主文件回写后出现原文漂移、编号断裂或一镜多插错位，先回到 `S5` 回滚并重插，不得把错误结果包装成“风格化写法”。
4. 若 `validation-report.md` 指出返工项涉及上游 grouped source 或分组锚点，直接暂停本层并回到上游修源，不得在本层强行弥补。

### Gate Summary 最低合同

| gate | PASS 信号 | 失败码 | 默认返工入口 |
| --- | --- | --- | --- |
| `边界门` | 当前任务确属 `1-分镜表现`，且未越权到 sibling 字段 | `FAIL-SBP-ROOT-01` | `S1` |
| `输入门` | 输入链完整、顺序清晰、可回查来源 | `FAIL-SBP-INPUT-02` | `S2` |
| `锚点门` | 每组 `panel_count` 与句段锚点成立且可解释 | `FAIL-SBP-ANCHOR-03` | `S3` |
| `内联门` | `分镜明细[]` 节点字段齐全、固定、无越权信息 | `FAIL-SBP-INLINE-04` | `S4` |
| `回写门` | 只 patch 同一份 `第N集.json`，原文守恒、编号连续、位置正确 | `FAIL-SBP-LAND-05` | `S5` |
| `闭环门` | PASS/FAIL、失败码、返工入口、unknowns 齐全 | `FAIL-SBP-HANDOFF-06` | `S6` |
