# 斯坦尼斯拉夫斯基体系·方法派表演技术参考

## Purpose

本细则是 `4-表演` 注入表演密度的方法论知识基底。它把斯坦尼斯拉夫斯基体系和好莱坞方法派的核心概念转译为 AIGC 制作中可执行的表演指令，供 LLM 在心理反应、五层表演控制、潜台词行为化、场景戏剧映射和演员任务规划时直接调用。

本文件不授权新增剧情事实、对白或事件结果；它只规定如何调用表演技术把上游已有信息翻译成演员可演、观众可感知的材料。

## 理论溯源与适用边界

### 斯坦尼斯拉夫斯基体系（俄式）

| 概念 | 原术语 | 核心含义 | 在 AIGC 中的投影 |
|------|--------|----------|-----------------|
| "假使" | *Если бы...* | 用假设条件激活角色真实反应 | 上游信息→感知锚定→情绪触发 |
| 肌肉松弛 | *мышечное расслабление* | 身体不紧绷才能传递真实信号 | 身体联动、呼吸、动作停顿 |
| 注意力集中 | *внимание* | 演员必须活在"舞台现实"而非观众 | 视线停点、道具特写、对手反应 |
| 形体动作分析 | *метод физических действий* | 通过外部动作触达内心状态 | 角色动作、对白画面、身体联动 |
| 情绪记忆 | *эффекты памяти* | 用个人情感记忆为角色注入真实 | 五层控制的 trigger 和 motive 层 |
| 最高任务 | *сверхзадача* | 角色的终极欲望/目标 | inner_motive、hidden_motive |
| 贯穿动作 | *сквозное действие* | 角色为达成最高任务采取的行动 | performance_task_map、行为策略 |
| 情感瞬间 | *предчувствие* | 情绪即将爆发前的临界点 | 微表情、身体僵住、沉默余波 |

### 方法派（美国化变体）

| 技术 | 代表人物 | 核心含义 | 在 AIGC 中的投影 |
|------|----------|----------|-----------------|
| 情感记忆调用 | 李·斯特拉斯堡 | 调用个人创伤经历激发角色情感 | 五层控制的 trigger→suppressed_emotion |
| 感官记忆 | 桑顿·怀尔德 | 用嗅觉/触觉/听觉重建真实感 | 环境声、身体触感、道具质感 |
| 自传性代入 | 马龙·白兰度 | 演员把自己的人生经验投射到角色 | 主角内心独白、第一人称视角 |
| 即兴具身化 | 李·斯特拉斯堡 | 用即兴练习探索角色行为边界 | 演员任务规划、行为策略 |
| 沉浸式停留 | 丹尼尔·戴-刘易斯 | 演员在片场保持角色状态 | 五层控制的 micro_dynamics |

**关键区分**：斯坦尼体系强调整体性和集体性，服务于戏剧整体；方法派强调个体化体验，服务于角色真实性。AIGC 制作应优先采用斯坦尼体系的"由外而内"（形体→内心），在需要深度情感注入时补充方法派的"由内而外"（内心→形体）。

## 感知锚定法（"假使"激活链）

### 触发路径

斯坦尼的"假使"是激活角色反应的起点。在 AIGC 中对应 PASS-PERF-03 的上游触发点定位：

```yaml
# 感知锚定执行链
感知锚定:
  step_1_锁定上游信号: 
    # 上游文字/动作/声音/道具/规则显影
    description: "找到场景中可被视觉/听觉/触觉/嗅觉接收的具体信息"
    examples:
      - "桌上锦缎折角的海浪纹在灯光下断开"
      - "门外传来两组脚步声，一快一慢"
      - "黑板最后一行的规则字迹被擦去一半"

  step_2_构建"假使"条件:
    # 把上游信号转化为角色感知
    description: "用'如果我是这个角色，此刻我的感知器官接收到了什么'激活真实反应"
    pattern: "如果我是<角色>，此刻<感知通道>告诉我<具体信号>"
    example: "如果我是阿真，此刻我的耳朵告诉我门外脚步变快了——有什么在靠近"
  
  step_3_身体先行:
    # 先写身体反应，再补心理
    description: "身体反应先于认知，不需要'理解'后才反应"
    priority: "可见/可听/可演 > 认知推理"
```

### 与五层控制的对应

| Pass 节点 | 斯坦尼概念 | 执行动作 |
|-----------|-----------|---------|
| `PASS-PERF-03` 心理反应可感知化 | "假使"激活 + 感官记忆 | 上游信号 → 感知锚定 → body_linkage 先写 |
| `PASS-PERF-04` 五层表演控制 | 情绪记忆 + 情感瞬间 | trigger → micro_expression → micro_dynamics |
| `PASS-PERF-06` 潜台词行为化 | 贯穿动作 + 最高任务 | 角色想达成什么 → 阻碍 → 行为策略 |

### 感知锚定变量字典

| 感知通道 | 激活信号示例 | 身体联动写法 |
|----------|-------------|-------------|
| 视觉 | 光影变化、道具位置变化、对方表情细微变化 | 视线停住、眨眼频率变化、瞳孔微缩 |
| 听觉 | 脚步声变化、器物碰撞声、语气停顿、声调异常 | 呼吸停顿、身体僵住、肩膀内收 |
| 触觉 | 器物温度变化（冷/热）、布料质感、风、地面湿滑 | 手部动作变化、重心偏移、脚尖位置 |
| 嗅觉 | 烟味、血腥味、香水异常、潮湿霉味 | 鼻翼轻张、喉结滚动、视线偏移 |
| 前庭/平衡 | 空间倾斜、地板震动、船/车晃动 | 手扶住、身体后仰、脚跟稳住 |

## 形体动作分析法（由外而内链）

### 核心方法

斯坦尼后期发展的"形体动作分析"强调：**通过做具体的外部动作来触达内心状态**，而非直接追求情绪。

```yaml
# 由外而内执行链
形体动作分析:
  step_1_拆解具体动作:
    # 不要抽象描述，要具体可执行动作
    description: "把'她在紧张'拆成'她把杯子换到另一只手'"
    forbidden: "紧张、害怕、愤怒、难过（情绪标签）"
    required: "动作+对象+空间位置"

  step_2_寻找动作与情绪的对应:
    # 哪些动作暗示哪些情绪状态
    description: "同一个动作可以暗示不同情绪，要回指上游确定"
    table:
      "手指在杯沿摩擦":
        可能指向: "紧张 / 思考 / 掩饰 / 等待"
        判定依据: "上游是威胁信号→紧张；上游是信息→思考；上游是追问→掩饰"
      
      "把东西慢慢放下":
        可能指向: "放弃 / 故意拖延 / 压制愤怒 / 确认安全"
        判定依据: "上游是谁的命令；放下的速度；放下后手的状态"

  step_3_用动作暗示而非说明:
    # 动作为载体，情绪通过行为逻辑暗示
    pattern: "<具体动作>+<动作完成质量>+<动作后状态变化>"
    example: "他把锦缎折角压平，指腹在断开的海浪纹上来回蹭过，手背却绷得很紧"
```

### 动作词汇库：按情绪功能分类

| 情绪功能 | 具体动作示例 | 动作质量描述 |
|----------|-------------|-------------|
| 紧张/警觉 | 眨眼加快、手指摩擦、肩颈内收、脚尖停住、呼吸变浅 | 动作变快、停顿变多、完成质量不完整 |
| 压制/强忍 | 把手藏进袖中、咬住后槽牙、视线绕开、身体反而静止 | 动作突然变慢、幅度变小、声音压低 |
| 试探/隐瞒 | 话说到一半停住、先看对方手再接物、整理无关物 | 动作衔接有裂隙、停顿在关键信息前 |
| 羞惭/心虚 | 避开正面、声线断掉、整理无关物、错开距离 | 动作先退后进、完成质量低 |
| 愤怒 | 指节发白、下颌绷住、身体前倾、动作变慢但力度加大 | 动作幅度增大、完成度高、停顿后突然启动 |
| 恐惧 | 身体后撤、手指松开、喉结滚动、视线固定在某处 | 动作变碎、不连贯、完成质量低 |
| 难过/不舍 | 视线不离开、眼眶发红但不移开、话尾音拖长 | 动作变慢、幅度变小、频率降低 |

## 情绪记忆调用法（方法派补充）

### 适用场景

当斯坦尼的"由外而内"不足以覆盖强情绪、压抑情绪或情绪切换时，补充方法派的情感记忆技术：

| 场景类型 | 优先方法 | 方法派补充 |
|----------|----------|-----------|
| 常规心理反应 | 形体动作分析 | 感知锚定 |
| 强情绪（愤怒/恐惧/悲伤） | "由外而内" + 微表情 | 情感记忆：调用"如果我曾经历过类似创伤" |
| 压抑情绪（强忍/掩饰） | "由外而内" + 三层动机 | 情感记忆：区分"表面平静"和"底层暗涌" |
| 情绪切换（忽然冷下来/忽然爆发） | 微动态限制 | 情感记忆：找到切换瞬间的"开关" |
| 未出口对白/想说不能说 | 身体先行 | 情感记忆：用真实咽回去的体验激活喉结、停顿 |

### 三层情绪执行模板

```yaml
情绪记忆调用:
  surface_emotion_layer:
    description: "表层正在展示什么（角色想让对方看到什么）"
    prompt: "如果演员此刻戴着一个面具，那个面具上写着什么表情？"
    examples:
      - "平静、冷淡、微笑、沉默、认真听"
    
  suppressed_emotion_layer:
    description: "被压住的真实情绪（角色自己知道但不让对方看到）"
    prompt: "如果面具忽然摘下来，此刻真正的情绪是什么？"
    examples:
      - "愤怒、恐惧、难过、不甘、羞耻"
    
  hidden_motive_layer:
    description: "角色为什么必须压住它（保护策略/身份风险/不能暴露）"
    prompt: "角色不直接表达真实情绪，是因为担心什么？"
    examples:
      - "不能暴露底牌"
      - "不能让对方觉得自己在意"
      - "不能激怒对方带来更大危险"
      - "不能让自己看起来软弱"
```

### 弱写法 vs 强写法

**弱写法**（只有情绪标签）：
```md
心理反应：阿真听到规则后很愤怒，但又不得不服从。
```

**强写法**（三层情绪 + 身体联动）：
```md
心理反应：阿真听见规则两个字时，嘴角还维持着笑，但指节在桌沿上慢慢收紧，像要把木头抠出痕迹；她没有抬眼，只把呼吸压到胸腔浅层，尾音比前一句轻了半度。

执行报告标注：
  surface_emotion: 平静（微笑、维持正常语速）
  suppressed_emotion: 愤怒（下颌绷住、指节收紧、呼吸变浅）
  hidden_motive: 不能让对方看出她在意这个规则
  body_linkage: 嘴角（笑维持）、指节（压桌沿）、呼吸（压到胸腔）
```

## 注意力集中与舞台现实法

### 核心方法

演员必须活在"舞台现实"中，而不是想着"观众在看"。在 AIGC 中对应：

| 斯坦尼概念 | 在 AIGC 中的投影 | 执行要点 |
|-----------|----------------|---------|
| 注意力集中对象 | 视线停点、道具状态、对手反应 | 不要写"看向远方"作为默认解，而是写角色真正在看什么 |
| 舞台现实 | 环境描写、空间细节、声音层次 | 让观众感受到角色所在的空间是真实的 |
| 真实交流 | 对手反应、对手对白后的停顿 | 角色不是对着空气说话，而是对"另一个活人"说话 |

### 视线操作规则

```yaml
视线作为注意力载体:
  优先级:
    1_对手的眼睛或面部: "正在真实交流时"
    2_道具或信息源: "需要处理信息时"
    3_空间中的威胁或机会: "感知到危险或希望时"
  
  视线转移触发:
    - "上一轮对白/动作结束后"
    - "空间压力改变时"
    - "情绪需要切换时"
    - "新的感知信号进入时"
  
  视线作为转场解的限制:
    "看向远方"只能在以下情况使用:
      - "观看对象是上游明确的信息源（窗户/门口/黑板）"
      - "空间侦察是角色的即时任务"
      - "威胁/希望的方位是情节核心"
    否则优先用声音桥、道具状态、空间变化、动作中断
```

## 情感瞬间与微动态控制

### 核心方法

斯坦尼强调情绪即将爆发前的瞬间（情感瞬间）最有戏剧张力。在 AIGC 中对应五层控制的 micro_dynamics：

```yaml
情感瞬间控制:
  前段:
    description: "情绪积累期，身体已有变化但意识在压制"
    典型写法: "呼吸变浅/动作变慢/手部有细微动作/视线绕开"

  触发点:
    description: "上游哪个信号打破了平衡"
    必须: "有明确上游触发点，不是凭空产生"
    
  临界点:
    description: "情绪即将泄露但被压住，或终于泄露一点"
    典型写法: "嘴角本来要抬起又压下去 / 眼眶湿润但没有落泪 / 话到嘴边停住"
    核心技巧: "不是爆发，是临界态"
    
  余波:
    description: "情绪通过身体某处泄露后，其余部分的状态"
    典型写法: "呼吸停顿一拍后恢复/手指慢慢松开/沉默延长半拍"
```

### 微动态词汇库

| 动态类型 | 典型写法 |
|----------|---------|
| 启动-停顿 | "他本要上前，脚尖却停在门槛边" |
| 启动-收回 | "嘴角本来要抬起，随即压回去" |
| 持续-增强 | "压在桌沿的手指慢慢收紧，指节发白" |
| 持续-减弱 | "笑意还挂在嘴角，但眼底的亮光一点点暗下去" |
| 切换-突然 | "她忽然停住，喉结动了一下，把后半句咽回去" |
| 切换-渐变 | "视线从他脸上滑开，落在窗外，声音也跟着变轻" |
| 余波-呼吸 | "话音落下后，她吸了一口气却没有吸满，手背抵到唇边又放下" |
| 余波-手部 | "她的手指在袖下松开，指尖却还在微微发颤" |

## 演员任务规划法（最高任务→贯穿动作）

### 核心方法

在规划层，每个角色的表演都可以用斯坦尼的"最高任务"+"贯穿动作"结构来组织：

```yaml
角色任务规划:
  最高任务（super-task）:
    description: "角色在这一集/这一场中最想达成什么"
    prompt: "如果用一句话描述这个角色在这个场景里最想要什么，那是什么？"
    example: "阿真最想要的是：不要让对方发现自己已经看穿了规则漏洞"
    
  贯穿动作（through-line）:
    description: "角色为达成最高任务采取的持续行动策略"
    prompt: "角色在整个场景中不断在做什么来接近这个目标？"
    example: "不断用追问来试探对方的底牌，同时用沉默来掩盖自己的底牌"
    
  即时目标（immediate objective）:
    description: "角色在每个对白/动作 beat 中的具体目标"
    prompt: "此刻角色想从对手那里得到什么？"
    
  阻碍（obstacle）:
    description: "角色为什么不能直接达成或说出来"
    typical: "身份限制/信息差/不能说出口的秘密/不能暴露的弱点"
    
  行为策略（tactic）:
    description: "角色用什么具体行为来应对阻碍"
    example: "反问/沉默/抢先递道具/后退半步/把问题压到最后/用玩笑卸力"
```

### 任务规划 vs 终稿的区别

| 层级 | 内容 | 是否进入终稿 |
|------|------|-------------|
| 规划层（可保留在证据中） | 最高任务、贯穿动作、即时目标、阻碍、行为策略 | 否（只保留在执行报告证据） |
| 执行层（必须进入终稿） | 具体动作、表情变化、身体联动、声音变化 | 是（必须内嵌到既有字段） |
| 禁止 | 在动作字段写"试图、想要、打算、意图"等主观预判 | - |

## 方法派沉浸技术（补充使用）

### 何时使用

当以下情况时，补充方法派的沉浸技术：

- 上游角色需要极度真实的状态（濒死、极度恐惧、创伤后）
- 需要演员在长段落中保持持续角色状态
- 主角的内心独白需要极度第一人称沉浸感

### 具身化技术

```yaml
具身化执行:
  1_感官重建:
    description: "用具体感官细节让角色"活在身体里""
    指令: "如果我是这个角色，我现在能闻到什么/摸到什么/听到什么"
    
  2_自传性代入:
    description: "调用自己真实经历中类似的情感记忆"
    注意: "不是发明新情绪，是找到自己曾有过的相似情绪"
    
  3_身体姿势先行:
    description: "先设计角色的身体姿势，再导出情绪"
    指令: "这个角色此刻的身体是什么状态的？紧张的姿态是什么样的？"
```

### 沉浸的边界

方法派沉浸技术不授权：
- 为角色新增与上游无关的个人历史
- 改变角色的目标或动机（必须服从上游）
- 创造上游没有的情绪类型
