## Reusable Heuristics

- 全局画面基调不是“审美愿望清单”，而是下游所有资产都能继承的底层渲染协议。越具体，越容易污染角色、场景和道具阶段。
- `色调与调色板` 维度可以在内部分析中出现，但最终 prompt 默认不写具体颜色词。内部可以判断“色彩纪律”，输出要落成“低饱和层级”“受控对比”“影调密度”等抽象可继承语言。
- 材质词最容易偷偷变成道具或服装设计。最终 prompt 应写表面行为和渲染机制，不写具体物质。
- 光影可以写结果，不写布置。例如可以写“体积光层次”“丁达尔效应”“漫反射扩散”，不要写灯从哪边打、什么焦段怎么拍。
- 大师锚点要像校准仪，不像署名堆栈。每个锚点至少回答：借哪一层、为什么适配、哪些部分不用。
- 参考图/视频分析先做“去内容化”：遮掉主体名称后还剩下的媒介、渲染、光影、颗粒、特效和运动质感，才适合进入全局协议。
- 剧本抽取视觉基因时，不要把剧情事件写进 prompt。应提取事件背后的秩序、压力、时间感、世界规则和情绪运行方式。
- 负面特征要足够制作化，例如避免塑料感、过度磨皮、廉价 3D 感、扁平贴图、随机光效、脏乱噪点，而不是“不要不好看”。
- 200-300 字 prompt 的稳定结构：媒介/流派 -> 渲染管线 -> 光影氛围结果 -> 情绪和时代抽象 -> 大师校准 -> VFX/质感控制 -> negative traits。
- 如果一个词只能影响某个具体镜头或具体物件，它不属于画面基调；如果它能影响全片所有资产的底层生成倾向，它才有资格进入全局 prompt。

## Repair Playbook

1. 先确认失败属于来源不足、风格属性抽取不足、因果链缺失、大师锚点缺证据、prompt 污染、字数不合规还是报告证据缺失。
2. 若来源不足，回到 `N1-INTAKE`，不要用热门风格或随机大师补空。
3. 若参考图/视频带来具象污染，回到 `N3-REFERENCE-EVIDENCE`，只保留媒介、渲染、光影、颗粒、VFX 和动画流派。
4. 若剧本分析只剩主题口号，回到 `N2-SCRIPT-EVIDENCE`，从题材机制、世界观规则、空间压力和情绪运行方式抽取视觉证据。
5. 若 prompt 出现颜色、材质、构图、摄影或具体内容，回到 `N7-PROMPT-DISTILL`，用风格层级表达替换。
6. 若大师参照只堆名字，回到 `N6-MASTER-ANCHORS`，补每个锚点的匹配维度和禁用边界。
7. 用户指出“脚本化/偷懒/未思考/未差异化”时，直接废弃候选协议回源层重做，不接受替换几个大师名或润色句子。
8. 修复后必须重新执行 `GATE-VT-01` 到 `GATE-VT-10`，并把失败码、返工点和复审结果写入 `Repair Log`。

## Contamination Scan Checklist

1. 删除具体颜色词、色温词和色彩符号；核心视觉符号例外必须有用户证据。
2. 删除具体材质、服装、物件、建筑、道具和生物名。
3. 删除构图法、景别、焦段、光圈、机位、推拉摇移、跟拍、环绕等摄影阶段词。
4. 删除具体角色身份、场景地点、动作事件和剧情结果。
5. 保留媒介、流派、渲染技术、光影结果、动画制作范式、艺术史锚点、VFX、负面特征。

## Positive Pattern

好的画面基调应该让下游读到后能判断：

- 这个项目更接近哪种媒介和渲染范式。
- 画面质感靠什么成立，而不是靠哪些具体物件成立。
- 光影和情绪如何组织，但不抢摄影执行权。
- 哪些艺术家/导演/摄影师/作品提供校准，哪些部分明确不借。
- 角色、场景、道具、分镜阶段仍保有各自设计空间。
