## Runtime Rule

所有域级业务真源统一写入对应域内 leaf 子目录：

```text
projects/aigc/<项目名>/7-设计/{场景,角色,道具}/{1-清单,2-设计,3-生成}/
```

域级包可以保留旧 JSON 作为兼容侧车，但最终验收只认：

```text
<域>/1-清单/<域>清单.md
<域>/2-设计/<主体名>.md
<域>/3-生成/<主体名>-*.json
```

## Incremental Runtime Rule

当 `6-分组` 分批完成或后续追加 `第N集.md` 时，`7-设计` 必须先执行增量对账，不得把既有产物视为可静默覆盖的一次性全量输出。统一规则见：

```text
.agents/skills/aigc/7-设计/references/incremental-reconciliation-contract.md
```

每个域可以维护 sidecar 状态索引：

```text
projects/aigc/<项目名>/7-设计/{场景,角色,道具}/design-manifest.yaml
```

`design-manifest.yaml` 只记录已消费上游、主体映射、设计稿路径、生成资产路径和缺口；清单主体真源仍以 `<域>/1-清单/<域>清单.md` 为准。
