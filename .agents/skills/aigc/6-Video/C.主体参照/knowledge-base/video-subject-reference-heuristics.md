# Video Subject Reference Heuristics

本文件保存 `C.主体参照` 可复用经验。强制合同以 `SKILL.md` 与 `references/` 为准。

## Heuristics

- `全能参照` 的旧三件套适合当兼容来源，但新包 runtime 根应统一落到 `C.主体参照/<episode>/`，不要继续分散在旧编号目录下。
- 主体参照最容易出错的点是把“视觉主体”泛化成一切词汇；只把可见、可识别、可由资产承载的对象放入主体索引。
- 角色和服装要关联，但不要合并；角色资产回答“是谁”，服装资产回答“当前长什么样”。
- 道具只有在镜头中被持有、使用、强调或承担叙事功能时才值得成为主体参照。
- 场景参照服务空间连续性，不应覆盖组级风格、氛围或镜头事实。
- 故事板和单帧图在本包里是辅助资产，不是第一绑定目标；如果它们才是用户要的主参照，应改走 A/B 融合包。
- `subject-index.json` 比自然语言报告更适合 provider handoff 回放；报告负责解释取舍，JSON 负责稳定消费。
- 无资产命中不是失败，猜测绑定才是失败。
