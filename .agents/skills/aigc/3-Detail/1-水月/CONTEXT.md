## Repair Playbook

1. 先确认 grouped script 是不是唯一真源，避免拿错输入。
2. 再核对 `Init / Global` 是否已被压成写作约束，而不是文案标签。
3. 若成稿发散，先重做组锚点，不要直接修句子。
4. 若人物成立但空间发虚，优先补 `运动表现 + 氛围表现`，不要继续堆情绪词。
5. 若画面好看但不稳，先压回 `品味` 与 `逻辑性`，再决定是否保留冲击力表达。
6. 超字数时优先删除重复修辞和解释性句子，最后才删主冲突。

## Reusable Heuristics

- `1-水月` 的核心不是“写得更长”，而是“把 grouped script 变得更可拍、更可演、更可感受”。
- `1-水月` 的锚点属于内部收束工具，不属于最终成稿字段；落盘时看起来必须还是剧本。
- 四条维度链应该像树根供养同一段 prose，而不是各自争抢段落版面。
- 最稳的组级扩写写法是：先锁一条主行动线，再给它加人物压力、空间气候和视觉抓手。
- `Init / Global` 最好不要以名词标签进入正文，而要变成语气、动作尺度、景物细节和感官温度。
- 想压字数时，先删抽象解释，保留能被镜头直接感知的内容。
- `冲击力` 不是靠形容词堆出来的，通常来自一个更明确的动作节点、视线节点或空间反差。
- `意境` 一旦离开 `层次` 和 `空间诗学`，很容易变成悬空的抒情句。
- 对手戏最稳的写法不是解释两人关系，而是让一个人怎么逼近、另一个人怎么回避或失手。
- 有台词的对手戏若想写得活，不能只交代台词内容；最稳的是把抢话、吞话、断句、换气失衡也写成权力变化证据。
- 子模块真源若只剩说明文，执行时就会退回临场发挥；最稳的做法是把每个分支和叶子都压成 `module-spec.yaml`，写清触发、问题栈、patch 槽位和裁剪优先级。
- 节点包长期要稳，必须把 `module-spec.yaml` 和 `module-guide.md` 配成双文件：前者锁合同，后者锁解释。
- 当创作类 `module-guide.md` 只剩“定位 + 使用方法 + 失真修正”三段时，解释层仍然太薄；更稳的写法是补齐 `判型入口 + 思维·执行主线 + 节点 + 延展`，把审美判断压成可重复的思考顺序。
- 想把创作写得更鲜明，不是四条链一起加码，而是先用 `route-profile.yaml` 决定谁主谁辅。
- `视觉强化` 的真正价值不是“写得更美”，而是把戏剧收益转成更具像、更可被 AIGC 下游稳定消费的视觉信息。
- `运动表现` 的真正价值不是“动作更顺”，而是把人类默认知道、但 AI 容易漂移的连续性与逻辑关系提前言明。
- 当 `角色表现` 命中 `动作戏` 主链时，默认按香港武侠动作电影黄金武指的尺度写：每次出手都要有爆发点、拆招关系、角色门路和改局结果。
