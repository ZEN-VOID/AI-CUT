# Volume Rhythm Framework

## 默认方法

卷级节奏固定采用六拍机制：

1. `卷钩`
2. `铺展`
3. `首回报`
4. `中卷反拧`
5. `卷末冲顶`
6. `尾钩/收束`

## 六拍问题

| beat | 本卷必须回答的问题 |
| --- | --- |
| `卷钩` | 本卷一开始用什么新 promise 或新压力抓住读者 |
| `铺展` | 本卷主战线如何展开，人物/场景/道具如何进场 |
| `首回报` | 前半铺设第一次怎样兑现，不让本卷前段空转 |
| `中卷反拧` | 本卷中段怎样改向，而不是只加码 |
| `卷末冲顶` | 卷末冲突、任务或关系如何推到峰值 |
| `尾钩/收束` | 本卷完成什么，又把什么交给下一卷 |

## 输出要求

- `本卷节奏曲线` 必须用这六拍标注卷内波形。
- Mermaid 图至少能看出卷内起势、改向、冲顶、交接四个变化点。

## 卷级节奏真正要回答的规划问题

- 本卷作为“一个追读单元”到底承诺什么，不只是承接整书总纲。
- 本卷前半怎样尽快成立局面、人物关系和任务压力，而不是若干章后才真正开卷。
- 本卷第一次兑现落在哪一段，避免连续多章都只铺不收。
- 中卷如何反拧，让本卷中段不是单纯加码或拖时长。
- 卷末冲顶后，本卷到底完成什么、又把什么转交给下一卷。

## 强化方向

- 卷级节奏优先处理“本卷波形和章节职责分配”，不是单章句段编排。
- `章划分` 应能回指六拍，让每章知道自己是在起势、铺展、首回报、反拧、冲顶还是交接。
- 卷级要显式说明哪些线索、任务、关系在本卷必须首兑，哪些只做下一卷压力转移。
