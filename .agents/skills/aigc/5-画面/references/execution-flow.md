# 5-画面阶段执行流程细则

## Canonical Landing

- 阶段根目录：`projects/<项目名>/5-画面/`
- 阶段验证报告：`projects/<项目名>/5-画面/validation-report.md`
- 分镜故事板：`projects/<项目名>/5-画面/分镜故事板/`
- 分镜帧：`projects/<项目名>/5-画面/分镜帧/`
- 漫画：`projects/<项目名>/5-画面/漫画/`

## 输入真源合同

### 必需输入

- `projects/<项目名>/编导/第N集.json`
- `.agents/skills/aigc/_shared/director_episode_output.schema.json`
- 上游至少已完成 `3-明细/subtypes/1-分镜表现`
- 本轮至少能锁定 `final_output.main_content.分镜组列表[]`、`分镜组ID`、`分镜ID` 或等价既有画面锚点之一

### 推荐输入

- `projects/<项目名>/3-明细/第N集.md` 作为人工可读 sidecar
- `projects/<项目名>/4-主体/` 下的角色、场景、道具参考资产
- `projects/<项目名>/5-画面/` 下已有的 prompt、manifest、历史图片与失败记录
- `projects/<项目名>/2-组间/` 下的风格、导演意图、节奏说明
- 用户补充的参考图、风格图、负面约束或模型偏好

### 输入处理原则

1. `5-画面` 的核心消费源是 `projects/<项目名>/编导/第N集.json`，不是重新猜测镜头事实。
2. `3-明细/第N集.md` 若存在，只作为人工可读 sidecar，不构成第二份 episode/group/shot 真源。
3. `4-主体`、已有参考图与历史出图只作为一致性参照与资产锚点来源，不改写镜头事实。
4. 若 `5-画面/` 下已存在 prompt、manifest 或历史图片，优先将其作为连续迭代锚点，而不是另起一套新口径。
5. 若上游 `编导/第N集.json` 尚未形成合法 `分镜组列表[] / 分镜明细[]`，必须先回到 `2-组间 / 3-明细`。

## Mandatory Workflow

1. 读取上层 `.agents/skills/aigc/SKILL.md + CONTEXT.md`。
2. 锁定 `projects/<项目名>/编导/第N集.json`、`4-主体/` 与已有 `5-画面/` 产物中哪些文件是本轮权威输入；必要时回读 `3-明细/第N集.md` 做人工校对。
3. 根据用户任务与输入状态，在 `分镜故事板 / 分镜帧 / 漫画` 中输出唯一主入口。
4. 组合本轮 prompt 包，明确正向约束、负向约束、一致性锚点与复用哪些已有图像。
5. 将对应产物落到 `projects/<项目名>/5-画面/<子路径>/`，并在阶段级 `validation-report.md` 记录路由、prompt 组合、一致性与验收。
6. 返回唯一推荐的下一步入口，而不是模糊候选集合。

## Handoff Rule

- `5-画面` 的直接上游是 `编导/第N集.json` 的共享分镜组/分镜明细、`4-主体` 的一致性锚点，以及本阶段既有 prompt / 图片资产。
- `分镜故事板` 是整阶段模式的默认主入口；`分镜帧` 与 `漫画` 是同源分支。
- 当前阶段完成后，默认 handoff 方向是 `6-视频` 的分镜参照消费面；若画面仍不稳定，可继续留在 `5-画面` 内做迭代复跑。
