# 创作型技能包发布级别合同

## 1. 目的

本文件定义创作型技能包综合评估后的推荐发布级别，用于避免只用 `PASS/FAIL` 粗糙表述成熟度。

## 2. 发布级别

### `R0` 合同成形期

- 仅说明合同开始成形
- 允许静态审阅
- 不允许直接进入生产

典型状态：

- 主要依赖 `L0-L1` 证据
- 四层中至少一层明显未成立
- 缺少足够动态证据

### `R1` 可试跑期

- 允许受控试跑
- 结果只适合作探索与补证
- 不适合持续复用

典型状态：

- 至少有 `L1-L2` 证据
- 静态结构基本成立，但动态结果不稳
- repair entry 已存在，但回归链路未稳

### `R2` 受控生产期

- 可用于限定范围生产
- 需要明确边界、题材范围与操作条件
- 允许带人工复核

典型状态：

- 至少有 `L2-L3` 证据
- 四层主结构大致成立
- 已有 baseline 与 regression 的动态证据
- 已知风险可被定位与返工

### `R3` 稳定复用期

- 可跨多个相近任务重复复用
- 质量下限清楚，回归机制存在
- 可作为团队稳定工作流的一部分

典型状态：

- 至少有 `L3` 证据
- baseline、boundary、regression 至少部分稳定
- 路由与知识拓扑已基本健康
- 局部升级不会立即破坏全局

### `R4` 可推广演化期

- 可跨题材、跨版本、跨场景推广
- 既有稳定下限，也有清晰升级路径
- 能作为共享方法或平台能力输出

典型状态：

- 达到 `L4` 证据
- baseline、boundary、stress、adversarial、regression 的动态证据长期齐备
- 四层结构成熟，且升级经济性良好
- shared carrier、template、schema、runbook 已形成稳定协同

## 3. 推荐规则

- 发布级别由四层主评分、横切专项检查、证据等级共同决定。
- 任一 `FAIL-COVENANT` 不得高于 `R0`。
- 仅有 `L0-L1` 证据时，不得高于 `R1`。
- 缺少动态回归证据时，不得高于 `R2`。
- 路由质量或知识拓扑专项明显失效时，不得高于 `R2`。
- 若质量上限高但质量下限不稳，优先下调发布级别，而不是维持高等级。

## 4. 用户输出要求

推荐发布级别时，必须同时说明：

- 当前建议级别
- 为什么不是更高一级
- 升到下一等级所需最小动作
