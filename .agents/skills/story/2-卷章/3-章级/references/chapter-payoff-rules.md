# Chapter Payoff Rules

本文件承载 `story-plan-chapter-level` 的章级爽点设计规则。爽点设计与节奏设计并列，但必须高度关联：爽点回答“读者为什么爽、爽在哪里兑现、爽后留下什么余味”，节奏回答“这一章怎样跑、怎样把结构义务交给 drafting”。

## Positioning

章级爽点设计只做 planning，不直接写正文。它必须在 `本章冲突` 之后、`本章节奏曲线` 之前完成裁决，并把结果回流给：

- `payoff_type`
- `micro_payoff`
- `selected_mode`
- `mode_selection_reason`
- `rhythm_intensity`
- `exit_hook`

## Required Payoff Slots

`本章爽点设计` 必须至少包含：

- `reader_desire`：读者此刻最想看到什么被满足。
- `promise_source`：该爽点上承哪个卷级 promise、角色欲望、任务欠账、关系欠账、信息欠账或世界观期待。
- `genre_payoff_profile`：本章所属小说类型或子类型下，爽点应该采用什么口味、禁忌和兑现尺度。
- `character_anchor`：该爽点绑定哪个角色的个性、欲望、缺陷、惯常反应、关系姿态或成长压力。
- `payoff_mode`：本章采用的爽点形态。
- `payoff_variation_axis`：本章高潮点相对近邻章节的差异轴，避免同类爽点重复。
- `duel_variation_axis`：若本章是高超对决，必须说明本次对决与近邻章节的差异轴。
- `build_up`：爽点前需要压什么、欠什么、误导什么、延迟什么或铺什么反差。
- `delivery_action`：本章实际完成的兑现动作，必须可验证。
- `satisfaction_delta`：兑现前后，角色状态、局势、认知、关系或世界感发生什么可见变化。
- `exaggeration_logic`：本章爽点如何被夸张、放大或极致化，以及为什么仍然符合角色与情境逻辑。
- `cost_or_aftershock`：爽点之后留下的代价、余波、新问题或情绪残响。
- `aftertaste_hook`：读者读完后还想翻下一章的残余牵引。

## Character-Bound Payoff Rule

爽点必须与角色个性高度相关。它可以夸张、锋利、癫狂、华丽、荒诞、热血、暧昧或极端，但不能像外部插件一样贴到任意角色身上。

设计时必须先回答：

1. 谁是本章爽点的承担者。
2. 这个爽点放大了 TA 的哪一种个性、欲望、缺陷、恐惧、关系姿态或成长压力。
3. 夸张之后，角色是否仍然会这么做、这么想、这么反应。
4. 若角色做出反常举动，情境压力、误判、诱因或成长节点是否足够支撑。

允许的夸张方式：

- 个性放大：把角色平时的锋利、温柔、贪婪、忠诚、羞怯、傲慢、好奇或疯劲推到更高强度。
- 欲望放大：让角色更接近想要之物，或更痛地失去它。
- 缺陷放大：让角色的盲点、执念、误判或习惯性反应带来强烈后果。
- 关系放大：让角色在亲密、对抗、嫉妒、保护、背叛、默契中暴露真实姿态。
- 成长放大：让角色在本章做出比过去更进一步、但仍能上溯的选择。

禁止的夸张方式：

- 为了爽点让冷静角色无因莽撞、让怯懦角色无因英勇、让深情角色无因背叛、让克制角色无因失控。
- 只写奇观、暴力、情色、反转或热血，不说明它为什么属于这个角色。
- 用“剧情需要”“读者需要”代替角色动机。

## Genre Payoff Profile

爽点必须先经过类型画像校准。同样的动能式、势能式或浪能式，在不同小说类型里不应长成同一种爽法。

类型画像至少回答：

1. 本项目或本卷的主类型是什么。
2. 当前章是否存在副类型或临时类型变奏。
3. 该类型下读者默认期待什么爽点。
4. 该类型下哪些爽点会显得串味、过界或破坏承诺。

常见类型适配：

| genre_payoff_profile | 动能式偏向 | 势能式偏向 | 浪能式偏向 |
| --- | --- | --- | --- |
| `xuanhuan_fantasy` | 高超对决、境界压制、法则碰撞、资源争夺、奇观战斗 | 道心震荡、天命压迫、禁忌真相、血脉/因果显影 | 宗门日常、奇境游历、灵物体验、师友情义 |
| `wuxia_xianxia` | 招式拆解、身法博弈、剑意/道法对撞、江湖救援 | 侠义抉择、心魔、誓约、牺牲、师门因果 | 江湖烟火、酒肆舟行、师徒/同道默契 |
| `urban_power` | 打脸反制、资源到手、身份亮明、职场/商业翻盘 | 阶层压迫、秘密身份、欲望膨胀、代价浮出 | 城市生活质感、暧昧拉扯、社交场景、小确幸 |
| `mystery_suspense` | 追捕、脱身、现场反制、证据争夺 | 谜面逼近、误判修正、真相裂缝、心理压迫 | 侦查日常、伙伴默契、环境氛围、软线索偶得 |
| `romance` | 主动保护、关系行动、情敌/误会现场处理 | 情感压迫、吃醋、误读、告白前压力、秘密逼近 | 暧昧、陪伴、照顾、生活甜点、默契升温 |
| `comedy_slice` | 小比赛、小任务、小反击、手艺展示 | 尴尬升级、误会累积、角色执念暴露 | 日常趣味、关系玩笑、生活质感、轻盈治愈 |
| `political_strategy` | 朝堂/商战/组织行动、局部清算、棋局落子 | 权谋压迫、忠诚测试、背叛疑云、价值选择 | 阵营内部默契、权力日常、局后余味 |
| `horror_weird` | 逃生、封印、反制怪异、危险现场突破 | 精神污染、认知崩坏、禁忌显影、不可名状压迫 | 安全屋短暂喘息、异常生活化、诡异余味 |

类型机制只校准“爽法口味”，不拥有最终裁决权。最终爽点仍必须同时通过角色一致性、节奏式和卷级 promise 三道门。

## General Payoff Variation Matrix

所有高潮点都需要多样化设计机制。若近邻章节连续使用同一类 `payoff_mode`，必须至少改变两个通用差异轴；若本章是阶段性大高潮，建议改变三项以上。

| variation_axis | 可选方向 | 设计问题 |
| --- | --- | --- |
| `satisfaction_object` | 角色、关系、任务、资源、身份、真相、世界规则、情绪状态、读者预期 | 本章满足的对象和上一章是否不同 |
| `payoff_mechanism` | 反杀、打脸、揭秘、升格、获得、失去、和解、牺牲、保护、诱导、逃脱、修复、展示、承认 | 爽点靠什么机制兑现，而不是只靠声量 |
| `scale_shape` | 小巧、局部、现场、组织级、城市级、世界级、精神级、关系级 | 本章规模是放大、缩小、转私密，还是转公共 |
| `timing_shape` | 即时兑现、延迟兑现、反向兑现、半兑现、误兑现后纠正、先苦后甜、先甜后刺 | 爽点释放的时序是否变化 |
| `participant_relation` | 主角独自、主配合力、队友接力、敌我互相成就、旁观者见证、关系对象参与、群体反应 | 谁参与了爽点，见证关系是否变化 |
| `emotional_color` | 热血、冷酷、荒诞、暧昧、悲壮、治愈、压抑、轻松、惊悚、庄严、羞耻后反弹 | 读者感到的情绪颜色是否重复 |
| `cost_shape` | 无代价、小代价、身体代价、关系代价、资源代价、秘密暴露、道德代价、长期伏笔 | 爽完之后付出什么或欠下什么 |
| `aftermath_shape` | 余威、余痛、余甜、余谜、余惧、余债、余笑、余温、新方向 | 章末余味和上一章是否不同 |
| `information_state` | 读者先知道、角色先知道、双方都误判、对手先知道、只给软线索、揭开一半 | 信息分配是否产生新鲜感 |
| `environment_use` | 场景奇观、生活物件、制度规则、自然地形、人群舆论、仪式流程、空间限制 | 爽点是否利用了不同环境与道具 |

常见高潮点的变体提示：

- 反杀：可变对手类型、反杀方法、是否公开、是否留情、代价、后续追杀。
- 打脸：可变见证人、羞辱尺度、证据来源、反转时机、是否让对方自证其错。
- 揭秘：可变信息来源、谁先知道、揭开比例、真相带来的情绪颜色和后果。
- 关系升温：可变亲密动作、误会处理、被看见的弱点、共同完成的小事、未说出口的余味。
- 牺牲：可变牺牲对象、主动/被迫、公开/隐秘、即时收益、长期债务。
- 治愈：可变修复对象、修复方式、是否被对方察觉、余温和下一步行动力。
- 奇观：可变奇观功能、尺度、观看者、规则信息、是否改变局势。

多样化不是每章都换类型，而是让同类爽点在对象、机制、尺度、时序、情绪和后果上出现新组合。

## Rhythm-Linked Payoff Families

### 动能式爽点

动能式爽点偏外部化、现场化、结果化。它的核心满足来自“事情发生了、力量释放了、局面被改写了”。

常见爽点形态：

- 暴力释放：战斗、反杀、惩罚、破坏、夺取、逃脱、救援、突破。
- 高超对决：高手过招、招式拆解、战术博弈、法则碰撞、棋局互算、谈判交锋、技艺碾压、临场破招。
- 权力兑现：压制对手、身份亮明、资源到手、谈判翻盘、规则被强行改写。
- 感官刺激：情色张力、危险现场、仪式、盛宴、怪物、灾变、视觉奇观、尺度奇观。
- 任务结果：成功、失败、代价落地、关键物品到手、敌我态势改变。
- 团队协作：配合成功、牺牲掩护、背刺反背刺、临场信任建立。

设计重点：

- `build_up` 要制造阻碍、压迫、时间限制或力量差。
- `delivery_action` 必须是外部动作，不是纯心理判断。
- `satisfaction_delta` 要能看见局势变化，例如赢、输、得、失、暴露、逃离、夺回、毁掉。
- `cost_or_aftershock` 不得缺失；动能爽点越强，后果越要留下牵引。

常见误判：

- 只有打斗过程，没有结果变化。
- 只有“强者对强者”的名义，没有体现双方技艺、判断、反制和临场调整的高超性。
- 只写“高潮很燃”，没有可验证的兑现动作。
- 把感官奇观当成独立目的，忘记它必须服务任务、权力、情绪或世界观。

#### 高超对决变体矩阵

多章连续出现“高超对决”时，必须改变至少两个变体轴，避免每章都变成同一种强者碰撞。

| variation_axis | 可选方向 | 设计问题 |
| --- | --- | --- |
| `opponent_type` | 力量型、速度型、技巧型、智谋型、资源型、规则型、心理型、群体型、镜像型、克制型 | 这个对手强在哪里，和前一个对手有什么本质不同 |
| `contest_domain` | 武力、法则、谈判、棋局、推理、追捕、潜入、手艺、商业、舆论、仪式、团队协作 | 本次比的不是“谁更强”，而是在哪个场域更高明 |
| `win_condition` | 击败、拖延、夺取、保护、识破、诱导、脱身、迫使对手犯错、拿到承认、改变规则 | 本章到底怎样算赢，是否允许惨胜或局部胜 |
| `power_gap_shape` | 碾压、弱胜强、互有克制、信息不对称、环境偏置、规则不利、代价不对等 | 爽点来自碾压、破局，还是以小博大 |
| `turning_method` | 临场破招、预判反制、借力打力、规则漏洞、心理诱导、队友配合、牺牲换位、隐藏资源、认知翻转 | 本章转折靠什么技术含量成立 |
| `spectacle_surface` | 视觉奇观、招式美感、逻辑精密、语言锋利、情绪爆燃、沉默压迫、生活化巧劲 | 读者感知到的“高超”是什么质地 |
| `cost_shape` | 受伤、暴露、失去资源、欠人情、关系裂痕、精神污染、规则反噬、留下更强敌人 | 高超之后付出什么代价或留下什么后果 |
| `repeat_avoidance` | 与上一场在对手、场域、胜法、代价、情绪色彩上至少两项不同 | 本章如何避免重复上一场 |

高超对决不要求每次都升级声量。可以用更小、更巧、更阴、更温柔、更荒诞、更精密的方式制造差异。

### 势能式爽点

势能式爽点偏内在化、精神化、认知化和压力化。它的核心满足来自“我终于看懂了、我被逼到了、我被点燃了、我意识到某种奇特或极端的东西”。

常见爽点形态：

- 认知震荡：揭秘、反转、误判修正、隐藏代价浮出、危险确认。
- 精神奇观：奇特、癫狂、扭曲、变态、异质、崇高、神秘、不可理喻的心理或价值观压迫。
- 情绪升压：热血、慷慨激昂、情义、牺牲、复仇心、信念被点燃。
- 道德压迫：选择变窄、两难显形、代价加重、角色被迫承认某件事。
- 关系张力：信任裂缝、忠诚证明、秘密逼近、亲密与危险同时增长。

设计重点：

- `build_up` 要先制造信息差、心理压力、价值冲突或隐性威胁。
- `delivery_action` 可以是发现、判断、承认、发誓、背负、看见真相或精神转向，但必须造成可见状态变化。
- `satisfaction_delta` 通常表现为认知、信念、关系、退路或压力结构的变化。
- `aftertaste_hook` 应来自更大的压力自然逼近，而不是生硬断章。

常见误判：

- 把势能式写成“什么都没发生”。
- 只有阴暗、扭曲、变态的气氛，没有角色状态或认知变化。
- 只写热血口号，没有牺牲、欠账、承诺或行动前压力。

### 浪能式爽点

浪能式爽点偏体验化、关系化、生活化和余味化。它的核心满足来自“我愿意待在这里、和这些人一起多走一会儿”。

常见爽点形态：

- 生活质感：美食、旅行、风景、集市、节日、住所、手作、装备、日常秩序。
- 关系暖点：陪伴、默契、暧昧、玩笑、照顾、被理解、被接纳、轻微吃醋、友情升温。
- 情绪修复：休息、疗愈、松弛、释然、重新获得安全感或行动力。
- 趣味体验：玩耍、试错、小比赛、小误会、轻喜剧、角色反差、可爱笨拙。
- 世界感打开：新地点、新风俗、新规则、新物种、新职业、新生活方式。
- 软线索顺手获得：不靠硬揭谜，在逛、聊、吃、玩、看、试的过程中自然获得信息。

设计重点：

- `build_up` 不一定压危机，可以压疲惫、陌生、尴尬、距离感、好奇或小期待。
- `delivery_action` 要是可感知的体验兑现，例如吃到、看到、玩到、靠近、放松、被安慰、发现小线索。
- `satisfaction_delta` 通常表现为关系更近、状态更好、世界更立体、心情更亮或下一站更有期待。
- `cost_or_aftershock` 可以很轻，例如小麻烦、轻微误会、隐约线索、未说出口的心事或下一处风景。

常见误判：

- 把浪能式写成无推进日常。
- 只有景物和食物清单，没有角色关系、状态或世界感变化。
- 用突然危机破坏松弛感，导致浪能爽点被硬拽回动能或势能。

## Payoff Arbitration

当一个章节同时有多种爽点时，按以下顺序裁决主爽点：

1. 先看读者当前最强期待：想看到行动结果、认知震荡，还是关系/体验/恢复。
2. 再看类型画像要求的爽法口味与禁忌。
3. 再看本章上承的卷级 promise 或任务欠账。
4. 再看承担爽点的角色个性、欲望、缺陷或关系姿态是否支撑这个兑现。
5. 再看本章 `delivery_action` 最终改变了什么。
6. 最后才选择 `payoff_type` 与 `selected_mode`。

允许副爽点，但 `payoff_mode` 和 `payoff_type` 只能各自裁定一个主导项；副爽点写入 `mode_selection_reason` 或 `cost_or_aftershock`，不得堆成安全列表。

## Planning-Only Boundary

- 可以写爽点机制、兑现动作、状态差、余味和牵引。
- 不得写可直接进入正文的暴力、情色、对白、描写段落或句段级桥接。
- 涉及情色、暴力或精神极端感时，只能作为叙事功能和类型信号描述，不输出露骨正文。
