# 解说剧 Presentation Type

本类型卡定义 `解说剧` 呈现方式在 `4-编剧` 中的策略偏置。它不是子技能，不拥有输出路径、节点、gate 或最终审查权。

## Identification

| field | value |
| --- | --- |
| `type_id` | `presentation-jieshuoju` |
| `axis` | `presentation_axis` |
| `screenplay_mode` | `jieshuoju` |
| `aliases` | 解说剧、解说模式、旁白解说剧、解说短剧 |
| `trigger_rule` | 只在用户或项目长期记忆显式要求时启用 |

## Strategy

| strategy_slot | rule |
| --- | --- |
| `voice_policy` | 陈述性 source 必须全部落为 `旁白（叙述者/指定主体）` + `旁白画面`；上游已有对白仍逐字冻结为对白。 |
| `source_unit_policy` | 进入正文前必须建立 `jieshuoju_source_unit_coverage_map`，覆盖已有对白、可见动作、环境状态、陈述事实、背景说明、时间桥、关系状态、结果概括、规则/系统信息和混合句。 |
| `field_variety_policy` | 不得连续 4 组以上无承托 `旁白 + 旁白画面`；含 3 条及以上旁白对的场景必须有非旁白视觉字段。 |
| `dramatization_policy` | 不新增 source 不支持的戏剧回合；只能通过旁白节拍、画面承托、转场、声音、状态连续性增强观看流畅度。 |
| `report_evidence` | `Screenplay Mode Decision`、`Jieshuoju Source Unit Coverage Map`、`Jieshuoju Field Variety Map`、`Narration To Voice Adaptation Map`。 |

## Prohibited Drift

- 不得把陈述性 source 改写为派生对白、独白或内心独白。
- 不得使用 `summary`、`fact_drop`、`cause_reorder`、`new_exposition` 或 `tone_rewrite` 替代 source 承接。
- 不得把可见动作和环境重复旁白化；它们应优先进入正式画面字段。
- 不得用方括号叙事小标题代替正式场景标题和字段节奏。

## Combination Notes

`解说剧 x 题材类型` 的组合重点是：题材材料只影响旁白画面承托、场景材料、节奏组织和尾钩落点，不允许题材包新增 source 之外的剧情回合。
