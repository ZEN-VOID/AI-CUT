# 逻辑性 模块说明

## 定位

- 本叶子负责给动作链补上“为什么是现在、为什么能接下去”的因果短链。
- 它不负责增添新桥段，只负责让已有动作不显得凭空发生。

## 具体创作方法

- 每次都锁定 `cause -> action_bridge -> effect` 三点，但三点都要尽量落在一个短动作链里，而不是拆成说明书。
- `cause` 优先取当前场上已经存在的压力源，例如被发现、被逼近、视线相撞、门将关上、脚步逼近、对方先出手。
- `action_bridge` 是从触发到动作之间最短的一步，例如“顺势侧身”“下意识后退”“借着回身把门掩住”“被逼得抬手一挡”。
- `effect` 只写对下一拍真正有用的结果，例如距离被拉开、节奏被抢先、遮挡形成、对方被迫停住。
- 若一个动作本身已经自带因果，不必硬补第二层解释。逻辑性的价值在于消除“凭空发生”，不是把节奏拖慢。

## 思维·执行节点

1. `触发点识别`
   先找动作不是凭空发生的证据。没有触发点，动作就会像为了画面而强行出现。
2. `动作桥补写`
   在触发点和主动作之间补一个最短桥，让转折显得顺势而不是突兀。
3. `结果落位`
   明确这个动作把局面推向哪里，给下一拍留下可接的结果。
4. `节奏复核`
   读一遍，看这条因果链是否比原文更顺，而不是更慢。

## 节点延展

- 当一组动作包含追、挡、抢、躲、停、交接等推进型行为时，逻辑性应适度上浮，优先解释动作发生的时机。
- 当场面重心是情绪压迫或关系试探时，逻辑性只需补最短触发，不要把内在动机全拿过来解释。
- 可延展的是“动作为什么接得上”，不可延展的是“人物完整人生和背景史”。

## 失真与修正

- 若动作只是“为了好看所以发生”，说明缺失触发点。
- 若开始解释太多背景，说明因果链写长了。
- 若因果成立但节奏变慢，保留最关键的触发点和结果即可。
- 若 effect 不能为下一拍提供承接，说明当前逻辑链仍然没有真正落地。
