# 水月 创作评审 Rubric

## 评审维度

| 维度 | 核心问题 | 过线标准 |
| --- | --- | --- |
| 人物成立度 | 人物压力、目的和关系是否可见、可演 | 至少有一处可拍信号，且不是纯情绪标签 |
| 动作与空间清晰度 | 谁往哪里动、为什么现在动是否清楚 | 读取者能在一次阅读内形成动作和方位图 |
| 场感承载力 | 氛围是否由空间、物象和感官条件托住 | 情绪温度不悬空，空间参与叙事 |
| 字段可消费性 | `角色表现 / 运动表现 / 氛围表现 / 视觉强化` 是否都可直接写回 | 每条都能回指固定 `剧本正文` 的组内锚点 |
| 槽位分工度 | 同一字段内部各槽是否真的在回答不同问题 | 不出现同句跨槽复写、轻微改词复写或把组级导演意图原样塞进槽位 |
| 收束质量 | 四链是否汇成同一组 branch-owned canonical | 输出像可写回 JSON 的字段块，而不是模块拼贴 |

## 一票否决

- 只剩抽象情绪词，没有可拍动作或神态
- 运动表现和方位关系读不清
- 氛围完全脱离空间承载
- 为了“好看”牺牲了主冲突或主行动线
- 同一 branch 内多个槽位机械复写同一句，导致字段分工失效
- 输出像第二份正文，而不是 branch-owned patch

## 评分锚点

- `可交付`：人物、动作、氛围都基本成立，四个 canonical 字段都可直接被父层 assembly。
- `有明显收益`：父层能明显感到比 grouped script 更可拍、更可演、更可感受。
- `出彩`：主行动线、人物压力、空间气候和视觉强化被压进同一组 canonical 字段，写回 `分镜明细[]` 后仍然清晰有力。
