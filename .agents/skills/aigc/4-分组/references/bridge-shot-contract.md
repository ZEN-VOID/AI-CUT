# Bridge Shot Contract

本文件定义 `入场画面：` 与 `出场画面：` 的设计规则。

## Purpose

分镜组之间如果只用原剧本内容直接生成视频，上一组结尾容易出现刹车感。补位画面用于在组尾和下一组组首之间建立视觉惯性：它是短镜头、尾帧/首帧垫片，不是新增剧情段落。

## Mandatory Shape

| field | requirement |
| --- | --- |
| `入场画面：` | 从第 2 组开始出现实质内容；每集第 1 组直接省略 `入场画面：` 段，不写 `无` |
| `出场画面：` | 每组都必须写实质内容，除非用户明确允许最后一组无出场补位 |
| duration | 1-2 秒 |
| dialogue | 一般为非对白式；只有用户明确要求或原声画强绑定时才可出现极短声音描述 |
| continuity | 第 i 组 `出场画面` 必须与第 i+1 组 `入场画面` 是同一画面内容 |
| truth boundary | 不新增剧情事实、不新增关键道具、不新增未出场角色、不改写原剧本 |

## Bridge Pair Rule

相邻两组的桥接必须成对设计：

```text
组 i
出场画面：
<桥接画面 A>

组 i+1
入场画面：
<桥接画面 A>
```

文本应尽量完全一致。若为了语法衔接做轻微称谓变化，必须保证画面内容完全一致，并在执行报告中说明。

## Connection Criteria

桥接画面必须同时满足：

- 能自然接上本组最后一个原尾帧。
- 能自然接入下一组第一个原首帧。
- 画面短、可拍、低信息，不抢正文戏剧任务。
- 具备视觉或声音惯性，例如共同物件、光线延续、运动方向、焦点停留、声音余波、构图形态或环境压力。

## Good Bridge Sources

- 灯光：灯管轻颤、光斑滑动、过曝残影、阴影压下。
- 环境结构：门缝、窗帘、走廊线、课桌边缘、黑板中轴。
- 道具：红珠反光、课本扉页、粉笔字、手机屏幕、教鞭末端。
- 身体：未落定的视线、停住的手指、群像同步转头后的静止。
- 声音：脚步声余波、广播底噪、日光灯嗡鸣、翻页声的尾音。

## Anti-Patterns

- 写成长动作段落，超过 2 秒或包含新剧情。
- 写成对白、旁白解释或心理独白。
- 为了连接而加入上游没有的新人物、新道具或新规则。
- 只写抽象气氛词，例如“气氛更加紧张”，没有可见画面。
- 每集第 1 组输出 `入场画面：无`，无意义占用字数。
- 上一组出场与下一组入场画面不一致，导致下游视频仍然断裂。
