# Transition Design Contract

本文件定义 `3-摄影` 中“有明确画面动机的转场”的判定与执行边界。转场不是独立炫技项，而是镜头连续性、节奏变化、场景切换和下游视频可执行性的交接机制。

## Core Rule

场景变化固定属于“有明确画面动机时才使用转场”的范畴。只要上游从一个场景、空间、时间段、叙事段落或明确观看对象切换到另一个场景、空间、时间段、叙事段落或观看对象，`3-摄影` 必须在内部形成 `transition_profile`，并决定是显式写出转场、使用低强度桥接，还是用普通切镜保持克制。

转场不等于必须高能。场景变化时至少要处理“上一画面如何交出、下一画面从哪里进入”；只有在形态、声音、动作方向、光色、文字、空间压力或信息显影存在可执行接口时，才显式写成转场特效。

## Motivated Transition Triggers

以下情况视为可用转场动机：

| trigger | required evidence | preferred strategy |
| --- | --- | --- |
| 场景变化 | 场景标题、空间、时间段或叙事段落发生切换 | 建立镜头、声画桥、光色桥、普通切镜或匹配剪辑 |
| 空间重置 | 人物位置、轴线、门窗/走廊/教室等空间关系需要重新定位 | 建立镜头、过肩重置、主观视角、反应切 |
| 注意力转交 | 观看焦点从人物转到道具/文字/危险源/群体反应 | 焦点拉移、插入特写、反应切、动作匹配 |
| 动作方向承接 | 推门、翻页、抬眼、转身、跌落、敲击等动作可咬合下一画面 | 动作匹配剪辑、甩镜后落点、结果钉镜 |
| 声音先行 | 下一画面的铃声、脚步、敲击、广播、嗡鸣先于画面入侵 | 声音先行转场、声画桥、空镜承压 |
| 形态/颜色匹配 | 两个画面共享轮廓、色块、光斑、文字形态或运动方向 | 形态匹配剪辑、光变转场、文字显影转场 |
| 信息显影 | 规则、系统字、板书、隐藏信息从不可读变为可读 | 文字显影、焦点拉移、光变、微距转场 |
| 高点断裂 | 惊吓、惩罚、规则打断、认知翻转或奇观冲击需要节奏断裂 | 急停、光变、甩镜、反应切，但必须给清楚落点 |

## Transition Profile

每个命中转场动机的 `visual_unit` 在输出前形成内部 `transition_profile`：

| field | question |
| --- | --- |
| `transition_need` | none / implicit_cut / soft_bridge / match_cut / high_energy |
| `trigger_source` | 场景变化、空间重置、注意力转交、动作承接、声音先行、形态匹配、信息显影或高点断裂 |
| `outgoing_anchor` | 上一镜交出的可见主体、动作、声音、光色、文字、空间出口或反应 |
| `incoming_anchor` | 下一镜进入的主体、空间、声音、光色、文字、动作或反应 |
| `continuity_risk` | 是否存在跳轴、跳色、景别断崖、空间迷失或表演停顿被破坏 |
| `explicitness` | 是否需要在 `分镜N` 明写转场；普通场景切换可只写建立入口或声画桥 |

## Decision Rules

1. 场景变化默认触发 `transition_profile`，不得直接让下一场景凭空开始。
2. 低信息场景变化优先使用 `implicit_cut` 或 `soft_bridge`，用清楚的建立入口完成交接，不强行高能。
3. 同一场景内部的普通动作承接不必显式写转场，除非动作、声音、光色或信息发生切换。
4. 表演停顿、规则读秒、道具可读性优先于炫技转场；若转场会抢走表演或信息，必须收敛。
5. 高能转场必须具备至少一个可执行接口：形态、动作、声音、光线、颜色、文字或明确空间出口。
6. `分镜N` 中出现转场时，必须能反推出上一镜的交出点和下一镜的进入点。
7. 不得使用“丝滑转场”“高级转场”“电影感转场”等空泛词替代具体接口。

## Output Guidance

| weak/random | functional |
| --- | --- |
| `用高级转场进入下一场。` | `铃声先压进当前画面，镜头停在学生僵住的手上，下一镜从走廊顶灯的冷白闪烁进入。` |
| `画面丝滑切到教室。` | `上一镜的白光吞掉门缝，切到教室日光灯同样的冷白亮面，黑板边缘先入画。` |
| `快速转场表现紧张。` | `教鞭点桌的动作成为切点，下一镜直接落在粉笔灰震落的黑板字上。` |

## Review Gate

转场合格必须同时满足：

- 场景变化、空间重置或注意力转交已被识别，并有清楚交出点/进入点。
- 转场强度与 `rhythm_profile` 匹配；低信息不硬高能，高点不被压平。
- 转场接口具体可执行，能被 `4-分组`、`6-图像` 和 `7-视频` 继承。
- 没有为了炫技破坏演员停顿、文字可读、空间方向或项目视觉母题。
