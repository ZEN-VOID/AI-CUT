# 3-Detail Template Field Guide

## Root-Level Fields

### `meta`

- `剧名`：项目 canonical 名称。
- `集数`：当前 episode id。
- `组数`：当前集命中分镜组总数。
- `总时长`：当前集所有 group 的总时长。

## Group-Level Fields

### `global`

- `全局风格`：写项目级稳定画面母体和整体调性。
- `类型元素`：写当前组的类型抓手。
- `导演意图`：写这组戏到底要观众看到什么。

### `detail.分镜数`

- 不是越多越好，也不是越少越高级。
- 判断标准是：剧情动作、视线关系、信息转折和时间推进是否真的需要切镜。

## Shot-Level Fields

### `时间`

- 使用 `0-3秒` 这类直接时间窗。
- 前后镜必须连得上。

### `剧本正文`

- 每镜都保留当前镜真正承接的正文，不再额外维护桥接字段。
- 不要整组复制，也不要写成摘要口号。

### `主体锚定`

- `场景`：当前镜的空间锚点。
- `角色`：当前镜最关键的角色实体。
- `道具`：当前镜必须被看见的物件锚点。

### `分镜构图`

- `构图形式`：画面组织方法。
- `景别景深`：距离和层级感。
- `镜头类型`：当前镜的观看任务。

### `运镜手法`

- `变化`：运动方式。
- `速度`：快慢与节奏。
- `组合`：与人物动作和构图如何配合。

### `角色表现`

- `动作戏`：角色此镜最可见的行为和表演动作。
- `对话戏`：对白攻守、失口、抢话、吞话等。
- `内心戏`：能被观众从身体或神态读到的隐性波动。

### `氛围表现`

- `层次`：前景/中景/后景、内外、远近的环境层。
- `空间诗学`：空间如何参与叙事。
- `意境`：不是抽象赞美，而是由空间、光、声、材质共同形成的气息。

### `摄影表现`

- `光影`：光从哪里来，压什么，露什么。
- `色彩`：不是报色名，而是写色调关系和叙事作用。
- `质感`：空气、皮肤、物件、材质怎么被看见。

### `转场特效`

- `特效`：只有必要时写。
- `组内`：镜内到镜内如何接。
- `组间`：这一组如何把下一组挂起来。
