# 类型元素思维链与字段主表

本文件承载 `类型元素` 的字段主表、可见思维快照、工具后反思与返工入口真源。

## 模式与对象

| 项目 | 当前合同 |
| --- | --- |
| 默认模式 | `思维链优化设计`；若整包重建可短暂降级为 `原生思维链设计`，但最终仍需回到本文件接口 |
| 目标对象 | 项目级类型协议真源，服务 `projects/<项目名>/编导/第N集.json` 中 `分镜组列表[].组间设计.类型元素` 字段 |
| 默认模型策略 | 若模型具备 reasoning / thinking 能力，优先使用高层裁决 + 可见快照 + Gate；普通模型仅保留少量显式步骤支架 |
| 可见性边界 | 只保留可审计、可回写、可返工的类型判断快照；不要求外显完整内部推理 |
| 工具 / 证据依赖 | 强依赖完整 `第N集.json` 中已稳定的 `全局风格` 字段、项目初始化结论、规划阶段结构结果 |
| 受保护接口 | 不破坏 `FIELD-TE-01` 到 `FIELD-TE-06` 与六区块主输出；不越权写成风格母体或按集承载的逐组导演设计 |

## Think-Think Design Snapshot

### 启发式工作链

- `不可删性`: 删掉这轮判断后，项目会失去“这到底是什么片、该先给观众什么体感、后续部门该怎么执行”的统一类型协议。
- `最先推动`: 默认先推动 `FIELD-TE-01` 与 `FIELD-TE-02`；若主副秩序与观众合同未锁住，禁止提前美化 `FIELD-TE-03` 到 `FIELD-TE-06`。
- `先砍什么`: 先砍题材标签堆砌、市场黑话、平均分配式混搭、只会形容质感却没有导演后果的空话。
- `真正比较尺`: 比较的不是哪个类型词更好听，而是谁最能同时锁住故事驱动力、观众预期管理、世界成立性与下游执行稳定度。
- `立场纠偏`: 当前判断默认站在 `导演意图 / 3-明细 / 部门执行者` 的消费立场，而不是站在项目包装文案的立场。
- `工具后反思`: 读取 `全局风格 / 初始化 / 规划结果` 后，先问世界与类型是否互相支撑、是否存在上游冲突、是否要先回上游补件，再继续写类型协议。
- `落盘门`: 每条保留判断都必须落到 `FIELD-TE-01` 到 `FIELD-TE-06`、`Gate Summary`、`Fail Code` 或 `Rework Entry` 之一；落不到就删。

### 三轴三重裁决

| 裁决角色 | 在 `类型元素` 的具体问题 | 主要落点 |
| --- | --- | --- |
| `方向轴` | 项目首先应该被当成什么类型片观看，观众第一时间该拿到什么情绪与期待管理 | `FIELD-TE-01`、`FIELD-TE-02` |
| `成立轴` | 这个主副类型秩序是否被故事驱动力、世界规则、关系方式与情绪承诺共同支撑 | `FIELD-TE-01`、`FIELD-TE-03`、`FIELD-TE-04` |
| `优选轴` | 在多个成立解中，哪一个最能降低下游漂移、最容易翻译成放大/克制与 handoff | `FIELD-TE-03`、`FIELD-TE-04`、`FIELD-TE-06` |
| `硬门禁轴` | 是否出现平均混搭、世界与类型互撞、无误配禁区、无下游导演后果等强制否决条件 | `FAIL-TE-01` 到 `FAIL-TE-06` |

| 收敛层 | 本层先裁什么 | 服务 `field_id` | 本层收窄了什么 | 正确产物 |
| --- | --- | --- | --- | --- |
| `粗裁决` | 这部项目首先是什么片，副类型如何从属，观众先拿到什么体感 | `FIELD-TE-01`、`FIELD-TE-02` | 收窄掉题材标签堆、平均混搭、没有观看体感的空类型名 | 唯一 `主副类型裁决 + 观众合同` |
| `细裁决` | 该类型会怎样改变表达与执行，哪些元素必须放大、哪些必须克制、最怕混入什么污染 | `FIELD-TE-03`、`FIELD-TE-04`、`FIELD-TE-05` | 收窄掉只会说氛围、不含导演后果、没有污染防护的伪类型协议 | `类型承诺 + 放大与克制 + 误配禁区` |
| `离散裁决` | 哪些类型判断必须交给 `导演意图 / 3-明细` 继续继承，哪些不能被下游稀释或改写 | `FIELD-TE-06` | 收窄掉空泛 handoff、只提醒不规定、越权代写下游内容的错误下沉 | `下游阶段指导` |

### 可见 / 隐藏分层

| 层级 | 保留内容 | 去向 |
| --- | --- | --- |
| `可见快照层` | `主副类型裁决`、`观众合同`、`类型承诺`、`放大与克制`、`误配禁区`、`下游阶段指导` | 进入目标文档与字段表 |
| `Gate Summary 层` | 当前阻塞、主导失败码、返工入口、confidence、unknowns | 进入验收记录或 sidecar |
| `隐藏推理层` | 被淘汰的备选类型组合、未采用的混搭解释、过程中的比较废案 | 仅留在内部推理，不进可见合同 |

## 工具后反思与 Gate Summary

### 工具后反思

| 触发节点 | 必问反思 | 继续条件 | 不满足时回退 |
| --- | --- | --- | --- |
| 读取 `全局风格 / 初始化 / 规划结果` 后 | 世界母体、故事驱动力、关系方式与类型判断是否同向；若冲突，冲突来自哪个上游真源 | 能明确回答“这片首先是什么、观众先拿到什么、为什么不是相邻类型” | 回上游补件或暂停写作 |
| 完成 `主副类型裁决 + 观众合同` 初稿后 | 主副秩序是否唯一；是否已经排除至少一种错误混搭或错误观看契约 | 不再出现“什么都有一点”的平均分配 | 回 `FIELD-TE-01` / `FIELD-TE-02` 重写 |
| 准备写 `下游阶段指导` 前 | 下游是否能直接继承类型导演后果，而不是重新发明一套风格语言 | `导演意图` 可提炼 `主题表达 / 情感基调 / 叙事视点 / 视听法则 / 表演尺度`，`3-明细` 可继续放大 | 回 `FIELD-TE-03` 到 `FIELD-TE-06` 收束 |

### Gate Summary Contract

| 项目 | 必须回答 |
| --- | --- |
| `status` | `green / yellow / red` 三选一，不能省略 |
| `dominant_fail_code` | 当前最主要的失败码或 `none` |
| `rework_entry` | 若失败，必须指回 `S1` 到 `S6` |
| `confidence` | `0-1` 浮点值，说明当前类型协议稳定度 |
| `unknowns` | 尚未被上游真源解决的缺口，最多 3 条 |

## Field Master

| field_id | 输出位置/字段 | 内容要求 | 默认责任 Step | 质量维度 | 失败码 |
| --- | --- | --- | --- | --- | --- |
| FIELD-TE-01 | `主副类型裁决` | 锁定本项目首先属于哪种类型、谁主谁副、混合秩序如何成立；必须排除至少一种错误混搭，不得平均分配 | S1 | 类型清晰度 | FAIL-TE-01 |
| FIELD-TE-02 | `观众合同` | 明确观众先拿到的体感、观看承诺与预期管理；要能说清“先被什么吸住、而不该先得到什么” | S2 | 承诺准确性 | FAIL-TE-02 |
| FIELD-TE-03 | `类型承诺` | 把类型判断翻译成项目级导演后果，说明本片应如何被执行、被推进、被观看 | S3 | 执行导向性 | FAIL-TE-03 |
| FIELD-TE-04 | `放大与克制` | 写清哪些元素必须被持续放大、哪些表达必须被克制，以及取舍依据 | S4 | 指导可执行性 | FAIL-TE-04 |
| FIELD-TE-05 | `误配禁区` | 列出最怕混入的错误寄生、题材污染与错误口气，至少覆盖一种高风险误配 | S5 | 防污染能力 | FAIL-TE-05 |
| FIELD-TE-06 | `下游阶段指导` | 给 `导演意图 / 3-明细` 的 handoff，明确必须继承的类型后果、继续放大方向与禁止稀释项 | S6 | 交接完整性 | FAIL-TE-06 |

## Thought Pass Map

| step_id | 聚焦字段 | 核心问题 | 生成动作 | 未达标信号 |
| --- | --- | --- | --- | --- |
| S1 | FIELD-TE-01 | 这项目首先到底是什么片，副类型怎样从属才不打架 | 锁主副类型，并排除至少一种错误混搭 | 只写题材标签堆，或主副平均分配 |
| S2 | FIELD-TE-02 | 观众最先应该拿到什么体感，什么不该先进入 | 写观众合同、观看承诺与错误体感排除项 | 只有类型名，没有体感或预期管理 |
| S3 | FIELD-TE-03 | 类型如何改变项目级导演执行，而不是停在概念层 | 写类型承诺并翻译成导演后果 | 仍停在概念词、宣传词层 |
| S4 | FIELD-TE-04 | 该放大与克制什么，取舍依据是什么 | 写阶段性取舍与执行偏向 | 无法指导下游继续继承 |
| S5 | FIELD-TE-05 | 最怕混进什么污染，一旦混入会破坏什么 | 写误配禁区与破坏后果 | 没有误配清单，或只写模糊提醒 |
| S6 | FIELD-TE-06 | 如何把类型判断交给后续阶段继续放大且不被稀释 | 写 stage handoff 与继承重点 | 无下游指令，或 handoff 只有空话 |

## Validation Matrix

| 检查项 | 通过标准 | 失败信号 |
| --- | --- | --- |
| 上游一致性检查 | 已读取并对齐 `全局风格 / 初始化 / 规划结果` 三类上游真源 | 缺任一上游仍继续写正文 |
| 主副类型唯一化 | 能明确回答“首先是什么片、为什么不是相邻类型、谁主谁副” | 类型标签并列陈列，没有主从 |
| 平均混搭压测 | 把主副顺序互换后，文档会明显失配 | 对调后仍几乎成立 |
| 导演后果追踪 | `类型承诺` 与 `放大/克制` 能被翻译成具体执行倾向 | 全文只剩类型形容词，没有导演后果 |
| 误配禁区检查 | 至少列出一种高风险污染及其破坏机制 | `误配禁区` 缺失，或只有“注意别跑偏” |
| 下游 handoff 测试 | `导演意图 / 3-明细` 可直接抽取继承重点与继续放大方向 | 下游仍需重新发明风格语言 |
| 可见 / 隐藏分层检查 | 可见层只保留类型协议快照，不混入废案思路历程 | 正文开始讲大量被放弃的类型比较过程 |
| 工具后反思检查 | 每个关键上游读取节点后，都发生过“是否继续 / 是否回退 / 是否补件”的二次判断 | 上游结果一回来就机械继续 |
