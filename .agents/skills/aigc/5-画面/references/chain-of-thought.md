# aigc 5-画面 / Chain Of Thought

本文件承载 `aigc 5-画面` 的最新思维链合同真源，负责把阶段级判断固定为 `既有文件归属 -> 唯一画面类型路由 -> prompt 组合与一致性 -> 字段落盘 -> Gate Summary` 的可执行链路。

## 模式与任务对象

| 模式 | 触发条件 | 在 `5-画面` 的默认用法 | 主要消费者 |
| --- | --- | --- | --- |
| `思维链优化设计` | 现有父级思维链只剩字段表、步骤表，无法承载真实路由压力 | 默认模式；优先修父级阶段判断、唯一路由、prompt 组合与阶段落盘合同 | `5-画面` 根技能、三个子路径、阶段验收者 |
| `原生思维链设计` | 某个新阶段或新父技能还没有稳定的阶段级思维链真源 | 用于从零建立阶段级裁决骨架 | 新建父技能或新阶段模块 |
| `推理模型适配设计` | 执行主体迁移到 reasoning / adaptive thinking 模型后，旧链条明显过度脚本化 | 把“先做哪个画面产物”改写为“先裁什么、看什么证据、何时停手” | 使用推理模型执行 `5-画面` 的代理 |

硬规则：

1. 父级 `5-画面` 的默认模式是 `思维链优化设计`，除非该模块完全缺失。
2. 父级思维链必须优先服务“既有文件归属 + 唯一路由 + prompt 组合/一致性 + validation-report 闭环”四个总判断，而不是直接下沉到 leaf 的局部生成细节。
3. 若当前任务无法回答“为什么是这个子路径、为什么不是其他 sibling、为什么这些文件足够支撑当前画面”，视为父级思维链未成立。

## Think-Think Design Snapshot

### 启发式工作链

- `不可删性`：删掉父级思维链后，执行者会失去“先锁既有文件、再判唯一子路径、再组合 prompt 与一致性、最后回写阶段验收”的总判断能力。
- `最先推动`：本轮最先要推动的不是具体画面词，而是当前任务是否真的属于 `5-画面`，以及它更该服务 `分镜故事板`、`分镜帧` 还是 `漫画`。
- `先砍什么`：先砍掉“无锚点也先做图”“跳过已有 prompt / 历史图”“三个子路径一起盲开”“把 `分镜帧` / `漫画` 当成 `分镜故事板` 下游”以及“只出图片不留追溯”的冲动。
- `真正比较尺`：比较候选子路径时，以“哪一路最符合当前消费面、最少返工、最不越权重写上游事实、最有利于一致性稳定”为主尺，而不是以“哪一路看起来更炫”做判断。
- `立场纠偏`：父级思维链必须站在阶段 orchestrator、维护者与下游消费者立场，而不是站在单次产物作者的临场发挥立场。
- `工具后反思`：读取 `编导/第N集.json`、`.agents/skills/aigc/_shared/director_episode_output.schema.json`、`4-主体/`、已有 `5-画面/` 产物、`validation-report.md` 和用户任务后，必须先反思“证据是否支持继续当前层”，再决定继续、回退还是改路由。
- `落盘门`：如果某个判断不能落到 `输入边界 / 唯一路由 / prompt 组合与一致性 / canonical landing / validation-report` 之一，它就不该留在父级思维链里。

### 三向三重自省流

| 裁决角色 | 在 `5-画面` 的具体问题 | 主要落点 |
| --- | --- | --- |
| `方向轴` | 当前最先服务的是输入归属、子路径路由，还是阶段闭环 | `FIELD-VIS-ROOT-01`、`FIELD-VIS-ROOT-02` |
| `成立轴` | 既有脚本锚点、任务目标词、组级/单帧/漫画边界与一致性锚点是否足以支撑当前路由 | `FIELD-VIS-ROOT-01`、`FIELD-VIS-ROOT-02` |
| `优选轴` | 在多个候选都可行时，哪一路最稳、最少返工、最符合默认主链与一致性要求 | `FIELD-VIS-ROOT-02`、`FIELD-VIS-ROOT-04` |
| `硬门禁轴` | 是否存在无锚点、路径漂移、互斥需求未裁决、只出图不留追溯或 sibling 关系误判等强制否决条件 | `FIELD-VIS-ROOT-01`、`FIELD-VIS-ROOT-03`、`FIELD-VIS-ROOT-04` |

| 收敛层 | 本层先裁什么 | 绝不带进下一层什么 | 正确产物 |
| --- | --- | --- | --- |
| `粗裁决 / Objective Framing` | 当前这轮要解决的是哪一个阶段级问题，服务哪个下游消费者 | “先做点分镜再说”的扩写冲动 | 阶段终局句、输入边界、主要未知项 |
| `细裁决 / Candidate Narrowing` | 哪个子路径应成为本轮唯一主入口，哪些 sibling 应被排除 | “三个子路径都可以一起做”的混开冲动 | 唯一路由裁决、排除理由、默认主链偏离说明 |
| `离散裁决 / Visible Landing` | 哪些内容进入可见快照，哪些进入阶段验收与返工入口 | 没有落盘位置的抽象判断 | 阶段写位、验收状态、下一入口、返工入口 |

层内自省问题：

- `为什么这轮是 5-画面，而不是还停留在 3-明细 或已经进入 6-视频？`
- `为什么是这个子路径，不是相邻 sibling？`
- `如果回到默认主入口更稳，当前是否有足够证据偏离默认路由？`
- `当前 prompt 组合与一致性锚点是否真的来自既有文件，而不是临时凭空发明？`
- `删掉这轮保留项，下游会失去哪一种可续跑能力？`

## 工具后反思与 Gate Summary

### 工具后反思

当父级判断依赖文件读回、上游状态或中间验收结果时，必须执行以下反思顺序：

1. 先判断 `projects/<项目名>/编导/第N集.json` 是否已具备合法 `分镜组列表[] / 分镜明细[]`，且可被 `.agents/skills/aigc/_shared/director_episode_output.schema.json` 解释。
2. 再判断用户要的是组级 storyboard、单帧锚点还是漫画页改编，是否足以支持唯一路由。
3. 检查 `4-主体/`、已有参考图与历史出图是否足以形成一致性锚点；不足时只允许弱一致性执行。
4. 若 `validation-report.md`、manifest 或历史 prompt 与当前路由冲突，先回到路由裁决重判，而不是继续往下游叶子层硬写。

### Gate Summary Contract

| gate_state | 触发条件 | 动作 |
| --- | --- | --- |
| `PASS` | 上游锚点已锁、唯一路由明确、prompt 与一致性可解释、阶段落点稳定 | 允许进入命中子路径或给出唯一下一入口 |
| `WARN` | 路由暂时可判，但一致性锚点不足、需求模糊或已有验收结果要求补证据 | 先补证据或显式保守路由，再进入下游 |
| `FAIL` | 无合法 `分镜组列表[] / 分镜明细[]`、互斥需求未裁决、路径沿用旧仓、只出图不留追溯，或把 sibling 误判成下游链 | 立即停止进入叶子层，先修父级阶段合同 |

推荐评测动作：

- `唯一路由快检`：不看子路径名，只看证据是否能推出“只能先做这一层”。
- `锚点就绪检查`：删掉 `编导/第N集.json` 的 `分镜组列表[] / 分镜明细[]` 证据后，当前阶段是否应立即回退。
- `一致性快检`：删掉 `4-主体`、参考图与历史图后，是否还能解释本轮为什么这样做 prompt 组合。
- `sibling 关系检查`：是否仍明确 `分镜故事板` 是默认主链，`分镜帧 / 漫画` 是同源分支。
- `落点追踪`：所有判断能否回到 `projects/<项目名>/画面/` 与 `validation-report.md`。
