# Chapter Rhythm Handoff Contract

`2-卷章/3-章级` 与 `3-初稿` 的 `Step 2 / 2-节奏优化` 共享这一份章级节奏移交合同。

## Ownership Split

### Planning Owns

- 章级爽点设计对 `payoff_type / micro_payoff` 的前置裁决
- `selected_pack`
- `selected_mode`
- 七步职责映射
- `mode_selection_reason`
- `payoff_type`
- `rhythm_intensity`
- `previous_next_contrast`
- 本章 `entry_promise / conflict_axis / micro_payoff / exit_hook`
- 哪些段位属于必须兑现的规划义务
- 哪些段位只是建议写法、允许 drafting 在不破义务时重排

### Drafting Step 2 Owns

- 把已锁定 handoff 兑现成正文里的 `pulse_ladder`
- 段落并拆、局部前后重排、`reaction bridge`、`middle build`
- 在不破坏 planning 义务与设定物理的前提下强化读感起伏
- `rhythm_type` 写回与最终合规审计

## Shared Base Spine

章级统一七步骨架固定为：

1. `入场`
2. `推动`
3. `转折`
4. `发展`
5. `升级`
6. `高潮`
7. `尾钩`

默认节奏包固定为：

- `动静结合`
  - `势能式`
  - `动能式`
  - `浪能式`

## Required Handoff Slots

`2-卷章/第N卷/第N章.md` 的 `本章节奏曲线` 段必须至少包含以下槽位：

- `selected_pack`
- `selected_mode`
- `mode_selection_reason`
- `payoff_type`
- `rhythm_intensity`
- `previous_next_contrast`
- `七步职责映射`
- `规划义务`
  - `entry_promise`
  - `conflict_axis`
  - `micro_payoff`
  - `exit_hook`
- `义务段位`
- `建议写法`
- `Mermaid` 节奏图

## Slot Semantics

- `selected_pack`
  - 说明本章调用的节奏包；当前默认值为 `动静结合`
- `selected_mode`
  - 说明本章当前 mode；合法值固定为 `势能式`、`动能式` 或 `浪能式`
- `mode_selection_reason`
  - 用 1-3 条 planning 证据说明为什么选这个 mode；证据必须来自本章冲突、任务状态、信息压力、情绪调性、旅程/游玩状态或章末牵引，不得只写“节奏需要”
- `payoff_type`
  - 说明读者读完本章获得哪种主要满足；合法值为 `认知 / 行动 / 情绪 / 关系 / 世界感 / 软线索 / 状态修复`
- `rhythm_intensity`
  - 说明本章节奏强度；合法值为 `低 / 中 / 高`，用于控制本章在连续章节波形中的发力程度
- `previous_next_contrast`
  - 说明本章如何承接上一章节奏，以及为下一章预留怎样的节奏变化；必须同时包含“承接上一章”和“预留下一章”
- `七步职责映射`
  - planning 负责回答每一步在本章究竟承担什么功能，而不是只写标签
- `规划义务`
  - 是 drafting 不得静默删失的结构义务
- `义务段位`
  - 只负责锁“必须出现什么 / 至少要完成什么”，不锁死具体段落长度与文面顺序
- `建议写法`
  - 只提供推荐落法、节奏取向与编排提示；drafing 可在合法边界内重排

## Non-Drift Rules

1. planning 锁的是义务与 mode，不是 Step 1 当前写法的逐段排版。
2. drafting 不得重新发明第二套 `selected_pack / selected_mode / mode_selection_reason / payoff_type / rhythm_intensity / previous_next_contrast / 七步职责映射`。
3. 若正文要偏离已锁 handoff，必须回源到 `2-卷章/3-章级` 修 planning，而不是在 drafting 的 Step 2 静默改法律。
4. `micro_payoff` 不等于必须大高潮；只要局面真实改变，就可视为有效兑现。
5. `micro_payoff` 是章级爽点设计中的 `delivery_action / satisfaction_delta` 被压缩到节奏义务后的结果；不得绕过爽点设计另行发明。
6. `exit_hook` 必须从本章现有压力自然长出，不得靠生硬断章伪造牵引。

## Chapter Wave Quality Rules

### `payoff_type`

`payoff_type` 用于裁定本章真正交付给读者的满足类型，只允许选择主导项；若存在副项，可在 `mode_selection_reason` 中补充，不得把所有类型堆成列表。

| payoff_type | 使用场景 | 常见匹配 |
| --- | --- | --- |
| `认知` | 读者或角色知道了关键新事实、误判被修正、谜面变清楚 | 多见于 `势能式` |
| `行动` | 任务推进、对抗结果、资源得失、战场态势改变 | 多见于 `动能式` |
| `情绪` | 开心、舒缓、酸楚、释然、治愈或情绪释放 | 多见于 `浪能式`，也可用于低强度势能 |
| `关系` | 角色距离变化、信任变化、暧昧/友情/同盟升温或破裂 | `浪能式` 与 `势能式` 都可承接 |
| `世界感` | 新地点、新规则、新风俗、新生活质感被打开 | 常见于 `浪能式` 或探索型章节 |
| `软线索` | 不靠硬揭谜，而是在日常、游玩、对话、景物中自然落下信息 | 常见于 `浪能式` |
| `状态修复` | 角色休整、疗愈、恢复行动力、从高压中重新获得呼吸 | 常见于 `浪能式` 或低强度过渡章 |

### `rhythm_intensity`

`rhythm_intensity` 描述本章在连续章节中的发力程度，不等于篇幅长短。

- `低`：轻波、过渡、恢复、日常、柔和蓄势；必须仍有微兑现，不能空转。
- `中`：标准章强度；有明确推进、转折或关系/信息变化。
- `高`：强转折、强行动、强压迫、强情绪或阶段性爆点；不得连续滥用，否则削弱高点。

### `previous_next_contrast`

每章必须说明它处在连续波形中的位置：

- `承接上一章`：说明它是接着升压、转行动、松弛换气、余波消化，还是反向变奏。
- `预留下一章`：说明它把下一章推向爆发、行动、舒缓、揭示、关系推进或新地点。

好小说的章感来自连续波形，而不是每章都把七步写满；本字段用于防止章章同构。

## Mode Selection Rules

### `势能式`

用于本章主要任务是蓄压、逼近、显影或收窄选择，而不是立刻完成外部行动结果。

选择信号：

- 冲突核心来自信息差、心理压力、关系张力、环境压迫或威胁逼近。
- 章节推进重点是发现异常、确认危险、暴露代价、缩小退路或让下一章具备爆发条件。
- `micro_payoff` 偏向认知变化、风险确认、局面变重或角色选择被迫收窄。
- `exit_hook` 指向即将爆发的下一步压力，而不是靠突兀断章制造悬念。

七步偏向：

- `入场` 建立不安、目标或隐性压力。
- `推动` 让角色靠近压力源。
- `转折` 打破原判断或暴露异常。
- `发展` 扩大信息差、代价或关系张力。
- `升级` 让隐藏风险浮出。
- `高潮` 完成危险确认、认知翻转或艰难判断。
- `尾钩` 让更大的爆点自然逼近。

### `动能式`

用于本章主要任务是行动执行、外部对抗、任务突破或局面即时改写。

选择信号：

- 冲突核心来自追逐、交锋、谈判、夺取、逃脱、突破、拦截或现场阻碍。
- 章节推进重点是角色做事、对手反制、策略切换和结果落地。
- `micro_payoff` 偏向拿到、失去、击退、暴露、失败、完成阶段任务或战场态势改变。
- `exit_hook` 来自行动后果引出的新麻烦，而不是单纯氛围加压。

七步偏向：

- `入场` 明确行动目标或现场限制。
- `推动` 立即执行。
- `转折` 遇到阻碍、反制或意外变量。
- `发展` 调整策略继续推进。
- `升级` 提高代价、暴露风险或扩大冲突。
- `高潮` 完成正面碰撞、关键动作或决定性取舍。
- `尾钩` 让行动结果引出新问题。

### `浪能式`

用于本章主要任务是松弛、游玩、旅行、休整、轻松互动、愉快探索或舒缓过渡，而不是蓄压或强行动推进。

选择信号：

- 冲突核心很轻，章节主要提供轻松、愉快、欢乐、松弛、舒缓、旅游、玩耍或悠哉悠哉的体验。
- 章节推进重点是角色关系升温、世界感展示、情绪恢复、生活质感、旅途见闻、轻信息顺手获得或小目标轻盈完成。
- `micro_payoff` 偏向心情变好、关系更亲近、获得休息、发现趣味、打开新地点、软线索自然浮现或角色状态被修复。
- `exit_hook` 保持轻牵引，可以是下一处风景、轻微好奇、温柔期待、小麻烦露头或舒缓后的新方向，不靠硬断章制造危机。

七步偏向：

- `入场` 建立轻松场域、旅途目标、游玩约定或休整理由。
- `推动` 让角色进入体验，开始逛、玩、吃、聊、看、试或慢行。
- `转折` 出现小惊喜、小误会、小发现或轻微偏航。
- `发展` 扩展趣味、关系互动、风景层次或生活细节。
- `升级` 让快乐、松弛、亲密、趣味或软信息更饱满，而不是突然拉高危机。
- `高潮` 完成一个轻盈 payoff：笑点、暖点、风景高点、关系小进展、状态恢复或软线索落点。
- `尾钩` 留下温和牵引：下一站、下一顿、下一件趣事、隐约线索或轻微新期待。

### Quick Gate

- 若本章主要回答“压力如何蓄起，危险如何逼近”，选 `势能式`。
- 若本章主要回答“人物如何行动，局面如何改写”，选 `动能式`。
- 若本章主要回答“角色如何松弛体验，关系/世界/心情如何轻盈展开”，选 `浪能式`。
- 若三者都有，先按章级爽点设计中的主导 `payoff_mode` 与 `delivery_action` 裁决，再用本章 `micro_payoff` 落到节奏义务：认知/压力兑现优先 `势能式`，行动/结果兑现优先 `动能式`，情绪/关系/旅途/游玩兑现优先 `浪能式`。
