# Action-First Continuity Contract

## Purpose

本共享合同定义 AIGC 影视链路中的人物动作链优先规则。它用于 `3-导演`、`4-表演`、`5-摄影` 和 `6-分组`，避免氛围、道具、反射、静物或漂亮镜头抢走人物正在发生的动作，造成上下文断裂、空间不可达或下游视频生成角度混乱。

本合同是抽象源层规则，不绑定任何具体场景、道具、角色名或示例。执行时必须从当前上游正文和本组/本镜证据中判断，不得把某个既有案例写成固定模板。

## Core Rule

每个关键场景、表演 beat、分镜明细和分镜组开端，都先锁定人物动作链，再决定环境、道具和镜头承托。

动作链的最小连续结构：

1. `entry_state`：人物从什么姿态、位置、朝向和空间关系开始。
2. `action_vector`：人物正在向哪里移动、转身、伸手、停住、退开、靠近或保持。
3. `reachable_target`：若人物触碰、取用、查看或避开对象，该对象必须在空间上可达，且方向、距离、遮挡关系清楚。
4. `interaction_or_no_interaction`：对象若没有互动、关键信息、规则/证据/危险源或必要环境功能，不得升级成焦点、特写、反射主体、声响节点或组间衔接节点。
5. `exit_state`：当前 beat 结束时人物的姿态、位置、朝向、身体接触和注意力落点，必须能成为下一 beat、下一镜或下一组的开端。

## Stage Consumption

| stage | must do |
| --- | --- |
| `3-导演` | 在戏剧实质、受控增强、氛围意境和终结画面判断中，先确定角色目标、阻碍、入场状态和动作向量；环境/道具只作承托，不替代人物行动。 |
| `4-表演` | 把心理、潜台词和沉默落成演员能做的连续身体动作；同一 beat 避免堆叠过多擦泪、抽泣、避视、摸物等动作，优先保留最能改变状态差的 1-2 个外显动作。 |
| `5-摄影` | `分镜明细` 必须承接人物姿态、运动方向、身体接触和注意力落点；道具插入镜只在通过准入或人物互动时成立，不能为了物件改角度打断动作。 |
| `6-分组` | `定场镜头` 和组间连接必须从上一组结束状态或连接件抵达首态开始，继承人物姿态、站位、朝向、身体接触和未完成动作，再展开本组空间。 |

## Admission Checks

### Prop And Environment Admission

环境和道具可以进入正文，但只能承担以下功能之一：

- 空间定位：说明人物站坐、距离、方向、进出路径或遮挡关系。
- 互动承托：人物正在触碰、移动、交接、取用、查看、避开或被该对象阻碍。
- 信息功能：对象是上游已有的关键信息、规则显影、证据、危险源、身份标记或剧情因果节点。
- 必要交代：当前场景首次需要建立环境或关键对象，且该交代直接服务后续动作。
- 状态变化：对象状态发生上游支持的变化，并改变人物下一步动作、注意力或风险。

不满足以上功能时，该对象只能作为背景轻写，不能成为独立镜头、焦点终点、反射主体、声音放大点、影子关系或组间锚点。

### Spatial Reachability

人物与对象发生互动前，当前 beat 或上一 beat 必须能读出：

- 人物在哪里，站/坐/蹲/躺/靠的状态是什么。
- 对象相对人物的前后左右、远近、高低、内外或遮挡关系。
- 人物的手、身体、视线或移动方向如何到达对象。
- 如果空间关系发生变化，变化动作必须可见，不得静默瞬移。

### Emotional Action Economy

强情绪不等于动作越多越好。一个情绪 beat 默认只保留能改变状态差、能指导演员、能被镜头承接的少数外显动作：

- 先选身体核心动作：呼吸、重心、肩颈、下颌、眼神、手部、声线、停顿。
- 再判断是否需要对象互动；若对象没有准入理由，删除对象反应。
- 避免同一 beat 连续堆叠多个低信息动作，导致表演拖慢或下游切镜碎裂。
- 如果需要多个动作，必须形成清楚起点、过程和结果，而不是并列堆砌。

## Review Questions

交付前至少追问：

- 删掉环境/道具细节后，人物动作链是否更顺？若是，删除或降级该细节。
- 人物从上一状态到当前状态是否有可见路径？是否出现坐站、朝向、距离、身体接触的无证据变化？
- 道具镜头是否有互动、信息、规则、证据、危险源或必要环境功能？若没有，是否正在抢镜头角度？
- 当前镜头、定场或连接件是否能把 `exit_state` 交给下一镜/下一组？
- 这条规则是否由当前文本证据推出，而不是从某个旧示例硬套？

## Failure Codes

| failure | symptom | repair |
| --- | --- | --- |
| `FAIL-ACTION-FIRST-01` | 氛围或物件细节抢走人物动作链，删掉后反而更连贯 | 删除、降级为背景，或改成有准入理由的互动/信息细节 |
| `FAIL-ACTION-FIRST-02` | 人物与对象空间不可达，伸手、取物、靠近或离开缺少位置关系 | 补 entry_state、方向、距离、遮挡和动作路径 |
| `FAIL-ACTION-FIRST-03` | 同一情绪 beat 动作堆叠，表演拖慢或镜头被迫碎切 | 收敛为 1-2 个状态差最强的外显动作 |
| `FAIL-ACTION-FIRST-04` | 上一镜/组 exit_state 无法成为下一镜/组 entry_state | 回看上一结束状态，补承接或明确动作变化分界 |
