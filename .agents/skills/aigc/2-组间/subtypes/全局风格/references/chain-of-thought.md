# 全局风格思维链设计台

## 设计立场

- 本模块只定义 `全局风格` 的可见思维合同，不要求外显完整内部推理全文。
- 可见层必须服务三个落点：`projects/<项目名>/编导/thinking/全局风格.md`、`统一根文件中的全局风格字段区块`、`validation-report.md`。
- 思维链的职责不是把参考词写得更漂亮，而是稳定回答：这部片整体怎样被看见、如何被观众进入、该采用什么媒介与渲染管线、项目级节奏和导演协议如何锁定、如何避免串味、如何被下游继承。
- 默认模式下，可见思维链还必须回答：`style_motherbody.style_backbone_en` 为何这样蒸馏，以及它如何再派生 `style_prompt`；显式锁定模式下则必须回答为何应保留原文而不是继续净化。

## Think-Think Design Snapshot

### 模式判定

- 默认模式：`原生思维链设计`
- 适配口径：优先使用高层目标 + 可见快照 + Gate，自由内部推理不外显成全文脚本
- 可见性边界：对外只保留项目级风格裁决快照，不保留冗长过程性自述

### 启发式工作链

- `不可删性`：删掉这轮思考，项目会失去哪条统一风格母线，导致下游各写各的？
- `最先推动`：当前最先要锁定的是 `项目风格一句话`，还是 `媒介/美学协议`，还是 `节奏/导演协议`？
- `先砍什么`：哪些参考词、作品联想、对象级细节会把项目级风格写成 moodboard 或设定表，必须先删？
- `真正比较尺`：多个风格方向都成立时，最终用什么来比较其叙事服务力、媒介成立性、继承稳定性和抗串味能力？
- `立场纠偏`：当前判断是在服务观众进入方式、项目真源，还是只是在满足文档作者的审美炫技？
- `工具后反思`：读取 `north_star`、规划结果、用户偏好后，哪些新信息足以改变媒介/美学/节奏/项目级导演协议，哪些只该进入局部备注？
- `蒸馏门`：当前内容应进入 `style_backbone_en`、`style_prompt`，还是应只保留为 `Canonical Style Lock` 证据？
- `落盘门`：本轮结论最终落到 `统一根文件中的全局风格字段区块` 哪个固定区块；若落不到，就不应保留在可见合同中。

## 三轴三重自省流

### 三轴定义

- `方向轴`：当前判断首先服务项目级风格真源，而不是某一集、某一角色、某一镜头。
- `成立轴`：当前风格结论是否由 `north_star / 规划结果 / 用户显式偏好` 共同支撑，而不是凭空拼贴参考。
- `优选轴`：多个候选风格都能成立时，优先选择更易被 `类型元素 / 导演意图 / 3-明细` 稳定继承的方案。
- `硬门禁轴`：若出现对象级越权、题材串味、空泛参考词堆积、不可回写下游，则强制否决。

### 三层收敛

| 裁决层 | 本层先裁什么 | 绝不带进下一层什么 | 正确产物形态 |
| --- | --- | --- | --- |
| `粗裁决 / Objective Framing` | 这部片整体怎样被看见，观众以什么距离进入 | 无终局的参考词清单、对象级细节冲动 | 项目级风格终局句、观演关系、主要未知项 |
| `细裁决 / Candidate Narrowing` | 哪组风格基础协议、风格母体、污染防护值得进入正式合同 | 只好看但不可继承的风格炫技、无法抗串味的混搭 | 风格基础协议、风格母体、禁区列表、比较尺 |
| `离散裁决 / Visible Landing` | 哪些内容进入 sidecar、哪些进入 `统一根文件中的全局风格字段区块`、哪些进入 Gate | 冗长过程叙述、不可审计的灵感碎片 | 可见快照、字段落点、失败码、返工入口 |

### 层内自省问题

- 为什么当前这条风格母体比其他候选更稳？
- 如果不用这条母体，下游最可能在哪个入口开始漂移？
- 删掉当前保留项，`统一根文件中的全局风格字段区块` 会失去哪条关键继承信息？
- 如果把这一层写回参考词堆或流程脚本，为什么会更差？

## 工具后反思合同

当本技能读取上游材料或外部输入后，必须先做一次工具后反思，再继续写可见产物：

1. `读取项目种子后`
   - 判断是否已经足以形成项目级风格命题。
   - 若仍只有情绪碎片，没有统一观看方式，先暂停写美学细节，回到 `项目风格一句话`。
2. `读取规划结果后`
   - 判断世界、叙事规模与观演距离是否改变了原先风格判断。
   - 若项目体量、视角结构已改变，必须重审 `观演关系` 与 `风格基础协议`。
3. `读取用户偏好后`
   - 判断偏好属于项目级母体，还是只是对象/局部灵感。
   - 若偏好与项目气质冲突，优先写入 `污染防护` 或显式说明“局部可借、整体不可继承”。

## 可见快照结构

对外可见的思维快照只允许保留以下结构：

### Global Style Design Snapshot

- `项目终局句`：一句话说明这部片整体怎样被看见。
- `叙事与世界约束`：哪些故事与世界条件在决定风格边界。
- `观演进入方式`：观众以什么距离、速度、情绪温度进入故事。
- `风格基础协议`：最终采用的媒介属性、渲染技术栈、美学范式与其叙事理由。
- `节奏与导演协议`：项目级节奏档位、信息密度基线与 `director_intent` 母合同摘要。
- `风格主裁决`：最终采用的风格母体，以及为何应蒸馏为 `style_backbone_en` 或转入 `Canonical Style Lock`。
- `污染否决点`：哪些常见误配必须被明确排除。
- `下游继承钩子`：`类型元素 / 导演意图 / 3-明细` 分别应继承什么，不该继承什么。

硬规则：

1. 快照是裁决摘要，不是全文思维实录。
2. 任何写不进上述结构的段落，默认不进入可见合同。
3. `统一根文件中的全局风格字段区块` 只承接项目级稳定结论；过程性比较说明优先留在 `projects/<项目名>/编导/thinking/全局风格.md`。

## Gate Summary 要求

- `mode`: `default / exact-lock`
- `status`: `pass / conditional-pass / fail`
- `confidence`: `0.0 - 1.0`
- `unknowns`: 仍缺哪些上游信息
- `dominant_risk`: 当前最主要的串味或漂移风险
- `next_entry`: 唯一推荐下一入口，只能是一个

## 反脚本化禁令

- 禁止把思维链写成“先想 A 再想 B 再想 C”的冗长流水账。
- 禁止把项目级风格写成参考作品罗列、镜头语言炫技或对象设定清单。
- 禁止只有字段，没有“为何这样裁”的比较尺与否决逻辑。
- 禁止把 `projects/<项目名>/编导/thinking/全局风格.md` 写成不可消费的散文日记。
