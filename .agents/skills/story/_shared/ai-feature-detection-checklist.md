# AI 腔系统化特征检测清单

本文件是 `story2026` 的 AI 写作腔系统化检测合同。它将原本依赖经验的 AI 腔判断转化为结构化、可量化、可复现的五层检测清单，供 `3-初稿/N6-AUTO-ACCEPTANCE` 和 `4-润色/P5-AUTO-ACCEPTANCE` 在内置验收中系统性检查。

本文件不拥有正文写权，不是独立阶段。

## 总览：五层检测架构

```
词法层 → 句法层 → 篇章层 → 人物层 → 叙事层
(底层)                                        (高层)
微观坏点                                   宏观坏点
```

## Detection Dimension 1: 词法层 AI 腔（Lexical AI Features）

词法层检测关注单个词汇/短语级别的 AI 腔。

### 1.1 过量因果连接词

| 检测项 | 检测方法 | 阈值 | 示例 |
|--------|----------|------|------|
| "因此"密度 | 统计全文"因此"出现次数 / 总段落数 | ≥0.3 次/段 | "他感到很害怕，因此他后退了一步。" |
| "于是"密度 | 统计全文"于是"出现次数 / 总段落数 | ≥0.25 次/段 | "门开了，于是他走了进去。" |
| "所以"密度 | 统计全文"所以"出现次数 / 总段落数 | ≥0.3 次/段 | "天很黑，所以他没有看见那个人。" |
| 连续因果句 | 检测连接词在相邻3句内重复出现 | ≥2次 | 3句内出现2次以上因果连接词 |
| 因果连接词总量 | 所有因果连接词（因此/于是/所以/因而/故此/是以/从而/故/以至于）总次数 | ≥全文句子数的 10% | — |

**修复指引**：因果连接词本身不是问题，问题在于机械使用。好的叙事中，动作序列用动词本身传达因果，不必显式连接。

### 1.2 死板的程度副词模式

| 检测项 | 检测方法 | 阈值 | 示例 |
|--------|----------|------|------|
| "XX地"状语密度 | 统计"XX地+动词"模式出现次数 / 总句子数 | ≥0.15 | "他小心翼翼地推开门，缓缓地走了进去，轻轻地关上了门。" |
| "非常/极其/特别"密度 | 统计强化副词（非常/极其/特别/十分/格外/异常）次数 / 总句子数 | ≥0.1 | "他非常愤怒，声音极其冰冷。" |
| "微微/略略/隐隐"密度 | 统计弱化副词出现次数 / 总句子数 | ≥0.08 | "她微微皱眉，嘴角隐隐抽动。" |

**修复指引**：用具体的身体动作、呼吸变化或直接描写替代程度副词。"他很愤怒" → 写他怎么愤怒。

### 1.3 死板的时间过渡词

| 检测项 | 检测方法 | 阈值 | 示例 |
|--------|----------|------|------|
| "就在这时"密度 | 统计全文出现次数 | ≥2次/章 | "就在这时，门外传来了脚步声。" |
| "突然/忽然"密度 | 统计全文出现次数 | ≥3次/千字 | "他突然意识到......她忽然转过头来。" |
| "与此同时"密度 | 统计全文出现次数 | ≥1次/章 | "与此同时，在城市的另一端......" |
| "不一会儿"密度 | 统计全文出现次数 | ≥2次/章 | "不一会儿，天就亮了。" |

**修复指引**：用场景切换自然暗示时间流逝；突然事件用视觉/听觉细节直接呈现，不必加"突然"。

### 1.4 万能连接词/填充词

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| "不过"密度 | ≥4次/千字 |
| "然而"密度 | ≥3次/千字 |
| "显然"密度 | ≥2次/千字 |
| "毫无疑问"密度 | ≥1次/千字 |
| "也就是说"密度 | ≥1次/千字 |
| "毕竟"密度 | ≥3次/千字 |

## Detection Dimension 2: 句法层 AI 腔（Syntactic AI Features）

句法层检测关注句子结构级别的 AI 腔。

### 2.1 异常完整的主谓句比例

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| 完整主谓宾句比例 | 统计主语+谓语+宾语完整句子占全部句子的比例 | ≥70% 判定为过整 |
| 省略主语句比例 | 统计合法省略主语的句子比例（中文叙事中常见） | <5% 判定为过整 |
| 非完整句比例 | 统计片段句、断裂句、独词句的比例 | <3% 判定为过整 |
| 连续完整句串 | 检测连续 ≥5 句均为完整主谓宾句 | 出现 ≥3 处 |

**修复指引**：中文叙事中，从上下文可知的主语可以省略；短促的动作句、断裂的情绪句、独词反应句是正常的叙事节奏。

### 2.2 句长分布均匀度

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| 句长标准差 | 计算全文句子长度（字符数）的标准差 | <15 字符判定为过均 |
| 短句比例（≤15字） | 统计短句占比 | <10% 判定为缺短句 |
| 长句比例（≥50字） | 统计长句占比 | <5% 判定为缺长句 |
| 极端句比例（≥80字） | 统计极长句占比 | 0 判定为缺极端句 |

**修复指引**：好的叙事需要长短交替。短句用于快节奏/紧张/情绪爆发；长句用于描述/沉思/氛围铺陈。

### 2.3 句式重复模式

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| 连续相同主语开头 | 检测连续 ≥3 句以同一主语开头 | 出现 ≥3 处 |
| 连续"他/她"开头 | 连续 ≥4 句以第三人称代词开头 | 出现 ≥2 处 |
| 段落首句重复 | 检测相邻段落首句使用相同句式结构 | 出现 ≥3 处 |
| "XX了XX"模式密度 | 统计"了"字句占比 | ≥全文句子数的 40% |

### 2.4 异常关联词/引导词

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| "并不是...而是..." | 统计全文出现次数 | ≥3次/章 |
| "不仅是...更是..." | 统计全文出现次数 | ≥2次/章 |
| "虽然...但是..." | 统计全文出现次数 | ≥3次/章 |
| "如果...那么..." | 统计全文出现次数 | ≥2次/章 |
| "只要...就..." | 统计全文出现次数 | ≥2次/章 |

## Detection Dimension 3: 篇章层 AI 腔（Discourse AI Features）

篇章层检测关注段落和整体结构级别的 AI 腔。

### 3.1 段落长度异常均匀

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| 段落长度标准差 | 计算全文章节段落长度（字符数）的标准差 | <80 字符判定为过均 |
| 连续等长段落 | 检测连续 ≥4 段长度差 ≤20% | 出现 ≥2 处 |
| 段落数量与字数比 | 总段落数 / 总字数 | <1段/200字 判定为段过长；>1段/100字 判定为段过碎 |

### 3.2 场景转换模板化

| 检测项 | 检测方法 | 示例 |
|--------|----------|------|
| "画面一转" | 搜索全文 | "画面一转，来到了......" |
| "镜头切换到" | 搜索全文 | "镜头切换到......" |
| "与此同时，在XX" | 搜索全文 | "与此同时，在城主府......" |
| "视线回到" | 搜索全文 | "视线回到主角身上。" |
| 分镜式空行分段 | 检测连续多段每段 ≤3 句 | 出现 ≥5 处连续段落 |

**修复指引**：场景转换应通过叙事自然过渡，不要用影视术语"画面/镜头/视线"做机械切换。

### 3.3 开头/结尾公式化

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| 环境开头 | 统计以环境描写/天气开头的章占比 | 连续 ≥2 章 |
| "XX感到/觉得"开头 | 统计以人物心理感受开头的章 | — |
| "夜/清晨/傍晚"结尾 | 统计以时间标志结尾的章 | — |
| "XX不知道的是"结尾 | 检测章末"XX不知道"预告句 | ≥1次/章 |

### 3.4 信息分布异常

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| 解释段比例 | 统计纯解释/说明/回忆段落占全文比例 | ≥25% |
| 动作-反应比例 | 动作描写段 / 反应描写段 | <0.5 或 >3.0（过于偏向一方） |
| 对白比例 | 对白段字数 / 总字数 | <0.1（对白太少）或 >0.7（变成剧本） |
| 描写比例 | 描写段字数 / 总字数 | >0.6（描写过量，叙事停滞） |

## Detection Dimension 4: 人物层 AI 腔（Character AI Features）

人物层检测关注人物塑造级别的 AI 腔。

### 4.1 角色共用作者口吻

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| 同质化对白长度 | 统计不同角色的平均对白长度是否相近（标准差） | <5 字判定为同质 |
| 同质化对白复杂度 | 不同角色的对白句子复杂度是否相近 | 标准差 <3 判定为同质 |
| 共享修辞模式 | 不同角色是否共享相同的修辞手法/比喻类型 | ≥2 种共享 |
| 共享认知视角 | 不同角色是否共享同一套观察世界的逻辑 | — |

**修复指引**：每个角色应有独特的声口——农民不说成语，小孩不分析局势，急性子不铺陈解释。好的对白让读者不看标签也知道谁在说话。

### 4.2 情绪标签直贴

| 检测项 | 检测方法 | 阈值 | 示例 |
|--------|----------|------|------|
| "感到+情绪词" | 统计全文出现次数 | ≥5次/章 | "他感到愤怒""她感到恐惧" |
| "心中涌起" | 统计全文出现次数 | ≥2次/章 | "心中涌起一股暖流" |
| "XX的心情" | 统计全文出现次数 | ≥2次/章 | "怀着忐忑的心情" |
| 直接情绪词密度 | 统计直接情绪词（愤怒/恐惧/喜悦/悲伤/焦虑/惊讶/厌恶/羞耻/嫉妒/绝望）总次数 | ≥15次/章 | — |
| 情绪解释句 | 检测"因为[情绪]，所以[行为]"因果模式 | ≥3次/章 | "因为他害怕，所以他的手在颤抖。" |

**修复指引**：情绪应通过行为、身体反应、对话和场景压力体现，而非直接标签。"他愤怒"可以被握拳、呼吸变重、声音变低、视线变窄、问句变短等替代。

### 4.3 脸色捷径

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| "脸(色)一白" | 统计全文出现次数 | ≥3次/章 |
| "脸(色)一红" | 统计全文出现次数 | ≥3次/章 |
| "脸色(微)变" | 统计全文出现次数 | ≥2次/章 |
| "面色XX" | 统计全文出现次数 | ≥3次/章 |
| 脸色变化总密度 | 所有脸色变化描写 / 总字数 | ≥1次/500字判定为过度 |

### 4.4 对白功能退化

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| 纯信息对白 | 统计只承载信息传递、无人物态度/潜台词的对白句占比 | ≥40% |
| "XX说"重复 | 统计"XX说/XX道"作为对话标签密度 | ≥人物数×5次/章 |
| 对白+解释 | 检测对白后紧接心理/背景解释的句对 | ≥对白句数的 30% |
| 功能性对白 | 检测纯粹为推进剧情的工具性对白（人物只是信息载体） | ≥3处 |

**修复指引**：好的对白同时承载信息、态度和人物关系。每句对白都应该让读者感知到：说话人现在什么状态、对听话人什么态度、真实想法和说出的话是否一致。

## Detection Dimension 5: 叙事层 AI 腔（Narrative AI Features）

叙事层检测关注叙事策略级别的 AI 腔。

### 5.1 解释性插入语

| 检测项 | 检测方法 | 阈值 | 示例 |
|--------|----------|------|------|
| 旁白式解释 | 检测叙事者跳出故事直接向读者解释的句子 | ≥3处/章 | "原来，这一切都是因为......" |
| 心理活动解释 | 检测"他心想/他暗想/他在心中"后接的完整逻辑分析句 | ≥3处/章 | "他在心中盘算着......" |
| 背景说明插入 | 检测在动作/对话中插入的背景说明段落 | ≥3处/章 | 打斗中插入长段能力/背景说明 |
| "要知道"引导 | 搜索"要知道/别忘了/须知"引导的解释性句子 | ≥2处/章 |

### 5.2 预判式叙述

| 检测项 | 检测方法 | 阈值 | 示例 |
|--------|----------|------|------|
| "不会想到"预判 | 搜索"XX不会想到/XX不知道/XX没想到"模式 | ≥3处/章 | "他不会想到，这将是他们最后一次见面。" |
| "殊不知" | 搜索"殊不知/却不知/哪知" | ≥2处/章 |
| "这还只是开始" | 搜索章末"这只是开始/这才刚开始/好戏还在后头" | ≥1处/章 |
| 上帝视角旁白 | 检测叙事者全知视角评论人物内心/未来 | ≥3处/章 |

### 5.3 过度流程化

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| 动作清单式叙述 | 连续 ≥4 句均以"XX了XX"模式描述动作序列 | ≥2处/章 |
| 冗余动作链 | 检测从"推门→走进→坐下→倒水→喝水→放下杯子"的完整物理动作链 | ≥3处/章 |
| 无信息量动作 | 检测不推进剧情、不塑造人物、不承载情绪的纯粹动作描写 | ≥5处/章 |
| 流程词密度 | 统计"接着/然后/随后/之后/接下来/再/便"出现密度 | ≥全文句子数的 8% |

### 5.4 万能结尾模式

| 检测项 | 检测方法 | 示例 |
|--------|----------|------|
| 落点式章末 | 检测章末"夜/晨/风/雨/光/影"落点句 | "夜色渐深""晨光微熹""风轻轻吹过" |
| 顿悟式章末 | 检测章末角色突然理解/明白/意识到的句子 | "他突然明白了一个道理。" |
| 疑问式章末 | 检测章末以疑问句结尾的"悬念制造" | "这一切，到底意味着什么？" |
| 翻转式章末 | 检测章末"XX不知道的是......""然而XX并不知道......" | — |

### 5.5 缺乏场景压力纹理

| 检测项 | 检测方法 | 阈值 |
|--------|----------|------|
| 信息驱动而非压力驱动 | 检测场景推进主要靠信息揭示而非角色压力（欲望/恐惧/关系张力）的场景占比 | ≥40% |
| 关系静态场景 | 检测人物关系在一整个场景中没有发生任何变化的场景 | ≥30% |
| 无阻碍场景 | 检测人物做成一件事从头到尾没有遇到有效阻碍的场景 | ≥2处/章 |
| 压力来源单一 | 一个场景中只有单一压力源（而非多重压力交织） | ≥60% |

## Integrated Detection Workflow

### 在 3-初稿 N6-AUTO-ACCEPTANCE 中

```
1. 加载本清单
2. 逐层检测（词法→句法→篇章→人物→叙事）
3. 每层输出：detection_report.{layer}.detected_features[]
4. 汇总：anti_ai_features_score（0-10分）
5. 将 detection_report 嵌入 stage_acceptance_packet
```

### 在 4-润色 P5-AUTO-ACCEPTANCE 中

```
1. 回读 3-初稿 验收中的 detection_report
2. 对润色稿重新执行五层检测
3. 对比 before/after 检测结果
4. 确认 AI 腔坏点是否被具体定位并修复
5. 不允许用泛化"更自然"替代具体坏点检测
```

## Scoring Rubric

| 分数 | 五层检测描述 |
|------|-------------|
| 0-2 | 三层及以上存在严重 AI 腔（多项超标），文本整体呈 AI 写作特征 |
| 3-4 | 两层有严重 AI 腔，或三层有轻度超标 |
| 5-6 | 一层有轻度超标，其余层在阈值内；基本通过但需关注 |
| 7-8 | 所有五层均在阈值内，文本无感知到的 AI 腔 |
| 9-10 | 五层均远低于阈值，句群有自然起伏，人物声口各异，叙事有"人味" |
