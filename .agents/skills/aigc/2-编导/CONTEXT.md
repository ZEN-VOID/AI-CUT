| B 路线被误用成自由新增 | 受控增强边界层 | 删除新增对白、事件、规则、线索、因果或人物动机；只保留有上游锚点的环境、反应、表演、调度、声音、道具和余波承托 | `controlled-enrichment-contract.md` 固定 B/C 分界与 `controlled_enrichment_ledger` | 每个新增项有 `source_anchor`、`target_field`、`purpose`、`risk_check` |

## Repair Playbook

1. 先锁定 `source_episode_path`，确认输出对应的上游 `第N集.md`。
2. 抽取上游 `【剧本正文】` 后正文，按原顺序建立事实承接清单。
3. 检查所有中文双引号里的对白，逐字回对上游原句；没有上游依据的新增对白必须删除或改为非 canonical 候选说明。
3.1. 检查所有 `对白` 字段标题：必须是 `对白（真实角色名，语态/状态短语）`；`原文角色`、`角色名`、`某人`、`说话者` 等只属于模板占位，不能进入终稿。第二项可以是自然短语，不强制一词或“地”字尾。
4. 检查每个声音字段后是否就近出现对应画面字段；缺失时补画面承托，不把声音文本复述到画面行。
5. 检查 `动作画面`、`角色动作` 是否可拍；心理和判断转入 `独白`、可感知 `心理反应`、可执行 `表演提示`。`心理反应` 不能成为抽象内心解释容器，必须写演员能演、观众能从画面或声音 GET 到的反应。
5.0.a. 检查关键情绪 beat 是否只写情绪标签或模板表情；若只有“愤怒/难过/紧张/害羞/开心”或皱眉、瞪眼、流泪、大笑，按 `actor-performance-control` 补触发点、情绪动机、眉眼嘴/咬肌/鼻翼、手/肩/呼吸/重心/声线、环境声和微动态限制。
5.0. 检查主角内心想法和主角视角判断：不得删掉；不得写成客观第三方概括；优先投影为 `内心独白（主角）` + `内心独白画面`。
5.0.0. 检查内心独白人称：`内心独白（主角）` 引号内若承接第三人称小说叙述，主角自指必须改为第一人称；只在第三人称指向其他角色、引用文本或刻意自我疏离并留证时保留。`内心独白画面` 不按此规则改成第一人称。
5.0.1. 检查动作客观性：`角色动作` / `动作画面` 中出现“试图、想要、打算、意图、为了掩饰、准备借此”时，删除主观意图词，改成可拍动作、神态、语气、生理反应或空间运动。
5.0.2. 检查直接情绪词：`感到恶心/难受/愤怒/害怕/崩溃` 不直接进终稿，改成微表情、肢体动作、生理反应、声线变化或主角内心独白。
5.1. 检查动作速度和力度：上游出现“一闪、骤然、缓缓、忽然、迟疑、停住、挣出、扑倒”等信号时，`角色动作` / `动作画面` 必须保留相应快慢、停顿、节奏或力度。
5.2. 检查 `道具特写`：不要把当前项目的具体物件清单写成通用规则；只写物件的可见信息、状态、痕迹、归属和变化，解释性推理转入 `心理反应` 或声画承托。
6. 建立 slugline 去重表，合并同一集内完全相同的场景标题。
7. 最后才做编剧质量 polish：强化场景目的、冲突压力、表演任务和可见承托，但不改变事实和对白。
8. 执行一次“编导创作内核 pass”：每个关键场景先问它的戏剧问题是什么、人物被迫选择什么、观众此刻比角色知道多还是少、信息该揭开还是压住、用什么动作/空间/道具/声音发动这场戏；答不出来就不能进入草稿。
9. 执行一次“高潮画面 pass”：若上游存在行动结果、认知翻转、关系暖点、规则显影、奇观、怪异落点或高超对决，至少让一个高点落入强画面字段和必要声音/沉默承托；没有真实高点时只处理最强 `micro_payoff`，不硬造大高潮。
10. 若本轮启用 subagents，先把 `team.yaml` 监制组相关智能顾问团作为编导监制请教，让其代入角色意识、创作风格和专业水准；再把当前思维·执行节点的判断、动作、证据、路由、gate 与失败回路转成顾问任务，不使用脱离节点的固定题型清单，也不问泛泛“好不好”。
11. 交付前专门执行一次“画面纯度 pass”：所有 `*画面`、`环境描写`、`道具特写`、`表演提示` 必须落到可见或演员可执行事项；抽象信息转入 `心理反应`、`独白`，或删除。
11.1. 交付前专门执行一次“占位泄露 pass”：删除所有内部规则句、模板句、任务说明和尖括号占位；这些内容不能作为剧本正文保留。
11.2. 交付前专门执行一次“环境纯度 pass”：`环境描写` 只保留地点、空间结构、自然条件、空气介质、光照状态、承载面、围护面、开口边界、静置物件等写景材料；咬、凑近、说、握住、别过头、收袖、拔刀、奔跑、灌酒等人物动作必须拆出。
11.2.1. 交付前专门执行一次“环境刷新 pass”：检查同一 slugline 内是否存在真实空间/背景/光线/空气/材质焦点变化；若有，从开篇环境描写之外追加环境刷新。不要为每个动作机械插环境；也不要因为场景标题未变而忽略从室内到门廊、从主厅到角落、从桌案到窗边、从船舱到船舷这类背景变化。
11.3. 交付前专门执行一次“景境氛围 pass”：关键情绪场、离别场、压迫场、危险场和宁静场不应只有地点点缀；在符合情境时，用飘雪、落叶、朝露、风沙、雨丝、雾气、日影、火光、尘土、潮气等自然景物承托心境。若该景物上游未明写，按 B 路线留 `controlled_enrichment_ledger`，并确认不造成新事件、线索、阻碍或因果。
11.4. 交付前专门执行一次“画面美学 pass”：关键场景先定一个核心画面，再定视觉气质、主次层级、母题变化、对比轴、节奏和留白；这些判断只能拆入 `环境描写`、`角色动作`、`对白画面`、`道具特写`、`群像画面` 或声音承托字段，不能新增字段或摄影方案。
11.5. 交付前专门执行一次“整集视觉主轴 pass”：从上游中抽出 3-5 个可回指的核心视觉母题，形成 `episode_visual_spine`；母题可以变化和呼应，但不能为了统一审美强行塞进每场。
11.6. 交付前专门执行一次“创作证据 pass”：执行报告至少覆盖 `novel_expression_transform_evidence`、关键场景的 `director_substance_evidence`、整集 `episode_visual_spine` 和若干 `visual_aesthetic_evidence.scene_items`；只有机械修复类任务可记录降级原因。
11.6.1. 交付前专门执行一次“思维·执行节点 pass”：为本轮经过的关键节点补 `thinking_action_node_ledger`，不得只写节点名或 checklist；每条记录至少说明判断问题、执行动作、证据字段、出口路由、gate 状态和 source owner。
11.6.2. 若本轮新增或显著修改学习型合同，交付前执行“学习集成验证 pass”：补 `learning_integration_review_evidence`，没有真实项目样例时标注 `static_only`，并写明残余风险和下一次生产运行观察点。
11.7. 交付前专门执行一次“小说表述转译 pass”：抽查作者评论、主角视角判断、心理内视、文学比喻、抽象概括、往日常态句、背景说明、因果解释、关系结论、感官散文和武侠/玄幻抽象是否已转成可拍声画、表演、空间、道具、群像、主角内心独白、短旁白或留白；不得改写引号内对白，不得把小说叙述变成新增台词，不得新增无关人物过往、物品来历或回忆性信息。
11.8. 交付前专门执行一次“终结画面 pass”：最后一组字段必须有迷你彩蛋尾钩；先判 `final_image_type_profile`，再选择环境描写式、道具特写式、情绪酝酿式或高潮结尾式。可读取下一集时只取关联方向，不搬运事件、台词、结果；不可读取时标注本集局部推断。
12. 交付前专门执行一次“声音本体 pass”：所有 `音效` 引号内必须是可听见的声音，不得是时间、事件或描述性概括。
13. 交付前把 review finding 当成修复输入，而不是报告附件；阻断项先在本阶段修复并复审，仍失败再阻断或回源层。
14. 对关键场景执行“状态差 pass”：从上游中标出进入状态、压力源、转折点、退出状态；如果只是过渡场，记录它承接什么信息或空间变化。
15. 对心理和潜台词执行“演员任务 pass”：把“不信任/试探/隐瞒/心虚/压迫/想保护”等标签改成角色目标、阻碍、策略和外显行为。
16. 对权力关系执行“场面调度 pass”：用站坐、高低、远近、门口/讲台/黑板占位、视线压力和道具归属表达关系，不写摄影机位、景别或分镜。
17. 对沉默和反应执行“余波 pass”：上游没有对白时，不新增对白；用呼吸、停顿、纸张声消失、手部动作、对手不接话或群体迟半拍反应承托。
17.1. 对转场和未出口对白执行“变量去重 pass”：若已使用视线转移承托潜台词，下一处优先换成道具、声音、群像、身体距离、动作中断、空间阻隔或环境刷新；不要连续写“看向远方 / 顺着视线望去”。
18. 若进入 B 路线，执行“受控增强 pass”：每个新增承托细节先写 `source_anchor` 和 `risk_check`；写不出来就删掉，不把它推进正文。
19. 交付前执行“内嵌 pass”：删除场景末尾总结式 `表演提示/场面调度`，把其中的指节、视线、道具、位置、低声、群体反应拆回对应剧本句段。

## Reusable Heuristics

- `2-编导` 的“创作”主要发生在投影方式、画面承托、场景标题和表演调度上，不发生在剧情事实重写上。
- 对白标签也是表演入口：角色名解决“谁说”，第二项解决“以什么状态说”；它应灵动、自然、鲜活，但只服务表演展示，不能改对白原文。
- 好的影视化不是把小说写短，而是把小说信息拆成可拍字段，让下游能解析、调度和表演。
- 真正的编导干货不是“写得更像剧本”，而是让每场关键戏有问题、有压力、有选择、有信息释放、有观众位置和可执行的动作/声音/空间发动机。
- references 细则不能只停在说明文档里；每条强制细则都应在 `steps/directing-workflow.md` 中找到 consumed_by 节点、planning evidence、blocking gate 和 route-back，否则执行时会退回泛泛 checklist。
- 思维·执行节点的价值在于“判断后执行、执行后留证、失败能回源层”；凡只列动作、不列判断问题、证据、路由和 gate 的节点，都还只是 checklist。
- 学习资料吸收不能只靠新增 reference；每次把学习成果晋升为合同，都要同时留下真实样例、等价 smoke 或 `static_only` 残余风险说明，避免把静态接入误判为运行稳定。
- 声音字段要像录音部门能拿走的材料，画面字段要像导演和摄影能拍出来的材料。
- 角色内心可以保留，但要转成有主体的独白、心理反应或表演提示；不要假装内心解释是可拍动作。
- 主角内心尤其要保留：主角对他人行为的判断、怀疑、误读和确认，是主观视角资产；它应进入 `内心独白（主角）`，而不是被删掉或改成全知客观结论。
- 内心独白不是旁白。把第三人称小说叙述投进 `内心独白（主角）` 时，主角自指要变成“我/我的/自己”；画面承托再用第三人称写可拍动作。
- 动作字段是镜头事实，不是动机说明书；“试图/想要/打算/意图”通常说明写作者还没有找到可拍外显，应回到手、眼、呼吸、停顿、声线、重心和道具。
- 直接情绪词是草稿信号，不是终稿画面；恶心可以是喉头、嘴角和手背，愤怒可以是指节、下颌和停顿，难受可以是呼吸和重心。
- 往日常态和人物前史容易把剧本拖回小说解释；除非当前主线必须承接，否则删掉，只保留当下能拍到的物、声、人和反应。
- slugline 是下游分组的锚点，不是剧情 beat 标题；同一个空间和时间段内只需要一个场景编号。
- 质量提升的顺序是：保真 -> 可拍 -> 声画清楚 -> 表演可执行 -> 下游可分组。
- 团队顾问的价值是“创作前照亮取舍”，不是接管剧本；最终执行口径必须同步于当前思维·执行节点，落成节点判断、执行取舍、证据补强、字段投影、表演任务、声画承托和风险禁区，并作为后续 LLM 投影、阶段内修复与复审的上下文继续使用。
- 高潮画面强化的顺序是：先找上游已有满足兑现点，再找角色锚点和状态差，最后才强化画面、声音、表演和余波；不要为了“更炸”新增事实。
- `2-编导` 不使用 `分镜明细预设` 字段；导演意图必须落到画面、动作、表演或声音字段，摄影/分镜方案交给下游阶段。
- “画面”不是解释画面的意义，而是把意义放进可见物：手、眼、门、纸、黑板、光线、座位、空位、屏幕字、身体距离。
- 小说二次画面化的关键不是把句子改得更美，而是把小说句子的功能转译出去：作者判断给空间和群体，心理给演员身体，比喻给物象和材质，概括给 beat，背景给画面证据与短旁白。
- 台词是冻结真源；二次加工只能发生在台词外的说话状态、对白画面、动作、停顿、空间距离、道具反应和环境余波里。
- `环境描写` 不是万能承接字段；它写景，人物做事进 `角色动作`，对白附近反应进 `对白画面`，群体状态进 `群像画面`。
- `环境描写` 也不是“一场一次”的字段；开篇环境建立场景，后续环境刷新承接同一场景内新的可见背景。室内到门廊、主厅到角落、桌案到窗边、船舱到船舷、雨声贴近、雾气加重、火光变暗，都可以触发新的环境描写。
- `环境描写` 也不是地点标签；优秀景境可以主动烘托心境。飘雪、落叶、朝露、风沙这类自然景物可在符合情境时补入，但只能改变氛围密度，不能改变剧情条件。
- `角色动作` / `动作画面` 不是动作清单；关键动作要有速度和力度，让演员知道是骤发、迟疑、缓慢压下、猛地扑出，还是停半拍后才动。
- `道具特写` 不是“某几个固定道具”的字段；它按功能承接信息载体、规则显影、线索痕迹、归属压力和状态变化。
- 画面美学不是多写形容词，而是先定核心画面和主次层级：观众记住什么、什么只做背景、哪里该重复成母题、哪里该留白。
- 视觉母题要变化，不要机械复读；同一扇门、同一盏灯、同一条雨线，每次出现都应改变压力、距离、归属或情绪温度。
- 整集视觉主轴不是统一滤镜，而是变化链：一个物件、一种材质、一片自然景物或一种动作节奏，在不同场景里承担不同压力，观众才会记住它。
- 终结画面不是“下一集预告”，而是迷你彩蛋尾钩：最后一个环境、道具、情绪或高点余波落点，应让观众感觉下一集已经在门外，但不能告诉观众下一集具体发生什么。
- 尾钩要先判型再写：环境描写式适合压力方向，道具特写式适合未解信息，情绪酝酿式适合关系和选择未尽，高潮结尾式适合代价与余波。
- 终结画面可以是一段尾场小段落，不必压成一两个镜头；意境来自可见物、可听声、可演停顿和空间方向，不来自小说式比喻或抽象评论。
- 下一集可读时只借“关联方向”，不可读时只做 `episode-local inference`；任何具体事件、台词、答案、新角色身份或结果都不该提前进入本集终结画面。
- 执行报告里的创作证据不是官样文章；它防止 skill 只在文档中有创作力，实际输出仍停留在字段整理。
- 对比轴比空泛“高级感”更可靠：明暗、静动、冷暖、空满、近远、整洁与破败、喧闹与寂静，都能让画面有审美张力。
- 占位句是比空字段更严重的失败：宁可少写一条字段，也不要把“本场按上游...”或“说话者的视线...”这类规则句当正文。
- “音效”不是声音的说明书；把“八点四十分”这类信息放到画面上下文，把引号内留给真正能听见的声音。
- `2-编导` 的终稿感来自阶段内闭环，不来自另开二改阶段：candidate 必须被审计、修复、复审后才交给 `3-摄影`。
- 高阶表演不是把心理词写得更文学，而是让演员知道“我要达成什么、不能暴露什么、用什么行为绕过去”。
- 角色演技不是情绪词堆叠，而是触发点、动机、微表情、身体联动、环境声音和微动态限制共同工作；缺任一层都容易退回 AI 模板脸。
- 微表情优先找眉眼嘴、咬肌、鼻翼、下颌和眨眼频率；非面部联动优先找手指、肩颈、呼吸、重心、道具停点和声线。
- 强情绪不等于大表情；愤怒可以是指节和下颌，难过可以是笑意压回去和吸气不完整，紧张可以是杯沿摩擦和声音变轻。
- `心理反应` 的标题不是抽象心理许可证；它应像演员表演 beat：谁、身体哪里变了、声音哪里断了、道具怎么被处理、对手如何接不到回应。
- 潜台词优先通过停顿、避视、反问、道具动作、身体距离和沉默余波表现；不要用新增对白解释潜台词。若同场已经用过避视或看向远方，下一次必须换变量。
- 转场不是只能靠人物看远方：声音桥、道具状态、群像欢呼变弱、动作中断、空间阻隔、环境刷新和未完成动作都能把下一场压力引进来。
- 场面调度是权力关系和信息差的可见形态：谁站着、谁坐着、谁占据讲台/门口/黑板、道具停在谁手边，比“他掌控全场”更可拍。
- 每场戏最好有状态差；若没有大转折，也应明确这一场承接了什么空间、信息、关系或压力变化。
- B 路线的价值是补“承托”，不是补“剧情”：环境微细节、群体反应、道具位置、停顿和余波可以新增；对白、桥段、因果、规则和线索不能新增。
- 判断受控新增是否安全，先问“删掉这项后剧情事实是否完全不变”；如果答案是否定的，它就不是 B 路线。
- 高段位表演不能靠末尾说明“角色想要什么”完成；要让观众从手、眼、声音压低、道具停点、众人反应里看出来。
