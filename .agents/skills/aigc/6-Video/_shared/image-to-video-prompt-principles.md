# Image-to-Video Prompt Principles

本文件是 `6-Video` 组级与帧级提示词蒸馏的共享句法原则真源。

- 适用范围：
  - `.agents/skills/aigc/6-Video/1-提示词蒸馏/全能参照/`
  - `.agents/skills/aigc/6-Video/1-提示词蒸馏/首帧参照/`
- 职责边界：
  - 本文件定义跨兄弟叶子共享的 `图生视频` 句法取向、信息排序与压缩原则。
  - 各叶子 `prompt-assembly-spec.md` 只负责在本原则下给出本地 specialization，不再各自平行发明第二套总体句法。

## Shared Principles

1. 内容来源保持当前设定：
   - 只消费 `projects/aigc/<项目名>/3-Detail/第N集.json` 的现有组级与镜级字段。
   - 优先消费 branch-owned canonical 字段；legacy 字段只作 fallback。
   - 不得引入新的事实源，不得虚构角色、场景、动作、道具、光线或镜头信息。
2. 句法取向更偏 `AIGC 图生视频`：
   - 文本应更像“给视频模型的画面与运动指令”，而不是字段抄录或编导复述。
   - 优先让模型先读到：主体识别锚点、场景/风格锚点、镜头起势、可见动作与空间关系、运动方式与速度、氛围/光感/道具、构图重心与镜间衔接。
3. 共享语义顺序：
   - `剧情/桥段锚点`
   - `主体识别与服装锚点`
   - `风格与世界观锚点`
   - `镜头起势：分镜构图 -> 运镜手法 -> 速度/视角/镜头类型兼容`
   - `主体动作与空间关系：角色表现 -> 运动表现`
   - `状态与环境：氛围表现 -> 摄影美学 -> 必要道具状态`
   - `视觉组织：视觉强化 -> 镜头框架/镜头类型等兼容补充`
   - `可选衔接：转场特效`
4. 表达体裁：
   - 默认使用可直接朗读的自然句。
   - 除 `分镜组ID / 分镜ID` 外，不暴露字段标题，不写成 `字段标题：字段值`。
   - 不把多个字段硬裁成关键词串；只有预算真的吃紧时，才允许局部短语化。
5. 压缩优先级：
   - 先压 legacy projection 与兼容补充句
   - 再压 `视觉强化 / 摄影美学 / 转场特效` 中的非关键修饰
   - 最后才允许影响 `分镜构图 / 运动表现 / 角色表现`
   - 主体位置关系、动作路径、镜头起势和主要视觉重心必须尽量保持清晰可辨
6. `转场特效` 使用原则：
   - 仅在上游非空时挂句
   - 只作为镜间连续性补充，不得抢占主体、动作与镜头控制信息
   - `ultra` 默认省略

## Shared Mapping

| prompt role | primary upstream fields | fallback fields |
| --- | --- | --- |
| 主体识别锚点 | `出场角色及穿搭`、`视觉强化` | `分镜构图`、`分镜表现` |
| 风格锚点 | `全局风格`、`类型元素`、`导演意图` | 无 |
| 镜头起势 | `分镜构图`、`运镜手法` | `景别`、`镜头视角`、`镜头速度` |
| 可见动作与空间关系 | `角色表现`、`运动表现` | `角色站位走位`、`角色背景面` |
| 状态与环境 | `氛围表现`、`摄影美学` | `场景氛围`、`道具及状态` |
| 视觉组织 | `视觉强化`、`分镜构图` | `镜头框架`、`分镜表现`、`镜头类型` |
| 连续性补充 | `转场特效` | 无；更老 `转场策略` 只允许在 runner 层做容错读取 |

## Canonical Governance Note

- 若兄弟叶子需要共同调整整体 `图生视频` 句法取向，应先改本文件，再改各自 `prompt-assembly-spec.md` 的本地 specialization。
- 若只是某个叶子的局部预算、桥段或落盘策略变化，则只改对应叶子 spec，不回写本文件。
- 若 leaf 仍强依赖 legacy 字段，必须在本地合同中明确写成 `fallback`，不得把 legacy projection 重新升格为 canonical source。
