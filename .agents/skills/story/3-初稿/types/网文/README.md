# 网文类型包索引

本目录保存 `story-drafting` 根技能使用的网文题材固定上下文包。执行时先加载本索引，再按 `north_star.yaml.genre_contract`、章级 planning 与用户请求选择一个或多个具体题材目录。

---

## Runtime Selection Rule

- 未能识别更窄题材时，加载本索引作为网文默认边界，并报告具体题材缺口。
- 能识别题材时，加载对应 `types/网文/<题材>/` 中的 Markdown 文件；若同一题材目录存在多个文件，全部作为该题材固定上下文。
- 起草、续写、重写、局部修复和 review 返工都必须保留原项目题材包，不得因为修复模式改写题材真源。
- 多题材命中时，按 `主类型 > 融合类型 > 场景功能` 优先级收束，最多同时加载 3 个题材包。

| **生存博弈** | 末世、无限流、黑暗题材 | 生存压力下的抉择快感、资源稀缺的张力 |

## 平台分发建议

### 番茄小说（免费+广告模式）

**高匹配题材**：系统流、都市异能、高武、末世、无限流、直播文、规则怪谈
**特征要求**：
- 开篇300字必须出核心冲突或金手指
- 每3章至少1个小爽点（打脸/突破/收获）
- 章节字数2000-3000字，标题必须有信息量
- 主角成长速度翻倍（传统修仙200章结丹 → 番茄80章）
- 对话占比 > 40%，减少大段描写

**适配策略**：
1. 把传统题材的"慢热开局"压缩为3章内让主角获得第一桶金
2. 每章至少留2个信息缺口（读者必须看下一章才能闭合）
3. 打脸场景增加"围观群众震惊"的多角度描写
4. 避免连续3章无实质进展（升级/收获/关系推进）

### 起点中文网（付费订阅模式）

**高匹配题材**：修仙、历史古代、科幻、玄幻剧、武侠、侦探剧
**特征要求**：
- 设定硬核、逻辑自洽是第一优先级
- 读者容忍慢热但必须每卷有高潮
- 章节字数3000-5000字
- 战力体系不能崩坏
- 配角必须有独立行动逻辑

**适配策略**：
1. 强化世界观细节和设定合理性
2. 每卷末尾必须有"值得付费"的高潮（大场面战斗/关键揭秘/重要突破）
3. 反派智商在线，不能写成只会送经验的工具人
4. 保持更新节奏稳定（日更4000+）

### 晋江文学城（付费+积分模式）

**高匹配题材**：狗血言情、青春甜宠、豪门总裁、宫斗宅斗、古言剧、幻想言情
**特征要求**：
- 感情线必须清晰且有张力
- 男主必须有苏感（不油腻、不爹味、不中央空调）
- HE是默认选项，BE需预警
- 人设不能崩
- 章节字数3000-6000字

**适配策略**：
1. 前3章必须确立男女主核心互动模式
2. 感情线不能静止超过3章——每次同框必须有推进
3. 副CP谨慎处理，不能抢主CP风头
4. 虐文注意"虐不过三章必给甜"

### 七猫/免费阅读平台

**高匹配题材**：系统流、都市异能、重生、神医、鉴宝、赘婿逆袭
**特征要求**：
- 极其快节奏，每章必须有爽点
- 章节字数1500-2500字
- 标题必须是信息流风格（"我重生后，前夫跪求复合"）
- 开篇10章定生死

### 知乎盐选

**高匹配题材**：悬疑短篇、反转故事、脑洞短篇、情感短篇
**特征要求**：
- 全文8000-20000字
- 开篇300字必须出核心冲突或高概念
- 全文至少3次反转或高潮
- 第一人称优先
- 结局必须有"值得转发"的力度

## 题材热度与趋势（2025-2026）

### 上升趋势题材

| 题材 | 趋势 | 驱动因素 | 创作建议 |
| --- | --- | --- | --- |
| 规则怪谈 | ↑↑ | 短视频助推、低门槛高概念、适合碎片化 | 强化规则设计的游戏性和可验证性 |
| 无限流 | ↑ | 副本多样性、跨类型融合空间大 | 每个副本换一个子题材，保持新鲜感 |
| 年代文 | ↑ | 怀旧情绪、真实历史代入感 | 90年代题材已饱和，向80年代/00年代初转移 |
| 脑洞类（都市/历史/现言脑洞） | ↑ | 设定新奇是第一生产力 | 核心脑洞必须在第一章就抛出来 |
| 知乎短篇 | ↑ | 碎片化阅读、高传播性 | 强化反转密度，全文至少3次大反转 |
| AI/科技题材 | ↑↑ | 时代热点、读者好奇 | 避免技术说明文，保持人物驱动 |

### 稳定长青题材

| 题材 | 状态 | 关键竞争力 |
| --- | --- | --- |
| 修仙 | 长青但高度内卷 | 需要差异化金手指或世界观 |
| 狗血言情 | 永恒刚需 | 人设新颖即可 |
| 历史古代 | 稳定 | 考据扎实+权谋智商在线 |
| 末世 | 稳定 | 生存细节真实感人 |

### 下行/饱和题材

| 题材 | 状态 | 原因 |
| --- | --- | --- |
| 赘婿逆袭 | ↓↓ | 过度同质化，读者审美疲劳 |
| 神医下乡 | ↓ | 套路固化，平台推荐权重下降 |
| 兵王回归 | ↓ | 老题材，新读者不买账 |
| 传统穿越（无系统/脑洞） | ↓ | 不够差异化 |

## 类型识别决策树

```
north_star.genre_contract 存在？
├── 是 → 直接路由到对应题材目录
│        ├── 单题材 → 加载对应目录全部文件
│        └── 多题材 → 按 type_stack 优先级加载主类型+融合类型
│
└── 否 → 从用户请求/章级 planning 提取题材信号
         ├── 存在金手指/系统 → 系统流
         ├── 存在境界/修炼 → 修仙 or 玄幻剧 or 高武
         ├── 存在感情线为核心 → 言情类（按背景选）
         ├── 存在案件/谜题 → 侦探剧 or 悬疑
         ├── 存在生存/资源危机 → 末世 or 无限流
         ├── 存在历史朝代 → 历史古代 or 历史脑洞
         └── 无法识别 → 默认不加载题材包，按通用规则
```

## 题材包质量门禁

每个题材目录的核心文件必须满足：
- [ ] 有明确的 **核心卖点**（一句话讲清读者为什么看）
- [ ] 有至少 3 种 **流派细分**（覆盖该题材的主要写法）
- [ ] 有 **爽点套路**（至少 3 个可操作的场景模板）
- [ ] 有 **常见雷区**（至少 4 条"必须避免" + 对应"建议做法"）
- [ ] 有 **大纲结构参考**（覆盖从头到尾的章节点分布）
- [ ] 有 **实体标签扩展**（为该题材定制的 XML 标签模板）
- [ ] 有 **追读心理分析**（读者留存的关键心理机制）
- [ ] 有 **经典作品参照**（该题材的代表作和可借鉴点）
