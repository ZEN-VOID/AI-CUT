# 来源归一与模式选择

本文件是 `漫画剧本改编` 的来源处理细则，不与主 `SKILL.md` 竞争真源。

## 1. 来源判型总表

| source_type | 先抽什么 | 常见风险 | 默认改编模式 |
| --- | --- | --- | --- |
| `text` | 情节链、人物关系、情绪核、转折点 | 复述味重、太守原顺序 | 剧情强化改编 / 原著重排改编 |
| `image` | 视觉奇点、人物关系、动作瞬间、前后因 | 只有美术感、没有剧情 | 单帧推演改编 |
| `video` | 时间序列、镜头切换、对白、动作节点 | 时间线散、炸点被埋没 | 镜头序列改编 |
| `news_event` | 已知事实、时间线、相关方、公众情绪、未解点 | 伪造事实、立场漂移 | 事实锚定改编 |
| `hot_search` | 热词爆点、讨论焦点、情绪极点、争议缺口 | 只追热度、不成故事 | 热点议题衍生改编 |
| `mixed` | 主锚点、辅证据、重复段、冲突优先级 | 一稿多主、节奏曲线失控 | 多源归一改编 |

## 2. 多源归一顺序

1. 先定主锚点：哪个来源最接近“不可被改丢的核心卖点”。
2. 再定辅锚点：哪些来源只负责补视觉、补情绪、补事实。
3. 去重：同一事实不重复出现多次，不让不同来源互相抢主线。
4. 汇总成统一 `source_digest`：
   - `已知事实`
   - `可戏剧化关系`
   - `可视化场景`
   - `情绪核`
   - `最大缺口`

## 2.1 虚构来源的 comic-first 姿态

对小说、网文、故事梗概、人物设定文、影视剧情等虚构来源，默认采用“保核但不保表层顺序”的进入姿态。

可默认执行：

- 把慢热开场改成悬念开场或炸点开场。
- 把原本后置的名场面、关系失衡或异常显形前置到前 1-3 场。
- 合并多个功能重复的过桥桥段。
- 把长解释拆散到对白、动作和后续钩子里。
- 提高视觉奇观、压迫感和翻页驱动力。

不可默认破坏：

- 角色核心动机。
- 关键关系位。
- 关键代价与主题方向。
- 对现实来源的事实边界。

## 3. 来源转剧情的最低问题集

每种来源至少回答下面五个问题：

1. 谁是最值得被跟拍的主角？
2. 他/她现在最想得到什么？
3. 有谁或什么在阻止他/她？
4. 这个素材里最容易让读者停下来的反常点是什么？
5. 读完第一章后，读者最想知道什么？

## 4. 默认模式选择

在完成来源归一后，还要再裁决一次剧本格式：

- `scene-script`
  - 素材已具备明确动作/对白/转场，优先写成场景推进型漫画剧本。
- `narration-script`
  - 素材概述密度高或目标兼容解说漫，优先写成旁白推进型漫画剧本。
- `compare`
  - 输入模糊时内部比较双路，只保留一个 canonical 结果。

### 文本

- 优先抽“冲突主线”，不要先修辞润色。
- 如果原文已有故事雏形，重点做“节奏重排 + 炸点前置 + 章末钩子强化”。
- 如果原文只是梗概，重点做“角色欲望 + 场景化落地”。
- 如果原文铺垫过长，优先问“哪一场最适合成为前 3 页内的抓停点”，而不是顺着原顺序慢慢讲。

### 图片

- 不要只写“她穿着什么、站在哪里”。
- 要先推断：
  - 这一刻之前刚发生了什么。
  - 这一刻之后最可能爆发什么。
  - 画面中谁在占上风，谁在隐忍。
- 单图更适合走“悬念开场”或“关系错位开场”。

### 视频

- 优先拆成 `起势 -> 升压 -> 失衡 -> 悬停` 四段。
- 有对白时优先保留冲突性句子。
- 无对白时优先用动作与目光替代解说。
- 若高势能镜头出现在后段，可前置作为冷开场，再倒回解释。

### 新闻 / 热搜

- 先做事实表，不急着起戏。
- 最稳的改编方式通常是：
  - `faithful`：纪实外壳 + 戏剧组织
  - `inspired_by`：事实锚点 + 虚构人物承载线
  - `free_reimagining`：只保留议题核与情绪核

## 5. 类型路由启发

| 来源信号 | 更稳的类型路由 |
| --- | --- |
| 强关系拉扯、误会、照顾、占有欲 | 青春恋爱 / 情感关系剧 |
| 身份错位、信息差、死者回归、监控证据 | 推理悬疑 / 恐怖怪谈 |
| 奇观、法则、面具、力量代价 | 少年战斗冒险 / 黑暗奇幻 / 科幻机甲 |
| 明确背叛、上一世经验、毒茶、婚约、权谋 | 重生 / 穿越 / 复仇 |
| 热搜舆论、城市话题、品牌危机、公共事件 | 都市成长 / 现实悬疑 / 议题衍生 |

## 5.1 默认改写姿态建议

| 来源类型 | 默认姿态 | 默认顺序策略 | 说明 |
| --- | --- | --- | --- |
| `text` | `comic-first` | `adaptive` | 优先抓冲突核和名场面潜力 |
| `image` | `spectacle-first` | `aggressive` | 先把画面炸点转成事件发动机 |
| `video` | `comic-first` | `adaptive` | 优先重切节奏点，再补连续性 |
| `news_event` | `faithful-core` | `preserve` | 事实顺序尽量稳，戏剧组织可强化 |
| `hot_search` | `inspired_by -> comic-first` | `adaptive` | 用虚构承载线放大追更性 |
| `mixed` | `comic-first` | `adaptive` | 先裁主锚点，再设计刺激曲线 |

## 6. 写前自检

若出现以下任一情况，先别写正文：

- 还说不清主角是谁。
- 还说不清本稿最核心的卖点是什么。
- 还说不清新闻事实哪些能改、哪些不能改。
- 还说不清第一章收尾要留什么缺口。
- 还说不清哪些原桥段可以删、并、换序或前置。
