## Repair Playbook

1. 先判断本章卡点属于“章内无推进”“章末无牵引”，还是“高压存在但没有被投影成 reader pull”。
2. 若章内弱，优先补微兑现与推进脉冲，不要先堆章末钩子。
3. 若章内其实不缺事件，而缺“会不会被看见 / 说破 / 失手 / 亵渎 / 登台”的临界感，先选 dominant pressure core，再改句子。
4. 若章末弱，先决定单一主钩，再显影读者最关心的问题；偏文学章也一样，不允许只留漂亮余韵。
5. 若近几章已经连续使用同构主钩，先做差异化，再做强度升级。
6. 若 type-pack 未启用，不要硬套题材专属偏好，只用通用 taxonomy。
7. 若主角启用了成长系统，额外回答：本章的压力、微兑现和章末 pull 分别把 `技能 / 心路 / 情感` 哪些轴推向了下一步。

## Reusable Heuristics

- 追读力不是“更刺激的文案”，而是“局部满足 + 新的未完成”同时存在。
- 最稳的 Step 7 改法，通常不是到处加事件，而是让已有事件更早显影代价、更晚收束结果。
- 章末最强的钩子往往只有一个；真正的专业感来自收束，而不是堆叠。
- 主角成长系统在 Step 7 的证据，通常不长在“他想明白了”这类总结句里，而长在“他被什么逼着暂时不再按旧方法行事”。
- 对偏文学、偏人物高压的章，reader pull 也必须能一句话说出来，例如：他会不会被看穿、会不会说破、会不会毁掉那个对象、会不会在众目下失控。
- 高压 reader pull 最稳的做法，不是制造更多事故，而是让角色知道再拖一拍就要暴露、失手或付出身体代价。
- 若一章的气氛已经足够强，下一步通常不是再加象征，而是把象征压成单一危险和单一牵引。
- 借三岛式镜片时，只借“假面 -> 欲望压力 -> 身体兑现 -> 公开场面”的推进语法，不借和风摆设或自毁魅力。

## Promotion Candidates

- 若更多章节验证“dominant pressure core + 单一主钩 + 微兑现”稳定有效，可把 pressure-core 诊断顺序提升到 `SKILL.md` 更靠前的位置。
