# 一致性 模块说明

## 定位

- 本叶子负责锁定人物状态、道具状态和动作承接不漂移。
- 它不负责新增动作，只负责确认上一拍留下的状态有没有被接住。

## 具体创作方法

- 先从上一拍回收一个“必须接住”的状态，不要贪多。常见高价值状态有三类：身体姿态、手中物件、情绪或关系受压状态。
- 判断这个状态在当前拍里是延续、变化还是被打断。只有三类之一成立时，才写入承接句；不要把所有细节都当连续性。
- 最好的写法不是解释“保持一致”，而是直接把承接嵌进动作里，例如“他仍半侧着身，没有完全转过来”“她手里的杯子一直没放下，只是指节更紧”。
- 若上一拍状态已经自然隐含在当前动作里，可只补最容易丢失的那一点，不必重复全量复述。

## 思维·执行节点

1. `上一拍回收`
   找到上一拍结束时真正仍在场的状态，而不是把整拍内容都搬过来。
2. `当前拍比对`
   判断当前文字有没有接住这个状态，或者是否发生了合理变化。
3. `承接句落点`
   把状态写进当前动作、姿态或手部细节里，避免单独写成解释句。
4. `漂移预警`
   若该状态极容易被下游忽略，就明确标出，例如“仍”“还”“没有松开”“并未转身”这类承接词。

## 节点延展

- 可延展的重点是“状态如何被带着走”，而不是“再创造一个新动作”。
- 当人物处在强压力或对峙场面中，情绪连续性比物件连续性更重要；当场景有交接、威胁或遮挡时，物件和身体朝向往往更重要。
- 若上一拍留下多个状态，只保留最会影响下一拍理解的那个，其余交给汇流层裁剪。

## 失真与修正

- 若出现“失忆式跳变”，说明上一拍状态没有被回收。
- 若为了补一致性开始扩写新动作，说明越权。
- 若承接点过多，先保留最影响主冲突的那一个。
- 若你写成“他保持上一拍的状态”这种元说明，说明还没有把一致性转成可见文本。
