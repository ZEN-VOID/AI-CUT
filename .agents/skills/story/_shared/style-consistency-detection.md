# Writing Style Consistency Detection Contract

本文件是 `story2026` 的写作风格一致性自动检测合同。定义风格指纹的提取、比对和偏离检测方法，用于验收时自动检测写作风格的异常变化。

本文件不拥有正文写权，不是独立阶段。

## Core Principle

长篇小说的写作风格可能会有意变化（不同卷/不同场景/不同POV），但无意中的漂移（同一POV同一场景类型的风格偏移）会破坏作品的整体性。风格一致性检测的目标是识别后者——无意的风格漂移。

## Style Fingerprint Dimensions

### 1. 句法指纹（Syntactic Fingerprint）

| 指标 | 计算方法 | 正常波动范围 |
|------|----------|-------------|
| 平均句长 | 全文句长均值（字符数） | ±20% |
| 句长标准差 | 全文句长的标准差 | ±25% |
| 短句比例（≤15字） | 短句数 / 总句数 | ±15个百分点 |
| 长句比例（≥50字） | 长句数 / 总句数 | ±10个百分点 |
| 对话句比例 | 对话句子数 / 总句数 | ±15个百分点 |

### 2. 词汇指纹（Lexical Fingerprint）

| 指标 | 计算方法 | 正常波动范围 |
|------|----------|-------------|
| 词汇丰富度 | 不重复词数 / 总词数（type-token ratio） | ±20% |
| 高频虚词密度 | "的/了/是/在/和/就/也/都"总次数 / 总词数 | ±10个百分点 |
| 成语密度 | 四字成语数 / 总词数 | ±50% |
| 比喻/修辞密度 | 比喻句数 / 总句数 | ±40% |

### 3. 段落指纹（Paragraph Fingerprint）

| 指标 | 计算方法 | 正常波动范围 |
|------|----------|-------------|
| 平均段落长度 | 段落字符数均值 | ±30% |
| 段落长度标准差 | 段落长度的标准差 | ±35% |
| 纯对话段比例 | 纯对话（无叙述）段落 / 总段落数 | ±15个百分点 |
| 长段落比例（≥500字） | 长段落数 / 总段落数 | ±10个百分点 |

### 4. 叙事指纹（Narrative Fingerprint）

| 指标 | 计算方法 | 正常波动范围 |
|------|----------|-------------|
| POV切换频率 | 每章POV切换次数 | ±50%（取决于章节类型） |
| 内心独白密度 | 心理描写字数 / 总字数 | ±20个百分点 |
| 环境描写密度 | 环境/场景描写字数 / 总字数 | ±15个百分点 |
| 动作描写密度 | 动作/事件描写字数 / 总字数 | ±15个百分点 |

## Baseline Establishment

### 基线章选择
- 第1卷第1章作为初始基线
- 每卷选择1-2章作为该卷的基线（建议选"标准强度"章，避免极端章）
- 卷内其他章与该卷基线比对

### 基线更新策略
- 如果风格有意变化（如north_star.style_contract中声明的卷级风格转变），建立新基线
- 无意的漂移不更新基线，标记为WARN

## Drift Detection

### 单章偏离检测
当本章的任意指纹指标偏离基线超过正常波动范围时：

1. 标记偏离维度和偏离幅度
2. 判断是否属于有意变化（场景类型差异、POV差异、节奏差异）
3. 若为无意漂移 → 记录 WARN + 偏离维度
4. 多维度偏离（≥3个指标同时超范围）→ 记录 FAIL

### 卷级趋势检测
当多章的同一指纹指标呈单向趋势性漂移时（连续3章以上同方向变化）：

1. 标记为"渐进式风格漂移"
2. 记录趋势方向和幅度
3. 建议回源到 north_star.style_contract 确认是否符合预期

## Integration

### 在验收包中

```json
{
  "style_consistency": {
    "baseline_ref": "第1卷/第1章",
    "drift_detected": false,
    "drift_details": [],
    "trending_drifts": [],
    "warnings": []
  }
}
```

### 评估时机
- `3-初稿/N6-AUTO-ACCEPTANCE`：初稿验收时检测
- `4-润色/P5-AUTO-ACCEPTANCE`：终稿验收时检测
- 风格偏离为 **诊断参考**，不影响 PASS/FAIL（除非 north_star 中明确声明了风格硬约束）

### Token Budget
- 风格检测不需要额外加载文件
- 基于当前章和基线章的文本统计
- 输出控制在 200 tokens 以内
