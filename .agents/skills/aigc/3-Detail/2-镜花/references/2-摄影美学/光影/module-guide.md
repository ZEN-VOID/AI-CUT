# 光影 模块说明

## 定位

- 本叶子负责确定主光位、辅助光、逆光、影调关系和组内光影推进逻辑。
- 它不负责凭空制造唯美光，只负责让冲突、空间和人物状态更可见。
- 它默认消费上游 `cinematography_brief + visual_control_line + cinematography_strategy_note`，而不是独立发明一套光线审美。

## 具体创作方法

1. 先回收光影锚。
   从 `visual_control_line + cinematography_strategy_note` 里确认哪条光影判断必须稳定贯穿整组，例如“主体始终被侧逆光压住”或“空间前后层一直靠冷硬顶光拉开”。
2. 再锁主光来源。
   要回答这束光为什么合理存在，以及它为什么是当前冲突最有效的显化方式。
3. 再补辅助光和逆光。
   它们不是为了“更有层次”，而是为了让主体剥离、轮廓显形、压迫感成立或空间深度可读。
4. 再定照明类型与影调。
   明确当前组更适合 `硬光 / 柔光 / 混合光` 哪一种，并说明高光、暗部和阴影关系对应的戏剧功能。
5. 最后补组内光影流动。
   不是每镜都重新发明布光，而是回答组内镜头怎样沿着同一条光影语法推进。

## 思维·执行节点

| node_id | objective | thinking_focus | execution_action | output |
| --- | --- | --- | --- | --- |
| `LGT-N1-INHERIT` | 回收光影锚 | 哪条光影判断必须稳定贯穿整组 | 从 `visual_control_line + cinematography_strategy_note` 提炼本组不可漂移的光影约束 | `lighting_anchor` |
| `LGT-N2-KEY` | 锁主光 | 主光从哪里来最合理、最有戏剧收益 | 指明主光方向、位置和主要照亮对象 | `key_light_logic` |
| `LGT-N3-SUPPORT` | 补辅助光与逆光 | 哪些轮廓和空间层需要被拉开 | 补充辅助光、逆光或环境补光逻辑 | `support_light_logic + back_light_logic` |
| `LGT-N4-TYPE` | 锁照明类型 | 当前冲突更适合硬、柔还是混合 | 写明照明类型及其戏剧用途 | `lighting_type` |
| `LGT-N5-SHADOW` | 安排影调 | 阴影和高光该遮住什么、放出什么 | 压清高光落点、暗部比例和阴影功能 | `shadow_use` |
| `LGT-N6-PROGRESSION` | 形成组内推进 | 光影怎样跟随情绪或信息推进 | 总结镜序间的光影变化轨迹 | `lighting_progression` |

## 延展问法

- 这束主光是揭示人物，还是把人物困在空间里？
- 若只保留一条稳定光影判断，它应该是哪一条，为什么？
- 如果主光不变，辅助光和逆光要承担的是“轮廓显形”还是“观看压力”？
- 阴影应该让哪些信息暂时看不清，才能为后续镜头留出张力？
- 本组更需要光线稳定，还是允许局部突然变硬、变冷、变斜？
- 当动作变强时，光影是跟着加压，还是故意维持冷静对照？

## 写法落点

- 单镜层优先写“光从哪来，照到谁，遮住什么”。
- 组级层优先写“这套光影如何在镜序中推进”。
- 若需压缩，优先保留“主光源 + 辅助光/逆光职责 + 照明类型 + 组内流动”。
- 若只有一个关键信息，优先保留主光与影调功能，不必强行写满辅助光细节。

## 失真与修正

- 若光线没有来源，说明已经漂成审美空话。
- 若没有先回答哪条光影判断必须稳定贯穿整组，说明还没真正继承 branch 策略。
- 若只写主光，不写辅助光或逆光，说明空间剥离和轮廓关系还没成立。
- 若照明类型只有“有层次、电影感”，说明没有真正回答戏剧功能。
- 若阴影关系压没了动作信息，说明光影过度抢戏。
- 若组内每镜都像不同摄影师拍的，说明没有把局部布光继续收束成组级推进。
- 若光影判断和空间逻辑冲突，优先保住空间可读性。
