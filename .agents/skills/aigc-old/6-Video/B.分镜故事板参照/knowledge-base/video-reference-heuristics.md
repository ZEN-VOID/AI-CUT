# Video Reference Heuristics

本文件保存 `B.分镜故事板参照` 可复用经验。强制合同以 `SKILL.md` 与 `references/` 为准。

## Heuristics

- 组级故事板参照最容易漏的是“逐镜边界”：故事板图能提供连续性，但 prompt 仍必须保留每镜时间、动作、镜头控制与回链。
- `全能参照` 的旧三件套适合当兼容来源，但新包 runtime 根应统一落到 `B.分镜故事板参照/<episode>/`，不要继续分散在旧编号目录下。
- 绑定故事板图时，`group_id` 是第一候选；角色和服装只做辅助，不能替代分镜组视觉连续性锚点。
- `Assets/分镜画板/漫画/` 可以作为组级风格/构图参考，但不得把漫画分格内容反向写成新的剧情事实。
- provider handoff 的价值是留下可复跑计划；只给“用某 provider”不算完成。
