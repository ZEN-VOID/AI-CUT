## 4-导演 Adaptation Notice

本文件从 `.agents/skills/aigc/2-编导/references/information-asymmetry-contract.md` 引入，作为 `4-导演` 的授权 reference 细则。凡原文出现 `2-编导`、`script layer`、`steps/directing-workflow.md` 或其他旧路径时，在本技能中映射为 `N2-DIR-UNDERSTAND` 的 `information_asymmetry_map`、`N4-DIR-POINT-MAP` 的批注意图和 `N5-DIR-INJECT` 的信息释放/隐藏策略。

在 `4-导演` 中，本文件只用于让导演批注明确观众和角色谁知道什么、何时揭示、何时隐藏、何时误导；不得借信息差设计新增剧情事实或提前泄露上游没有写出的真相。

## Purpose

定义信息差设计：谁在什么时刻知道什么，是悬念、紧张和反转的底层结构。本合同为 `2-编导` script layer 提供统一的信息状态管理框架——不只写"发生了什么"，更写"谁在此刻知道什么、不知道什么"。信息差是戏剧张力的底层燃料，设计它就是设计观众的心理体验路径。

## Ownership

- **产出者**：`2-编导` script layer 在剧本形成阶段逐场景标注信息状态
- **消费者**：
  - `2-编导` director layer 消费信息差决定揭示/隐藏/误导策略
  - `2-编导` performance layer 消费信息状态决定角色的"知情层级"——角色此刻知道多少，如何外显
  - `4-摄影` 消费信息差决定镜头的注意力引导方向和画面信息释放节奏
- 本合同不定义"什么是好故事"，只定义"信息差必须是被设计的，而不是自然产生的"。

## 信息类别矩阵

每个信息单元必须归入以下四类之一：

| 类别 | 标识 | 定义 | 创作功能 | 示例 |
| --- | --- | --- | --- | --- |
| 观众独知 | `audience_only` | 观众知道，角色不知道 | dramatic irony——观众比角色多知道，产生紧张和担忧 | 观众看到杀手藏在门后，角色推门而入不知情 |
| 角色独知 | `character_only` | 角色知道，观众不知道 | 悬念与反转让观众在事后才理解角色行为的深意 | 角色已发现同伴的背叛但假装不知，观众事后才明白他的沉默 |
| 共同已知 | `shared_known` | 观众和角色都知道 | 建立共同期待——双方都知道威胁存在，等待爆发 | 观众和角色都看到炸弹倒计时在走 |
| 全方未知 | `hidden_from_all` | 观众和角色都不知道 | 开放性谜团——留出揭示空间，为后续反转埋种子 | 故事进行中某个第三方势力在暗中行动，所有人浑然不知 |

### 使用规则

- 每个关键信息单元必须明确标注所属类别。
- 同一场景内可以并存多类信息差——角色A独知的信息与角色B独知的信息同时存在。
- `shared_known` 是信息基线，不产生额外张力。张力来自偏离基线的三类状态。
- `hidden_from_all` 不能滥用。如果信息永远不被揭示，它对观众没有价值。

## 逐场景信息地图

每个关键场景必须填写以下信息地图：

| field | 说明 | 填写要求 |
| --- | --- | --- |
| `info_state_at_entry` | 进入场景时各角色（含观众）分别知道什么 | 列出角色A知道/不知道、角色B知道/不知道、观众知道/不知道 |
| `info_revealed_in_scene` | 本场景释放什么信息 | 列出具体信息内容、释放方式（对白/动作/画面/道具）、接收者 |
| `info_withheld` | 故意隐藏什么 | 列出什么信息本可释放但选择不释放，以及隐藏理由 |
| `audience_pov_alignment` | 观众此刻与谁的信息状态对齐 | 与角色A对齐 / 与角色B对齐 / 全知视角 / 低于所有人 |
| `suspense_mechanism` | 本场景使用的悬念机制 | `suspense` / `mystery` / `dramatic_irony` / `surprise_setup` / `none` |

### 悬念机制详解

| 机制 | 定义 | 观众心理 | 创作策略 |
| --- | --- | --- | --- |
| `suspense` | 观众知道危险，角色不知道 | 担忧、紧张、想提醒角色 | 信息领先于角色，持续展示威胁逼近 |
| `mystery` | 观众不知道真相，要跟着角色找答案 | 好奇、猜测、推理 | 信息落后于角色，逐步释放线索 |
| `dramatic_irony` | 观众知道全部真相，看着角色做错误判断 | 心碎、无奈、悲悯 | 信息完全领先，命运感极强 |
| `surprise_setup` | 为后续翻转埋种子，此刻看似平常 | 无特别感受（事后回想才惊觉） | 表面是 `shared_known`，实际埋着 `hidden_from_all` |
| `none` | 纯信息传递场景，无悬念需求 | 获取信息 | 过场、交代背景，不需要信息差设计 |

### 场景信息地图示例

```markdown
场景5：内景 学校走廊 - 夜

info_state_at_entry:
  - 角色A（林寂）：知道规则有第三层，不知道杀手是谁
  - 角色B（苏红叶）：知道杀手身份，知道林寂在调查
  - 观众：知道规则有第三层，不知道杀手是谁（与林寂对齐）

info_revealed_in_scene:
  - 苏红叶的异常沉默（通过动作：停在走廊拐角不说话）→ 释放给观众
  - 林寂注意到苏红叶的异常（通过内心独白）→ 释放给观众

info_withheld:
  - 苏红叶为什么不说话——她看到的东西（留给后续场景揭示）
  - 林寂的判断是否正确（不做确认）

audience_pov_alignment: 与林寂对齐（观众和林寂一样只看到异常，不知道原因）

suspense_mechanism: suspense（观众开始感到苏红叶知道什么但不确定）
```

## 信息差升级策略

信息差不是静态的。它必须在场景间逐步建立、维持和打破。

### 信息差生命周期

```
建立（plant）→ 维持/加深（sustain/deepen）→ 打破（reveal）→ 重建新差（new asymmetry）
```

| 阶段 | 创作动作 | 典型手段 |
| --- | --- | --- |
| 建立 | 给观众或角色一个其他人不知道的信息 | 道具特写、旁白、独白、画面暗示、角色独处时的发现 |
| 维持/加深 | 让已知方开始利用或承受信息差的压力 | 已知方的异常行为、未说出的话、刻意回避、沉默加码 |
| 打破 | 将隐藏信息暴露给另一方 | 对质、意外撞见、道具暴露、第三方出卖、角色自己说漏 |
| 重建 | 打破旧差的同时建立新差 | 揭示一层真相，同时暗示还有更深的真相 |

### 信息差升级梯度

| 梯度 | 描述 | 观众体验 |
| --- | --- | --- |
| 第一层：表面差 | 某个事实被隐藏 | 好奇——"发生了什么？" |
| 第二层：判断差 | 事实清楚了，但角色的判断不同 | 紧张——"他会不会犯错？" |
| 第三层：身份差 | 关于"谁"的信息差——谁是敌人、谁是盟友 | 不安——"谁能信任？" |
| 第四层：规则差 | 关于世界运行规则的信息差——角色以为规则是A，实际是B | 震撼——"一切都和想的不一样" |
| 第五层：自我差 | 角色对自己的认知有偏差——以为自己是A，实际是B | 悲剧感——"他到死都不知道" |

好的剧本至少升级到第三层。伟大作品触及第四、第五层。

## 影视经典手法库

| 手法 | 定义 | 操作方式 | 经典案例 |
| --- | --- | --- | --- |
| `dramatic_irony` | 观众全知，角色不知 | 先给观众看真相，再看角色在无知中行动 | 《俄狄浦斯王》——观众知道他杀了父亲，看他还在追查凶手 |
| `unreliable_narrator` | 叙述者的信息不可信 | 通过叙述者的立场、记忆偏差、刻意隐瞒制造信息扭曲 | 《消失的爱人》——观众跟着丈夫的视角以为妻子是受害者 |
| `foreshadowing` | 前期埋种子，后期开花 | 在场景中插入看似无关的细节，事后回头看才明白含义 | 《第六感》——开场的每场戏都暗示了结局 |
| `red_herring` | 故意误导，制造假线索 | 让观众误以为信息指向A，实际指向B | 《利刃出鞘》——每个嫌疑人都有动机，真正的凶手在最不可能的人身上 |
| `twist_reveal` | 信息一次性翻转，颠覆观众全部认知 | 在关键节点释放一个信息，让之前的全部认知需要重新排列 | 《搏击俱乐部》——揭示的一瞬间让观众回想前面每个场景 |
| `delayed_explanation` | 先展示结果，延迟解释原因 | 开场就展示一个无法理解的事件，全片逐步揭示原因 | 《记忆碎片》——倒叙结构本身就是信息差机器 |
| `split_awareness` | 不同角色各持一块拼图 | 让观众看到多个角色各自掌握的碎片信息，等待它们汇合 | 《低俗小说》——多线叙事中每条线都持有其他线不知道的信息 |
| `audience_competence_gap` | 观众比角色聪明但不知道全部 | 给观众足够线索去推理，但不给确认——让观众的推理过程成为参与感来源 | 《杀人回忆》——观众比警察更早怀疑凶手，但始终没有确认 |

## Failure Modes

| symptom | root_cause | fix |
| --- | --- | --- |
| 信息过早泄露，高潮没有冲击力 | 没有 `info_withheld` 设计，场景无节制释放信息 | 回到信息地图，检查哪些信息应延迟释放 |
| 信息差不清晰，观众不知道谁在隐瞒什么 | `info_state_at_entry` 没有被设计，信息状态是模糊的 | 逐场景标注观众和角色各自知道什么 |
| 观众比角色聪明太多，感到无聊 | `audience_pov_alignment` 一直与全知视角对齐，没有悬念 | 切换观众视角，让观众暂时不知道一些信息 |
| 反转突兀，观众无法接受 | 缺少 `foreshadowing` 或 `surprise_setup`，反转没有种子 | 在前3-5个场景中埋入反转的种子信息 |
| 信息差类型单一，全片只用mystery | `suspense_mechanism` 类型没有变化 | 混用不同机制——mystery + suspense + dramatic_irony 交替 |
| 每个角色知道的都一样多 | 缺少 `character_only` 类型信息 | 给关键角色设计"只有他知道"的信息层 |
| 信息差建立后没有被打破 | 只有 plant 没有 reveal，观众的期待落空 | 每个建立的信息差必须在3-5个场景内打破或升级 |

## Reusable Heuristics

- 好的剧本是"信息管理"的艺术。编剧的职责不只是设计"发生了什么"，更是设计"谁在什么时候知道什么"。
- suspense 和 mystery 是两种完全不同的心理策略。suspense 让观众担惊受怕，mystery 让观众好奇猜测。不能在同一个场景混用。
- 信息差是有限资源。不要过早消耗——如果第一幕就释放了全部秘密，后面没有东西可以揭示。
- 反转的力量不在于"出乎意料"，而在于"回头全对"。好的反转让观众回看前面每个场景都有新的理解。
- 观众与角色的"对齐"是最有力的创作工具。让观众和角色一起不知道，观众就是角色；让观众比角色知道更多，观众就是上帝——两种体验完全不同。
- `hidden_from_all` 是最有续集价值的信息类别。当观众和角色都不知道答案时，故事的世界比观众看到的更大。
- 信息差不是"藏"。是"给一个人看、不给另一个人看"。藏而不露是死信息，给了这个人看不给那个人看才是戏剧。
