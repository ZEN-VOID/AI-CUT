# 自动剪辑 PRP：测试项目配音字幕成片

## 目标

基于 `projects/测试/素材/` 中的视频、文案和旁白配音，参照 `projects/测试/示例/` 的剪辑节奏，输出完整成片到 `projects/测试/结果/`。

## 输入

- 参考成片目录：`projects/测试/示例/`
- 主视频：`projects/测试/素材/视频/content-01-ancient-male-dragon-battle.mp4`
- 字幕文案：`projects/测试/素材/文案/直接他妈大结局了.md`
- 旁白配音：`projects/测试/素材/音频/MiniMax_2026-06-20_23_26_01_Unrestrained_Young_Man.mp3`

## 输出

- 最终成片：`projects/测试/结果/直接他妈大结局了_final.mp4`
- 工作目录：`projects/测试/结果/auto-edit/edit/`
- 字幕文件：`projects/测试/结果/auto-edit/edit/master.srt`
- 剪辑计划：`projects/测试/结果/auto-edit/edit/edl.json`
- 执行报告：`reports/自动剪辑-测试项目-20260620.md`

## 技能路由

- 主编排：`.agents/skills/video-use/SKILL.md`
- 参考节奏与切点分析：`.agents/skills/media/timestamp-extraction/SKILL.md`
- 剪辑执行：`.agents/skills/media/video-editing/SKILL.md`
- 字幕时间轴：`.agents/skills/wis/wjs-transcribing-audio/SKILL.md`
- 字幕烧录与最终合成：`.agents/skills/wis/wjs-burning-subtitles/SKILL.md`

不调用 `.agents/skills/wis/wjs-dubbing-video/`，因为用户已经提供旁白音频，不需要生成新配音。

## 执行阶段

### 1. 工具链与素材检查

- 检查 `ffmpeg`、`ffprobe`、Python 运行环境。
- 读取参考视频、主视频、旁白音频的时长、分辨率、帧率和音频参数。
- 创建工作目录，不改动原素材。

通过条件：
- 所有输入文件可读。
- 主视频、旁白音频均可被 `ffprobe` 正常解析。

### 2. 参考节奏分析

- 对 `projects/测试/示例/*.mp4` 提取基础节奏指标：时长、fps、分辨率、镜头/黑屏节奏、首尾结构。
- 采样参考帧，判断是横屏、宽屏裁切还是黑屏字幕卡。
- 形成 `reference_notes.md`，作为本轮剪辑风格依据。

通过条件：
- 能得出本轮剪辑的目标时长、画幅、黑屏补位策略和字幕风格。

### 3. 字幕与旁白对齐

- 以用户文案为字幕真源。
- 以旁白音频时长作为主时间轴。
- 优先用中文 ASR 生成时间戳；如凭证或网络不可用，则用文案标点、文本长度和音频总时长做比例分配，生成可读 SRT。
- 字幕控制为短句分块，避免长行溢出。

通过条件：
- `master.srt` cue 编号连续、时间不重叠、毫秒逗号格式正确。
- 字幕总时长贴近旁白音频总时长。

### 4. 画面重剪与缺失补位

- 以旁白时长为目标，把主视频切成若干节奏段。
- 参考示例的平均段落长度，保留快节奏推进。
- 若视频素材无法覆盖对应节奏或内容，使用黑屏/深色字幕卡补足。
- 原视频声音默认关闭，最终音频使用旁白配音。

通过条件：
- `edl.json` 明确记录每个画面段来源、起止时间和类型。
- 输出片长与旁白音频基本一致。

### 5. 最终合成

- 合成画面与旁白音频。
- 字幕最后烧录，避免被其他图层遮挡。
- 输出 H.264/AAC MP4。

通过条件：
- 生成 `projects/测试/结果/直接他妈大结局了_final.mp4`。
- 文件可播放，包含视频轨和音频轨。

### 6. 自检与报告

- 抽查首尾、中段和最长字幕帧。
- 检查字幕是否溢出、音频是否存在、片长是否合理。
- 写入执行报告，说明实际路径、降级点和验证结果。

通过条件：
- `ffprobe` 能正常读取最终成片。
- 报告记录源层路由、执行结果与剩余风险。

## 降级策略

- ASR 不可用：使用文案驱动的比例时间轴生成 SRT，并在报告中记录。
- 字幕烧录 libass 不可用：使用静态 ffmpeg fallback；若仍失败，保留无字幕合成和 SRT。
- 示例节奏无法机器识别：使用基础媒体参数、抽帧观察和人工判断形成节奏策略。
- 主视频内容不足：允许黑屏/深色字幕卡补位，这是用户明确允许的路径。

## 验收标准

- 最终成片存在于 `projects/测试/结果/直接他妈大结局了_final.mp4`。
- 成片使用用户提供旁白音频。
- 成片包含与用户文案对应的硬字幕。
- 成片节奏参照示例目录，不只是简单替换音轨。
- 缺失画面用黑屏/字幕卡处理，不产生空洞时间。
