# CONTEXT.md

本文件沉淀 `文体读感` 维度的局部经验。它不改写父层 review gate，也不替代 `结构兑现`、`人物一致性` 或 `逻辑自洽校验`。

## Type Map

| failure_or_outcome_type | root_cause_layer | immediate_fix | systemic_prevention | verification_point |
| --- | --- | --- | --- | --- |
| 正文信息完整但像提纲复述，没有读者能进入的场面 | drafting prose layer | 回到 `3-场景和氛围渲染`，补现场物件、环境反作用和人物即时反应 | system prompt 明确 planning 是义务，不是正文句法 | 每个关键推进都有可感知触发物 |
| 文本顺滑但像模型整理，句群长度平均、段落没有呼吸 | polishing layer | 回到 `8-润色`，恢复长短句变化、停顿和段落重心 | 润色只做局部修补，不做通用顺滑化 | 段落之间有节奏差异 |
| 人物心理只报标签，如“他很愤怒/她很震惊” | psychology drafting layer | 回到 `6-心理活动描写`，改成身体动作、误触、犹豫、视线和未说出口的话 | 心理必须绑定处境、欲望、恐惧和选择成本 | 情绪能被动作/选择证明 |
| 对白像互相说明真相，没有潜台词和关系动作 | dialogue drafting layer | 回到 `5-对白优化`，压缩解释，加入试探、回避、施压、安抚或索证动作 | 声纹表不得只有标签，必须落成句式和关系动作 | 每段对白有说话目的 |
| 题材类型包把爽点写成机械公式，文本变廉价 | type package layer | 以 `genre-trope-quality-filter.md` 压制公式，重写为角色代价、局势后果和关系压力 | 类型包只提供原料，不拥有压过项目风格的命令权 | 爽点服务人物与剧情，不是独立表演 |

## Repair Playbook

1. 先剥离 frontmatter、标题和流程术语，只看小说正文。
2. 若问题是“没写成场面”，优先打回 `3-场景和氛围渲染`，不要只要求润色。
3. 若问题是对白同质化，交给 `5-对白优化`；若问题是情绪标签化，交给 `6-心理活动描写`。
4. 若问题是 AI 腔、过度总结、句群平均、模板脸色反应或镜头残留，交给 `8-润色` 做最小修补。
5. 若题材套路污染文本，先回查 `_shared/genre-trope-quality-filter.md`，再要求当前 lane 重写受污染段落。

## Reusable Heuristics

- “写得顺”不是小说质量终点；真正的 PASS 需要现场、动作、信息、欲望、阻力和余波在段落里互相咬住。
- 读感问题常被误塞进结构或人物维度；文体读感只管文本肌理，不抢结构义务和人物动机裁决权。
- 好的现场发现不是装饰描写，而是能改变人物选择、暴露信息、制造误读或把危险推近的细节。
- 模板化“脸色大变”通常说明作者在逃避具体反应；优先查手、呼吸、步伐、物件误触、话语断裂和身份相关动作。
- 润色阶段最危险的失败是把活气修成整齐说明文；文体读感要保护初稿里有用的粗粝、停顿和口语弹性。
