# Shot Duration Decision Contract

本文件定义 `4-摄影` 的镜头时值裁决。`beat-analysis-contract.md` 决定“哪里切”，`visual-rhythm-analysis-contract.md` 决定“快慢与张弛”，本合同决定“每个 `分镜N` 需要在屏幕上停留多久才算完成观看任务”。

## Core Rule

每个 `分镜N` 在进入 `shot_design_plan` 前必须形成内部 `shot_duration_decision`，并在逐集摄影稿正文中以 `分镜N（约X秒）:` 显式落盘。`X秒` 是面向下游执行的近似时值，不是精剪级绝对帧码；它必须由观看任务、短剧·AIGC 节奏偏置、对白台词量、动作完成、信息可读、表演停顿和下游 `5-分组` 的 15 秒组内节奏共同决定。

本仓库默认生产对象是 AIGC 短剧·电影，不默认套用传统影视剧的宽停顿节奏。除非用户、项目 `MEMORY.md`、导演风格或当前高点明确要求慢节奏，本阶段采用 `short_drama_aigc_duration_bias`：默认让镜头更短、更早落点、更少氛围性长停；只有文字/规则必须读清、台词必须听完、动作必须完成、表演变化必须停住、重大认知翻转或高点余波成立时，才允许进入 `held / long_hold`。

判断标准固定为：

> 如果把这一镜缩短一半，观众是否会看不清、读不懂或感受不到当前必要内容？

- 如果不会，当前镜头过长，应收敛、合并或删掉冗余停顿。
- 如果会，当前镜头需要保留时值，并在分镜文字中体现可读性、动作完成或情绪停顿。
- 如果一镜内部同时要求快速冲击和信息可读，应拆成“冲击镜 + 落点/反应/读秒镜”，而不是让单镜承担互相冲突的时值任务。
- 如果该镜承载对白、旁白或需听清的台词内容，必须先满足台词完整听完所需的时间，再决定画面运动和切点；不得为了视觉节奏提前切走正在发生的关键台词，除非明确设计为声画分离或跨镜延续。
- 如果只是慢推、氛围、普通空间压迫或普通表情，而没有新增可读信息、动作结果或表演变化，短剧·AIGC 模式下不得自动超过 `standard`。

## Duration Fields

| field | values | use |
| --- | --- | --- |
| `duration_class` | `instant / short / standard / held / long_hold` | 当前分镜的时值等级 |
| `estimated_seconds` | `0.3-0.7 / 0.7-1.3 / 1.3-2.2 / 2.2-3.5 / 3.5-5` | 短剧·AIGC 默认内部估算范围 |
| `display_seconds` | `约0.5秒 / 约1秒 / 约1.5秒 / 约2秒 / 约2.5秒 / 约3秒 / 约4秒 / 约5秒` | 逐集摄影稿正文中写入 `分镜N（约X秒）:` 的近似时长 |
| `duration_reason` | `impact / action_completion / readability / performance_pause / spatial_orientation / reaction_absorption / handoff_breath` | 为什么需要这个长度 |
| `dialogue_time_budget` | `none / inherited / local_line / voiceover` | 当前镜头是否承载对白、旁白、画外音或跨镜延续台词 |
| `dialogue_seconds_floor` | `<number>` | 由台词量估算出的最低时长，若无台词则为 `none` |

| `screen_information_load` | `single / readable_detail / layered / dense` | 观众需要读的信息量 |
| `compression_risk` | `none / too_short / too_long / conflicts_with_15s_group` | 当前时值风险 |
| `duration_mode` | `short_drama_aigc / traditional_cinematic / project_override` | 当前时值是否采用短剧·AIGC 默认压缩、传统电影停顿或项目显式覆盖 |

## Duration Matrix

| shot_function | default duration | expand when | compress when |
| --- | --- | --- | --- |
| 惊吓、危险确认、硬断裂 | `instant / short`，约 0.3-0.8 秒 | 需要给反应落点或后果钉镜 | 只是制造刺激、没有新信息 |
| 普通动作承接 | `short / standard`，约 0.7-1.5 秒 | 动作有预备、执行、结果需要看清 | 动作只是过场，且下一镜能承接结果 |
| 道具、文字、规则显影 | `standard / held`，约 1.2-2.2 秒 | 文字、细节、材质或异常必须可读 | 信息已在前镜读清，只需交出锚点 |
| 表演微反应、沉默、呼吸 | `standard / held`，约 1.5-2.8 秒 | 情绪变化靠眼神、肩膀、手指或呼吸完成 | 只是复述心理结论，没有可见变化 |
| 空间建立、群像关系 | `standard / held`，约 1.5-2.5 秒 | 观众需要重新定位空间、轴线或多主体关系 | 空间已建立，当前只需单一反应 |
| 高潮读秒、认知翻转、艰难判断 | `held / long_hold`，约 2.5-4 秒 | 需要让观众读懂结果、代价或规则翻转 | 高点已经由前后镜承托，当前不应重复 |
| 复杂调度、连续运镜 | `standard / held`，按动作完成，优先 1.5-3 秒 | 运动路径本身承担信息转交 | 组合运镜超过 2-3 个动作且无新信息 |
| 边界交出、声画余波 | `short / standard`，约 0.8-2 秒 | 需要给 `5-分组` 留可见锚点 | 会抢走当前表演停顿或信息可读性 |
| 对白/旁白承托画面 | 由台词量决定下限，常见约 1.5-4 秒 | 角色说话、旁白、画外音或对白落到反应画面上 | 台词已跨镜延续，当前镜只承担反应或交出 |

## Short Drama AIGC Bias

短剧·AIGC 模式的默认目标不是“充分停留”，而是“必要信息尽快成立”。执行时采用以下偏置：

- 默认优先 `short / standard`；`约3秒` 以上且 `tempo` 不为 `slow_burn/hold` 时，必须有明确的台词、读秒、表演变化、复杂调度、空间重置或高点证据。若 `tempo` 标记为 `slow_burn` 或 `hold`（如崩溃/震惊/醒悟/强忍情绪场景），时长由情绪释放节奏决定，不受此条约束。
- `held` 不是“更电影感”的同义词；只有观众需要等到某个可见变化发生，才允许使用。
- `long_hold` 是强例外，通常只服务重大认知翻转、关系停顿、压迫性长镜、明确长对白或项目显式要求。
- AIGC 工具片段时长不是单镜叙事时长。若视频工具要求 4 秒、5 秒或更长，应在 `5-分组` 或视频阶段把多个短分镜组合进同一生成片段，而不是把 `4-摄影` 的单镜叙事时值拉长。
- 若某镜在 AIGC 生成中容易主体漂移、动作发散或表演失控，优先缩短、拆镜或给清楚落点，不用长镜硬撑。

## Dialogue Duration Rule

对白台词量是镜头时值的硬输入，不是后期才考虑的附加项。命中以下情况时，必须形成 `dialogue_time_budget`：

- 当前 visual unit 是 `对白画面`、`旁白画面`、`独白画面` 或与对白/旁白相邻并明显承载说话过程的画面。
- 上一行或同组附近存在 `对白（角色）`、`独白（角色）`、`旁白` 等台词字段，且该画面是这句台词的可见承托。
- 当前画面虽然不是说话人正脸，但台词以画外音、声画分离或反应镜头方式继续存在。

台词量估算采用内部近似，不追求字幕级精确：

| speech_type | default speech rate | use |
| --- | --- | --- |
| 快速短句 / 惊慌喊话 | 约 5 字/秒 | 短促命令、惊叫、打断 |
| 常规对白 | 约 4 字/秒 | 普通中文对白默认值 |
| 低声、压迫、迟疑、情绪停顿 | 约 3 字/秒 | 恐惧、试探、关系高点、规则宣判 |

`dialogue_seconds_floor` 的内部估算：

```text
dialogue_seconds_floor = ceil(台词中文字数 / speech_rate) + pause_allowance
pause_allowance:
  0-0.5 秒：短促命令或硬切前台词
  0.5-1 秒：常规对白后的反应落点
  1-2 秒：迟疑、沉默、关系停顿或规则宣判
```

裁决规则：

- `display_seconds` 不得低于 `dialogue_seconds_floor`，除非台词明确跨多个分镜延续。
- 一句长对白可以拆成多个视觉分镜承托，但必须在 `shot_design_plan` 内说明每镜承载哪一段台词、反应或声画延续。
- 若一个视觉分镜只承载“听完对白后的反应”，而台词已在前镜完成，可按反应时值裁决，不必继承整句台词时长。
- 对话场景不能每句都机械切正反打；台词量决定最低时值，情绪压力和反应落点决定是否拆分为说话镜、听者反应镜、道具/空间压力镜。

### Long Dialogue Visual Budget

当上游提供 `long_dialogue_beat_map` 或 `long_dialogue_delivery_map` 时，`4-摄影` 必须形成 `long_dialogue_visual_plan`。长对白的时值不等于把整段对白塞进一个长镜头，而是把文本节拍、表演气口和镜头焦点分配到可连续观看的视觉链。

**裁决规则**：
- 每个长对白视觉镜头必须标明承载哪个 `beat_id`、哪一段 `exact_text_segment_ref` 或哪一个反应/沉默余波。
- 单镜可以承载多个连续短节拍，但必须形成 `trigger_merge_exception`：同镜内有明确框内变化、焦点接力、动作相位或反应变化，不损失观看结果。
- 长对白不默认 `long_hold`。只有一口气压迫、故意不给对手插话、关系停顿或复杂框内调度成立时，才允许单镜承载较长段落。
- 默认优先把长对白拆为：说话者状态镜、听者反应镜、手部/道具压力镜、空间关系镜、群像压力镜、画外声源/沉默余波镜；具体数量按节拍需要，不固定。
- 每 2-3 个长对白节拍至少应有一次非说话者视觉承托，除非上游明确设计为说话者压迫式独白且镜内能通过身体和空间变化维持观看。

**失败模式**：
- 一个 `分镜N（约X秒）` 正面拍说话者说完整段长对白，正文没有焦点变化、听者反应或框内动作推进。
- 多个长对白分镜只是“说话者中景 -> 说话者近景 -> 说话者特写”，没有观看信息变化。
- 每个节拍机械正反打，忽略手部、空间、沉默、群像或道具压力。

## Decision Procedure

1. 从 `shot_function` 判断当前镜头的首要时值任务：冲击、动作完成、信息可读、情绪停顿、空间定位、反应吸收或边界交出。
2. 读取 `rhythm_profile.tempo / density / movement_complexity`，初步决定时值等级；快节奏不等于每镜都短，慢节奏也不等于每镜都长。
3. 检查 `dialogue_time_budget`：当前镜是否承载对白、旁白、画外音或反应镜头中的台词延续；若命中，先算 `dialogue_seconds_floor`。
4. 检查 `screen_information_load`：文字、系统字、规则、微表情、复杂群像和空间重置必须给足可读时间。
5. 检查 `action_phase`：预备、执行、结果、反应是否能在当前镜内完成；不能完成时拆镜，已经完成时不拖长。
6. 检查 `peak_shot_profile`：认知翻转和关系高点优先读秒/停顿，动作高点优先冲击后钉镜，恐怖入侵优先断裂后给清楚落点。
7. 检查连续性和 15 秒分组风险：连续多个 `held / long_hold` 会拖慢组内速度；连续多个 `instant / short` 会让信息不可读。必要时让一个镜头收敛、另一个镜头承担读秒或反应。
8. 应用 `short_drama_aigc_duration_bias`：能用 `short / standard` 成立的，不升级为 `held`；`tempo=slow_burn/hold` 的情绪类镜头可进入 `held / long_hold`，但必须由可见微动态、静止压力、极慢运动或框内变化证明；能跨镜承托的长对白，不让单镜吞完整句；能用动作结果落点交出，不追加氛围尾巴。
9. 形成 `display_seconds`，并把最终正文写成 `分镜N（约X秒）:`；运镜速度、停点、句子长短必须服从该时值裁决。

## Duration Class Guide

| duration_class | estimated_seconds | use when | output cue |
| --- | --- | --- | --- |
| `instant` | 0.3-0.7 | 惊吓切点、闪现、急停确认、硬断裂 | 文字短，强调骤停、闪过、切到、撞上 |
| `short` | 0.7-1.3 | 单一动作、视线转交、反应入口、边界锚点 | 一句完成，清楚起落，不铺陈 |
| `standard` | 1.3-2.2 | 普通揭示、动作完成、空间关系承接 | 有明确路径和落点，信息能读清 |
| `held` | 2.2-3.5 | 表演停顿、规则读秒、空间重置、群像压力 | 使用静止、极慢、慢推、焦点停留或少量框内运动 |
| `long_hold` | 3.5-5 | 高潮读秒、关系暖点、重大认知翻转、压迫性长镜 | 必须有可见变化或压力累积，不可只是延长气氛 |

`estimated_seconds` 是内部估算，`display_seconds` 是正文落盘时长。正文可以使用近似整数秒或半秒，推荐 `约0.5秒` 到 `约4秒`；`约5秒` 以上必须有明确台词、长调度、重大认知高点或项目特殊要求。

## Split And Merge Rules

- 同一分镜既要“快速惊吓”又要“读清规则文字”时，拆成快速入侵镜和规则/反应读秒镜。
- 同一分镜只是重复上一镜已经读清的信息时，合并或缩短为交出锚点。
- 多分镜内部必须形成时值接力：短镜制造冲击，标准镜交代动作，held 镜给反应或信息读秒；不得每镜同长。
- 低信息动作不因慢推而自动变成长镜；长镜必须有动作完成、空间压迫、表演停顿或信息可读的理由。
- 高点不默认更长。动作高点可以短促，认知高点通常需要读秒，关系高点通常需要停顿，恐怖高点通常需要断裂后落点。
- 短剧·AIGC 模式下，普通氛围尾巴优先删除；若需要余波，写成清楚交出锚点或反应落点，不把单镜拖成长停。

## Output Projection

最终 `分镜N` 必须显式写近似时长，格式固定为：

```text
分镜1（约3秒）: <自然中文分镜明细>
```

同时，正文仍必须能从镜头语言读出时值，不能只靠括号里的秒数成立：

- `instant / short`：句子更短，动作更明确，落点更硬。
- `standard`：包含起点、路径和落点，信息能被看清。
- `held / long_hold`：通过静止、极慢运动、焦点停留、读秒、框内细微变化或反应延迟体现停留，而不是写“停留很久”。

## Review Questions

审查每个 `分镜N` 时必须追问：

1. 这一镜为什么需要这个时值等级？
2. 缩短一半会丢失什么？拉长一倍会增加什么，还是只会拖慢？
3. 该镜的时值是否和相邻镜头形成节奏接力？
4. 文字、道具、微表情、空间关系是否有足够可读时间？
5. 对白/旁白是否有足够时间被听完？是否需要跨镜延续或拆成说话/反应/空间压力镜？
6. `分镜N（约X秒）` 是否和文字中的运镜速度、停点、读秒感一致？
7. 是否出现连续多镜同长、同速、同密度，导致整段没有呼吸？
8. 当前时值是否会让下游 15 秒分组过挤或过慢？
9. 当前镜头是否只是沿用传统影视停顿？在短剧·AIGC 模式下，压短到 `short / standard` 是否仍然成立？
10. 当前 `约3秒` 以上的镜头是否有台词、读秒、表演变化、复杂调度、空间重置或高点证据？若 `tempo=slow_burn/hold`，是否能明确读出咬肌、眉心、呼吸、肩膀、手指、泪光、视线等可见微动态或框内压力变化？

## Anti-Patterns

- 只判断分镜数量，不判断每镜时值。
- 用“慢推”把低信息动作拖成长镜。
- 用“快速切换”带走必须读清的文字、道具或微表情。
- 每个分镜都写同样长度、同样速度，导致节奏平板。
- 高潮一律加速，或一律拉长，没有区分动作高点、认知高点、关系高点和恐怖高点。
- 逐集正文缺少 `分镜N（约X秒）`。
- 只机械写秒数，却没有通过画面运动、停点、台词承托和可读性体现时值。
- 对白画面不看台词量，导致台词还没说完就切走，或短句被拖成长镜。
- 把 AIGC 工具片段时长误当作单镜叙事时长，导致每个分镜都膨胀到 4-6 秒。
- 把传统电影的慢停顿默认搬到短剧·AIGC 输出里，导致过场、普通反应和氛围镜普遍偏慢。
