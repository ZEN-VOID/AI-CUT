# Natural Shot Detail Writing Contract

本文件定义 `3-摄影` 的最终成稿语气。它不削弱 `shot_design_plan`、节拍、连续性、功能性影视投影或摄影执行要求；它只规定这些判断如何被写成自然、可读、可拍的中文分镜明细。

## Core Rule

`shot_design_plan` 是内部工程图，`functional_payload` 是下游消费骨架，`sequence_profile` 是段落连续性参考，`分镜明细：` 是给创作链继续使用的镜头文字。最终正文必须让读者先看到画面和动作，再自然读出摄影策略；不得把内部字段逐项翻译成参数清单，也不得牺牲功能 payload 或逐画面点归属去追求句式好看。

## Internal vs Output

| internal must know | output should show |
| --- | --- |
| 景别、景深、视角、镜头类型、运镜速度 | 只显式写当前节拍最关键的 1-2 个摄影选择，其余藏进自然动作和画面关系 |
| 起点、路径、落点、交出点 | 读起来能反推即可，不要求每句都写“从/以/变化到/最终” |
| 段落级 `sequence_profile` | 只内化为当前块可消费的视觉母题、材质光色、进入点或交出点；不得把多个画面点揉成一条长镜 |
| `unit_ownership_map` | 当前 `分镜明细` 必须只写正上方画面句子拥有的主体、动作、道具/文字/身体锚点和对白承托 |
| 节拍动机、连续性入口 | 只在跳轴、强转场、跨空间或高点强化时短写动机；普通承接不机械解释 |
| 构图、光影、色彩 | 用具体物、位置、遮挡、亮暗和颜色落点表达，少用抽象形容词 |
| 主体、动作相位、下游消费点 | 必须在画面文字中具像存在，不能被“氛围”“电影感”“高级运镜”替代 |
| AI 视频执行稳定性 | 不显式输出完整提示词分栏，但成稿应能改写为镜头先行、方向清楚、光线有结果、表演有微动态的视频提示词 |

## Writing Style

- 句子优先短而清楚；一个 `分镜N` 通常 1 句，复杂高点最多 2 句。
- 每条分镜只抓一个主要观看动作：靠近、停住、转交、显影、压住、断裂或落地。
- 每条分镜先服从正上方画面句子的归属，再服从段落流畅；整段好看不能抵消当前块失主。
- 少用连接词堆叠；避免连续出现“从……以……变化到……最终……”。
- 不把“景别/景深/视角/镜头类型/速度”机械全列出来；只有当前画面离不开它时才显式写。
- 技术词服务画面，不抢画面。能写“黑板压到脸后方”，就不要写“低角度压迫构图增强制度感”。
- 光线写结果，不写口号。能写“右侧窗光照亮半张脸，左脸落进柔和阴影”，就不要只写“右侧柔光”或“电影感光线”。
- 方向写参照，不写含混位移。能写“从画面左侧进入，停在右侧三分之一处”，就不要只写“从左边进来”。
- 表演写泄露，不写心理标签。能写“咬肌慢慢收紧，手指抓住桌沿”，就不要只写“她很愤怒但压抑”。
- 保留中文口语的顺滑度：先给可见画面，再补必要摄影动作。
- 自然不是文学化。成稿必须能被下游抽取为生图/视频提示的主体、动作、构图、光色和运动。
- 材质和身体锚点优先服务当前画面点：湿木、鱼鳞、衣摆、脚趾、手指、酒葫芦等可以承接段落，但必须是当前字段里可见或被同一行承托的信息。
- 对白画面的身体锚点应落在说话、听话或同一行可见反应上；不能为了镜头顺滑借用后文尚未出现的动作。

## Good Output Patterns

- `长焦压扁后排课桌，镜头慢慢贴近他的手。指节发白时，背景冷白灯虚成一片。`
- `镜头不动。她从门口走进画框，红色胸牌先被灯照亮，随后才露出没有表情的脸。`
- `教鞭声先进来，画面切到黑板最下方的粉笔灰；镜头轻轻上摇，停在新写出的规则字上。`
- `手持跟着人群后退半步，晃动在门框前突然稳住，留下走廊尽头那块红光。`
- `镜头从她正前方极慢靠近，前两秒几乎不动；咬肌收紧时，右侧窗光只留住半张脸，左脸沉进桌沿投下的阴影。`
- `他从画面左侧穿过前景，停在门框内侧；走廊冷白灯照亮肩线，脸被门框阴影切掉一半。`

## Stiff Output Anti-Patterns

- `从中景平视起，以长焦浅景深慢推变化到近景，最终形成压迫感。`
- `使用低角度、浅景深、慢速推轨和冷白光，表现人物紧张。`
- `镜头以丝滑转场承接上一镜，并通过组合运镜强化戏剧效果。`
- `先建立空间关系，再进行焦点转移，最后完成情绪落点。`
- `镜头与构图：手持中近景。人物与动作：男人很紧张。光线：电影感侧光。不要夸张表情。`
- `人物向前走，光线很好，气氛压抑。`

## Repair Rule

若连续 2 条以上 `分镜N` 出现相同句法骨架、参数堆叠、抽象效果词或明显模板腔，必须回到 `N7-INJECT` 重写成自然成稿；不得只替换少量形容词。

若一句话好读但无法抽取 `visible_subject / action_phase / camera_movement_plan / composition_anchor / light_color_material / continuity_handoff`，不是自然成稿问题，而是功能性投影失败；必须先回到 `functional-cinematic-projection-contract.md` 和 `N6.5-SHOT-PLAN`。

若一句话能抽取主体和动作，但动作先于镜头、方向含混、光线只写光源或抽象效果、情绪没有面部/身体/呼吸/手部/视线微动态，必须回到 `ai-video-prompt-execution-contract.md`；不得把分栏提示词或“不要夸张表情”这类命令式负向约束直接塞进 `分镜明细`。

若一组分镜读起来像完整段落预告，但其中某条无法回指正上方画面句子，或提前吞入后文主体动作、对白反应、记忆段、道具揭示、跨场景连接方案，不是润色问题，而是逐画面点归属失败；必须回到 `visual-sequence-alignment-contract.md`、`unit_ownership_map` 和 `shot_design_plan` 重写。
