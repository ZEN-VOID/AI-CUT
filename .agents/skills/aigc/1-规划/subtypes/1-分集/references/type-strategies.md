# Type Strategies

Canonical module for `1-分集` typed routing strategies.

- target skill: `.agents/skills/aigc/1-规划/subtypes/1-分集`
- canonical scope: `P1 / P2 / P3 / unknown`

## 类型策略总览

### 输入处理基线

- 全部路由只负责“按集切分 + 证据落盘”，不负责清洗、改写或剥离上游已有的场次、镜头、运镜描述。
- 若输入已是分镜脚本或混合剧本，相关场次、镜头组、运镜提示、视角切换与时间跳转都视为原文证据，可用于边界判断，也允许保留进入 `【剧本正文】`。
- 标准剧本化或分镜结构化属于下游显式任务；若没有明确下游要求，本阶段不得把输入擅自改写成纯小说叙述。

## P1 预设驱动

### 触发信号

- 用户明确指定总集数、关键断点或不能拆开的剧情包。
- `north_star.yaml` 或 `init_handoff.yaml` 已写明总集数规模、关键卡点或交付波次。
- 预设足以形成唯一规模约束。

### 必备证据

- 预设原句或字段位置
- 总集数、关键边界、禁拆条件、合并条件
- 若预设只部分完整，必须明确缺口

### 执行动作

1. 抽取全部预设条款，形成“预设约束表”。
2. 先锁总集数与硬剧情包，再求解局部切点。
3. 若原文已带场次、镜头或运镜描述，保留其证据身份，不做预清洗。
4. `P2/P3` 在这里只能辅助求边界，不能反推翻预设规模。
5. 每个最终边界都要回答：
   - 是否满足预设条款？
   - 为什么落在这个结构点而不是相邻点？

### 回退条件

- 预设互相冲突
- 只剩模糊愿景，没有可执行约束
- 用户最新明确要求与旧预设冲突且无法解消

### 示例

- `north_star.yaml` 写明“首季 6 集，每集保留完整案件闭环”，则先锁定 6 集，再按案件推进单元求解边界。

## P2 结构驱动

### 触发信号

- 没有可直接执行的预设边界。
- 源文本已有稳定的章、幕、场、seq、镜头组或视角分段。
- 天然断点足以承担大部分集边界。

### 必备证据

- 结构清单
- 每个结构块的主事件、字数、事件完整性
- 合并/拆分理由

### 执行动作

1. 先生成结构清单，不直接按阅读体感切集。
2. 给每个结构块标记：
   - 结构强度
   - 事件完整性
   - 可并/可拆性
3. 以天然断点为第一候选边界。
4. 只在粒度明显过粗或过细时，才允许合并或块内二次切分。
5. 若结构信号来自场次、镜头组或运镜提示，仍按原文证据处理，不得先行剥离。
6. 若需要微调，只能在结构链路上借 `P3` 补节奏，不得跳过结构证据。

### 回退条件

- 标题存在但叙事价值极弱
- 结构块内部松散，无法承担边界解释
- 两轮合并/拆分后仍不能形成可交接的分集表

### 示例

- `第1-2章` 共同构成潜入任务的准备与进入，可合并为同一集；`第3-4章` 构成任务升级与首次失败，也应合并成一集。

## P3 节奏驱动

### 触发信号

- 用户与 `0-Init` 没有提供可执行边界。
- 源文本缺少稳定结构，或结构标记只有形式编号。
- 必须依赖冲突闭环、反转、悬念与代价显现点来补位。

### 必备证据

- 文本密度分级：`low / medium / high`
- 冲突触发点、升级点、反转点、悬念点
- 边界前后为何比相邻位置更适合作为集尾

### 执行动作

1. 先做密度分级，再确定字数窗口：
   - `low`: 1500-2200 字
   - `medium`: 1200-1800 字
   - `high`: 900-1400 字
2. 先用冲突单位拼出粗分集草案，再用字数窗口微调。
3. 每一集至少满足以下之一：
   - 有 1 个核心冲突闭环
   - 有 1 个关键反转
   - 集尾落在悬念点、决策点或代价显现点
4. 即使最终主要依赖节奏补位，原文已有的场次、镜头或运镜描述仍可保留进逐集正文，不必先小说化。

### 回退条件

- 连续两轮补位仍没有成立边界
- 只能得到按字数均分的机械切法

### 示例

- 对没有章节标题的连续叙述，可按“接到委托 -> 潜入失败 -> 再次布局 -> 真相揭露”四个冲突阶段组织边界。

## Unknown / 暂停澄清

- 当 `P1/P2/P3` 都无法形成唯一主裁决时，不得继续切文。
- 必须输出：
  - 当前缺失的输入
  - 为什么无法锁主路由
  - 继续分集所需的最小补充信息
