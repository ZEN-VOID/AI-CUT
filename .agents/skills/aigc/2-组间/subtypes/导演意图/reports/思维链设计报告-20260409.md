## 2. 基线与触发原因

- 基线事实：
  - 旧版目标文件已经有 `Think-Think Design Snapshot`、字段主表、thought pass 与 validation matrix。
  - 但它仍停在上一代显式快照层，缺少更完整的 `模式与对象`、`可见 / 隐藏分层`、`工具后反思` 与 `Gate Summary Contract`。
  - 现有 `FIELD-DI-01` 到 `FIELD-DI-05` 五字段接口不宜破坏。
- 触发原因：

  - 当前问题不是“没有思维链”，而是“已有思维链，但还没完全升级到最新合同”。

## 3. 启发式工作链

- 先锁 `FIELD-DI-01` 与 `FIELD-DI-02`，没有集级命题和观看契约时，禁止提前美化组级文案。
- 先砍项目口号复读、类型词堆砌、组间可互换文案和自嗨式导演腔。
- 真正的比较尺不是“谁更文艺”，而是谁最能同时锁观看契约、拉开组级差异、降低 `3-明细` 的继承成本。
- 站在 `3-明细` 与部门执行者的消费立场判断，而不是站在文档作者的表演立场判断。
- 关键上游真源读完后，先判断是否需要回上游补件，再决定是否进入组级构思。
- 每条保留判断都必须落到 `FIELD-DI-01` 到 `FIELD-DI-05`、`Gate Summary`、`Fail Code` 或 `Rework Entry`。

## 4. 三轴三重设计

### 三轴

- `方向轴`：锁定这集如何被观看。
- `成立轴`：确认导演判断被 `全局风格 / 类型元素 / 分组容器` 共同支撑，并已翻译成部门后果。
- `优选轴`：在成立解里，优选最能帮助整集收束、组级拉开与下游继承的方案。
- `硬门禁轴`：缺容器、组间可替换、无部门后果、越权写成 shot list 时直接否决。

### 三重

- `粗裁决`：收窄出唯一的 `本集总导演命题 + 本集观看方式`。
- `细裁决`：收窄出不可互换的 `分组导演构思`。
- `离散裁决`：收窄出 `情绪推进 + 下游加载提示`，并把不可稀释项交给 `3-明细`。
