## Purpose

本细则定义 `2-编导` performance layer 中角色发展弧线的表演规划语义：从第一集到当前集，角色经历了什么变化，表演如何体现这种变化。它不是剧作层面的故事大纲或人物小传，而是把角色弧线翻译为可执行的表演变量——声音、身体、微表情、空间占位从之前集到本集的具体差异。

核心原则：角色变化不靠旁白说"他变了"，也不靠台词解释"我已经不是从前的我了"。观众必须通过同一类动作在不同集里做出不同选择来感知角色弧线。表演规划的任务是确保演员在每一集的表演状态与角色弧线位置匹配，让角色"变了"成为观众自己看出来的结论。

## Ownership

- 本文件拥有角色弧线的表演层规划：弧线阶段定位、表演强度曲线、弧线节点的表演标记法、多角色弧线交织时的表演优先级。
- `psychological-reaction-contract.md` 负责单场景内心理反应的可感知化；本文件负责跨集的心理-表演连续性，确保角色的微表情基线、身体习惯和空间行为反映弧线演变。
- `actor-performance-control-contract.md` 提供五层表演控制公式；本文件决定五层中的参数从第一集到当前集发生了哪些系统性变化。
- `performance-and-scene-craft-contract.md` 负责场面调度和演员任务；本文件决定弧线节点场景中的特殊调度需求（如转折点需要的隔离、终局需要的收敛）。
- `../references/field-routing-and-audio-visual-contract.md` 提供角色弧线的故事层定义；本文件只负责把故事层弧线翻译为表演层变量。
- `../references/` 中的剧作大纲和人物关系文件定义角色走向；本文件只负责把走向翻译为"演员这一集怎么演"。

## Character Arc Profile

每个主要角色在每集的弧线定位应形成以下结构：

| field | definition | allowed_values | examples |
| --- | --- | --- | --- |
| `arc_stage` | 角色在本集处于弧线的哪个阶段 | `初始` / `上升` / `转折` / `下降` / `终局` | "上升"——角色正在经历信念动摇但尚未做出选择 |
| `arc_direction` | 弧线整体运动方向 | `成长` / `堕落` / `觉醒` / `崩溃` / `坚守` | "觉醒"——角色从蒙蔽走向看清真相 |
| `key_arc_moment_in_episode` | 本集最能体现弧线变化的具体 moment | 场景编号 + 触发事件 + 表演变化描述 | "S3：林寂看到黑板上第三条规则被改，手指在课本边缘压出白痕后又松开——第一次没有选择逃避" |
| `performance_evolution` | 声音/身体/微表情/空间占位从之前集到本集的系统性变化 | 四维度对比描述 | "声音：从低沉回避变为偶尔提高音量；身体：从缩肩变为肩线展开；微表情：从闪躲变为凝视；空间：从退到角落变为站到桌前" |
| `arc_anchor_scene` | 本集最能体现弧线的关键场景 | 场景编号 + 锚定理由 | "S7：独处场景——角色第一次主动翻看之前不敢碰的物件，手指从犹豫到果断的过渡就是弧线本身" |

## Arc Stage To Performance Intensity Map

弧线阶段决定了表演强度的基准线和变化模式：

| arc_stage | intensity_baseline | change_pattern | performance_direction |
| --- | --- | --- | --- |
| `初始` | 较低强度，角色尚在舒适区 | 缓慢建立，微小细节累积 | 建立角色的"日常表演基线"——说话速度、手势习惯、空间偏好 |
| `上升` | 中等强度，冲突开始积累 | 渐进增强，压力逐集叠加 | 基线开始出现裂痕：说话变快或变慢、手势变多或变少、空间偏好被挑战 |
| `转折` | 高强度，角色面对关键选择或遭遇 | 突变——表演状态在转折场景中出现可感知跳变 | 同一动作做出不同选择（之前松手现在握紧，之前回避现在直视） |
| `下降` | 中高强度，转折后果开始显现 | 波动——角色在新状态和旧习惯之间拉扯 | 表演出现不稳定：声音在两种模式间切换、身体在两种姿态间摇摆 |
| `终局` | 收敛或爆发，取决于弧线方向 | 汇聚——所有累积变化在最终场景中定格 | 表演到达弧线终点：要么平静到极致（收敛），要么释放到极致（爆发） |

### Performance Intensity Curve Rules

- 初始阶段的表演强度不应超过转折点的70%，否则后续没有上升空间。
- 转折点必须有一个表演突变——观众不需要被告知"这是转折"，但必须能从表演中感知到状态跳变。
- 终局的收敛或爆发必须有前期铺垫，不能从天而降。
- 同一集内如果有多个弧线阶段并存（如角色A在上升、角色B在转折），必须通过表演强度差异让观众区分轻重缓急。

## Arc Node Performance Marking

如何在表演层面让观众感知到"角色变了"：

### The Same Action, Different Choice

弧线标记的第一手段：让角色在不同集面对同类情境时做出不同选择。

| marking_technique | description | example |
| --- | --- | --- |
| 同动作不同执行 | 同一个动作（如接过物件），第一集是犹豫接、本集是果断接 | 第一集：手伸出又缩回，停了两拍才接；本集：直接伸手接住，没有停顿 |
| 同空间不同占位 | 同一个空间（如教室），第一集坐后排角落，本集坐前排中央 | 角色从画面边缘走进画面中心，空间位置就是弧线的可视化 |
| 同沉默不同质感 | 同样是沉默，第一集是恐惧的沉默，本集是坚定的沉默 | 第一集沉默时手指在抖、呼吸很浅；本集沉默时呼吸平稳、目光没有移开 |
| 同台词不同节奏 | 同一句或类似台词，节奏、重音、气口完全不同 | 第一集说"我知道了"时尾音下沉、视线移开；本集说同样的话时尾音上扬、直视对方 |
| 同关系不同距离 | 同一对关系，物理距离和身体朝向发生系统性变化 | 第一集与对手保持一臂距离，身体微侧；本集距离缩短到半臂，身体正对 |

### Performance Fossil Technique

在弧线转折后的场景中保留旧表演模式的"化石"——角色已经变了，但偶尔会闪回旧习惯。这比彻底抹除旧模式更真实。

```yaml
# 正确：转折后保留旧模式化石
# 角色已从回避型转变为直面型
角色动作：林寂直视黑板上的新规则，没有移开目光；但他左手的拇指在课本边缘蹭了一下——那是他以前回避时的习惯动作。

# 错误：转折后旧模式完全消失
角色动作：林寂直视黑板上的新规则，没有任何犹豫。
```

## Multi-Character Arc Priority

当多个角色在同一集到达弧线节点时，必须确定表演优先级：

### Priority Rules

| scenario | priority_rule | execution |
| --- | --- | --- |
| 主角弧线与配角弧线同时到达节点 | 主角优先，配角用反应承托 | 配角在主角的弧线场景中做出反应性表演，自己的弧线节点放在独立场景或主角不在场的场景 |
| 两个主要角色弧线同时到达节点 | 以叙事功能更关键的一方为焦点 | 另一方用身体/声音变化标记自己的弧线，但不抢注意力焦点 |
| 多角色同时经历同一事件但弧线不同 | 群戏中用表演强度差异区分 | 弧线阶段更高的角色表演更强烈，其他角色用反应层次（前景反应者/中景知觉者）标记自己的弧线 |
| 角色弧线交叉（A的变化触发B的变化） | 先行者为驱动，后行为反应 | A的弧线节点是主动表演，B的弧线节点是反应性表演 |

### Anti-Clustering Rule

- 同一集中不应有超过两个主要角色同时到达弧线转折点或终局。
- 如果剧本层面确实多角色同时到达，必须通过场面调度分散：一个在前景完成弧线，其他人在背景或中景用单一标记技术（如空间占位变化或沉默质感变化）简化处理。
- 所有角色在同一集同时到达弧线高峰是严重的表演规划失败——观众无法同时关注多个弧线高潮。

## Cross-Episode Performance Continuity

角色弧线的表演规划必须维护跨集连续性：

### Baseline Tracking

| tracking_variable | what_to_track | how_to_maintain |
| --- | --- | --- |
| 声音基线 | 语速、音量范围、气口频率、尾音习惯 | 每集开篇场景的说话方式应与上集结尾的状态衔接 |
| 身体基线 | 肩线位置、手势频率、步态、坐姿习惯 | 新集开始时角色的身体应带有上集结尾状态的残留 |
| 微表情基线 | 眉头松紧、眨眼频率、嘴唇状态、目光停留时间 | 角色的"默认表情"应反映累积经历 |
| 空间基线 | 与他人的物理距离、偏好的位置、进入空间的方式 | 角色的空间行为应反映关系变化和自信心变化 |

### Residual Effect Rule

- 如果上集结尾角色经历了重大情绪事件，本集开场表演必须带有生理残留（详见 `physiological-realism-contract.md`）。
- 弧线变化不是开关式的——角色不会在两集之间从A状态跳到B状态。变化必须有过渡痕迹。
- 回忆场景或闪回场景中角色的表演应匹配该时间点的弧线位置，而不是当前位置。

## Failure Modes

| failure | symptom | fix |
| --- | --- | --- |
| 角色表演跨集无变化 | 第一集和第五集的表演完全一样，观众看不出角色经历了什么 | 检查 `performance_evolution` 是否有至少两个维度的具体变化 |
| 弧线转折没有表演突变 | 编剧层面写了"角色做出重大决定"，但表演上没有任何可感知跳变 | 在转折场景中使用"同动作不同执行"或"同沉默不同质感"标记技术 |
| 所有角色在同一集同时到达弧线高峰 | 多个角色同时爆发，观众不知道看谁 | 使用 Multi-Character Arc Priority 的反聚集规则分散处理 |
| 转折后旧模式完全消失 | 角色在转折后像换了一个人，没有任何旧习惯残留 | 使用 Performance Fossil Technique 保留旧模式碎片 |
| 弧线变化依赖旁白或台词说明 | "我知道我不再是以前的我了"——用台词替代表演 | 删除说明性台词，用表演标记技术让观众自己感知 |
| 跨集连续性断裂 | 新集开场时角色的状态与上集结尾完全对不上 | 检查 baseline tracking 的四个变量是否与上集衔接 |
| 终局表演没有前期铺垫 | 最终场景的爆发或收敛像是从天而降 | 从转折点开始逐集铺垫表演强度的走向 |

## Reusable Heuristics

- "角色变化不靠旁白说'他变了'，靠的是同一种动作在不同集里做出不同选择。"
- "转折点需要表演突变，但突变不等于突然——观众应该能从前期铺垫中预感到变化正在逼近。"
- "旧习惯不会一夜消失。转折后的角色应该偶尔闪回旧模式，这才是活人。"
- "多角色弧线交织时，永远先回答'这一秒观众应该看谁'，再决定谁的弧线表演更强烈。"
- "弧线是累积的：每一集的表演都在为下一集的变化做铺垫，不存在孤立的表演决定。"
- "终局的收敛是把之前所有的爆发收回体内；终局的爆发是把之前所有的收敛一次释放。无论哪种，前面必须有东西可收或可放。"
