# 北京电影学院 Agent Usage Protocol

## 默认读取顺序

1. 先判任务属于导演组织、镜头组织还是摄影支撑。
2. 先读最贴近问题的一级证据，再读二级补强。
3. 只有当项目内已有成熟范式时，才调用 `项目级动态最佳实践.md` 做风格收束。

## 何时必须先读知识库再发言

- 当要判断“为什么这场戏要这样组织”时，先读 `导演手册/`。
- 当要判断“镜头怎么拍、怎么切、怎么调度”时，先读 `分镜脚本/`。
- 当要判断“光色、焦段、构图和视觉策略如何支撑叙事”时，先读 `电影摄影/`。
- 当任务是对话场景、权力关系或多人站位时，先读 `一流对话场景.md`。

## 哪些内容可以作为公开依据

- 可以公开引用：
  - 景别、视点、轴线、构图、机位、走位、镜头意图、光色动机、节奏组织这类可验证规则。
- 应谨慎公开：
  - 来自 `项目级动态最佳实践.md` 的项目内经验，只在命中相似场景时引用，并说明这是项目经验，不是通用教材定律。

## 哪些内容只能作为潜台词支撑

- 由多份教材共同抽象出来的学院气质，可以用于加强判断语气，但不能伪装成某一页教材原句。
- 对用户方案成熟度的内部判断，只能影响建议的锋利度，不能变成公开羞辱。

## 禁止硬套

- 不要把任何单条规则机械套在所有场景上。
- 不要把对话场景技巧直接套进动作段落。
- 不要把摄影风格词汇当成剧情结构替代品。
- 不要把项目级最佳实践当作跨项目通用真理。
