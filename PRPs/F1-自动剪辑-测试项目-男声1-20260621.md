# F1 PRP: 测试项目自动剪辑（男声1）

## Goal

参照 `projects/测试/示例` 的短视频处理节奏，使用 `projects/测试/素材/视频` 中的内容视频和工具使用视频，配合用户文案与 `男声1.mp3` 旁白，生成一条字幕与旁白对齐、内容画面和工具画面合理组合的硬字幕成片。

本 PRP 对应 `$F1` 的 `plan_only` 路由：本轮只落执行计划，不直接渲染。

## Execution Stages

1. `N1-INTAKE`
   - Re-check all input paths.
   - Read `视频说明.yaml` and verify every selected `videos[].segments[]` points to a real file.
   - Confirm existing result files; do not delete or overwrite old artifacts.

2. `N2-PRP`
   - Use this PRP as the execution plan.
   - Before rendering, require explicit user confirmation to move from plan to execution.

3. `N3-REFERENCE-RHYTHM`
   - ffprobe all 5 reference clips.
   - Summarize target rhythm from reference range: short-form voiceover, about 42-60s, fast hard cuts, frequent visual proof shots, strong ending hook.
   - Reference clips influence timing and style only; they are not source footage for final.

4. `N4-SUBTITLE-TIMELINE`
   - `男声1.mp3` is the master clock; target final duration is `49.15s +/- 0.25s`.
   - Script text is the subtitle truth, including口语、粗口 and the literal phrase `别 time 追着我要工具了`; do not silently correct suspected typos.
   - Preferred path: Volcano ASR word timestamps -> align script spans to ASR word/speech spans -> build `master_male1.srt`.
   - Fallback path: `silencedetect` speech intervals -> LLM semantic chunking -> bind each subtitle cue to a real speech interval -> split only intervals longer than 4s.
   - Forbidden: full-video proportional subtitle timing as final output.
   - Required evidence: `dialogue_alignment_male1.json` with cue text, `script_span`, `audio_span`, source (`asr_word_span` or `speech_interval`), and verdict.

5. `N5-VISUAL-PLAN`
   - Select visual clips from `视频说明.yaml` at `segment_id` level, not just by file name.
   - Target balance: about 45-55% tool display and 45-55% AIGC content footage.
   - Keep category switches motivated by the script; avoid rapid tool/content alternation every second.
   - Every EDL range must record:
     - `video_id`
     - `file`
     - `segment_id`
     - `source_start`
     - `source_end`
     - `output_start`
     - `output_end`
     - `selection_reason`
     - `subtitle_risk`

6. `N6-RENDER`
   - Output: horizontal `1280x720`, 30fps, H.264 video, AAC audio.
   - Drop source video audio unless a separate bed is explicitly requested; voiceover is the main and only semantic audio.
   - Add short audio fades at segment joins if any source audio bed is retained; otherwise keep voiceover continuous.
   - Burn subtitles last in the final filter chain.

7. `N7-VERIFY`
   - Full decode check.
   - ffprobe final duration, resolution, fps, video stream, audio stream.
   - SRT validation: sequence, no overlap, comma milliseconds, max cue <= 4s.
   - Dialogue alignment validation: every cue maps to a script span and an audio span.
   - Frame checks: first cue, midpoint cue, tail cue, longest cue, and any selected segment with `subtitle_safe_zone.risk_level: high`.

8. `N8-REPORT-WRITEBACK`
   - Write execution report.
   - Append `project.md` session summary.
   - Run Source Sync Check; only update skill/source contracts if a new reusable failure mode appears.

## Visual Rules

- Tool-display segments are high subtitle-risk; prefer shorter 2.5-4.5s windows and verify subtitle readability.
- Content battle segments carry emotion and proof-of-output; place them on rhetorical impact words, not on detailed tool explanations.
- Use hard cuts or short flash-match cuts; avoid decorative transitions unless the reference analysis shows a clear need.
- Do not use `projects/测试/示例` as final footage.
- Do not mutate any file under `projects/测试/素材` or `projects/测试/示例`.

## Subtitle Style Plan

- Default delivery platform assumption: Chinese mobile/social platform, so hard subtitles are preferred.
- Current ffmpeg has both `ass` and `subtitles` filters available.
- Start style: PingFang SC or system CJK font, white text, black outline, no opaque box by default.
- Line discipline: Chinese line length about 12-15 chars per line; two short lines are preferred over one long line.
- Because tool clips have interface text, execution must test `MarginV` and vertical placement on actual selected frames before full render.

## Fallbacks

- ASR unavailable: use speech-interval fallback, but bind subtitles interval-by-interval; never use whole-duration proportional timing.
- ASR text conflicts with script: keep user script as subtitle text; use ASR only for timing evidence and report the mismatch.
- libass unavailable: switch to a libass-enabled ffmpeg build or the subtitle-burning skill fallback before promising full hard-sub render.
- Reference rhythm cannot be machine-detected: use ffprobe, sampled frames, and written observation notes as rhythm evidence.
- Selected tool segment causes subtitle clash: move subtitle position, split the cue shorter, or replace with a lower-risk segment.
- Existing final path conflict: write to `_男声1_final.mp4` by default; only overwrite generic final after backup.

## Acceptance

- Final duration differs from `男声1.mp3` by no more than `0.25s`.
- Voiceover is the main audio track.
- Subtitles match the user script text and are burned into the final video.

- `dialogue_alignment_male1.json` proves cue-to-audio/script alignment.
- `edl_male1.json` records segment-level material provenance from `视频说明.yaml`.
- Content footage and tool footage both appear in meaningful script-matched positions.
