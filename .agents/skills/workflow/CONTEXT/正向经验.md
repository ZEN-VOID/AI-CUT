# 正向经验

本文件只保存从正例抽象出的可复用模式，不承载 workflow 规范真源。

## PE-WF-01 流程型教学的目标教学形态

- 适用范围：AIGC 创作、工具链实操、软件流程、内容生产流水线等流程型教学视频。
- 模式：优先构建 `结果/学习承诺 -> 明确步骤路线图 -> 分步骤实操演示 -> 关键选择为什么这样做 -> 重复操作折叠为代表样本 -> 参数/素材保存提醒 -> 最终效果证明`。
- 复用条件：源素材中存在足够的结果展示、步骤表达、工具操作、参数说明或最终效果证据；若某一环节缺失，应在 plan/report 中标记 unavailable，而不是凭空补写。
- 验证方式：N2 产出 teaching-role tags；N3 产出 teaching-shape summary；N6 检查是否缺少源素材支持的 roadmap、代表性演示、参数/理由或 final proof。
- 晋升状态：已晋升到 `SKILL.md` 和 `templates/output-template.md`；后续若出现不同类型教程的稳定目标形态，可拆分更多 segment role。

## PE-WF-02 教学字幕先分类再定样式

- 适用范围：带口播字幕、章节卡、步骤标签、参数说明、资源提醒或结果证明的教学视频。
- 模式：先判断文字覆盖层承担的教学角色，再用稳定字体栈、字号层级、位置和安全区区分，不用随机字体或装饰动画制造层级。
- 复用条件：源 UI、屏幕操作或最终结果需要被学习者看清；字幕必须辅助理解而不是遮挡证据。
- 验证方式：N5 写出 subtitle-style spec；N6 抽样检查可读性、对比度、两行限制、时间同步和是否遮挡操作/结果。
- 晋升状态：已晋升到 `SKILL.md` 的 `Subtitle Style Contract`、`templates/output-template.md`、`review/review-contract.md` 和 `test-prompts.json`。

## PE-WF-03 补充配音是素材生成不是内容生成

- 适用范围：原声噪声大、讲解啰嗦但画面高价值、精简字幕需要转成旁白、两个源片段之间需要短桥接讲解的教学视频。
- 模式：先在 N3 由 LLM 基于源素材/指南写出配音文案与使用理由，再在 N4 调用 `mmx-cli` 生成音频文件，默认落到 `projects/素材/` 的版本化文件名，随后把该音频当作可追踪素材进入 N5 重组剪辑。
- 复用条件：补充配音能提升教学清晰度，并且不会遮盖源素材中有价值的讲解、操作声音或判断依据。
- 验证方式：检查 voiceover script、generated-audio manifest、mmx 输出路径、音频文字对齐、混音区间和 N6 抽样 QA。
- 晋升状态：已晋升到 `SKILL.md` 的 `Supplemental Audio Material Contract`、Peer Skill Routing、`templates/output-template.md`、`review/review-contract.md` 和 `test-prompts.json`。
