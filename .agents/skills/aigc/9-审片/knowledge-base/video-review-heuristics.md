# Video Review Heuristics

- 先看视频实际内容，再决定是否需要改组；不要从 prompt 预设里反向美化素材。
- 一条 15 秒视频如果出现 4 个以上独立叙事事件，通常不是“丰富”，而是分镜组过载。
- 书册、名册、符纸近景容易生成伪字；除非剧情必须读字，否则让文字作为纹理而非可读信息。
- 人物近景、手部特写和门口剪影很容易把环境建立镜头变成剧情蒙太奇。
- 同组多个变体都错，优先查上游；只有一个变体错，优先 rerun。
- prompt 清楚但模型未执行，通常先 rerun；prompt 本身缺焦点、矛盾或过载，才修 `5-分组` 或 prompt 组装。
- 审片要反平庸：无硬伤不等于通过，通用库存感、无记忆点、无叙事推进的素材应标为创作质量弱。
- 用户提供好/坏示例时，先提炼维度再评价目标片，避免把“像不像参考”退化为一句主观判断。
