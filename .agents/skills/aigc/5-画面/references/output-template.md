## 上游共享真源

- 唯一 episode/group/shot 真源：`projects/<项目名>/编导/第N集.json`
- shared schema 真源：`.agents/skills/aigc/_shared/director_episode_output.schema.json`
- `5-画面` 各子路径只允许消费、摘录或派生该共享真源，不得再私造第二份导演主文件

## 阶段级硬规则

1. 三个子目录名都不带数字前缀，仓库默认把它们视为 `unordered` sibling。
2. 正式执行时不得三个子路径一起盲开，必须按任务目标裁决唯一主入口。
3. 当用户只说“做 5-画面”时，默认先进入 `分镜故事板`；`分镜帧` 与 `漫画` 解释为同源分支，而不是其下游。
4. `分镜帧` 与 `漫画` 默认直接消费 `编导/第N集.json + 4-主体 + 已有 5-画面 资产`，必要时可回读 `3-明细/第N集.md`，但不以 `分镜故事板` 为必经前置。
5. 不得只有图片没有 prompt 包、来源锚点或一致性说明。
6. 不得凭空重写上游镜头事实；所有画面产物都必须能回链到既有脚本锚点或既有图像资产。

## 阶段验收最小面

1. 已说明为什么当前任务属于 `5-画面` 而不是其他阶段。
2. 已说明本轮权威输入文件、唯一主入口与排除其他 sibling 的理由。
3. 已说明 prompt 组合、一致性锚点与本轮交付模式。
4. 已说明本轮产物落点与阶段级 `validation-report.md`。
5. 已给出默认下一步 handoff 方向。
