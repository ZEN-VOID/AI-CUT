# Character Identity Type Map

## 类型包加载边界

- 每次调用本技能时，必须依据本文件识别并加载同目录 `types/` 中选中的类型包（单选或多选）。
- `types/` 中命中的类型包作为固定上下文加载；`knowledge-base/` 只作为按需检索、切片或向量召回的知识库，不替代类型包。


## Identity Types

| type_id | observed_form | merge_strategy | evidence_needed | output_handling |
| --- | --- | --- | --- | --- |
| `ID-NAME` | 明确姓名 | 作为 canonical 名称优先候选 | YAML 原值即可，冲突时查同组正文 | `名称` 写姓名 |
| `ID-ALIAS` | 绰号、代号、昵称 | 归并到已确认姓名或最稳定称呼 | 同组正文、项目上下文或连续出现证据 | `主名（别名：绰号）` |
| `ID-ROLE-TITLE` | 职务或身份称呼，如老师、班长 | 仅在指向唯一主体时归并 | 同组正文明确指向或项目设定确认 | 低置信度则保留独立或报告待确认 |
| `ID-RELATION` | 亲属或关系称呼 | 归并到姓名，或作为稳定称呼保留 | 同组正文关系链可确认 | `主名（别名：妈妈）` |
| `ID-PRONOUN` | 他、她、那人等代称 | 只在同组上下文唯一回指时归并 | 同组正文明确回指 | 无法回指则不强行入表 |
| `ID-GROUP` | 学生们、村民、路人等群体 | 判断是否为设计主体 | YAML 角色字段与下游资产需要 | 可作为群体角色，或报告不纳入 |
| `ID-AMBIGUOUS` | 含糊称呼或冲突称呼 | 不强归并 | 需要人工确认 | 执行报告风险区 |

## Decision Priority

1. 用户或项目记忆已确认的 canonical 名称。
2. 上游 YAML 中出现的明确姓名。
3. 最具体、最早出现、可复查的稳定称呼。
4. 群体设计主体名称。
5. 待确认风险，不强行落入 canonical 行。

## Anti-Patterns

- 只因两个条目同为 `老师` 就合并。
- 只因代词性别相同就合并。
- 只因同一场景出现就合并。
- 为了清单整洁删除低置信度风险。
