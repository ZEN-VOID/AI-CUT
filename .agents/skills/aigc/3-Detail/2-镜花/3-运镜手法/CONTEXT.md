## Purpose & Loading Contract

- 本文件保存 `3-运镜手法` 的经验层。
- 命中本 skill 时必须和同目录 `SKILL.md` 一起加载。

## Repair Playbook

1. 先锁前置：当前 root、`分镜构图`、可选 `摄影美学` 是否已就位。
2. 先做静镜基线，再决定镜头是否真的要动。
3. 回读 `references/电影镜头调度-运镜判型.md`，把默认路线归到单一运动家族。
4. 默认路线锁定后，再串行细化 `变化 -> 组合 -> 速度`。
5. 所有 side input 先过兼容性过滤，不兼容就放弃。
6. 先写本地 `camera_patch`，再投影为 canonical `运镜手法`。
7. 若需要挑战变体，只保留比较结论，不得覆盖默认路线。

## Reusable Heuristics

- 运镜最稳的起点不是“这里能怎么动”，而是“这里其实能不能不动”。
- 一个镜头的默认路线最好只服务一个主观看收益；同时想服务三件事，通常已经过量。
- 真正有用的运镜词汇不是越多越好，而是先落到一个可复用的运动家族，再让 `变化 / 组合 / 速度` 各管一层。
- `变化` 决定运动路径，`组合` 决定观看流，`速度` 决定节奏呼吸；三者混写通常会失真。
- 长焦 / 中焦 / 广角、遮挡 / 反射 / 纵深等知识最好当兼容线索使用，而不是另起一条设计主线。
- 揭示、伴行、逼近、空间搜索、失衡断裂这些词，比“更电影、更高级”更接近可执行的镜头调度判断。
- `academy_hit_note` 最好的用法是当过滤后的补强提示，而不是再次发明镜头。
- 本 branch 的本地装配槽位可以叫 `camera_patch`，但项目级业务真相只能叫 `运镜手法`。
