# Unified Shot Design Workflow Contract

本文件是 `5-摄影` 的统一分镜设计工作流。它整合了原 `shot-planning-integration-contract.md`、`shot-detail-dimension-contract.md` 和 `natural-shot-detail-writing-contract.md` 的核心内容，形成从内部计划到外部成稿的完整链路。

## 文档定位

本文档是 `5-摄影` 的**设计层核心文档**，而非独立的合同契约。执行时应以本文档为主，原三份文件作为历史参考保留，待框架稳定后可逐步废弃。

---

## 第一部分：设计前强制汇流（Shot Design Plan）

### 1.1 核心规则

每个 `visual_unit` 在写出 `分镜1（约X秒）:`、`分镜2（约X秒）:` 前，必须先形成内部 `shot_design_plan`。该计划不默认输出到正文，但必须支配最终 `分镜N（约X秒）` 的数量、顺序、入口、单镜时值、摄影语法、运动、落点、衔接、逐画面点归属、下游消费 payload 和视频执行稳定性。

禁止直接从画面句子跳到 `分镜N` 文案。若无法说明某个 `分镜N` 对应的节拍触发、节奏密度、时值等级、段落归属边界、连续性入口、摄影语法选择、功能 payload、AI 视频执行检查、运镜路径和交出点，该分镜视为随机分镜，必须删除或重写。

### 1.2 必填字段清单

| 字段 | 来源契约 | 用途 |
| --- | --- | --- |

| `unit_ownership_map` | `visual-sequence-alignment-contract.md`、`visual-matching-contract.md` | 当前 visual_unit 拥有哪些主体、动作、道具/文字/身体锚点、对白承托和禁止跨块外溢项 |

| `prop_shot_admission` | `visual-matching-contract.md`、本文档第二部分 | 判断道具、反射、倒影、涟漪或普通物件是否允许成为焦点、特写、反射主体或交出锚点；无互动无关键信息则降级为背景或删除 |
| `ai_video_prompt_execution_profile` | `ai-video-prompt-execution-contract.md` | 为每个分镜锁定镜头先行执行顺序、方向参照、动作在镜头内部完成、光线结果、表演微动态和提示词模板边界 |
| `beat_sequence` | `beat-analysis-contract.md` | 每个分镜对应一个观看策略变化，不能按固定数量灌水 |
| `shot_count_decision` | `beat-analysis-contract.md`、`visual-rhythm-analysis-contract.md`、`sequence-density-curve-contract.md` | 明确本 visual_unit 为什么是 1/2/3/4 镜；2 镜不得作为默认值；5-6 镜只允许 set_piece_chain 例外 |
| `rhythm_profile` | `visual-rhythm-analysis-contract.md` | 决定分镜数量、句子密度、运动复杂度、边界清晰度和停顿感 |
| `duration_profile` | `shot-duration-decision-contract.md` | 决定每个分镜N的时值等级、正文显示秒数、短剧·AIGC 默认压缩、对白台词量预算、停顿/压缩理由和 15 秒分组节奏风险 |
| `shot_duration_decision` | `shot-duration-decision-contract.md` | 为每个计划分镜说明为什么是 instant/short/standard/held/long_hold、正文写约X秒，以及缩短或拉长会损失什么 |
| `dialogue_time_budget` | `shot-duration-decision-contract.md` | 对对白/旁白/画外音/反应镜头，裁决台词量下限和跨镜承托关系 |
| `continuity_entry` | `shot-continuity-contract.md` | 当前画面从上一注意力落点、声音、动作、光色或空间轴线如何进入 |
| `handoff_profile` | `transition-design-contract.md` | 场景变化、空间重置、注意力转交、动作/声音/形态/光色/文字接口需要记录哪些交出锚点、进入提示和连续性风险 |
| `camera_grammar_plan` | `cinematic-technique-library.md`、`dynamic-lens-language-contract.md`、`shot-continuity-contract.md` | 为每个节拍裁决景别梯度、景深/焦点、镜头视角、镜头类型、构图、光影、色彩、运镜方式、速度曲线、停点和变化动机 |
| `technique_selection` | `cinematic-technique-library.md`、`dynamic-lens-language-contract.md` | 从 camera_grammar_plan 中选择最小充分且必须显式进入成稿的 1-2 个摄影选择 |
| `dimension_coverage` | 本文档第二部分 | 裁决每条计划分镜覆盖哪些维度家族（角色表演、非角色动态、镜头技术、光影精细、焦点精细、节奏同步），以及为什么 |
| `natural_output_strategy` | 本文档第三部分 | 决定哪些参数必须显式写、哪些内化为画面动作，避免模板腔和参数清单 |
| `unit_ownership_check` | `visual-sequence-alignment-contract.md` | 每条计划分镜能回指当前上游画面句子；不存在后文主体动作、对白反应、记忆段、道具揭示或转场方案提前外溢 |
| `intra_unit_handoff` | `dynamic-lens-language-contract.md` | 同一 visual_unit 内相邻分镜N如何从上一个终点接到下一个起点 |

### 1.3 功能性 Payload（Functional Payload）

每个分镜N必须能从正文中抽取以下 payload：

| payload | 必须含义 |
| --- | --- |
| `shot_function` | 建立空间、执行动作、显影信息、承托表演、制造危险、完成反应中的一种或组合 |
| `visible_subject` | 当前画面真正可见的主主体；可以是人物、群体、道具、文字、门窗、光源或危险源 |
| `action_phase` | 预备、执行、结果、反应、停顿、显影、进入、退出、转交中的明确相位 |
| `camera_movement_plan` | 运镜方式、运动方向、速度曲线、停点、焦点/景别变化和交接接口；静止镜头也必须说明"为什么不动" |
| `shot_duration_decision` | 时值等级、内部估算范围、正文显示秒数和时值理由 |
| `dialogue_time_budget` | 当前镜头是否承载对白、旁白、画外音或听者反应；如命中，显式秒数必须满足台词量下限 |
| `camera_plan` | 机位/景别/景深/镜头类型中最影响该镜功能的选择 |
| `composition_anchor` | 画面构图的稳定锚点：前景遮挡、门框、课桌行列、黑板、手、眼、道具、文字、光斑等 |
| `light_color_material` | 可被图像和视频工具继承的光线、颜色、材质或视觉母题 |
| `continuity_handoff` | 从上一镜如何进入，或把注意力交给下一镜/下一画面的哪个可见接口 |
| `unit_ownership` | 当前分镜归属的上游画面句子，以及没有提前吞入后文动作、对白反应、记忆段或道具揭示 |
| `prop_admission_reason` | 若物件、反射、倒影或涟漪成为焦点，说明它是互动、关键信息、规则/危险或必要环境交代；普通背景物件为 `none` |
| `ai_video_prompt_execution` | 镜头是否从起点包裹动作、方向参照是否明确、光线是否写成可见结果、情绪是否落到表演微动态 |

## 第二部分：分镜维度覆盖（Dimension Coverage）

### 2.1 核心规则

每个分镜N的内部裁决必须审视以下六个维度族群，判断当前画面句子中哪些维度自然存在、值得被摄影机表达。存在则写入，不存在则跳过——不为凑数而硬塞。维度选取遵循"应有则有、没有不必强制"原则。道具、反射、倒影、涟漪、餐具声、桌面纸角等物件类维度还必须通过 `prop_shot_admission`；无互动、无关键信息、非必要环境交代的普通物件不得因为“光影精细”或“焦点精细”而进入成稿。

### 2.2 六大维度家族

#### 维度家族 1：角色表演（character_performance）

| 维度 | 描述 | 执行要点 |
| --- | --- | --- |
| 微表情变化 | 瞳孔收缩、嘴角微颤、眉心紧锁、鼻翼扇动 | 正面双眼特写（眉骨到鼻尖），禁止侧面单眼 |
| 呼吸节奏 | 呼吸加速/屏息/深呼吸/叹息 | 肩膀起伏、胸腔扩张、鼻翼张合作为可见指标 |
| 沉默压力 | 无台词时的情绪张力 | 停顿的时长、眼神回避、手指动作承托沉默 |
| 表情层次 | 多层情绪叠加 | 用景深分层：嘴部镇定在焦点内，眼底慌乱在浅景深中可读 |
| 情绪词转译 | 抽象情绪词→可见动作 | 强制规则：不得写"紧张""愤怒""压抑"；必须转译为咬肌收紧鼓起、眉心竖纹加深、手指抓紧衣角等 |
| 眼部特写 | 泪光、瞳孔变化、眼睑颤动 | 正面双眼特写：瞳孔、泪光、眼睑颤动必须以双眼对称呈现 |
| 姿态语义 | 前倾=压迫、后仰=防御、侧身=回避、僵直=恐惧 | 让姿态变化成为叙事动作而非静态摆拍 |
| 走位路径 | 入场方向、停留位置、退场路径 | 绑定空间锚点：从门到讲台、从座位到窗边 |
| 手部动作 | 握拳、松手、指尖触碰、手指颤抖 | 手部动作与镜头焦点同步：先给手，再给动作结果 |
| 身体距离 | 人物间物理距离变化 | 距离缩短=压迫/亲密，距离拉远=隔离/逃离 |
| 语速变化 | 加速=紧张/愤怒、减速=强调/威胁、停顿=转折/犹豫 | 语速变化必须对应镜头运动速度变化 |
| 语气层次 | 低语、正常、提高、嘶吼、气声 | 语气强度决定景别尺度：低语用特写/近景，提高用中景 |
| 直视镜头 | 打破第四面墙、挑衅、求助、宣告 | 只在有明确戏剧动机时使用，保持稀缺 |
| 视线回避镜头 | 被观察感、隐瞒、不安 | 角色目光从镜头方向滑开，暗示被监控或内心不安 |
| 主观镜头锚定 | 角色的视线成为镜头运动的起点 | 先给角色眼神方向，再沿视线运动到目标物 |

#### 维度家族 2：非角色动态（non_character_dynamics）

| 维度 | 描述 | 执行要点 |
| --- | --- | --- |
| 道具运动 | 门被推开、窗帘被风吹、粉笔字自行改变 | 先确认道具运动由上游动作、规则或环境支持；让道具运动有起点、过程和落点 |
| 机械运动 | 日光灯闪烁、钟表指针、投影仪光束 | 机械运动的节奏可与角色情绪形成同步或反差 |
| 自然运动 | 风、雨、灰尘、光影漂移 | 自然元素的运动方向和速度暗示空间状态 |
| 异常运动 | 文字生长、颜色渗透、物理违反 | 异常运动的速度和方式决定惊悚/奇幻基调 |
| 群体统一动作 | 全班低头、同时转头、集体起立 | 统一动作的同步程度暗示制度压迫强度 |
| 个体分散活动 | 不同学生各自的微动作 | 分散注意力时用深焦保持多点可读 |
| 陪体反应 | 旁观者的表情、姿态变化 | 陪体反应镜作为主事件的余波承托 |
| 前景遮挡运动 | 人物走过、门板开关、窗帘摆动 | 前景运动可创造"窥视"感或临时信息遮蔽 |
| 前景引导运动 | 前景元素向画面深处运动引导视线 | 前景运动方向决定观众注意力跟随路径 |
| 背景人物活动 | 走廊行人、窗外人影、远处学生 | 背景运动速度与前景主体形成节奏对比 |
| 背景光影变化 | 云层移动导致光线变化、灯管闪烁 | 背景光变可暗示时间流逝或异常入侵 |

#### 维度家族 3：镜头技术（camera_technique）

| 维度 | 描述 | 执行要点 |
| --- | --- | --- |
| 景别推近 | 大远景→远景→中景→近景→特写 | 推近对应信息聚焦、情绪加压、注意力收窄 |
| 景别拉远 | 特写→近景→中景→全景 | 拉远对应空间揭示、情绪释放、处境展示 |
| 景别跳切 | 直接从远景跳到特写 | 跳切对应惊吓、突然揭示、认知翻转 |
| 景别保持 | 同一景别内完成动作 | 保持对应观察、冷静压迫、制度不可抗拒 |
| 推/拉 | 镜头向主体靠近/远离 | 速度决定情绪：极慢推=悬疑逼近，快速推=冲击 |
| 摇/移 | 镜头水平/垂直转动或平移 | 摇镜跟随视线，移镜跟随走位 |
| 升降 | 镜头垂直位移 | 升起=释然/宏观，下降=压迫/聚焦 |
| 跟拍 | 镜头跟随运动主体 | 保持主体在画面中的相对位置不变 |
| 视角切换 | 平视→仰视→俯视 | 视角变化必须对应权力关系或空间认知变化 |
| 主观↔客观切换 | 从角色主观视角切到旁观客观视角 | 切换点对应信息揭示或情绪转折 |
| 窥视视角 | 门缝、窗缝、遮挡物之间的视线 | 窥视视角增强被观察感和空间层次 |

#### 维度家族 4：光影精细（light_refinement）

| 维度 | 描述 | 执行要点 |
| --- | --- | --- |
| 光线渐变 | 亮→暗、暗→亮、色温变化 | 渐变速度对应情绪变化节奏 |
| 光线突变 | 突然开灯/关灯/闪光 | 突变对应惊吓、揭示、规则入侵 |
| 阴影移动 | 阴影随人物或光源移动 | 阴影的运动方向和速度暗示威胁来源 |
| 镜面反射 | 玻璃、水面、黑板、瞳孔中的倒影 | 反射内容可承载信息揭示或情绪映射 |
| 折射变形 | 透过水杯/玻璃的变形 | 变形暗示感知扭曲或空间不可信 |
| 光斑/高光 | 镜头眩光、水面光斑、金属反光 | 光斑位置引导注意力或暗示危险 |
| 色彩反射 | 红色物体在白色表面的反射色 | 色彩反射可把危险母题渗透到安全区域 |

#### 维度家族 5：焦点精细（focus_refinement）

| 维度 | 描述 | 执行要点 |
| --- | --- | --- |
| 焦点拉移 | 从 A 拉到 B | 拉移速度决定信息揭示节奏：慢拉=悬疑，快拉=冲击 |
| 失焦再合焦 | 整体失焦后合到新焦点 | 失焦对应意识模糊/眩晕，合焦对应清醒/发现 |
| 景深变化 | 浅景深→深景深或反之 | 景深变化对应信息从集中到分散或反之 |
| 焦点呼吸 | 焦点在近处和远处之间微调 | 微调对应犹豫、注意力不稳定、感知异常 |

#### 维度家族 6：节奏同步（rhythm_sync）

| 维度 | 描述 | 执行要点 |
| --- | --- | --- |
| 声画同步 | 镜头运动与声音节拍对齐 | 推镜落在鼓点上，摇镜跟随音乐旋律线 |
| 呼吸同步 | 镜头运动与角色呼吸节奏同步 | 呼气时推近，吸气时停顿 |
| 动作同步 | 镜头运动与角色动作节拍同步 | 推门瞬间甩镜，举手瞬间急停 |
| 情绪同步 | 镜头运动与情绪曲线同步 | 紧张段加速运镜，释放段减速或静止 |
| 剪辑节奏同步 | 镜头内部运动与切换频率协调 | 快切段落内运镜精简，长镜内运镜丰富 |

### 2.3 维度覆盖输出规则

1. 维度覆盖是内部裁决，不出现在正文标签中
2. shot_design_plan 必须包含 dimension_coverage 字段，说明每条计划分镜覆盖了哪些维度以及为什么
3. 成稿时把覆盖的维度压缩为信息点密集的自然语句，融入分镜N的正文中
4. 同一 visual_unit 内不同分镜N的维度覆盖可以不同
5. 禁止为了满足维度数量目标而虚构画面中不存在的信息
6. 情绪转折类画面应覆盖"眼部特写"和"情绪词转译"维度
7. 涉及剧烈心理变化（震惊、崩溃、醒悟）的画面，维度覆盖应包含节奏同步（slow_burn/hold）

## 第三部分：自然成稿策略（Natural Output Strategy）

### 3.1 核心规则

`shot_design_plan` 是内部工程图，功能 payload 是下游消费骨架，序列配置文件是段落连续性参考，`分镜明细：` 是给创作链继续使用的镜头文字。最终正文必须让读者先看到画面和动作，再自然读出摄影策略；不得把内部字段逐项翻译成参数清单，也不得牺牲功能 payload 或逐画面点归属去追求句式好看。

自然中文不等于复述原句。每条分镜N必须通过源句复述扣除测试：去掉正上方画面句子里已经存在的人物、动作、道具和事实后，剩余文字仍能读出摄影机的观看策略。若剩余内容只剩"中景/特写/推进/氛围"等孤立术语，说明它只是画面内容拆写，不是真正的分镜明细。

### 3.2 内部 vs 输出对照

| 内部必须知道 | 输出应展示 |
| --- | --- |
| 景别、景深、视角、镜头类型、运镜速度 | 只显式写当前节拍最关键的 1-2 个摄影选择，其余藏进自然动作和画面关系 |
| 起点、路径、落点、交出点 | 读起来能反推即可，不要求每句都写"从/以/变化到/最终" |
| 段落级 sequence_profile | 只内化为当前块可消费的视觉母题、材质光色、进入点或交出点 |
| unit_ownership_map | 分镜明细必须只写正上方画面句子拥有的主体、动作、道具/文字/身体锚点和对白承托 |
| 节拍动机、连续性入口 | 只在跳轴、强转场、跨空间或高点强化时短写动机；普通承接不机械解释 |
| 构图、光影、色彩 | 用具体物、位置、遮挡、亮暗和颜色落点表达，少用抽象形容词 |
| 主体、动作相位、下游消费点 | 必须在画面文字中具像存在，不能被"氛围""电影感""高级运镜"替代 |
| AI 视频执行稳定性 | 不显式输出完整提示词分栏，但成稿应能改写为镜头先行、方向清楚、光线有结果、表演有微动态的视频提示词 |

### 3.3 写作风格

- 句子优先短而清楚；一个分镜N通常 1 句，复杂高点最多 2 句
- 每条分镜只抓一个主要观看动作：靠近、停住、转交、显影、压住、断裂或落地
- 每条分镜先服从正上方画面句子的归属，再服从段落流畅
- 少用连接词堆叠；避免连续出现"从……以……变化到……最终……"
- 不把"景别/景深/视角/镜头类型/速度"机械全列出来
- 技术词服务画面，不抢画面
- 光线写结果，不写口号
- 方向写参照，不写含混位移
- 表演写泄露，不写心理标签
- 抽象情绪词（"紧张""愤怒""压抑""心痛"）是 AIGC 分镜的杀手：必须转译为咬肌收紧、呼吸变浅、手指抓紧衣角等可见物理动作
- 眼睛特写必须写"正面双眼""正面上半脸"等明确景框词，避免只写"眼睛特写"或单眼侧面特写
- 跨分镜组时，下一组首镜必须携带上一组结尾的动作锚点（坐着/站着/抱着）
- 材质和身体锚点优先服务当前画面点

### 3.4 优秀输出范例

```
长焦压扁后排课桌，镜头慢慢贴近他的手。指节发白时，背景冷白灯虚成一片。
镜头不动。她从门口走进画框，红色胸牌先被灯照亮，随后才露出没有表情的脸。
教鞭声先进来，画面切到黑板最下方的粉笔灰；镜头轻轻上摇，停在新写出的规则字上。
手持跟着人群后退半步，晃动在门框前突然稳住，留下走廊尽头那块红光。
镜头从她正前方极慢靠近，前两秒几乎不动；咬肌收紧时，右侧窗光只留住半张脸，左脸沉进桌沿投下的阴影。
他从画面左侧穿过前景，停在门框内侧；走廊冷白灯照亮肩线，脸被门框阴影切掉一半。
正面特写：她双眼微张，瞳孔在冷白灯下缓慢收缩，泪光把灯管映成两条白线。
正面近景：他咬肌慢慢鼓起来，右手从她腰侧收紧，指节在深色毛衣上压出白痕——两人依然坐在床沿。
正面中景：两人面对面站着，他的手还搭在她肩上没松开；镜头极慢推近，推了三秒才到她微微发抖的嘴唇。
动作锚点携带：他扶着她在床边坐下——她的手还搭在他肩上没松开，镜头继续贴着她的侧脸。
```

### 3.5 僵硬输出反模式

```
❌ 从中景平视起，以长焦浅景深慢推变化到近景，最终形成压迫感。
❌ 使用低角度、浅景深、慢速推轨和冷白光，表现人物紧张。
❌ 镜头以丝滑转场承接上一镜，并通过组合运镜强化戏剧效果。
❌ 先建立空间关系，再进行焦点转移，最后完成情绪落点。
❌ 人物向前走，光线很好，气氛压抑。
❌ 她的眼睛特写。（单眼侧面特写，AI 视频生成器易出畸形五官）
❌ 他看起来很紧张。（抽象情绪词，AI 视频无法表现）
❌ 两人坐着，下一镜他站起来——缺少动作锚点，跨分镜组断裂
❌ 她的表情特写。（模糊词，应明确为正面近景或正面双眼特写）
```

### 3.6 复述修复范例

| 弱复述 | 强分镜明细 |
| --- | --- |
| 中景拍阿袖跑来，赤脚上都是血。 | 手持镜迎着阿袖倒退，机位压到膝下，脚掌每次落地都把湿沙和血点甩向画面下沿；镜头在她踉跄停步时急稳。 |
| 特写蜜薯，血泥沾住皮。 | 微距俯拍蜜薯压在潮湿沙面上，焦点从裂开的薯皮拉到血泥反光；火光只扫过边缘，让它像遗物一样从黑沙里浮出来。 |

## 第四部分：设计流程（Planning Procedure）

### 4.1 十八步设计流程

1. **锁定视觉单位功能**：先用一句内部判断锁定 `visual_unit_function`，不得把一个低信息动作硬做成高密度段落。

2. **生成节拍序列**：从 `Beat Trigger Matrix` 中选择真实触发点，生成 `beat_sequence`；弱触发合并，强触发拆开。

3. **消费段落密度曲线**：若相邻画面形成连续观看段落、速度阶段、动作链或声音打点，先读取 `sequence-density-curve-contract.md` 并消费内部 `sequence_density_curve`；判断当前画面属于 `conserve / measured / build / burst / hold / release` 哪个密度槽位。

4. **裁决分镜数量**：形成 `shot_count_decision`：先列出有效触发点，再按触发点数量和节奏槽位裁决 1/2/3/4 镜；第二个有效触发点、第二个观看结果、平台刺激或 AIGC 执行稳定性价值即可支撑 `分镜2`。关键揭示、群像扩散、动作分相、空间重置、高点承托、平台钩子连续推进或 AIGC 执行重置可继续扩展到 3-4 镜。若命中 `set_piece_chain_slot`，可扩展到 5-6 镜。

5. **校准节奏**：用 `rhythm_profile` 校准分镜数量：低信息收敛，关键揭示或高点发散；分镜变多必须带来新的注意力、信息、动作相位或情绪压力。

6. **裁决时值**：形成 `duration_profile` 和每个 beat 的 `shot_duration_decision`：先应用短剧·AIGC 默认压缩，再判断是否承载对白并估算台词量下限，继续判断缩短一半会丢失什么、拉长一倍是否只会拖慢。

7. **消费视觉序列**：若相邻 3-6 个画面单位存在共享空间、道具链、声音链、动作链、记忆插入或视觉母题，先读取 `visual-sequence-alignment-contract.md` 并消费内部 `sequence_profile`。

8. **建立归属地图**：为当前 `visual_unit` 建立 `unit_ownership_map`；当前块拥有的主体、动作相位、道具/文字/身体锚点、对白承托和禁止外溢项必须清楚。

9. **承接连续性**：用 `continuity_entry` 承接前 3 个画面单位中的最近落点；不能每个画面重新发明一套风格。

10. **建立交出档案**：若发生场景变化、空间重置、注意力转交、动作承接、声音先行、形态/颜色匹配、信息显影或高点断裂，先建立 `handoff_profile`。

11. **建立摄影语法计划**：建立 `camera_grammar_plan`；景别变化像呼吸，视角变化有权力/主观/观察/空间动机，景深和焦点负责注意力交接，镜头类型和构图服务空间压力或信息显影。

12. **建立功能投影**：建立 `functional_projection_plan`；没有主体、动作相位、运镜计划、构图锚点、光色/材质、连续性交接、显式时长、对白承托关系或下游消费意义的 beat 不能写成分镜。

13. **裁决维度覆盖**：为每个 beat 裁决 `dimension_coverage`，覆盖当前画面中自然存在的维度家族。

14. **建立 AI 视频执行档案**：建立 `ai_video_prompt_execution_profile`；镜头类型/运动/构图必须先于动作成立；人物位移必须有相对方向参照。

15. **选择技法**：为每个 beat 选择技法时遵守"最小充分"：只选择能服务当前 beat 的参数和运镜策略。

16. **执行归属检查**：执行 `unit_ownership_check`；每个 beat 必须回答"它服务哪条上游画面句子"。

17. **执行自然输出策略**：用 `natural_output_strategy` 压缩显式参数；每条分镜只写当前节拍最关键的 1-2 个摄影选择。

18. **组装分镜明细**：写分镜N时，让每条分镜的起点来自 `continuity_entry` 或上一条分镜的终点；让每条分镜的终点成为下一条分镜的入口或 `next_handoff`。

## 第六部分：输出充分性门槛（Output Sufficiency Gate）

每个分镜N必须至少满足以下七点：

1. **start 明确**：从哪个景别、焦点、空间位置、动作、声音或光色接口进入
2. **path 明确**：使用哪类镜头、运镜方向、速度曲线和焦点/景别变化
3. **end 明确**：落到哪个人物、道具、文字、危险源、反应或转场接口
4. **motivation 明确**：能看出该运动服务信息揭示、动作相位、情绪压力、空间关系或转场
5. **duration 明确**：每条分镜写成 `分镜N（约X秒）:`，能反推该镜是快速通过、标准承接、读秒停留还是长停顿
6. **dialogue_budget 明确**：若承载对白/旁白/画外音，显式秒数不低于台词量下限
7. **handoff 明确**：多分镜时相邻分镜首尾相接，最后一镜能交给下一画面
8. **downstream_payload 明确**：能抽取主体、动作、运镜、构图锚点，光色/材质和图像/视频可消费点
9. **camera_grammar 明确**：景别、视角、景深/焦点、镜头类型或构图变化至少有一个真实服务当前节拍
10. **ai_video_execution 明确**：镜头先行包裹动作、方向参照明确、光线写出结果、表演微动态可见
11. **naturalness 合格**：读起来不是字段展开、参数清单或模板填空
12. **unit_ownership 合格**：每条分镜能回指当前画面句子，没有把后文动作、对白反应、记忆段或道具揭示提前写进当前块
13. **dimension_coverage 合格**：覆盖了画面中自然存在的维度，没有虚构信息

## 第七部分：反模式（Anti-Patterns）

### 设计层面

- 只因为"想更电影感"就把一个画面拆成多个分镜
- 只因为模板里有 `分镜2` 就把每个画面写成 2 镜
- 只做单个画面句子的 `shot_count_decision`，没有为连续段落形成 `sequence_density_curve`
- `分镜1`、`分镜2`、`分镜3` 彼此都在重新描述同一个状态，没有新的观看策略
- 分镜数量确实变多了，但没有主体、动作、信息或情绪的递进
- 每条分镜都同样长度、同样速度，或长停顿没有可读性/表演/空间/高点理由
- 普通氛围镜、过场动作或常规反应沿用传统影视停顿，普遍写到 `约3秒` 以上却没有必要性证据
- 快速切走必须读清的文字、道具或微表情，导致分镜数量对了但观看时值错了

### 连续性层面

- 上一组分镜结尾两人坐着，下一组首镜直接写"他站在窗前"（没有起身动作描写）
- 上一镜结尾两人面对面站着，下一镜直接切到他们走路（没有手放下来的动作）
- 上一镜是沿课桌从左向右横移，下一镜无说明突然从右向左横移

### 表演/情绪层面

- 写"愤怒、紧张、难过"，但没有任何眼神、咬肌、鼻翼、嘴角、肩膀、手指、呼吸或身体联动
- 眼睛特写只写"眼睛特写"或单眼侧面特写，不写"正面双眼"

### AI 视频执行层面

- 把完整视频提示词分栏直接粘进 `分镜明细`
- 写"人物向前走、往后退、从左边进入"，但没有说明相对镜头、画面边界、前中后景或空间锚点
- 写"左侧光、顶光、电影感光线"，但没有说明照亮哪里、阴影在哪里、背景如何压暗、轮廓如何分离

### 归属层面

- 当前块为了承接段落运镜，把下一块的主体动作、对白反应、记忆插入、道具揭示或创意转场方案提前吞进来
- 分镜读起来顺，但无法判断它属于哪条上游画面句子

## 第八部分：修复规则（Repair Rule）

若连续 2 条以上 `分镜N` 出现相同句法骨架、参数堆叠、抽象效果词或明显模板腔，必须回到本文件的自然成稿策略部分重写成自然成稿。

若一条分镜去掉源句已有的人物、动作、道具和事实后，不能保留机位、构图、运镜路径、速度、停点、焦点、光影结果或连续性交接，必须回到功能性 Payload 部分重建功能 payload。

若一句话好读但无法抽取 `visible_subject / action_phase / camera_movement_plan / composition_anchor / light_color_material / continuity_handoff`，不是自然成稿问题，而是功能性投影失败。

若一句话能抽取主体和动作，但动作先于镜头、方向含混、光线只写光源或抽象效果、情绪没有面部/身体/呼吸/手部/视线微动态，必须回到 `ai-video-prompt-execution-contract.md`。

若一组分镜读起来像完整段落预告，但其中某条无法回指正上方画面句子，或提前吞入后文主体动作，不是润色问题，而是逐画面点归属失败。
