# Chapter Rhythm Engine

## Purpose

给 `3-Drafting/2-节奏优化` 用的单章节奏发动机。

本文件默认消费 planning 已声明的集节奏规则：

- 统一七步骨架：
  1. `入场`
  2. `推动`
  3. `转折`
  4. `发展`
  5. `升级`
  6. `高潮`
  7. `尾钩`
- 当前默认集节奏包：
  - `动静结合`
    - `势能式`
    - `动能式`

它不是题材模板，而是 chapter-level 的共性引擎；在未来新增其他集节奏包前，默认按 `动静结合` 去理解本集是偏阴还是偏阳。

## Seven-Step Base Spine

| base_step | semantic question | write move | anti-pattern |
| --- | --- | --- | --- |
| `入场` | 为什么第一屏就要继续读？ | 尽快给变化、压力、异常、强情绪或局势失衡 | 先讲背景和设定，真正的事很晚才开始 |
| `推动` | 这章到底要往哪笔交易推进？ | 让读者在前段知道本章问题、欲望、任务或压力 | 全章有动作没交易，读者说不出这章要干嘛 |
| `转折` | 第一轮改向在哪里？ | 至少一次价值变化、局势翻面、误判暴露或方向偏折 | 从头到尾一个方向推到底 |
| `发展` | 改向后怎样继续缠绕？ | 让中段矛盾、关系、迷阵或回声继续长出来 | 只有改向，没有后续延展 |
| `升级` | 哪个压力被继续推高？ | 让风险、代价、陷落、误会或外压真正再上一级 | 转折后直接跳高潮，中段发空 |
| `高潮` | 本章最高点如何形成？ | 抵达冲突峰值、局面兑现或不可逆高点 | 整章只铺不收，像下一章预告片 |
| `尾钩` | 读者为什么点下一章？ | 用 reveal / decision / threat / pressure transfer / quiet unease 留下余压 | 平收，或故意硬断在最热闹处但没有自然因果 |

### Semantic Overlay

为了兼容现有 drafting 判断，七步骨架默认叠加以下语义检查：

- `入场` 对应 `entry_hook`
- `推动` 对应 `chapter_promise + conflict_axis`
- `转折` 对应 `turn_or_reversal`
- `发展` 对应中段纠葛与 `reaction_decision` 的展开
- `升级` 对应压力继续加码、陷深或再翻面
- `高潮` 对应 `micro_payoff` 或不可逆高点
- `尾钩` 对应 `exit_hook`

## Default Pack: 动静结合

### 势能式

- `入场`：回忆、梦境、碎片蒙太奇、快闪事件等非稳态切口
- `推动`：醒来、转场、落回当下的平静表面
- `转折`：矛盾开始显形
- `发展`：无所谓、戏谑、轻慢、自嘲式应对
- `升级`：逃避、拖延、错置回应，导致纠葛越陷越深
- `高潮`：不可逆的矛盾高点
- `尾钩`：未化解压强被推向下一集

### 动能式

- `入场`：开场爽点、激突、突发高压事件
- `推动`：悬念、迷阵、错位信息或危险诱饵
- `转折`：第一轮反转
- `发展`：新局面继续推进，人物重新布线
- `升级`：再反转、再升压、战线外扩或代价上扬
- `高潮`：冲突高潮；以强冲击结果兑现前面积压
- `尾钩`：在高压余震或更大迷阵上收向下一集

## Practical Moves

### Entry Hook

- 第一屏优先处理“失衡”，不是处理“设定完整度”。
- 开头最常见的有效切口：
  - 某件事出错了
  - 某个秘密要露了
  - 某个欲望马上能摸到
  - 某个关系正要变
  - 某个危险已经逼近

### Chapter Promise

- Promise 不等于写作提纲，而是读者能感知到的“这章要给我什么”。
- 常见 promise：
  - 这次冲突谁赢
  - 这个误会会不会说破
  - 这条线索能不能拿到
  - 这个机会抓不抓得住
  - 这次压力会不会真正落下

### Conflict Axis

- 冲突不是“发生了事”，而是“人物的主动意图被某个力量阻住”。
- 最低配也要写清：
  - 角色这章想做什么
  - 谁/什么阻止他
  - 失败会损失什么

### Turn or Reversal

- 不是每章都要大翻盘，但每章最好至少有一次价值变化。
- 常见形式：
  - 原以为能赢，结果是假胜
  - 原以为完了，结果是假败后转机
  - 误会加深或被戳破
  - 时间压力突然落下
  - 外部压力转成内部压力

### Reaction Decision

- 冲突之后必须有人物吸收，不然章节只会越来越吵。
- 最常见的结构是：
  - 先反应
  - 再意识到新的两难
  - 最后作出下一步决定

### Pulse Ladder

`pulse_ladder` 不是第八个槽位，而是把七步骨架编排成读感起伏的方式。

常用编排：

1. `hook`
2. `promise visible`
3. `conflict`
4. `发展`
5. `升级`
6. `高潮`
7. `尾钩`

不是每章都必须 7 步等长齐全，但至少要有“引、推、变、升、收、续”。

### Micro Payoff

局部兑现不要求大高潮，但要求局面真的改变：

- 线索多知道了一格
- 关系更近或更坏了一格
- 主角得到了一个资源/位置/认可
- 某种情绪终于被戳破或释放

### Exit Hook

尾钩优先从本章现成压力里长出来。

优先级通常是：

1. `pressure transfer`
2. `reveal`
3. `decision`
4. `threat`
5. `quiet unease`

## Mobile-Read Notes

- 手机阅读优先短段、清句、快切换。
- “快”不等于全用短句，而是关键信息不要埋在厚重说明里。
- 当需要减速时，用在：
  - 关键情绪落点
  - 真正重要的感官/动作
  - 反转前的半拍停顿

## Anti-Patterns

- 第一屏还没有事件，只剩世界观说明。
- 角色想做什么不清楚，所以整章没有真正冲突。
- 有冲突，但没有转向，整章只是在加长同一件事。
- 有转向，但没有后续发展与升级，高潮像凭空出现。
- 全章只有 `入场` 和 `尾钩`，中间没有任何兑现。
- 为了快，把必要因果剪断。
- 为了更刺激，临时发明新设定。
- 尾钩只是作者不肯结尾，不是人物和局势的自然余波。
