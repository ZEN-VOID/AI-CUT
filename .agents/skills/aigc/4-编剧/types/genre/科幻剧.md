# 科幻剧 Genre Type

本类型卡定义 `科幻剧` 在 `4-编剧` 中的题材策略。它不是子技能，不拥有输出路径、节点、gate 或最终审查权。

## Identification

| field | value |
| --- | --- |
| `type_id` | `genre-kehuan` |
| `axis` | `genre_axis` |
| `aliases` | 科幻、硬科幻、软科幻、AI、实验、未来科技、时空异常 |
| `primary_signals` | 科技系统、实验装置、AI/算法、太空/星际、时间循环、空间异常、末世科技、数据证据、科研伦理 |

## Strategy

| strategy_slot | rule |
| --- | --- |
| `dramatic_pressure_bias` | 把冲突落到技术失控、证据验证、信息不对称、伦理选择、系统限制、倒计时和现实后果。 |
| `rhythm_bias` | 适合“异常出现 -> 数据/现象验证 -> 角色误判或隐瞒 -> 系统限制逼近 -> 可见后果/尾钩”的节奏。 |
| `controlled_supplement_bias` | 可补屏幕读数、警报、实验记录、设备状态、环境异常和角色验证动作；不得新增技术规则或解决方案。 |
| `field_material_bias` | 实验室、控制台、舱室、监控屏、数据日志、警报灯、冷却雾、断电、倒计时、隔离门、破损设备。 |
| `climax_hook_bias` | 高潮优先落到系统状态变化、数据证据反转、隔离失败、倒计时归零或未知信号出现。 |

## Prohibited Drift

- 不得新增 source 不支持的科学原理、设备能力、AI 意图、时间规则或技术解法。
- 不得用技术名词堆砌替代可见实验现象、角色选择和后果。
- 不得为了反转改变已经确立的因果链。

## Combination Notes

- `正剧 x 科幻剧`：可用角色验证动作、系统画面和对白解释技术压力。
- `解说剧 x 科幻剧`：旁白承接规则/背景/结果概括，画面用数据、设备、警报、隔离空间和现实后果承托。
