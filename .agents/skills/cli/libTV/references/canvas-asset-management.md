# 画布资产管理细则

本细则约束 LibTV 会话中的资产图上传、画布可见节点创建、节点命名修正与视频结果命名。它不替代 `SKILL.md` 的入口合同，只补充画布资产管理的执行细则。

## 触发范围

- 用户要求上传资产图、参考图、角色图、道具图、场景图、设定图或其他项目资产图片到 LibTV 画布。
- 用户要求新建会话画布并把本地图片资产放入画布。
- 用户反馈画布中看不到图片，或图片节点命名显示为 `素材图片`。
- AIGC 视频生成任务需要下载或归档结果文件。

## 资产图上传合同

当用户要求上传资产图时，必须同时完成两个层面的交付：

1. 使用 `upload_file.py` 将本地图片上传到 LibTV OSS，取得可访问 URL。
2. 将每个 OSS URL 创建为画布里的可见图片素材节点，而不是只把 URL 发进会话消息。

执行门禁：

- 本地路径只用于上传，不直接发送给 LibTV 会话。
- 会话消息必须包含 `把全部工作流和结果都放在画布上。`
- 每张资产图都必须有 `原文件名 -> OSS URL -> node_key` 的映射。
- 完成后必须查询会话，确认返回的工具结果中包含对应数量的 `type=image` 和 `node_key`。

## 可见图片节点创建

上传 OSS URL 后，必须让后端创建素材节点：

- 先通过会话请求后端使用 `resource_generator`。
- 对每张图片调用素材节点创建能力，任务类型为图片素材。
- 不生成新图，不改图，不做创意扩写；只把已有图片 URL 变成画布可见节点。

推荐提交语义：

```text
把全部工作流和结果都放在画布上。
请逐项调用 resource_generator，把下面每个 URL 创建为当前项目画布上的可见图片素材节点。
节点标题/显示名必须使用原文件名；不要重新生成，不要只保存链接。
文件名.png: https://...
```

若一次提交较长或返回 `会话正在进行中`，按批次提交并等待上一批完成后再提交下一批。批次大小建议 5-8 张。

## 节点命名修正

`resource_generator` 的默认节点结构会把图片素材节点的 `name` 写成固定值 `素材图片`。即使请求参数或 metadata 中传入原文件名，画布 UI 仍可能显示为 `素材图片`。

因此创建图片节点后必须进行命名核验与修正：

1. 查询会话消息，提取每个工具结果中的 `node_key` 与 `url`。
2. 使用本地映射还原 `node_key -> 原文件名`。
3. 请求后端查询画布节点详情，确认节点顶层 `name` 和 `data.name` 是否仍为 `素材图片`。
4. 若仍为默认名，要求后端调用 `nodes_connections_batch`，按 `node_key -> 原文件名` 批量更新节点：
   - 顶层 `name` 必须等于原文件名。
   - `data.name` 必须等于原文件名。
   - `url` 和 `action=image_resource` 不得改变。
   - 不得新建重复节点。

推荐修复语义：

```text
把全部工作流和结果都放在画布上。
修复命名：刚才创建的图片节点在画布里显示为“素材图片”。
请调用可更新画布节点的工具，按下面 node_key 精确把每个图片节点的显示名/标题/name 改成对应文件名。
不要重新生成图片，不要新建重复节点。
node_key: 文件名.png
```

完成门禁：

- 会话返回 `nodes_connections_batch` 或等价节点更新调用。
- 后端确认已按对应关系重命名。
- 若当前后端工具不支持重命名，必须明确报告 `不支持节点重命名`，不得声称已完成。

## 视频生成命名

AIGC 视频生成任务默认以分镜组 ID 命名。

- 分镜组 ID 使用仓库四段式分镜 ID 的组级前缀或用户明确给定的组 ID，例如 `1-1-1`。
- 下载单个视频结果时优先使用 `download_results.py --filename "分镜组ID.mp4"`。
- 不使用泛化前缀生成 `video_01.mp4`、`libtv_01.mp4` 或带 `_01` 的派生名，除非用户明确要求。
- 若任务缺少分镜组 ID，必须先从项目上下文、用户消息或对应阶段文件中确认；无法确认时按照默认命名继续，不阻断任务。

## Root-Cause 钩子

遇到画布资产问题时按以下链路追溯：

`画布不可见/命名错误 -> 只上传 OSS 或 resource_generator 默认名 -> 本细则资产图上传合同/节点命名修正 -> SKILL.md Reference Loading Guide -> AGENTS.md Skill 2.0 规则`
