# 编导字段桥接类型包

## Match Signals

- 来源或目标稿含大量对白、旁白、内心独白、音效、系统提示、规则文字、道具文字。
- 用户要求“声画清楚”“兼容编导”“可直接拆分镜”“字段对齐”。
- 漫画稿需要继承 `.agents/skills/aigc/2-编导` 的声画配对和字段纯度理解，但仍输出 `第N组.md`。

## Fixed Context

- 加载 `references/directing-field-projection-bridge.md`。
- 所有声音/文本条目必须有就近画面承托。
- 声音字段只写可听、可说、可显示文本；动作、表情、空间反应下沉到画面承托。
- 画面承托必须回答“漫画格里看见什么”，不能只写心理判断、主题解释或作者结论。
- 系统提示、规则文字、公告、黑板字、屏幕字优先投影为 `系统画面 / 规则显影 / 道具特写 / 旁白（系统提示）`。
- 场景锚点不等于影视 slugline；它是漫画组内稳定的空间、光线、核心物件、角色站位和最贵一格。

## Review Gate

- 至少能抽出本组的 `scene_anchor`。
- 对白、旁白、音效、系统提示没有与动作描写混写。
- `视觉焦点 / 分格建议 / 气泡负载建议 / SFX建议` 不停留在抽象概念。
