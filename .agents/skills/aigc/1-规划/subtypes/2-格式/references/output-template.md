# 2-格式 / Output Template

本文件是 `2-格式` 父技能的输出模板真源。

## Canonical Runtime Output

- 默认结果稿：`projects/<项目名>/规划/2-格式/第N集.md`
- 默认角色：基于 `1-分集` 已定集边界，在不改动原文措辞的前提下，为原文附加被选变体下的字段标题
- 说明性报告：默认降为按需 sidecar，不再作为主结果

## `第N集.md` Template

```markdown
# 第N集

## 来源画像
- 故事主源：
- 预设保留：
- 明细扩写：
- 锁定轴：
- 当前集继承锚点：（仅在上游显式提供时写出）
- 场景编号规则：按当前来源粒度写明

### 场景1：地点·时间
- 镜号范围：（仅在上游显式提供或锁轴要求保留时写出）
- 锚点继承：（仅在上游显式提供或锁轴要求保留时写出）

动作画面：

镜头语言预设：（仅在上游显式提供时写出）

对白（角色）：“……”

对白画面：
```

硬规则：

1. 必须是结果稿，不得写成合同摘要或样例说明。
2. 不新增故事事实，不越权写导演/摄影/主体细节。
3. 默认只允许“加标题，不改正文”：可添加场景标题、字段标题、锚点说明，但不得对原文做压缩、同义改写、润色或重述。
4. 必须保留给 `3-分组` 继续切组的场景边界与锚点顺序。
5. 若 `source_type in {storyboard_script, hybrid_story_text}` 且 `locked_preset_axes` 包含 `scene_boundary`，`场景号` 只能按连续时空编号；同一连续时空跨组续写时，允许使用 `场景X（续）`。
6. 混合源或分镜源的 `source_profile` 必须在结果稿顶部可见，不得只藏在 handoff 文件里。
7. `当前集继承锚点 / 镜号范围 / 锚点继承` 都是条件字段；只有上游显式提供，或当前锁轴明确要求保留时，才允许写入。
8. 若上游分镜脚本明确写出了运镜或镜头提示，必须优先保留为 `镜头语言预设：...`，并紧跟相关 `*画面` 条目；未明确写出时，该字段不得出现。
9. 对分镜源输入，默认应规范化整理既有画面/镜头结构，而不是先假定画面缺失再补齐第二套字段内容。
10. 场景标题若可从上游分镜结构稳定提取，必须优先复用原结构证据。
11. 本文件只定义 scene-first draft；不得在 `2-格式/第N集.md` 中提前写入 `## G01 / G02 ...`、`组目标`、`交接约束` 等组级容器。

## Dual-Track Addendum

若用户要求双案对照，在 `validation-report.md` 中新增：

```markdown
## 双案对照
- 标准剧入口：
- 解说剧入口：
- 推荐主案：
- 备选案：
- 推荐理由：
```

## Minimum Acceptance

- 至少存在一个推荐主变体
- 至少回链一个可用子技能合同
- 至少给出一个失败码或 PASS 结论

## Parent-Skill Relationship

- `2-格式` 的 `validation-report.md` 是格式裁决 sidecar，不是规划阶段最终集级主稿。
- 当父级 `1-规划` 走全链时，必须把这里的 `推荐主变体`、`格式边界` 与 `下游约束` 回写进 `projects/<项目名>/规划/第N集.md`。
- 若用户没有显式要求保留格式侧车报告，父级全链完成后可以只保留集级主稿中的格式裁决结果。
