# Visual Matching Contract

本文件定义 step1“画面匹配”的画面句子识别规则。

## Goal

从 `4-表演/第N集.md` 中找出所有需要分镜明细处理的可见内容。每一个画面句子都是一个独立 `visual_unit`，其下方必须新增一个 `分镜明细：` 块。

段落级连续运镜不得改变本合同的单位边界。即便相邻 3-6 个画面句子共享空间、道具链、声音链、动作链、记忆插入或视觉母题，也只能在 `visual-sequence-alignment-contract.md` 中形成内部 `sequence_profile`；最终仍按本文件识别出的 `visual_unit` 逐点落盘。

## Primary Label Match

字段标签包含以下关键词时，默认命中：

| keyword | typical fields |
| --- | --- |
| `画面` | `动作画面`、`群像画面`、`对白画面`、`音效画面`、`旁白画面`、`系统画面` |
| `动作` | `角色动作`、`动作画面` |
| `表演` | `表演提示` |
| `描写` | `环境描写` |
| `特写` | `道具特写`、`表情特写` |
| `显影` | `规则显影` |

## Semantic Visual Match

字段标签未命中但内容显然可被摄影机处理时，也可命中：

- 人物造型、服装、姿态、视线、微表情、呼吸、沉默、手部动作、身体距离。
- 场景空间、光线、色块、门窗、桌椅、黑板、屏幕、道具状态。
- 规则文字、系统字、符号、反光、投影、显影、异常物理变化。
- 画面闪回、新闻画面、现实灾难画面等可视化插入段。

### Prop Admission Overlay

道具或普通物件进入画面匹配后，还要单独判断是否允许成为分镜焦点。只有满足以下任一条件时，道具才可被升级为 `visual_unit_function=显影信息/道具运动/信息锚点` 或成为独立 `分镜N` 的主体：

- 人物与道具有明确互动：拿、放、推、压、递、藏、翻看、打碎、避开、指向、交接。
- 道具是上游明确的重要道具、证据、规则/系统显影、危险源、身份标记或剧情因果节点。
- 道具状态发生上游支持的可见变化：文字改变、屏幕亮起、门被推开、文件被推过桌线、物件被移动。
- 当前镜头必须先交代该物件作为环境/空间/职业/身份信息，且后续动作会消费它。

不满足准入时，道具只能作为环境背景或构图陪体，不能被新增为独立画面单位、焦点拉移终点、反射/倒影主体或多分镜切点。普通杯子、水面涟漪、餐具轻响、桌面纸角、玻璃倒影、人物影子压住物件等，如果没有互动或关键信息功能，不应触发分镜明细扩写。

## Non-Visual Exclusion

以下字段默认不直接挂 `分镜明细：`，除非同一行本身包含明确可见画面承托：

- `对白（角色）`、`独白（角色）`、`内心独白（角色）`。
- `音效（对象）` 的声音本体行。
- `心理反应` 中仅有判断、推理、回忆、恐惧结论的句子。
- frontmatter、执行报告、空行、场景标题。

## Unit Boundary

- 一行字段通常就是一个画面句子处理单位。
- 若一行中有多个强动作或信息揭示，可在 step2 内拆成多个分镜，不要拆改原文行。
- 不允许为了画面匹配而重写原 `4-表演` 字段文本。
- `分镜明细：` 必须紧跟在命中的画面句子下方；同一画面句子只挂一个分镜明细块。
- 不允许为了段落运镜流畅，把多个画面句子合并成一个 `分镜明细：` 块。
- 不允许把后文画面句子的主体动作、对白反应、记忆段或道具揭示提前写入当前画面句子的 `分镜明细：`；当前块最多保留下一画面的可见进入提示或交出锚点。

## Evidence

每个 `visual_unit` 至少保留：

| field | meaning |
| --- | --- |
| `source_line` | 原编导稿行文本 |
| `label` | 字段标签 |
| `match_reason` | 标签命中或语义命中原因 |
| `scene_anchor` | 所在场景标题 |
| `episode_anchor` | 所在 `第N集.md` |
| `ownership_boundary` | 当前 visual_unit 拥有的可见信息边界；用于阻止段落级连续运镜外溢 |
