# 3-Detail Heuristics

本文件保存 `3-Detail` 的稳定可复用经验。它不改写 `SKILL.md` 的入口路由，也不替代 `CONTEXT.md` 的经验层。

## Stable Heuristics

- `3-Detail` 最稳的主链是“先定镜数和镜级正文，再让其余字段往骨架上长”。
- `分镜构图` 在这里首先是结构字段，其次才是审美字段。
- 每镜保留自己的 `剧本正文`，比维护额外桥接字段更稳。
- `主体锚定` 的价值是给下游稳定抽取实体入口，不是写漂亮评语。
- 字段真正高质量时，通常同时具备四件事：可见、可拍、可连续、可被下游抽取。
- `角色表现` 回答“人物怎么演”；`氛围表现` 回答“环境如何施压”；`摄影表现` 回答“画面如何被光和质感组织”；`运镜手法` 回答“观众如何被带着看”。
- 学院派知识最有价值的不是名词更多，而是让镜级判断更可解释、更不撞车。

## Repair Playbook

| symptom | likely cause | repair |
| --- | --- | --- |
| 字段完整但很空 | 缺动作、空间、物件、光气或关系证据 | 回到字段 guide，补可见来源 |
| 知识证据有但像教材 | 没回译到字段对象 | 回 `references/电影学院派知识接线.md` |
| 镜头之间像独立卡片 | continuity 未验收 | 回 `N6/N7` 检查时间、运镜和转场 |
| 局部 patch 覆盖未命中组 | scope 未锁 | 回 `types/type-map.md` 和 `N1` |
| closure 只有“已完成” | 报告模板未对齐 | 回 `templates/output-template.md` 与 `review/review-contract.md` |

## Upgrade Heuristic

Skill 2.0 升级后，`references/思行网络.md` 仍保留为兼容入口；canonical 节点真源逐步收束到 `steps/detail-thinking-action-workflow.md`。运行中若两者出现冲突，以根 `SKILL.md` 与 `steps/` 的节点门禁为准，再更新兼容 reference。
