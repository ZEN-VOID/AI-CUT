# Planning Fractal Layout Contract

## Canonical Carrier Layout

`2-卷章` 当前 primary business truth 固定为三层 Markdown：

- `2-卷章/整体规划.md`
- `2-卷章/第N卷/卷规划.md`
- `2-卷章/第N卷/第N章.md`

## Layer Ownership

| layer | carrier | 负责回答 |
| --- | --- | --- |
| 部级 | `整体规划.md` | 整部故事承诺、整体大纲、故事编年史、卷划分、整部任务关系、整部冲突、整部节奏曲线、总规避 |
| 卷级 | `第N卷/卷规划.md` | 单卷故事大纲、本卷时间线、章划分、卷冲突、卷节奏、卷人物/场景/道具/任务线、卷末达成、卷规避 |
| 章级 | `第N卷/第N章.md` | 单章故事概要、本章时间推进、章冲突、章节奏、章人物/场景/道具/任务线、线索、伏笔、章末达成、章规避 |

## Upward Trace Rule

1. `第N卷/卷规划.md` 必须能回指 `整体规划.md` 中对应卷定位。
2. `第N卷/第N章.md` 必须能回指所属 `第N卷/卷规划.md` 的卷职责。
3. 任何章级判断不得越权推翻部级或卷级已锁定的主承诺。
4. 任务关系必须可上溯：章级 `上承卷级任务` 要能对回卷级 `主线/支线`，卷级 `上承部级主任务` 要能对回部级 `主任务树`。
5. 任何章级支流若未在本章汇聚，必须在章级显式写明去向，且该去向不能与卷级 `汇聚回主线` 冲突。
6. 时间线必须可上溯：章级 `本章时间推进` 要能对回卷级 `本卷时间线`，卷级 `本卷时间线` 要能对回部级 `故事编年史`。

## Compatibility Note

- 若项目仍保留 `全息地图.json / 卷分片/*.json`，它们仅视为兼容投影或历史工件。
- 新规划任务不得再以旧 JSON 结构作为 primary business truth。
