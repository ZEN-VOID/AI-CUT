# 角色提取

从 BYKJ `02-剧本处理` 输出中提取角色资产清单，处理姓名、称谓、代号和 speaker 标签的别名归并，并为每个 canonical 角色生成 JSON 设计规格。

默认输入：

```text
output/[项目名]/02-剧本处理/
```

默认输出：

```text
output/[项目名]/05-资产提取/角色提取/
├── 角色清单.json
├── 角色提取报告.json
└── manifest.json
```

核心规则：同一角色必须归并为一个 canonical 条目；证据不足的别名进入低置信池，不强行合并。设计规格参考 `aigc/7-设计/角色/2-设计`，但本阶段不默认生成 Markdown 设计稿。
