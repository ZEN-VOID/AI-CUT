# 电影学院派知识接线

## Purpose

本文件负责把 `knowledge-base/电影学院派/*` 接到当前 `3-Detail` 的固定 pass 上，避免知识库只被“挂名引用”。

原则只有三条：

1. 先判当前问题属于导演组织、分镜语法还是电影摄影。
2. 再选最小必需知识包，而不是整库通读。
3. 最后把知识翻译回当前 JSON 字段对象，不直接搬运教材口吻。

## Read Order

1. 先读 `knowledge-base/电影学院派/README.md`，确认三大知识域边界。
2. 如果当前在判“为什么这样组织剧情、空间、揭示、动作”，优先读 `导演手册/`。
3. 如果当前在判“镜头怎么切、轴线怎么守、机位怎么带”，优先读 `分镜脚本/`。
4. 如果当前在判“光色质、影调、透视、视觉重量怎么支撑”，优先读 `电影摄影/`。
5. 回到本地 `references/*`，把知识转成 `分镜构图 / 角色表现 / 氛围表现 / 摄影表现 / 运镜手法 / 转场特效` 的字段语言。

## Translation Rules

- `导演手册` 的知识，优先翻译成戏剧单元、支点、关系变化、揭示顺序和动作逻辑。
- `分镜脚本` 的知识，优先翻译成景别、方向、轴线、镜头句子、机位路径和观看任务。
- `电影摄影` 的知识，优先翻译成光位来源、影调、色彩关系、视觉重力、材质显影和空间层级。
- 若某个知识点不能落到当前字段对象，就说明这次不该硬用。

## Anti-Patterns

- 不要把“180°轴线、30°规则、三点布光、广角夸张”直接抄进字段，当成答案本身。
- 不要让 `摄影表现` 抢走 `分镜构图` 的结构权，也不要让 `运镜手法` 重写镜级正文。
- 不要因为知识库丰富，就把所有字段写成学院论文；`3-Detail` 仍然是给下游设计、生图、视频消费的镜级真源。
- 不要只读一个知识域就处理整组戏。对话压强、空间组织、光色质通常至少涉及两种知识域联动。

## Minimal Healthy Pattern

一组 detail 写得健康时，常见顺序是：

1. 用 `导演手册` 划清戏剧单元和支点。
2. 用 `分镜脚本` 锁镜数、方向线索、机位逻辑和观看路径。
3. 用 `电影摄影` 补足光色质与空间压强。
4. 回到本地 `references/*`，把判断收成当前字段对象。
