# Reading Power Execution Playbook

## Positioning

本文件是 `7-追读力强化` 的本地执行细则。

- 分类定义真源：`../../../_shared/reading-power-taxonomy.md`
- 爽点工程真源：`../../../_shared/cool-points-guide.md`
- 本文件只负责把 taxonomy 映射成 Step 7 的重写动作

## Bundle Selection Matrix

| 当前症状 | 首选动作 | 次选动作 | 禁止误用 |
| --- | --- | --- | --- |
| 有事件但没有紧张感 | 显影选择、代价、未知 | 前移压力节点 | 只加“危险/震惊”形容词 |
| 章内一直推进但没收获 | 插入微兑现 | 给角色一次阶段性确认或收获 | 一味拖延到章末 |
| 爽点打完就结束 | 补 aftermath 与新债务 | 转入新的主钩 | 高光后立刻硬切 |
| 章末过平 | 选一个主钩：危机 / 悬念 / 渴望 / 选择 / 情绪 | 强化“失去什么 / 想知道什么” | 同时塞两个以上主钩 |
| 悬念太多但兑现太少 | 先补 micro-payoff | 降低本章悬念密度 | 一章只挖坑不回收 |
| 有压抑、羞耻或美感，但就是不抓人 | 先选 dominant pressure core，再显影被看见后果 / 身体代价 / 物象风险 | 把压力挂到单一场面或物象上 | 用抒情堆叠代替推进 |
| 角色宣言很重，但读者不想追 | 补身体兑现 | 把判断改写成训练、站姿、触碰、伤口、节制、登台动作 | 继续加理念台词 |
| 场面有仪式感，但章末很轻 | 把私密冲突推到公开场面边缘 | 选“即将暴露 / 即将说破 / 即将失手 / 即将登台”之一做主牵引 | 只堆礼法、服装和气氛词 |

## Chapter Internal Routing

1. 先判断本章最缺的是：
   - 压力
   - 钩子
   - 微兑现
   - 爽点落地
   - 章末 pull
2. 再判断当前 reader pull 主要该走：
   - 事件 / 信息推进
   - 人物高压 / 假面暴露 / 身体兑现
3. 只补最关键的 1-2 个维度，不做满配堆叠。
4. 若项目存在 `type-pack` 投影，用对应 genre 偏好做 tie-break；否则用通用 taxonomy。

## Pressure-Core Projection

> 当章内已有压抑、羞耻、欲望、审美压力或公开场面潜势，但读者仍不够想追时，用这组投影语法把“气压”转成“续读力”。

| pressure_core | 核心问题 | 典型动作 | 章末适配 |
| --- | --- | --- | --- |
| `mask_exposure` | 谁会先看见这层假面裂开？ | 遮掩、强撑、转移、被盯住、差点露馅 | 即将被看穿 / 即将当众失态 |
| `desire_pressure` | 他会不会承认、靠近或越界？ | 试探、停手、靠近、避开、说到一半 | 即将说破 / 即将失手 |
| `beauty_pressure` | 他会保护、占有、亵渎还是毁掉这个对象？ | 凝视、占有、嫉妒、触碰、克制、破坏冲动 | 即将亵渎 / 即将失去 |
| `body_commitment` | 这份判断会逼出什么身体代价？ | 训练、站姿、伤口、熬忍、节制、触碰、扛住 | 即将撑不住 / 即将付代价 |
| `public_staging` | 私密欲望会不会在众目下失控？ | 演说、对峙、登台、行礼、被围观、公开测试 | 即将登台 / 即将当众暴露 |

### Projection Rules

1. 只选一个 dominant pressure core。
2. pressure core 不替代主钩，而是给主钩提供真正的戏剧压力来源。
3. 一旦选了 pressure core，就必须把它翻译成动作、后果、空间或观众关系，不能只留在心理形容词里。
4. 若使用 `beauty_pressure`，必须绑定单一物象、理想或关系对象；不要同时挂多个象征。
5. 若使用 `public_staging`，观众是谁、角色失去什么，必须具体。

## Intensity Ladder Projection

- 若当前章是普通推进章，优先保证 `B/C` 级回报稳定出现，不硬做 `S` 级大爆。
- 若当前章承担阶段转折，再考虑把主爽点拉到 `A` 级，并让余波直接打开下一轮 reader pull。
- 若当前章前面已经连续压抑 1-2 章，本章应优先考虑“虐点 -> 爽点”转换，而不是继续积压。

## Main + Secondary Cool-Point Rule

- 每章默认只允许一个主爽点。
- 若要叠加副爽点，副爽点必须服务主爽点，例如：
  - 主爽点：公开打脸
  - 副爽点：身份掉马 / 关系回暖 / 资源到手
- 若副爽点抢走了章末主问题，就删掉副爽点。

## Hook Selection Order

1. 先问“读者下一章最想立刻知道什么”。
2. 若答案是“会不会出事”，优先危机钩 / 选择钩。
3. 若答案是“真相是什么”，优先悬念钩。
4. 若答案是“能不能拿到 / 在一起 / 反杀”，优先渴望钩。
5. 若答案是“心里过不去”，优先情绪钩。

### High-Pressure Tie-Break

- 若读者真正想知道的是“会不会被看穿”，优先 `mask_exposure`，hook 常落在 `情绪钩 / 选择钩 / 悬念钩`。
- 若读者真正想知道的是“会不会说破或越界”，优先 `desire_pressure`，hook 常落在 `渴望钩 / 选择钩 / 情绪钩`。
- 若读者真正想知道的是“会不会毁掉那个对象/关系/理想”，优先 `beauty_pressure`，hook 常落在 `情绪钩 / 选择钩`。
- 若读者真正想知道的是“他还能撑多久”，优先 `body_commitment`，hook 常落在 `危机钩 / 选择钩`。
- 若读者真正想知道的是“会不会当众失控”，优先 `public_staging`，hook 常落在 `危机钩 / 情绪钩 / 选择钩`。

## Micro-Payoff Rules

- 每章至少给一个读者可感知的小收获。
- 小收获可以是信息、关系、资源、认可、情绪释放，不一定非得是大反转。
- 当章末主钩较强时，章内更需要至少一个微兑现，避免“全章只吊胃口”。
- 偏文学高压章的 micro-payoff 往往不是“赢了”，而是：更接近真相一步、关系失衡一步、体面裂开一点、欲望被承认一点。

## High-Pressure Ending Pull

- 章末若走人物高压路线，优先在以下收束中选一个：
  - 即将被看见
  - 即将说破
  - 即将失手
  - 即将亵渎或毁掉
  - 即将登台或被围观
- 这些收束必须已经被本章铺垫，不能凭空突降。
- 不要把“余韵”误当主钩；余韵只能服务主牵引，不能替代主牵引。

## Cool-Point Guardrails

- 爽点必须建立在已有铺垫和读者期待上。
- 迪化误解、身份掉马这类模式只有在前面已经埋过信息差时才可启用。
- 若当前章主任务是沉浸或情绪推进，爽点可以降级为轻量微兑现，不必硬做大爆点。
- 若近三章主爽点都落在同一来源轴，必须至少换一个副轴，避免审美疲劳。
- 爽点后必须补余波：评价变化、局面变化、关系变化或新债务，不能爆完即断。

## Regression Guard

- 不为追读力而越权新增主线。
- 不把 reviewer 风格建议误写成正文事实。
- 不让章末钩子抢走本章已经完成的情绪落点。
- 不重复近几章同构的主钩和主爽点，除非项目明确要求连续加码。
- 不把“每章都必须很炸”误解成“每章都堆最高强度爽点”。
- 不把高压 reader pull 偷换成和风装饰、自毁姿态或空洞象征堆叠。
