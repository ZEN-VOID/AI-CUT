# 文本来源类型包

## Match Signals

- 小说、网文、故事梗概、短篇帖文、剧本草稿、角色设定文、影视剧情摘要。
- 用户要求“改成漫画剧本”“更抓人”“按组改编”“进九刀流”。

## Fixed Context

- 默认先抽：情节链、人物关系、情绪核、转折点、名场面潜力。
- 虚构来源默认不保表层顺序，优先保 `人物关系核 / 情绪核 / 卖点核 / 关键代价`。
- 若原文前段铺垫过长，允许前置炸点、合并过桥、后移解释。
- 若用户明确要求忠于原文顺序，降级为 `faithful-core` 姿态。

## Review Gate

- 前 3 场内至少出现一个抓停点。
- 成稿不是复述摘要，而是有动作、对白、环境变化和可画场景。
