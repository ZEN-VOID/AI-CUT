# Visual Matching Contract

本文件定义 step1“画面匹配”的画面句子识别规则。

## Goal

从 `2-编导/第N集.md` 中找出所有需要镜头语言处理的可见内容。每一个画面句子都是一个独立 `visual_unit`，其下方必须新增一个 `镜头语言：` 块。

## Primary Label Match

字段标签包含以下关键词时，默认命中：

| keyword | typical fields |
| --- | --- |
| `画面` | `动作画面`、`群像画面`、`对白画面`、`音效画面`、`旁白画面`、`系统画面` |
| `动作` | `角色动作`、`动作画面` |
| `表演` | `表演提示` |
| `描写` | `环境描写` |
| `特写` | `道具特写`、`表情特写` |
| `显影` | `规则显影` |

## Semantic Visual Match

字段标签未命中但内容显然可被摄影机处理时，也可命中：

- 人物造型、服装、姿态、视线、手部动作、身体距离。
- 场景空间、光线、色块、门窗、桌椅、黑板、屏幕、道具状态。
- 规则文字、系统字、符号、反光、投影、显影、异常物理变化。
- 画面闪回、新闻画面、现实灾难画面等可视化插入段。

## Non-Visual Exclusion

以下字段默认不直接挂 `镜头语言：`，除非同一行本身包含明确可见画面承托：

- `对白（角色）`、`独白（角色）`、`内心独白（角色）`。
- `音效（对象）` 的声音本体行。
- `心理反应` 中仅有判断、推理、回忆、恐惧结论的句子。
- frontmatter、执行报告、空行、场景标题。

## Unit Boundary

- 一行字段通常就是一个画面句子处理单位。
- 若一行中有多个强动作或信息揭示，可在 step2 内拆成多个分镜，不要拆改原文行。
- 不允许为了画面匹配而重写原 `2-编导` 字段文本。
- `镜头语言：` 必须紧跟在命中的画面句子下方；同一画面句子只挂一个镜头语言块。

## Evidence

每个 `visual_unit` 至少保留：

| field | meaning |
| --- | --- |
| `source_line` | 原编导稿行文本 |
| `label` | 字段标签 |
| `match_reason` | 标签命中或语义命中原因 |
| `scene_anchor` | 所在场景标题 |
| `episode_anchor` | 所在 `第N集.md` |
