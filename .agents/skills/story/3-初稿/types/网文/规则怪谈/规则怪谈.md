# 规则怪谈题材模板 (Rule Horror)

> **核心卖点**: 异常规则 + 认知污染 + 逻辑破译。在规则束缚的世界中，每一步选择都关乎生死。

- 推荐约束包：Pack M22
- 通用叠加：Pack U04

## 1. 核心流派细分

### 规则破译流
- **特点**: 核心爽点来自发现/理解/利用异常规则
- **核心爽点**: 规则逻辑闭环、利用规则反杀
- **创作要点**: 规则必须可验证——读者可以和主角一起推演

### 禁忌生存流
- **特点**: 违反规则即死/即疯，在限制下生存
- **核心爽点**: 高压决策、绝境求生、遵守规则的焦虑
- **创作要点**: 每条禁忌必须在第一幕就被暗示——不能"突然冒出来"

### 组织管理流
- **特点**: 官方/民间组织管理异常——SCP基金会范式
- **核心爽点**: 收容管理、风险评估、秘密揭露
- **创作要点**: 组织内部的政治也是看点——收容不是技术问题，是政治问题

### 日常侵蚀流
- **特点**: 日常生活的规则被异常规则替换——"你家冰箱里的东西，晚上会多出来"
- **核心爽点**: 认知崩塌、日常反转
- **创作要点**: 从"最平常的地方"入手——镜子、楼梯、电梯、手机

---

## 2. 规则设计铁律

### 规则三要素
1. **触发条件** — 做了什么会触发
2. **效果** — 触发后发生什么
3. **漏洞** — 规则可以被利用的缝隙

### 规则暴露节奏
- 第1-3章: 暗示规则存在——有人消失了/变样了
- 第4-7章: 发现第一条规则（不完整版）
- 第8-15章: 验证规则+发现规则有漏洞
- 第16章+: 利用规则+发现更深层规则

---

## 3. 追读心理

| 驱动力 | 执行手段 |
| --- | --- |
| 规则好奇 | 每章释放一条新规则或一个规则验证 |
| 破译爽感 | 让读者有机会比主角先猜出规则——公平竞争 |
| 生存悬念 | 每个规则至少展示一次"违反规则的代价" |

## 4. AI 写作专项指南

### 常见陷阱
1. **规则逻辑不自洽**: 前面的规则和后面的矛盾 → 规则必须是"发现性"的而非"发明性"的
2. **规则万能化**: 规则可以解释一切 → 每条规则必须有边界和例外
