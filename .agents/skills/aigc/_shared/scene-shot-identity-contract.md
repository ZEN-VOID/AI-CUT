# Scene And Shot Identity Contract

本共享合同承接“画面至少有角色、环境、场景/镜头身份”的源层规则。它不替代各阶段主合同，只给导演、摄影、分组、图像和视频阶段提供同一套可回指的身份锁定方法，避免提示词只从“谁在做什么”开始，导致年代、空间、光线、声音和运动方向漂移。

## Core Rule

任何需要进入画面、分镜、图像 prompt 或视频 prompt 的单位，必须先锁定四类身份，再组织动作：

1. `character_identity`: 画面中是谁，当前身体状态、站位、朝向、动作动机和情绪压力是什么。
2. `environment_identity`: 这个空间的固定锚点、材质、光源、年代感、生活痕迹和可听见的环境基底是什么。
3. `scene_identity`: 这是什么类型的场景，属于什么年代/社会语境/功能空间，观众应从画面和声音里认出什么身份。
4. `shot_identity`: 摄影机在哪里、朝向哪里、以什么景别/焦段/运动方式观察，主体动作如何在镜头内部发生。

写作顺序不是机械模板，但判断顺序必须成立：先有场景和镜头身份，再让人物动作、表演微动态、光线结果和声音承托发生在这个身份内部。

## Scene Identity Fields

| field | question | examples |
| --- | --- | --- |
| `era_period` | 画面属于哪个年代、朝代或技术时代 | 年代明确的审问空间、当代公共机构走廊、旧式居民空间 |
| `location_function` | 空间功能是什么，人在这里通常受什么规则约束 | 审讯、卧室反锁、餐厅对峙、学校晨会 |
| `social_context` | 这个空间的权力关系、阶层、制度或生活方式是什么 | 公权审问、家庭控制、校园秩序、江湖客栈 |
| `soundscape_baseline` | 不新增事件的前提下，环境应有什么声音底色 | 荧光灯嗡鸣、远处餐厅闷响、老楼水管声、衣料摩擦 |
| `lighting_baseline` | 场景天然光源和可见光线结果是什么 | 顶灯压眼窝、窗光切侧脸、门缝光横切地面 |
| `material_age` | 材质新旧、磨损、潮湿、灰尘、反光如何体现身份 | 老木门掉漆、冷亮瓷砖、旧墙皮、磨损桌沿 |

## Shot Identity Fields

| field | question | execution note |
| --- | --- | --- |
| `camera_identity` | 摄影机是固定、跟随、推进、后退、过肩、主观还是微距 | 必须在动作前成立，不能作为动作后的补丁 |
| `screen_direction` | 人物朝镜头、远离镜头、从画面左/右侧进入，还是沿空间深处移动 | “左右”只有在镜头朝向已锁定后才有效 |
| `action_inside_camera` | 动作如何在一个连续镜头逻辑里完成 | 避免“人物先做完，镜头再补一句跟拍” |
| `lighting_result` | 光照亮了哪里、阴影落在哪里、主体如何从背景分离 | 只写光源不合格，必须写可见结果 |
| `continuity_entry` | 本镜头从上一状态继承什么姿态、站位、朝向和光色 | 跨组或跨镜都要避免凭空重置 |

## Lighting Direction Rule

光线必须写成“位置 + 照亮对象 + 阴影/分离结果”。只写“左侧光”“右侧柔光”“顶光压迫”不够稳定；若原图或场景参照已有明确光影，不要直接用新光源对抗旧光源。

执行口径：

- 生图/分镜：写清光源相对镜头或画面的位置、被照亮的脸/手/门/地面/墙面，以及阴影形状或主体分离结果。
- 图像修复或二次调光：若要改变已有光影，先说明去除/减弱原有光线或使用局部蒙版重构，不把“新光源”直接叠在冲突的旧光线之上。
- 视频：光线说明必须服务镜头内动作和主体可读性，不能为了“电影感”新增与场景身份冲突的灯光。

## Prompt Order Rule

AI 视频和生图模型更像按提示词顺序执行，而不是自动理解完整导演意图。最终提示词或分镜明细不一定要输出字段名，但应能还原以下顺序：

1. 场景/镜头身份：年代、空间功能、视觉/声音基底、镜头位置和朝向。
2. 主体空间：角色站位、身体朝向、前中后景、固定锚点。
3. 镜头运动：固定、推进、跟随、摇移、焦点变化等。
4. 人物动作：动作在镜头内部发生，方向相对镜头或画面。
5. 表演微动态：短暂泄露、情绪切换瞬间、身体联动。
6. 光线/声音结果：可见亮暗、阴影、轮廓、环境声基底和沉默。

## Failure Modes

| fail_code | symptom | repair |
| --- | --- | --- |
| `FAIL-SCENE-IDENTITY-01` | 只写人物动作，没有年代、空间功能或环境身份 | 补 `scene_identity`，让场景本身能被看出/听出身份 |
| `FAIL-SCENE-IDENTITY-02` | 年代、声音、材质或光影互相冲突 | 回到场景身份字段统一 era、soundscape、material 和 lighting |
| `FAIL-SHOT-IDENTITY-01` | 动作先发生，镜头只在后面补“推进/跟拍” | 重排为镜头先行，让动作发生在镜头内部 |
| `FAIL-DIRECTION-REF-01` | “前后左右”没有镜头或画面参照 | 改为朝镜头、远离摄像机、画面左/右、前景/后景或空间锚点方向 |
| `FAIL-LIGHT-DIRECTION-01` | 只写光源或空泛光效，没有可见结果 | 补照亮对象、阴影位置、亮暗交界和主体分离结果 |

## Owner Boundaries

- `3-导演` 拥有 scene identity seed 的主创判断，决定场景身份服务什么戏剧功能。
- `5-摄影` 把 scene/shot identity 转成分镜明细的内部摄影约束。
- `6-分组` 把 scene/shot identity 写入每组 `定场镜头：` 的开头，让独立消费的分镜组先回答“这是什么场景”，再承接上一组末尾和本组人物动作。
- `8-图像/A-分镜画面` 把 scene/frame identity 转成英文生图 prompt 的开头、空间、光影和材质锁。
- `9-视频/libTV画布流` 保持上游 prompt 顺序和镜头身份，不把分镜组重写成主体动作摘要。
