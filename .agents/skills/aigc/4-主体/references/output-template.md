# 4-主体输出契约

## 固定交付件

`4-主体` 阶段至少应稳定维护以下固定交付件：

1. `subject-stage-index.md`
2. `validation-report.md`
3. 当前命中子路径的 canonical 产物
4. 唯一下一入口说明

## 硬规则

1. 必须先回答当前进入的是哪个子路径，再执行具体产出。
2. `1-清单 -> 2-设计` 是默认主链，不得把 `3-审计 / 4-面板` 写成默认必经。
3. 所有阶段级产物必须回写到 `projects/<项目名>/主体/`，不得漂移到其他阶段。
4. 若顾问团启用，必须显式说明是否执行了 `策划前置` 与 `评审闸门`。
