# Chapter Rhythm Rules

## Shared Contract

章级节奏的共性真源已回收到：

- `../../../_shared/chapter-rhythm-handoff-contract.md`

本文件只补章级 planning 自身的落盘要求，不再平行维护第二套共性发动机。

## 章级写回重点

- `本章节奏曲线` 必须明确写出 `selected_pack / selected_mode`。
- `本章节奏曲线` 必须消费 `本章爽点设计`，不得绕过 `reader_desire / promise_source / genre_payoff_profile / character_anchor / payoff_mode / payoff_variation_axis / delivery_action / satisfaction_delta / exaggeration_logic` 另行发明读者满足。
- `selected_mode` 后必须写 `mode_selection_reason`，用本章冲突、任务、信息压力、情绪调性、旅程/游玩状态或章末牵引解释选择依据。
- `payoff_type` 必须裁定本章主导满足类型，并与 `本章爽点设计` 的主导爽点一致；不得把所有类型堆成安全列表。
- `rhythm_intensity` 必须标为 `低 / 中 / 高`，并能解释本章在连续章节波形中的发力程度。
- `previous_next_contrast` 必须同时写清 `承接上一章` 与 `预留下一章`，避免章章同构。
- `七步职责映射` 必须回答本章每一步承担什么推进功能，而不是只写步骤名。
- `规划义务` 必须锁清 `entry_promise / conflict_axis / micro_payoff / exit_hook`；其中 `micro_payoff` 必须是 `delivery_action` 与 `satisfaction_delta` 的节奏化移交，不得另起炉灶。
- `义务段位` 与 `建议写法` 必须分开，避免 drafting 把建议写法误读成不可动的法律。
- Mermaid 图至少能看出起势、转折、升级、高潮、尾钩。

## `selected_mode` 判定细则

### 全量章感字段

章级节奏不只回答“这一章是什么 mode”，还必须回答“读者得到什么、强度多大、和前后章如何形成波形”。

- `payoff_type` 合法值：`认知 / 行动 / 情绪 / 关系 / 世界感 / 软线索 / 状态修复`。
- `rhythm_intensity` 合法值：`低 / 中 / 高`。
- `previous_next_contrast` 必须包含：
  - `承接上一章`：升压、转行动、松弛换气、余波消化、反向变奏等。
  - `预留下一章`：爆发、行动、舒缓、揭示、关系推进、新地点等。

使用约束：

- `势能式` 常见 `payoff_type=认知 / 软线索 / 关系`，强度可低中高，但高强度不等于必须爆炸。
- `动能式` 常见 `payoff_type=行动 / 关系 / 认知`，强度中高最常见，低强度可用于小任务或轻行动。
- `浪能式` 常见 `payoff_type=情绪 / 关系 / 世界感 / 软线索 / 状态修复`，强度多为低中；若标高强度，必须说明它是强情绪、强关系或强世界感高点，而不是硬冲突。
- `rhythm_intensity=高` 不宜连续滥用；若上一章已高强度，本章要么给更大的质变，要么用中低强度形成呼吸。
- `previous_next_contrast` 不锁死正文排布，只给 drafting 一个连续波形意图。

### 选择 `势能式`

当本章的主要节奏任务是蓄压、逼近、显影、试探、收窄选择或让下一章具备爆发条件时，选 `势能式`。

判定信号：

- 冲突重心在信息差、心理压力、关系张力、环境压迫、威胁逼近或未知风险。
- 本章推进的是“知道了什么 / 确认了什么 / 局面变危险了什么”，而不是“完成了什么外部动作”。
- `micro_payoff` 是认知变化、危险确认、隐藏代价浮出、角色退路变窄。
- `exit_hook` 来自本章压力自然增长，读者感到下一步要爆，而不是突然切断。

七步职责写法：

- `入场`：建立不安、目标、欠账或隐性压力。
- `推动`：让角色靠近压力源，或让信息差开始压迫选择。
- `转折`：原判断被打破，异常第一次具备方向。
- `发展`：压力扩大，关系/信息/环境逐步收紧。
- `升级`：隐藏代价或更大风险浮出。
- `高潮`：完成危险确认、认知翻转或艰难判断。
- `尾钩`：把下一章爆发点自然推出。

### 选择 `动能式`

当本章的主要节奏任务是执行行动、外部对抗、任务推进、即时反制或局面实际改写时，选 `动能式`。

判定信号：

- 冲突重心在追逐、交锋、谈判、夺取、逃脱、突破、拦截、救援、破局或现场阻碍。
- 本章推进的是“做成了什么 / 失去了什么 / 打赢或打输了什么 / 局势被怎样改写”。
- `micro_payoff` 是资源到手、关系破裂、身份暴露、对手退让、任务失败或战场态势改变。
- `exit_hook` 来自动作后果引出的新麻烦，而不是单纯氛围加压。

七步职责写法：

- `入场`：明确行动目标、现场限制或即时危险。
- `推动`：角色立刻执行。
- `转折`：对手反制、阻碍出现或计划失效。
- `发展`：角色换策略继续推进。
- `升级`：代价变大，风险暴露，冲突扩大。
- `高潮`：发生关键动作、正面碰撞或决定性取舍。
- `尾钩`：行动结果引出新问题。

### 选择 `浪能式`

当本章的主要节奏任务是轻松、愉快、欢乐、松弛、舒缓、旅游、玩耍、休整或悠哉悠哉地展开角色与世界时，选 `浪能式`。

判定信号：

- 冲突重心很轻，章节不是为了蓄压或破局，而是为了让读者和角色一起换气、游逛、玩耍、休息或感受生活质地。
- 本章推进的是“关系如何变暖 / 心情如何恢复 / 世界如何打开 / 小趣味如何发生 / 软信息如何自然出现”。
- `micro_payoff` 是角色状态修复、关系亲近、轻松笑点、风景/美食/游玩高点、软线索顺手获得或下一站期待被点亮。
- `exit_hook` 来自温和好奇、下一处风景、小麻烦露头或舒缓后的新方向，不靠突然危机破坏松弛感。

七步职责写法：

- `入场`：建立轻松场域、旅途目标、游玩约定或休整理由。
- `推动`：角色进入体验，开始逛、玩、吃、聊、看、试或慢行。
- `转折`：出现小惊喜、小误会、小发现或轻微偏航。
- `发展`：扩展趣味、关系互动、风景层次或生活细节。
- `升级`：让快乐、松弛、亲密、趣味或软信息更饱满，而不是突然拉高危机。
- `高潮`：完成笑点、暖点、风景高点、关系小进展、状态恢复或软线索落点。
- `尾钩`：留下下一站、下一顿、下一件趣事、隐约线索或轻微新期待。

### 快速裁决

- `信息差 > 行动链`、`心理压力 > 外部对抗`、`危险确认 > 危险解决`：优先 `势能式`。
- `任务执行 > 情绪酝酿`、`外部阻碍 > 内心判断`、`结果落地 > 氛围加压`：优先 `动能式`。
- `松弛体验 > 强冲突`、`关系升温 > 任务破局`、`旅途玩耍 > 危险升级`、`舒缓恢复 > 结果落地`：优先 `浪能式`。
- 若三者都强，先以 `本章爽点设计` 的 `payoff_mode` 与 `delivery_action` 裁决，再用 `micro_payoff` 落到节奏义务：认知/压力兑现选 `势能式`；行动/结果兑现选 `动能式`；情绪/关系/旅途/游玩兑现选 `浪能式`。

## Chapter-Specific Rule

- 章级是 planning 的最细层，但仍只锁“这章怎么跑”，不直接写正文段落。
- `mode_selection_reason` 是 planning 证据，不是正文导语；不得写成可直接放入正文的叙述段。
