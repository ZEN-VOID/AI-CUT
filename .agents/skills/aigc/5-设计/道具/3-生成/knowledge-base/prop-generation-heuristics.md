# Prop Generation Heuristics

本文件保存 `道具/3-生成` 的稳定启发式。新经验优先沉淀到同目录 `CONTEXT.md`，稳定后再迁入本文件。

## Heuristics

- 主体稳定性优先于画面炫技；主图必须让单个道具被清楚识别。
- 多视图设计图不是海报，应展示同一主体的正面、侧面、背面、细节和尺度逻辑。
- 道具生成最常见漂移是加入人物手持、环境喧宾夺主、把一个道具扩展成产品合集；prompt 和 review 都要压住这些漂移。
- 参考图用于锁定形态、材质和识别点，不用于复制随机背景、光斑、噪点或错误构图。
- JSON 提示词应比聊天记录更可靠；后续复跑应能只凭 JSON 找到上游设计、参考图和目标输出路径。

## Failure Prevention

- 生图前先检查上游提示词是否英文、是否聚焦单个主体、是否已有 negative constraints。
- 多视图 prompt 中明确禁止人物、手、角色占据画面，除非上游设计文档明确要求用非主导尺度参照。
- 批量生成时按主体循环执行，不把多个道具塞进同一张图。
