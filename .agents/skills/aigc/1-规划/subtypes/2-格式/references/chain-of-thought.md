# 2-格式 / Chain Of Thought

本文件承载 `2-格式` 父技能的思维链细则，是主 `SKILL.md` 的专项细节真源。

## 思考目标

- 判断当前项目应进入 `标准剧` 还是 `解说剧`
- 把格式规划控制在规划层，而不是提前写正文
- 保证父技能只做“判模 + 交接 + 汇总”，不与子技能抢真源

## Pass Blueprint

### Pass 1: Scope Lock

- 识别当前任务是否真的是“格式规划”
- 若用户其实要写正文、改对白、调导演风格，应停止并回路由
- 先确认 `0-Init` 与 `1-分集` 是否足够支持判模

### Pass 2: Variant Diagnosis

- 看用户是否显式要求“解说剧 / 旁白主导 / 讲述者带路”
- 看项目是否更依赖表演承载，还是更依赖解释承载
- 若无强解说信号，默认保持 `标准剧`

### Pass 3: Contract Delegation

- 父技能不亲自细写每个字段模板
- 父技能只决定应进入哪个子技能，并确保子技能交付合同、样例和局部验证
- 若用户要求双案对照，允许并行生成，但必须在父级重新收束为“推荐主案 + 备选案”

### Pass 4: Handoff Clarity

- 父级报告必须能让下游知道：
  - 应按哪一套格式继续
  - 为什么不是另一套
  - 若后续发现判模错误，应回到哪个返工入口

### Pass 5: Anti-Drift Check

- 检查主 `SKILL.md` 是否只保留主合同
- 检查详细流程、类型策略、模板是否已下沉到 `references/`
- 检查父级是否没有变成隐藏的第二真源

## 失败信号

- 只写“标准剧 / 解说剧”两个名字，却没有唯一裁决标准
- 父级重新抄了一整套子技能字段模板
- 说要“格式规划”，结果输出成正文片段
- 双案对照时没有推荐主案
