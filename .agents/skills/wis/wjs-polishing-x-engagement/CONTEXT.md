## Repair Playbook

1. 先抓原文核心：它到底在说一个观点、一个感受、一个人/事/地，还是一个日常瞬间。
2. 围绕核心找一个具体且可核实的真事实；优先年份、数字、人名、地名、纪录或反常识事实。
3. 落笔前复核关键事实；拿不准就换事实或直说没查到可靠事实。
4. 用同一个事实写 2-3 个不同钩子版本：历史规律外推、提问、填空、反常识、二选一/站队按需组合。
5. 每版先删长：保留一两句，去掉书面腔、文艺腔、煽情、排比和不必要解释。
6. 检查钩子是否低门槛：读者扫一眼就能回答，最好能用个人经验、报数、补词或选边回应。
7. 输出时按 `SKILL.md` 格式标注版本和钩子说明，末尾附事实来源；有强画面资料时再补配图提示。

## Reusable Heuristics

- 最稳的结构是“一个真事实 + 一个缺口”；事实负责可信，缺口负责让人接话。
- 事实不是越多越好；一条推文通常只需要一个最能支撑原意的事实。
- 短比完整重要。能让读者补的，不要替读者讲完。
- 版本差异应来自钩子机制差异，而不是把同一句话改成三种语气。
- 提问钩要贴近日常经验；填空钩的空必须有唯一或很强的正确答案；二选一钩适合降低回复门槛。
- 历史规律外推钩需要至少两个真事实形成暗规律，否则容易变成空泛预测。
- “第一、唯一、最后、最多、超 X”这类词很有钩子，但必须严格核实。
- 配图只在能增强事实感时使用；没有强图时，纯文字更干净。

## Promotion Backlog

- 增加输出前自检清单：事实已核、来源已附、版本钩子不同、正文短白、无煽情文艺腔。
- 沉淀常见钩子模板库，但模板只描述互动机制，不替代联网核实。
- 建立高风险事实词提示：`第一`、`唯一`、`最后`、`100倍`、年份、人数、纪录类断言必须二次核对。
- 为配图提示补最小规则：只推荐公共领域、Wikimedia、官方档案或来源清晰的老照片/数据图。
