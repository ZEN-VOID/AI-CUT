# North Star Projection Contract

本文件定义从 `projects/aigc/<项目名>/0-初始化/north_star.yaml` 向每个分镜组组头投影风格字段的规则。

## Required Fields

每个分镜组组头必须直引三项，但输出时不显示字段标题：

| output order | north_star.yaml key |
| --- | --- |
| 第 1 行 | `全局风格.全局风格提示词` |
| 第 2 行 | `类型元素.类型元素提示词` |
| 第 3 行 | `细分风格.画面风格` |

直引优先级高于摘要。不得把三项字段改写成概括句，也不得把项目 `MEMORY.md` 或 `CONTEXT/` 的风格偏好替代 north_star 三项。

## Output Lines

组头固定写成三行：

```text
<全局风格.全局风格提示词原文>
<类型元素.类型元素提示词原文>
<细分风格.画面风格原文>
```

不得输出 `[全局风格]`、`[类型元素]`、`[画面风格]` 等可见标题字段；不得使用中文括号包裹字段内容；不得在三项之间添加逗号、顿号、连接词或其他装饰性拼接符。若原字段是 YAML 列表，应保持原列表语义，可用分号连接列表项，但不得删改内容。若字段是多行字符串，应保留主要换行或用空格拼接成可读单行，并在执行报告说明投影方式。

## Missing Field Handling

- 三项任一缺失时，不得猜测补写。
- 可读取相近字段时，只能在执行报告中列为候选，不得静默替代。
- 需要继续执行时，应先修复 `north_star.yaml` 或取得用户明确授权。

## Word Count Handling

north_star 三项计入 1980 字限制。若三项内容较长，应缩小本组分镜剧本正文范围，不能摘要 north_star 来规避字数限制。
