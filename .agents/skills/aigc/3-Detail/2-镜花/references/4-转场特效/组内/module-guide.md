# 组内 模块说明

## 定位

- 本叶子负责优化组内分镜之间的切接收益和视觉递进。
- 它不负责把普通剪切包装成花哨技巧，只负责让衔接自然顺滑。
- 它优先解决的是“哪些地方需要被观众感知到顺滑”，而不是“每一刀都要设计方法名”。

## 使用方法

- 先判断当前切接问题主要来自动作、视线、节奏还是情绪承接。
- 再比较保持直切是否已经足够；若直切最强，只需保留隐性的 handoff，不必硬写转场提示。
- 只有在直切不足时，再抓动作承接、视线承接和情绪推进最关键的切接点。
- 输出应让相邻镜头之间更顺，不必每刀都命名。

## 具体创作方法

### 1. 只挑“关键切点”，不要平均用力

- 先找本组里最容易断气的 1 到 3 个切点。
- 优先级通常如下：
  - 动作接力点
  - 视线对接点
  - 情绪抬落点
  - 节奏换挡点
- 如果每一刀都想照顾，最后常常没有一刀真正有效。

### 2. 先写承接逻辑，再决定是否显化

- 推荐顺序：
  - 先说上一镜留下了什么。
  - 再说下一镜接住了什么。
  - 最后才判断是否需要补一句 handoff hint。
- 例如不是先写“用一个漂亮转场”，而是先写“让抬手余势在下一镜落下时继续成立”。

### 3. 四类常见组内问题的稳妥写法

#### 动作承接

- 适合处理：抬手、转头、起身、扑上、回身、停顿后再动。
- 稳妥写法：
  - 以前一镜未尽动作带入下一镜完成。
  - 让动作方向和力线延续，不强调切换本身。
- 原则：动作线比技巧名更重要。

#### 视线承接

- 适合处理：看向某人、看向门外、看向物件、视线被吸走。
- 稳妥写法：
  - 让视线方向、落点或主客体关系在切后立即被接住。
  - 若后一镜能直接揭示视线对象，通常不必再写多余转场。

#### 节奏承接

- 适合处理：静到动、快到慢、密到疏、连续反应之间的节拍转换。
- 稳妥写法：
  - 让镜头切换顺着呼吸点、停顿点或动作尾音发生。
  - 若节奏断得太硬，可补“略带缓冲”的描述，但不要演变成拖慢节奏。

#### 情绪承接

- 适合处理：压抑到爆发、犹疑到决断、冷静到失控。
- 稳妥写法：
  - 让前一镜的情绪余波在下一镜里被接住或翻面。
  - 情绪承接要落在人和动作上，不要落在空泛效果词上。

## 思维·执行节点

### N1 锁关键切点

- 问：哪一刀最容易让动作、视线、节奏或情绪断掉？
- 产物：`key_cut_points`。

### N2 直切测试

- 问：如果什么都不补，只干净切过去，会不会更强？
- 若答案是“会”，就只保留极轻承接 note。

### N3 选择承接主轴

- 在 `动作 / 视线 / 节奏 / 情绪` 中只选当前最主要的一条。
- 其他维度只做辅助，不要四条一起写满。

### N4 写 handoff hint

- 只在必要时补一句说明：
  - 承接什么。
  - 为什么更顺。
  - 不额外发明新动作。

### N5 回删炫技词

- 删掉没有连续性收益的词，例如纯效果化、纯装饰化的转场措辞。
- 保证输出读起来更像剪辑判断，而不是花字说明。

## 节点延展

### 可复用的微策略

- 动作接力：前镜起势，后镜落势。
- 视线接力：前镜抛视线，后镜接落点。
- 构图押韵：前后镜主体位置或方向形成低调呼应。
- 节拍贴合：在呼吸点、顿点、反应点切换。

### 适合省略显式转场的情况

- 动作本身已自然连贯。
- 视线对象在下一镜一出现就能成立。
- 快节奏段落需要利落，不宜被说明性文字拖慢。
- 组内重点是信息清晰，不是形式表演。

## 失真与修正

- 若还没说明为什么不是直切，就直接写转场方式，说明前置判断缺失。
- 若普通剪切被写成效果展示，说明过度渲染。
- 若切接方式和动作连续性冲突，优先保动作线。
- 若组内衔接太多，保留最关键的几个切点即可。
