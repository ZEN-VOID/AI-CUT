## Durable Boundaries

- workflow 的规范真源在 `SKILL.md`；`CONTEXT/` 只保存可复用经验、反例、启发和长期记忆。
- workflow 是 HyperFrames-native 视频工作流，不恢复旧 F1 scripts、EDL、MoviePy 或 ffmpeg filter graph 作为主执行器。
- `workflow/video-to-manifest` 是直属卫星技能，只能作为可选素材证据入口，不替代 workflow 的 `asset_evidence.json`、storyboard 或 HyperFrames runtime。
- workflow 普通任务的完成态是 final MP4；除非用户明确要求 plan-only、audit-only、asset-evidence-only、no-render，或依赖阻断，否则不能停在可预览工程。
- workflow 默认输出比例为 `16:9`、`1920x1080`；竖屏素材、竖屏参考片或短视频语境不能自动改写比例，除非用户或项目真源明确锁定。
- workflow final 的默认观众对象是短视频平台 C 端用户或外部潜在客户，不是内部学习交流、项目复盘或团队汇报；入口、计划和报告都应保留这个 audience profile。
- final 成片画面不得默认加入 `workflow`、`HyperFrames`、文案编号、流程说明、参考说明或其他工具/工作流水印。
- 台词字幕必须以音频为主时钟；final 路线必须有 `dialogue_sync_validation.json` 或等价严格证据，不能只用 `manual_script_audio_duration`、全片等分或预览级 cue。
- 台词字幕必须同时满足时间同步和顺序同步：cue 时间、脚本顺序、音频 anchor 顺序和 HTML `data-cue-id` 时间线都要一致；字幕顺序错乱不能只靠交换 HTML 可见文字修复，必须回 `N4` 重建 `dialogue_alignment.json`。
- 社媒广告型成片不能退化成会议汇报式动态 PPT；有可用影像/工具/结果视频时，背景层默认优先使用视频或影像承托。
- 社媒广告、爆款口播和批量获客成片不能只按文案随机拼接素材；必须先锁 `hook_opening -> content_body -> private_traffic_cta` 节奏骨架，再落 `background_video`、`semantic_pip`、`dialogue_caption`、`editorial_overlay` 四层。
- 严格社媒/批量路线中，`projects/素材/开头素材/` 是 hook_opening 的必选候选池，选中素材要 5-10 秒完整展示；`projects/素材/引流素材/` 默认不得放大；PiP 默认要有同屏多窗网格和可读尺寸，不能退化为单个很小的装饰角标。
- 大字报/左上角标题必须是匹配文案 cue 的精华总结，且带 source cue/text/reason 证据；“工作流程”“内部学习交流”“项目复盘”等内部标题不得进入观众可见画面。
- `validate_visual_contract.py` 是观众可见文本、字幕完整性、overlay/caption 去重、PiP cue 绑定、manifest match 和批量 ledger/audit 的机械验收门。
- `projects/素材/` 和 `projects/示例/` 是外层可累积通用素材池，默认只读，不作为当日输出目录或 final 成片归档目录。
- `projects/素材使用监控.csv` 是跨批次素材使用的用户可读全局汇总表，固定四列 `素材名/文件路径/使用次数/使用程度`，其中 `使用程度` 只能是 `全片` 或 `部分切片`；只有本地 final 验证通过后的实际使用才计数。
- 当日 workflow 过程文件默认进入 `projects/output/<日期>/过程/`；单条 final 验证后归集到 `projects/output/<日期>/`；批量 final 验证后统一移动/归集到 `projects/output/<日期>/成片/`。
- `projects/素材/` 的预处理目录可以为空；空目录只代表候选落位，不能被 workflow 当作真实素材证据。
- `projects/内容/文案/` 与 `projects/内容/音频/` 是当前批量脚本和旁白主时钟池；同名 stem 是稳定配对键，`BGM.*` 只作为背景音乐候选，不是旁白主时钟。

## Stable Asset Taxonomy Memory

- 视觉素材分支：`开头素材/`、`收益素材/`、`漫剧素材/`、`大字报/`、`工作流素材/`、`引流素材/`、`资产图/`、`转场素材/`。
- 关键词分支：`核心关键词/`。
- 内容池分支：`projects/内容/文案/` 下的 `.txt` 是内容真源，`projects/内容/音频/` 下同 stem 的 `.mp3` 是对应主时钟；`BGM.*` 是背景音乐候选。
- 历史原始入口：旧式 `视频/`、`图片/`、`文案/`、`音频/` 应归入 `projects/素材/旧/` 或明确标记为 archive；不得在未获授权时自动搬迁到当前预处理分支。

## Promotion Signals

- 同一失败模式连续影响 final 验收，应从 `CONTEXT/负向经验.md` 晋升到 `Review Gate Binding` 或 validator。
- 同一正向做法连续稳定提升输出质量，应从 `CONTEXT/正向经验.md` 晋升到 `SKILL.md` 的节点、量化标准或输出合同。
- 路径、目录、registry、模板或脚本行为变化，应同步 `SKILL.md`、README、registry 和测试提示，而不是只修改本文件。
