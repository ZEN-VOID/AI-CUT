# Frame Image Heuristics

- 镜级生图不是把整组内容揉成一张图，而是为当前 `分镜N` 找到最能表达该镜头的单帧。
- 若源镜头是运动过程，prompt 应选择一个可绘制的峰值瞬间，例如“门缝扩大到红色高跟鞋即将入画前”。
- 若源镜头有两个分镜明细分句，优先选择视觉信息更强、剧情更关键、主体更明确的一句作为单帧主画面。
- 角色、场景、道具参照图不应压过当前镜头构图；但场景参照图必须额外承担视觉风格锁，约束画面风格、光影、色调和氛围。
- 同一场景下组织新 prompt 时，上一分镜已生成图是实际视觉状态；先 `view_image` 回看，再把角色站位、走位方向、朝向、遮挡、关键道具相对位置和镜头轴线转成当前镜头的连续性约束。
- 批量生图前先写完整 prompts 文档，能把镜级叙事、空间模型和参照槽位一次性摊开审查；imagegen 阶段只消费已审核 prompt block，并把 runtime 上一画面回看写入结果/报告。
- 场景空间一致性不是同一张背景；同场景每一镜都要在 3D 空间中定位角色起点、终点、移动轨迹、朝向、视线和遮挡。
- 分组场景名不是单镜主锚点。蒙太奇、插入镜、道具特写、转场镜头和路线图要从当前 `分镜N` 的可见主体重投影锚点，例如飞檐、城门、桅杆、海图朱线、纸边、刀尖、油绢纤维或酒葫芦停点。
- 正反打对话戏中，连续镜头背景面可能是相对的南北面、东西面或房间两端；只要对话轴线、视线闭合和角色相对位置自洽，就不是场景漂移。
- 不要只把“画面风格与场景参照一致”写成空泛口号；先看场景图，再抽出可执行的光源方向、光比、色温、主色/辅色、雾气、材质和暗部密度。
- 冷白日光灯、黑窗帘、低饱和黑白红、金色真相层是项目风格骨架；重复过多会浪费 1300 English words 额度，prompt 中应压缩整合。
