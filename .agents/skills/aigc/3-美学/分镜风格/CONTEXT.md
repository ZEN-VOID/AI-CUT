## Repair Playbook

1. 先确认失败属于来源不足、节奏证据不足、连接语法缺失、信息密度失衡、剧情越权、镜头编号越权、资产设计越权、摄影参数越权、字数不合规还是报告证据缺失。
2. 若来源不足，回到 `N1-INTAKE`，不要用通用剪辑术语补空。
3. 若剧本分析变成剧情复述，回到 `N2-STORY-EVIDENCE`，抽取段落压力、信息揭示、情绪转折和场景转换节奏。
4. 若参考视频带来照搬，回到 `N3-STYLE-REFERENCE-EVIDENCE`，只保留节奏、连接、信息密度和段落推进事实。
5. 若 prompt 出现镜头编号、剧情动作、资产词或摄影参数，回到 `N7-PROMPT-DISTILL`，用分镜组织层级表达替换。
6. 若连接语法不可执行，回到 `N5-CONNECTION-GRAMMAR`，为每条规则补适用场景、承接对象和禁用边界。
7. 若信息密度过高，回到 `N6-DENSITY-FLOW-CALIBRATION`，把复杂动作链拆成建立、推进、反应和尾钩功能。
8. 修复后必须重新执行 `GATE-SBS-01` 到 `GATE-SBS-10`，并把失败码、返工点和复审结果写入 `Repair Log`。

## Reusable Heuristics

- 分镜风格不是分镜正文的压缩版，而是后续写分镜时遵循的组织语法。
- “景别流转”可以写宏观倾向，不要写具体镜头列表。例如可写“建立镜头后压缩为反应与细节交替”，不要写“镜头1远景、镜头2近景”。
- 分镜风格可以继承摄影风格的能量，但不能继承摄影参数。焦段、光圈、机位、灯位应留给摄影风格或具体镜头阶段。
- 参考视频分析先做“去剧情化”：遮掉人物和事件后还剩下的节奏、承接、转场、信息密度，才适合进入协议。
- AI 视频下游通常怕过长动作链、快速主体切换和无承接转场；分镜风格应鼓励动作承接、视线承接、情绪承接和段落钩子。
- 约 100 字 prompt 的稳定结构：节奏密度 -> 景别流转 -> 镜头组合 -> 连接方式 -> 信息密度 -> 下游容错 -> negative traits。
- 如果一个句子只能指导某个具体镜头，它不属于分镜风格；如果它能指导一整段分镜如何推进和衔接，它才适合进入本协议。
- “流畅性”必须落成承接机制，例如视线匹配、动作余势、情绪反应、信息递进或节奏缓冲。
- “连贯性”必须考虑 AI 视频执行限制，避免把多个主体、复杂动作和场景变化压进同一短段。
- 负面特征应制作化，例如避免硬切堆砌、镜头跳轴、信息爆炸、过密动作链、无承接转场、随机景别变化。
