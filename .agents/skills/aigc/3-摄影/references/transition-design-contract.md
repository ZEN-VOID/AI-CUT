# Handoff Boundary Contract

本文件保留旧路径 `transition-design-contract.md`，但语义已迁移：`3-摄影` 不再主创组间或跨场景转场设计。本文件只定义镜头内部连续性之外的边界交出点、进入提示和连续性风险，供下游 `4-分组` 的组间首尾帧连接件消费。

## Core Rule

`3-摄影` 只回答三个问题：

1. 当前画面从哪里进入，是否需要延续上一画面的轴线、运动、声音、光色或焦点。
2. 当前画面最后一镜把注意力交给哪个可见主体、动作、声音、光色、文字、道具、空间出口或反应。
3. 若下一画面或下一场景需要连接，哪些可见锚点可被 `4-分组` 用来设计 3-4 秒连接件。

`3-摄影` 不裁决普通切镜、软桥接、匹配剪辑、高能转场、光变转场、声画桥转场或其他创意转场方案。跨场景、跨组和首尾帧缝合的创意落盘归属 `4-分组/references/bridge-shot-contract.md`。

## Handoff Triggers

以下情况只触发 `handoff_profile`，不触发转场主创：

| trigger | required evidence | 3-摄影输出边界 |
| --- | --- | --- |
| 场景变化 | 场景标题、空间、时间段或叙事段落发生切换 | 标记上一画面的交出锚点和下一画面的进入提示 |
| 空间重置 | 人物位置、轴线、门窗/走廊/教室等空间关系需要重新定位 | 给出建立空间所需的当前入口或末端落点 |
| 注意力转交 | 观看焦点从人物转到道具/文字/危险源/群体反应 | 写清焦点落在哪里，供下一阶段承接 |
| 动作方向承接 | 推门、翻页、抬眼、转身、跌落、敲击等动作可能连接下一画面 | 记录动作的方向、速度和落点 |
| 声音先行或余波 | 铃声、脚步、敲击、广播、嗡鸣等声音可作为连接素材 | 记录声音来源、持续状态和落点，不设计声画桥方案 |
| 形态/颜色/文字 | 两个画面共享轮廓、色块、光斑、文字形态或运动方向 | 记录可见形态和颜色/文字状态，不写匹配剪辑 |
| 高点余波 | 惊吓、惩罚、规则打断、认知翻转或奇观冲击后的反应 | 记录结果、反应或停点，供下游决定是否做断裂连接 |

## Handoff Profile

每个命中边界风险的 `visual_unit` 在输出前形成内部 `handoff_profile`：

| field | question |
| --- | --- |
| `boundary_need` | none / local_handoff / scene_boundary / group_boundary_candidate |
| `trigger_source` | 场景变化、空间重置、注意力转交、动作承接、声音余波、形态/颜色/文字、信息显影或高点余波 |
| `outgoing_anchor` | 当前画面最终交出的可见主体、动作、声音、光色、文字、空间出口或反应 |
| `incoming_hint` | 下一画面可能从哪里进入；若未知，只记录当前可交出的稳定锚点 |
| `continuity_risk` | 是否存在跳轴、跳色、景别断崖、空间迷失或表演停顿被破坏 |
| `downstream_note` | 给 `4-分组` 连接件可消费的简短锚点，不写完整连接方案 |

## Decision Rules

1. 场景变化默认触发 `handoff_profile`，不得让当前画面没有交出锚点就结束。
2. 同一场景内部的普通动作承接不必显式写边界说明，除非动作、声音、光色或信息会成为下一阶段连接素材。
3. 表演停顿、规则读秒和道具可读性优先；不要为了给下游连接件留素材而抢走当前画面要看的信息。
4. 高强度断裂只能作为当前画面的停点、反应或交出锚点存在，不在本阶段写成转场方案。
5. 不得使用“丝滑转场”“高级转场”“电影感转场”“匹配剪辑到下一场”等词替代具体可见锚点。

## Output Guidance

| weak/random | handoff-focused |
| --- | --- |
| `用高级转场进入下一场。` | `镜头停在学生僵住的手上，指节还压着桌沿；日光灯嗡鸣没有立刻消失。` |
| `画面丝滑切到教室。` | `上一镜的白光停在门缝边缘，画面最后只留下冷白亮面和黑色门框。` |
| `快速转场表现紧张。` | `教鞭点桌的动作成为当前画面的落点，粉笔灰在黑板前短暂悬住。` |

## Review Gate

边界交出合格必须同时满足：

- 场景变化、空间重置或注意力转交已被识别，并有清楚交出点或进入提示。
- 交出锚点具体可见，能被 `4-分组`、`6-图像` 和 `7-视频` 继承。
- 没有在 `3-摄影` 中落盘组间/跨场景创意转场方案。
- 没有为了边界交出破坏演员停顿、文字可读、空间方向或项目视觉母题。
