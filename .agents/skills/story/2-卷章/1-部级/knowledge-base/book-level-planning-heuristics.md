# Book-Level Planning Heuristics

本文件保存部级规划的稳定经验。强制执行规则仍以 `SKILL.md`、`references/` 和 `review/` 为准。

## Success Patterns

- 好的 `整体规划.md` 会让卷级无需重猜“这本书到底在追什么”，只需要把卷职责下钻。
- `卷划分` 先写功能，再写情节；功能清楚，情节才不会互相抢戏。
- `整部任务关系` 最好先树后簇再里程碑：主任务树决定方向，支流簇决定扩张，里程碑决定汇聚。
- `整体冲突` 要有长期压力走廊，不能只列一个反派或一个外部危机。

## Failure Patterns

- 把 Save the Cat 15 步写成百分比，会让长篇卷级规划变得僵硬。
- 只写“第一卷铺垫、第二卷发展、第三卷高潮”会导致卷级全靠二次发明。
- `规避` 若写成“避免拖沓、保持爽感”，下游无法执行；应写成具体禁飞区。
- 部级若复制角色卡、场景卡、物品卡、技能卡，会制造第二套真源并拖慢后续修订。

## Practical Checks

- 读完 `卷划分` 后，问：每一卷为什么非存在不可？
- 读完 `整部任务关系` 后，问：哪条支流何时回主线？
- 读完 `整体节奏曲线` 后，问：`Midpoint / All Is Lost / Finale` 分别改变了整部走势的什么？
- 读完 `规避` 后，问：卷级和章级能不能据此删除或避免某类写法？
