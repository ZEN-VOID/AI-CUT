# Volume Planning Heuristics

本文件保存稳定可复用经验。强制入口与门禁仍以 `SKILL.md`、`references/` 和 `review/` 为准。

## Heuristics

- 每卷必须有自己的追读承诺；如果只是在复述整部总纲，这卷还没有成立。
- 章划分的最小合格标准是“每章功能 + 承接关系”，不是章名清单。
- 六拍里最容易缺的是 `首回报` 和 `中卷反拧`；缺这两项时，本卷通常会变成连续铺垫。
- `支线` 需要解释为什么存在：服务主线、制造压力、转交下一卷，至少占一项。
- 卷末达成要同时写“完成了什么”和“留下了什么”，否则章级会不知道如何收束或续推。

## Common Repair Moves

| symptom | repair |
| --- | --- |
| 前半空转 | 提前设置 `首回报章节`，把一个小 promise 在卷前半兑现 |
| 中段疲软 | 让 `中卷反拧` 改变任务理解、敌我关系或代价结构 |
| 资源堆叠 | 删除不承担冲突、揭示、转向或交接功能的人物/场景/道具 |
| 支流游离 | 给每条支流标注 `汇聚回主线` 或 `转交下一卷` |
| 卷尾平 | 明确卷末冲突状态和下一卷压力入口 |
