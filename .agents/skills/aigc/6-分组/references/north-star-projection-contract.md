# North Star Projection Contract

本文件定义从 `projects/aigc/<项目名>/0-初始化/north_star.yaml` 向每个分镜组组头整理风格字段的规则。

`全局风格.全局风格提示词` 是指导整个作品全集各部分的总风格母稿，是并集式风格合同，不是每组简单照抄的交集前缀。`6-分组` 必须以它为总体依据，结合当前分镜组的场景类型、空间、时间、动作、光源、色彩、材质、空气、摄影和负面约束证据，抽取并重组当前组真正匹配的风格部分。

## Required Fields

每个分镜组组头必须整理并输出三项，但输出时不显示字段标题：

| output order | north_star.yaml key | output rule |
| --- | --- | --- |
| 第 1 行 | `全局风格.全局风格提示词` | 在行首前置固定词：`视频生成的画面风格，光影和氛围与场景参照图保持一致。需要生成现场物理互动音效、氛围感音效、环境声、自然现象声、动作声，不要生成任何字幕，不要生成背景音乐。`，再接当前分镜组专属的全局风格整理句；整理句必须是自然中文单段，300 字以内，不得简单照抄完整全局母稿 |
| 第 2 行 | `类型元素.类型元素提示词` | 直引原文 |
| 第 3 行 | `细分风格.画面风格` | 直引原文 |

第 1 行是“场景化整理”，不是摘要删减，也不是完整直引。第 2、3 行仍直引原文，除非用户或项目合同另行要求。不得把项目 `MEMORY.md` 或 `CONTEXT/` 的风格偏好替代 north_star 三项；它们只能作为当前组匹配和取舍的辅助证据。

## Global Style Extraction Rules

每个分镜组必须先形成内部 `group_style_projection`：

1. 读取 `全局风格.全局风格提示词`，必要时参考 `全局风格.光影逻辑`、`全局风格.画面质感`、`全局风格.场景化风格策略` 与 `全局风格.避免出现`。
2. 从当前分镜组取证：场景标题、入场镜头、画面构图、组内 `分镜N`、上一组尾帧、连接件首态，以及组内画面属性。
3. 判定当前组属于哪些可复用场景类型：例如室内、室外、夜景、正午、傍晚、雨雪、旷野、战场、武打、群像、对话、仪式、密室、旅途、追逐、静物证据、情绪停顿等。
4. 从全局母稿中只抽取与这些类型匹配的风格词：媒介/时代/质感底座可保留；光影、色彩、空气、材质、摄影、运动和负面约束必须与当前组场景证据匹配。
5. 写成一段自然语句，300 字以内；不使用标签列表、参数清单、字段名或“本组适用”等元说明；不得为凑风格加入当前组不存在的天气、光源、材质、角色或动作。
6. 若全局母稿没有覆盖当前场景类型，只能抽取通用底座和当前组可证实的风格逻辑，并在执行报告记录 `style_projection_gap`；不得凭空发明新的全局风格。

示例：全局母稿包含“室内单侧窗光/烛光硬光、室外正午硬顶光、傍晚金色逆光、武打动态追光”时，室内密谈组只取室内动机光、胶片质感、材质与低饱和厚重色调；旷野追击组才取风沙、硬顶光或逆光、尘土肌理和动作追光。

## Style Signature Retention Rule

场景化整理不得把作品级风格削弱成“当前场景物件 + 普通光影”的局部描述。只要 `north_star.yaml` 的全局母稿已经提供明确的作品风格签名，当前组第 1 行整理句必须同时保留：

- **作品身份锚点**：至少保留 2 个来自全局母稿的稳定身份词或等价中文表达，例如媒介、时代、地域/类型、核心审美风格、主情绪母题。
- **场景适配锚点**：保留当前组真实存在的空间、光源、色彩、材质、空气、动作或摄影证据，避免完整照抄母稿。
- **材质/摄影锚点**：至少保留 1 个对下游生成有强约束力的材质、镜头、质感或运动标识，例如真实 4K 质感、戏剧性明暗对比、克制推轨、危险时手持碎裂、玻璃/金属/血/布料等物理反应。
- **边界锚点**：在必要时保留当前项目的关键负面边界，例如不美化胁迫、暴力、羞辱或类型漂移。

执行上，推荐句式是“作品身份锚点 + 当前组场景适配 + 材质/摄影锚点 + 边界锚点”。例如 TMBR 这类已在 `north_star.yaml` 中声明 `live-action AIGC film / contemporary New York mafia neo-noir luxury melodrama / low-key luxury / dramatic light-dark contrast / restrained tracking pressure / sudden handheld danger / realistic material texture` 的项目，书房组不能只写“冷光、橡木桌、照片反光”，还应保留 `contemporary New York mafia neo-noir luxury melodrama`、`low-key luxury`、`dramatic light-dark contrast`、`restrained tracking pressure` 或等价强标识中的若干项。

若当前组整理句无法在 300 字以内同时保留作品身份和场景证据，应优先压缩局部物件枚举，而不是删除作品级风格签名。

## Output Lines

每个分镜组标题后先写场景标题行，再写 `入场镜头：` 和 `画面属性：`，随后组头固定写成三行：

```text
视频生成的画面风格，光影和氛围与场景参照图保持一致。需要生成现场物理互动音效、氛围感音效、环境声、自然现象声、动作声，不要生成任何字幕，不要生成背景音乐。 <当前分镜组专属全局风格整理句，300字以内>
<类型元素.类型元素提示词原文>
<细分风格.画面风格原文>
```

不得输出 `[全局风格]`、`[类型元素]`、`[画面风格]` 等可见标题字段；不得使用中文括号包裹字段内容；不得在三项之间添加逗号、顿号、连接词或其他装饰性拼接符。第 1 行固定前置词必须出现在当前组全局风格整理句之前，不得单独成第 4 行，也不得加到第 2 或第 3 行。若第 2、3 行原字段是 YAML 列表，应保持原列表语义，可用分号连接列表项，但不得删改内容。若字段是多行字符串，应保留主要换行或用空格拼接成可读单行，并在执行报告说明投影方式。

## Missing Field Handling

- 三项任一缺失时，不得猜测补写。
- 可读取相近字段时，只能在执行报告中列为候选，不得静默替代。
- 需要继续执行时，应先修复 `north_star.yaml` 或取得用户明确授权。

## Word Count Handling

north_star 三项与第 1 行固定前置词不计入组内 `时长估算`、1680 字目标上限、1980 字硬上限或 YAML `字数统计`。每组标题后的场景标题行、入场镜头、画面构图、构图分区和画面属性计入 YAML `字数统计`，但不计入 `时长估算`。组内时长只累计从 `5-摄影` 划定正文里的 `分镜N（约X秒）`；1680/1980 只作为场景标题行、入场镜头、画面构图、构图分区、画面属性加纯分镜剧本正文的辅助风险线。即便 north_star 的全局母稿较长，也不得删除固定前置词；第 1 行应通过场景化整理控制在 300 字以内，边界裁决只能回到分镜剧本正文的完整 atomic unit 与显式时长累计。
