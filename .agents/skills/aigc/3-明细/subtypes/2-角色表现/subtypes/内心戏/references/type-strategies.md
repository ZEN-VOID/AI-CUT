# aigc 3-明细 / 2-角色表现 / 内心戏 / Type Strategies

本文件承载 `aigc 3-明细 / 2-角色表现 / 内心戏` 的路由策略、VSM 与局部回退规则。

## Strategy Scope

- 本层处理的标准类型为：`内心OS / 回忆 / 幻想 / 梦境 / 闪回 / 闪进 / 抽象象征 / 音乐联想 / Q版`。
- 上述类型只作内部裁决标签，不得在正文显式输出标签前缀。
- `内心戏` 默认不是“奇观制造层”，而是把主观压力翻译成当前场景内可拍、可读、可回收的句段。

## Local Route Note

- 本技能是父级裁定后的唯一 leaf 入口，不在本层再分叉。
- 进入前提以主 `SKILL.md` 的 `When to Use / When Not to Use` 与父级路由结果为准。
- 若当前问题跨 sibling、输入前提不齐或边界不清，先回退父技能重新判路。

## Hard Gates

### 表达形态

- 正文只能使用自然句，不得出现 `类型名：`、`xxx画面：` 一类标签。
- 类型差异必须通过触发物、句法节奏、物象选择与现实回钩来体现。

### 不可变层

- 禁止改写对白、独白、旁白的既有文本内容。
- 禁止新增关键剧情事实、关键角色关系、关键世界规则。
- 禁止让非现实段落反客为主，替代现实叙事主线。

### 密度与连续性

- 默认每个场景融写 `1-3` 条，单段最多 `1` 条；超出时必须有“该场承担转折/创伤回收/决策演算”的必要性证据。
- 同一触发点不得重复融写同类画面，优先保留信息量最高、最易回收的一条。
- 非现实画面必须紧贴触发语句落盘，不允许跨场景漂移。

## Inner-Scene Micro-Performance Guidance

| 主观承载方式 | 更稳的使用场景 | 优先锚点 |
| --- | --- | --- |
| 静默 | 情绪压住不爆、场面暂时失声 | `呼吸 + 目光 + 姿态` |
| 物象投射 | 需要让内层张力落到死物上 | `手部/道具 + 目光 + 交互反馈` |
| 感官错位 | 极度紧张、专注或失衡时 | `呼吸 + 面部 + 感官聚焦` |
| 记忆碎片 | 当前刺激明确勾连过去时 | `目光 + 呼吸 + 回收钩子` |
| 轻度非现实化 | 现实感尚在，但主观已开始扭曲 | `姿态 + 感官 + 回收钩子` |

执行门禁：

1. 静场不等于空场，一个呼吸停顿、一次视线回收也算有效节拍。
2. 先试 `静默 / 物象 / 感官`，只有它们不够时再上 `记忆` 或 `轻度非现实`。
3. 若主观段写完回不到当下动作或关系对象，说明已经越界。

## NRVM-3D 裁决框架（Mandatory）

先判“这段主观画面承担什么叙事功能”，再根据 `情绪维度 + 剧情节点 + 题材风格` 联合裁决主类型；若存在明确音乐或音效 cue，再叠加音频触发校正。

### Emotion Axis

| 情绪态势 | 优先类型 | 落盘提醒 |
| --- | --- | --- |
| 极度悲伤、遗憾、悼失 | `回忆`、`闪回` | 先让现实触发物开门，再用短碎片回钩 |
| 强烈恐惧、焦虑、失控前兆 | `梦境`、`内心OS`、`抽象象征` | 优先压迫感与失真感，不先讲道理 |
| 狂喜、亢奋、愿望外溢 | `幻想`、`闪进` | 快速给愿景，再迅速打回现实 |
| 荒诞、缓冲、轻喜剧反差 | `Q版` | 只能短促存在，不可改写整体声部 |

### Plot Node

| 剧情节点 | 优先类型 | 作用 |
| --- | --- | --- |
| 关键转折、真相揭示前后 | `闪进`、`抽象象征` | 点亮压力或后果，不延长支线 |
| 创伤回收、因果补偿 | `闪回`、`回忆` | 用过去向碎片给当下加压 |
| 决策前高压停顿 | `内心OS` | 让临界状态变得可见 |

### Genre Style

| 题材风格 | 优先倾向 | 收敛规则 |
| --- | --- | --- |
| 悬疑、惊悚 | `闪回 + 梦境 + 低饱和象征物` | 防止过于抒情，保持压迫与可疑感 |
| 喜剧、轻快 | `Q版 + 节奏点 + 快速回收` | 只作缓冲，不吞掉现实层 |
| 实验、作者向 | `抽象象征 + 音乐联想` | 仍需单一主物象与明确回钩 |

### Audio Cue Override

- 当音乐、儿歌、噪声、机器鸣响或特定音效直接触发联想时，优先把 `音乐联想` 视为候选主类型或辅类型。
- 音频触发必须可追溯到当前段的真实声音来源，不能凭空插入。

## IOS-Visual 机制（Mandatory for `内心OS`）

当主类型裁决为 `内心OS` 时，不允许停留在“角色很纠结/很害怕”的解释句，必须把压力写成可见动作、物象或空间变化。

### Trigger Gate

- 决策高压：处于“做/不做、说/不说、进/退”的临界点。
- 表里反差：外层动作克制，内层冲突剧烈，可观察到微反应。
- 信息差承压：角色知道信息但不能明说。
- 生理失控：呼吸、指尖、咬肌、视线、吞咽、肩颈出现异常。

### OS 子类型梯度

| subtype | 适用情形 | 落盘策略 |
| --- | --- | --- |
| `OS-V0` | 触发弱或证据不足 | 不启用内心OS，回退其他承载方式 |
| `OS-V1` | 细微泄露最可信 | 用局部动作 + 近景物象显影 |
| `OS-V2` | 压力需要具象化 | 将内压压成单一物象或空间压缩 |
| `OS-V3` | 外冷内炸、反差极强 | 使用“慢外层 + 快内层”对撞，但严控夸张度 |
| `OS-V4` | 决策高压最强 | 用 `若A/若B` 的瞬时预演承载内算 |
| `OS-V5` | 记忆短促入侵当前 | 让记忆碎片掠过并在同段回收 |
| `OS-V6` | 喜剧轻快题材补味 | 只能叠加，不能单独做主类型 |

### 主副裁决优先级

- `OS-V4 > OS-V3 > OS-V2 > OS-V1 > OS-V5`。
- `OS-V6` 仅可叠加在已成立主类型上；严肃题材默认禁用。
- 若反差证据不足，则把 `OS-V3` 降级到 `OS-V1/OS-V2`。

### 句法骨架

- 句法骨架固定为：`触发锚点 + 可见变化 + 单一物象/空间异化 + 现实回收钩子`。
- 每条 `内心OS` 句内必须至少出现一个镜头可见的动作或物象。
- 未能在同段或邻句回扣现实动作，直接判 `FAIL-INN-OS-NO-RECOVERY`。

## Dual-Type Landing（Mandatory）

默认策略仍是单主类型优先。只有当单一类型无法同时承载“时间跳切 / 象征加压 / 音频联想 / 喜剧叠加”的第二价值时，才允许进入 `1 主 + 1 辅`。

### 允许组合

| 主类型 | 辅类型 | 第二价值 | blend_mode | landing_mode | 约束 |
| --- | --- | --- | --- | --- | --- |
| `内心OS` | `抽象象征` | 给内压补一个单一物象 | `inline-fuse` | `same-sentence` | 同句只留一个象征物 |
| `内心OS` | `闪进` | 决策演算的未来碎闪 | `adjacent-echo` | `adjacent-sentence` | 先现实触发，再短促预演 |
| `内心OS` | `音乐联想` | 用声音牵出主观层 | `inline-fuse` | `same-sentence` | 必须有可追溯音频 cue |
| `内心OS` | `Q版` | 喜剧题材的趣味叠加 | `inline-fuse` | `same-sentence` | 仅在 `OS-V6` 成立时允许 |
| `回忆` | `音乐联想` | 声音开门，记忆进入 | `adjacent-echo` | `adjacent-sentence` | 先 cue 后记忆 |
| `闪回` | `抽象象征` | 创伤回收后的压强收束 | `adjacent-echo` | `adjacent-sentence` | 先过去碎片，再象征封口 |
| `幻想` | `闪进` | 愿望外溢兼带后果预感 | `adjacent-echo` | `adjacent-sentence` | 必须快速收回现实 |
| `梦境` | `抽象象征` | 加强梦魇压迫感 | `inline-fuse` | `same-sentence` | 不再叠第二时间层 |
| `音乐联想` | `回忆` | 声音是主驱动，过去是被拉起层 | `adjacent-echo` | `adjacent-sentence` | 归因仍属于声音触发 |

### 禁止组合

- `回忆 + 闪回`：同为过去向时间层，信息重复。
- `闪回 + 闪进`：时间方向相反，单触发点易失焦。
- `幻想 + 梦境`：主观层级模糊，容易漂成独立支线。
- `Q版 + 抽象象征`：语气冲突，风格崩坏风险高。
- 任意 `主 + 辅 + 辅` 或套娃式结构。

### Landing Order

- `single-primary`: `trigger -> primary -> recovery`
- `inline-fuse`: `trigger -> primary/secondary 同句融合 -> recovery`
- `adjacent-echo`: `trigger -> primary句 -> secondary句 -> recovery`
- 允许的 `landing_mode` 只有 `single-line / same-sentence / adjacent-sentence`；其余一律视为 `FAIL-INN-LANDING-MODE`。

## Unknown Fallback

- unknown 默认路由：停止在父级判路层，不直接写本层产物。
- 若共享终稿、上游 handoff 或父级判路缺失，先补前提再恢复本层。
- 若类型裁决冲突、辅类型无第二价值、或回收钩子无法成立，直接降级为单主类型或回退父级重判，不保留摇摆方案。
