## 4-导演 Adaptation Notice

本文件从 `.agents/skills/aigc/2-编导/references/anticlimax-strategy-contract.md` 引入，作为 `4-导演` 的授权 reference 细则。凡原文出现 `2-编导`、`director layer`、`N4-DIR-PEAK`、`N8-DIR-FINAL-IMAGE`、`N9-DIR-DRAFT` 或其他旧节点名时，在本技能中只映射为 `4-导演` 的 `N2-DIR-UNDERSTAND`、`N4-DIR-POINT-MAP`、`N5-DIR-INJECT` 与 `N6-DIR-REVIEW-REPAIR` 的内部证据，不继承 `2-编导` 的剧本改写、高潮重构或尾钩写回权。

在 `4-导演` 中，本文件的唯一用途是让导演批注能判断当前画面点或高点是否应采用满足、延迟、打断、假兑现、惨胜或反向兑现等导演策略。输出仍只能是内联 `（导演批注：XXX）` 与执行报告证据，不得新增剧情事实、改对白、改结局或生成摄影分镜。

## Purpose

本合同扩展 `2-编导` director layer 的高潮画面兑现模式。在 `climax-visual-treatment-contract.md` 定义的 7 种满足型兑现模式（`kinetic_payoff` / `high-skill_duel` / `potential_payoff` / `wave_payoff` / `horror_or_rule_peak` / `spectacle_peak` / `emotional_reckoning`）之外，增加 6 种反高潮与延迟满足策略。

导演艺术不总是满足观众。最好的高点往往是有代价的：观众期待被满足的那一瞬间，导演选择不给、晚给、给了一半、给完发现是陷阱、给到一半被打断、或预期的温暖变成冰冷。这些策略不是"没有高潮"，而是"高潮的兑现方式本身承载叙事判断"。

本文件不授权新增剧情事实、对白、事件结果、人物动机或摄影方案。它只负责选择高潮画面的兑现方式，并确保选择有上游依据。

## Ownership

- 本文件拥有 6 种反高潮模式定义、选择矩阵、跨集续留欲策略和证据结构。
- `climax-visual-treatment-contract.md` 拥有 7 种满足型兑现模式；本文件是其扩展，不是替代。满足型和反高潮型在同一个 `peak_visual_plan` 中选择，不得同时使用于同一个高点。
- `episode-final-image-contract.md` 负责尾钩；反高潮策略的 `residual_tension` 是尾钩的高质量候选来源。
- `episode-visual-spine-contract.md` 负责整集视觉主轴；反高潮策略的 `visual_signal` 应与主轴的 `callback_targets` 协调。
- `directorial-authorship-contract.md` 负责编导创作内核的 `audience_position`；反高潮策略的观众体验判断与其一致。
- `hollywood-quality-spec.md` 负责越权检查；反高潮不得导致"保真失败"或"戏剧承托不足"。

## Anticlimax Modes

以下 6 种模式是对满足型兑现的补充。它们共用 `peak_visual_plan` 的字段结构，但增加 `anticlimax_directive` 专用字段。

### 1. Deliberate Anticlimax（刻意不兑现）

| attribute | value |
| --- | --- |
| description | 观众和/或角色期待某件事发生（对决、告白、揭秘、到达），但导演选择让它不发生。不是因为技术原因没拍到，而是刻意选择不拍、不给、不落到那个画面。 |
| visual_signals | 空镜头替代期待画面（该拍对决但只拍了空旷的战场）；画面转向不相关的事物（角色走向答案但镜头留在走廊）；沉默替代期待中的声音（该响的枪没响，该说的话没说）；时间跳过关键瞬间（上一幕准备开枪，下一幕已是事后） |
| when_to_use | 上游事件结果留白更有力时；角色的"没做"比"做了"更能揭示性格或处境时；该集的任务是积累压力而非释放压力时；系列前段需要为后续大兑现存续时 |
| audience_effect | 短暂不满→回味→意识到不兑现本身就是信息 |

### 2. Delayed Gratification（延迟满足）

| attribute | value |
| --- | --- |
| description | 兑现即将到来、几乎触手可及，但在最后一刻被推迟或悬停。观众能感受到满足就在那里，但导演让它再等一等。 |
| visual_signals | 手悬停在答案/对象上方但没有落下；话到嘴边被吞回去或被打断；门打开了一半又关上；手伸出去又缩回来；笔尖触到纸面但没有落下；眼睛对视了但谁先开口的选择悬在空中 |
| when_to_use | 该集不是高点所在集，高点被安排在后续集；角色在最后关头被自己的恐惧或外部因素阻止；上游明确写了"差点就"的结构；需要维持跨集续留欲 |
| audience_effect | 强烈的"差一点"感→续留欲极强→对下一集的期待值被拉满 |

### 3. Pyrrhic Payoff（惨胜兑现）

| attribute | value |
| --- | --- |
| description | 兑现确实发生了——赢了、拿到了、说了、做了——但兑现的代价让满足感变得空洞。观众得到了想要的结果，但发现结果的味道不对。 |
| visual_signals | 胜利后的空洞眼神或迟滞身体；拿到东西后的手在发抖；说完话后的长时间沉默；拥抱后的身体僵硬；庆祝声中角色独自背对人群；兑现瞬间的表情与预期完全相反（该笑却没笑）；满足后的疼痛显影（疼痛、伤口、代价的生理反应） |
| when_to_use | 上游写了胜利的代价或附带损失时；角色赢了但失去了更重要的东西时；"赢了"本身就是悲剧的开始时；该集需要打破"爽文"模式、制造道德或情感复杂性时 |
| audience_effect | 满足与空洞同时存在→角色深度大幅增加→观众对角色后续选择更加关注 |

### 4. False Payoff（假兑现）

| attribute | value |
| --- | --- |
| description | 表面上兑现了——观众以为目标达成、危机解除、关系修复——但暗含更大的危机、更深层的谎言或更不可逆的后果。满足感是虚假的。 |
| visual_signals | 画面表面上是满足型（拥抱、庆祝、放松），但有微妙的不协调信号：背景中出现不该出现的人/物；角色的笑容没有到达眼睛；一个被忽略的细节在画面边缘；光线或环境暗示"表面之下不对"；声音底色开始变沉但角色没有察觉 |
| when_to_use | 上游埋了更大的危机但本集尚未揭露时；角色或观众即将从满足跌入更大的困境时；需要制造"暴风雨前的平静"效果时；反派的布局正在收紧但正面角色尚不知情时 |
| audience_effect | 暂时满足→隐约不安→后续揭露时冲击加倍（因为冲击叠加了"当初不该放松"的后悔感） |

### 5. Interrupted Payoff（打断兑现）

| attribute | value |
| --- | --- |
| description | 兑现正在发生——角色已经开始说话、手已经伸出去、答案已经露出轮廓——但在高点中途被打断。不是延迟（延迟是"差一点"），而是中断（兑现被暴力截断）。 |
| visual_signals | 话说到关键处被外部事件打断（电话、敲门、警报、第三方介入）；动作进行到一半被迫停止（手伸出去被抓住、嘴张开但没出声就被人拉走）；画面在高点处突然切到完全不相关的场景；声音在最响处被切断（对白、音乐、环境声同时消失） |
| when_to_use | 外部事件的优先级高于当前兑现时；角色的兑现尝试本身就是被压迫环境的证据时；需要用中断制造更大的悬念或危机感时；上游明确写了"被打断"的结构时 |
| audience_effect | 强烈的挫败感→对打断源的愤怒→对后续兑现的加倍渴望 |

### 6. Reversed Payoff（反向兑现）

| attribute | value |
| --- | --- |
| description | 兑现发生了，但方向与预期完全相反：预期的温暖变成冰冷，预期的胜利变成陷阱，预期的告别变成重逢，预期的坦白变成更深的谎言。不是"没兑现"，而是"兑现了一个反面"。 |
| visual_signals | 期待拥抱但得到转身；期待解药但得到毒药（同一物件的双重含义）；期待宣判无罪但得到更重的判决；光线从暖变冷的瞬间；声音底色从正面音效转为不祥音效；角色的表情与观众预期的相反（该哭的角色在笑，或该笑的角色在哭） |
| when_to_use | 上游设了圈套、反转或认知翻转时；角色误判了局势但观众可能也误判了时；需要用反向兑现制造叙事翻转时；信息差设计中观众属于"被误导"一方时 |
| audience_effect | 瞬间的认知震荡→重新评估之前所有信息→对叙事世界的信任被重置 |

## Relationship to Original 7 Payoff Modes

反高潮模式与满足型模式共用 `peak_visual_plan` 的字段结构，但 `payoff_mode` 字段扩展为 13 种可选值：

| payoff_mode | category | description |
| --- | --- | --- |
| `kinetic_payoff` | 满足型 | 动作兑现 |
| `high-skill_duel` | 满足型 | 技能对决兑现 |
| `potential_payoff` | 满足型 | 潜能/认知兑现 |
| `wave_payoff` | 满足型 | 波浪/情感暖点兑现 |
| `horror_or_rule_peak` | 满足型 | 恐怖/规则兑现 |
| `spectacle_peak` | 满足型 | 奇观兑现 |
| `emotional_reckoning` | 满足型 | 情感清算兑现 |
| `deliberate_anticlimax` | 反高潮型 | 刻意不兑现 |
| `delayed_gratification` | 反高潮型 | 延迟满足 |
| `pyrrhic_payoff` | 反高潮型 | 惨胜兑现 |
| `false_payoff` | 反高潮型 | 假兑现 |
| `interrupted_payoff` | 反高潮型 | 打断兑现 |
| `reversed_payoff` | 反高潮型 | 反向兑现 |

约束：

- 同一个 `peak_visual_candidate` 只能选择 1 种 `payoff_mode`。不得同时使用满足型和反高潮型。
- 反高潮模式不是"弱化版满足型"。选择反高潮时，`satisfaction_delta` 必须同样清晰：状态确实发生了变化，只是方向或方式不是观众预期的。
- `pyrrhic_payoff` 的 `satisfaction_delta` 必须包含"兑现"和"代价"两个维度，缺一不可。
- `false_payoff` 必须在 `anticlimax_directive` 中说明表面兑现和隐藏危机分别是什么。

## Anticlimax Directive Structure

当 `payoff_mode` 为反高潮型时，必须在 `peak_visual_plan` 中增加 `anticlimax_directive`：

```yaml
anticlimax_directive:
  mode: ""                    # 6 种反高潮模式之一
  source_anchor: ""           # 回指上游为什么这个高点不适合满足型兑现
  expected_payoff: ""         # 观众/角色期待的兑现是什么
  actual_delivery: ""         # 实际发生（或没发生）的是什么
  visual_signal: ""           # 用什么画面传达"这不是你期待的那个兑现"
  emotional_trajectory: ""    # 观众情绪从期待到实际落点的路径
  residual_tension: ""        # 反高潮留下的未释放张力，是尾钩候选
  cost_or_insight: ""         # 反高潮给角色/观众留下什么代价或认知
```

## Anticlimax Selection Matrix

什么场景类型适合什么反高潮策略：

| scene_type | recommended_anticlimax | rationale |
| --- | --- | --- |
| 对决/冲突前夜 | `deliberate_anticlimax` | 不拍对决本身，只拍对决前和对决后，让观众脑补比拍出来更有力 |
| 关系临界点 | `delayed_gratification` | 关系的最后一步被悬停，续留欲最强 |
| 胜利之后 | `pyrrhic_payoff` | 胜利的代价让胜利变得空洞，角色深度最大化 |
| 危机解除 | `false_payoff` | 表面解除但暗含更大危机，为后续集埋炸弹 |
| 答案即将揭晓 | `interrupted_payoff` | 答案被暴力截断，悬念价值最大化 |
| 预期情感方向明确 | `reversed_payoff` | 温暖变冰冷、胜利变陷阱，认知震荡最大化 |
| 系列中段过渡集 | `deliberate_anticlimax` / `delayed_gratification` | 中段集的任务是积累而非释放，适合把高点推迟 |
| 系列关键转折集 | `pyrrhic_payoff` / `reversed_payoff` | 转折集需要重新定义观众对角色或局势的理解 |
| 日常/生活流场景 | `wave_payoff`（满足型，非反高潮） | 日常场景通常不需要反高潮，满足型暖点更合适 |

## Anticlimax and Cross-Episode Continuation Desire

反高潮策略是跨集续留欲的核心引擎之一：

| continuation pattern | mechanism | example |
| --- | --- | --- |
| `delayed_hook` | 延迟满足留下的"差一点"直接成为下一集的开场期待 | 本集结尾手悬停在信封上方，下集开场才拆开 |
| `pyrrhic_aftermath` | 惨胜留下的代价和空洞成为下一集的叙事压力 | 本集赢了但失去了同伴，下集在余波中开场 |
| `false_security` | 假兑现制造的安全感被后续打破，冲击加倍 | 本集以为危机解除，下集发现更大的危机正在逼近 |
| `interrupted_resume` | 打断兑现的"未完成"驱动续留欲 | 本集对白被打断，下集用另一场戏补完被打断的内容 |
| `reversed_reckoning` | 反向兑现的认知震荡需要下一集来消化 | 本集发现信任的人是叛徒，下集面对信任崩塌的后果 |

约束：

- 每集最多使用 2 种反高潮模式。反高潮过多会让观众产生"什么都得不到"的疲劳感。
- 如果本集的高潮使用了反高潮模式，尾钩（`episode-final-image-contract.md`）应使用满足型或延迟满足，避免整集都是未满足状态。
- 反高潮模式不应连续出现在超过 2 集中。连续 3 集的反高潮会让观众失去信任。
- 跨集续留欲的实现依赖 `residual_tension` 的具体性：必须说明"未释放的是什么张力"，不能只写"留悬念"。

## Boundary

**允许**：

- 为 `peak_visual_plan` 选择 13 种兑现模式中的任何一种，只要选择有上游依据。
- 在反高潮型兑现中说明观众的情绪路径和未释放张力。
- 利用反高潮策略制造跨集续留欲。
- 在反高潮后通过尾钩提供微小满足（避免全集未满足）。

**禁止**：

- 所有高点都使用满足型兑现（连续 3 集无反高潮→节奏单调）。
- 所有高点都使用反高潮型兑现（连续 3 集无满足→观众疲劳）。
- 把反高潮当成"没有高潮"来处理（反高潮必须有清晰的 `satisfaction_delta`，只是方向或方式不同）。
- 没有上游依据就选择反高潮模式（"这个集不适合爽"不是理由，必须回指上游为什么）。
- 让 `false_payoff` 的隐藏危机在本集中被观众识破（假兑现的力量在于观众暂时相信了表面，识破应在后续集）。
- 越过摄影边界，把反高潮的画面信号写成机位或镜头运动方案。

## Evidence Contract

当 `payoff_mode` 为反高潮型时，执行报告中除 `peak_visual_plan` 的标准字段外，`anticlimax_directive` 必须包含：

```yaml
anticlimax_directive:
  mode: ""                    # 必填
  source_anchor: ""           # 必填，为什么这个高点不适合满足型
  expected_payoff: ""         # 必填，观众期待什么
  actual_delivery: ""         # 必填，实际发生了什么
  visual_signal: ""           # 必填，如何让观众感知到"这不是你期待的"
  emotional_trajectory: ""    # 必填，观众情绪路径
  residual_tension: ""        # 必填，留下的未释放张力
  cost_or_insight: ""         # 必填，代价或认知
  continuation_pattern:       # 必填，续留欲实现方式
    type: ""                  # delayed_hook / pyrrhic_aftermath / false_security / interrupted_resume / reversed_reckoning
    next_episode_expectation: ""  # 观众下一集期待什么
  risk_check:
    anticlimax_without_anchor: false  # 是否有上游依据
    too_many_anticlimax: false        # 本集是否超过 2 种反高潮
    consecutive_anticlimax: false     # 是否连续 3 集使用反高潮
    anticlimax_as_no_climax: false    # 是否把反高潮处理成没有高潮
    residual_tension_vague: false     # 未释放张力是否具体
```

## Failure Cases

### Failure 1: 所有高点都是满足型

**症状**：连续多集的高潮画面全部使用满足型兑现模式，观众体验可预测，缺乏叙事张力变化。

**诊断**：`N4-DIR-PEAK` 默认选择满足型模式，没有考虑反高潮策略的可能性。

**修复**：检查近 3 集的 `payoff_mode` 分布。若连续 3 集无反高潮，至少将 1 集的非关键高点改为反高潮模式（推荐 `pyrrhic_payoff` 或 `delayed_gratification`，因为它们保留了部分满足感）。

**验证**：执行报告中近 3 集的 `payoff_mode` 分布包含至少 1 个反高潮型，且反高潮型有上游依据。

### Failure 2: 反高潮时机不对导致观众不满

**症状**：在系列的关键高潮集或观众长期等待的兑现点使用了反高潮模式，观众感到被欺骗或被愚弄，不是叙事层面的不满而是信任层面的不满。

**诊断**：`N4-DIR-PEAK` 在不该使用反高潮的地方使用了反高潮。关键高潮集的兑现点是观众与作品的契约，打破契约需要极强的叙事理由。

**修复**：检查反高潮是否用在了"观众与作品的契约点"上。若是，必须有极强的上游理由（如上游明确写了反转），否则改为满足型。延迟满足可以在契约点使用（因为满足最终会来），但刻意不兑现和反向兑现在契约点使用风险极高。

**验证**：执行报告中 `anticlimax_directive.source_anchor` 对于关键高潮集的反高潮选择，其理由比普通集更强、更具体。

### Failure 3: 反高潮没有留下未释放张力

**症状**：使用了反高潮模式，但 `residual_tension` 为空或只写了"留悬念"，没有具体说明什么张力没有被释放。

**诊断**：`N4-DIR-PEAK` 选择了反高潮模式但没有追踪反高潮的叙事后果。

**修复**：回到 `anticlimax_directive.residual_tension`，具体说明未释放的是什么：是一个未回答的问题？一个未完成的动作？一个未面对的真相？一个未解决的冲突？一个未兑现的承诺？必须具体到"观众在想什么"。

**验证**：执行报告中 `residual_tension` 至少包含 1 个具体的问题、动作或真相，不是抽象的"留悬念"。

### Failure 4: pyrrhic_payoff 缺少兑现或缺少代价

**症状**：惨胜兑现只有"赢了"没有代价，或者只有代价没有"赢了"。前者退化为满足型，后者退化为失败叙事。

**诊断**：`N4-DIR-PEAK` 没有同时追踪 `pyrrhic_payoff` 的两个维度。

**修复**：回到 `pyrrhic_payield` 的 `satisfaction_delta`，确认同时包含"兑现了什么"和"失去了什么"。若只有兑现，补充代价维度；若只有代价，确认角色确实得到了某个具体的结果（否则不是惨胜而是惨败）。

**验证**：执行报告中 `pyrrhic_payoff` 的 `satisfaction_delta` 包含至少 1 个兑现描述和 1 个代价描述。

### Failure 5: 反高潮连续出现超过 2 集

**症状**：连续 3 集或以上使用反高潮模式，观众感到"什么都得不到"，对作品失去信任。

**诊断**：`N4-DIR-PEAK` 在跨集层面没有检查反高潮的频率。

**修复**：检查近 3 集的 `payoff_mode` 分布。若连续 3 集有反高潮，将中间 1 集改为满足型，或确保该集的尾钩提供微小满足。

**验证**：执行报告中近 3 集的反高潮使用不超过连续 2 集。

### Failure 6: false_payoff 的隐藏危机在本集被观众识破

**症状**：假兑现的隐藏信号过于明显，观众在本集就已经识破了表面满足，假兑现退化为"明显有诈"，失去了暂时相信再被打破的冲击力。

**诊断**：`N4-DIR-PEAK` 在 `visual_signal` 中放入了过多不协调信号，或者不协调信号太明显。

**修复**：回到 `anticlimax_directive.visual_signal`，减少不协调信号的数量或强度。假兑现的隐藏信号应该微妙到观众在本集可能会注意到但不会确认，确认在后续集发生。

**验证**：执行报告中 `false_payoff` 的 `visual_signal` 描述的不协调信号不超过 2 个，且强度为"微妙"而非"明显"。

## Reusable Heuristics

- 导演艺术不总是满足观众；最好的高点往往是有代价的。
- 反高潮不是"没有高潮"。反高潮必须有清晰的状态变化，只是方向或方式不是观众预期的。
- 延迟满足是续留欲的最强引擎。"差一点"比"完全没有"更让人上瘾。
- 惨胜比完胜更有戏。角色深度往往来自代价，不来自奖赏。
- 假兑现的力量在于观众暂时相信了。如果观众从一开始就怀疑，假兑现就退化为"角色太蠢"。
- 打断兑现的戏剧价值来自中断源。中断本身必须比兑现更重要，否则只是在戏弄观众。
- 反向兑现的前提是观众已经被引导到某个预期方向。没有预期就没有反转。
- 反高潮是调味料，不是主菜。每 3 集中至少 1 集应该让观众得到满足，维持观众对作品的信任。
- 最高级的反高潮是观众知道反高潮来了，但仍然被打动——因为他们关心的是角色的命运，不只是自己的满足感。
