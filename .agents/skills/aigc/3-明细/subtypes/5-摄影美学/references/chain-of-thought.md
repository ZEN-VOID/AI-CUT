## Think-Think Design Snapshot

### 启发式工作链

- `不可删性`：删掉这轮父级判断后，下游会重新猜“当前该进哪个 leaf、多个 leaf 怎么串行、结果写到哪里、谁负责收口”。
- `最先推动`：默认先推动 `FIELD-CINE-ROUTE-03` 与 `FIELD-CINE-LINE-04`；若主入口与共写写位未锁定，禁止直接美化光影、色彩或参数措辞。
- `先砍什么`：先砍三个 leaf 同时并发落笔、审美形容词堆叠、平行摄影摘要、以及偷改 `焦距 / 光圈 / 景深 / 对焦点` 的越权写法。
- `真正比较尺`：比较的不是哪条描述更“美”，而是谁最能同时保证单一真源、感知层次不打架、下游可继承、边界不漂移。
- `立场纠偏`：当前判断默认站在 `leaf 执行者 / 第N集.json 消费者 / 6-转场特效与视频下游` 的消费立场，而不是站在文案作者想写一段漂亮摄影说明的立场。
- `工具后反思`：读取终稿、输出模板或 leaf 结果后，先问被改变的是 `主入口`、`写位合同`、`落盘真源` 还是 `交接留口`，再决定继续还是回退。
- `落盘门`：每条保留判断都必须落到 `FIELD-CINE-*`、镜级 `摄影美学` 字段、执行报告或 `Gate Summary` 之一；落不到就删。

### 三轴三重裁决

| 裁决角色 | 在 `5-摄影美学` 的具体问题 | 主要落点 |
| --- | --- | --- |
| `方向轴` | 当前最需要先裁决的是哪一个 leaf 入口，以及是否要进入多 leaf 受控串行 | `FIELD-CINE-ROUTE-03` |
| `成立轴` | 当前路由是否被命中分镜节点的静态骨架、终稿正文、组间 handoff、氛围结果和已存在摄影信息共同支撑 | `FIELD-CINE-INPUT-02`、`FIELD-CINE-ROUTE-03` |
| `优选轴` | 在都成立的候选里，哪种路由和写法最能稳住摄影感知层，又不制造第二真相或下游返工 | `FIELD-CINE-LINE-04`、`FIELD-CINE-LAND-05`、`FIELD-CINE-HANDOFF-06` |
| `硬门禁轴` | 是否出现多处 `摄影美学` 平行字段说明、leaf 越权、静态光学字段漂移、或无统一交接出口等强制否决情形 | `FAIL-CINE-ROUTE-03` 到 `FAIL-CINE-HANDOFF-06` |

| 收敛层 | 本层先裁什么 | 服务 `field_id` | 本层收窄了什么 | 正确产物 |

| `离散裁决` | 哪些内容写进同一块 `摄影美学` 字段，哪些只进入执行报告与留口 | `FIELD-CINE-LINE-04`、`FIELD-CINE-LAND-05`、`FIELD-CINE-HANDOFF-06` | 收窄掉平行摄影稿、散落字段、无下一入口的伪完成 | `摄影美学` 字段合同 + 真源落点 + 交接说明 |

### 可见 / 隐藏分层

| 层级 | 保留内容 | 去向 |
| --- | --- | --- |
| `可见快照层` | 父级边界、输入链、主入口裁决、受控串行顺序、`摄影美学` 字段写位合同、统一真源落点、交接说明 | 进入本文件字段表、执行报告与 `第N集.json` |
| `Gate Summary 层` | 当前阻塞、主导失败码、返工入口、confidence、unknowns | 进入执行报告或验收记录 |
| `隐藏推理层` | 被淘汰的 leaf 组合、未采用的形容词方案、过程中的比较废案 | 仅留在内部推理，不进可见合同 |

## 工具后反思与 Gate Summary

### 工具后反思

| 触发节点 | 必问反思 | 继续条件 | 不满足时回退 |
| --- | --- | --- | --- |
| 读取当前集终稿与命中分镜节点后 | 当前缺口到底是光影、色彩还是摄影参数；是否已有 `摄影美学` 字段需要 patch-in-place 而非新增第二处写位 | 能明确回答唯一主入口，或给出 `光影设计 -> 色彩设计 -> 摄影参数` 的受控串行理由 | 回 `S2-S3` 重判输入链与路由 |
| 读取 `.agents/skills/aigc/3-明细/references/output-template.md` 与 `type-strategies.md` 后 | 当前写位和路由是否仍然单一；是否出现多真相、越权改字段或非法并发 | 能维持同一块 `摄影美学` 字段与同一份 `第N集.json` 真源 | 回 `S3-S5` 修路由与落盘合同 |
| 读取 leaf 结果或准备回写前 | leaf 结果是否彼此打架；哪些应该进入 `摄影美学` 字段，哪些只能写成执行报告留口 | 光影、色彩、参数能收敛为同一块字段，且下一入口唯一 | 回 `S4-S6` 重写写位合同与交接 |

### Gate Summary Contract

| 项目 | 必须回答 |
| --- | --- |
| `status` | `green / yellow / red` 三选一，不能省略 |
| `dominant_fail_code` | 当前最主要的失败码或 `none` |
| `rework_entry` | 若失败，必须指回 `S1` 到 `S6` |
| `confidence` | `0-1` 浮点值，说明当前父级路由与共写合同的稳定度 |
| `unknowns` | 尚未被上游真源或 leaf 合同解决的缺口，最多 3 条 |

## Validation Matrix

| 检查项 | 通过标准 | 失败信号 |
| --- | --- | --- |
| 父级边界检查 | 能明确回答父级只做路由、共写治理与交接，不直接吞掉 leaf 细部 | 父级开始直接写光影、色彩或参数明细 |
| 唯一主入口检查 | 当前任务能给出唯一主入口，或给出受控串行的清晰理由 | 三个 leaf 并列开放，没有主从 |
| 单一写位检查 | 所有摄影信息最终收敛到同一块 `摄影美学` 字段 | 同一镜位出现多处摄影说明或旁路摘要 |
| 真源落盘检查 | 所有改动只 patch `projects/<项目名>/编导/第N集.json` 与执行报告 | 新增长期并行摄影稿或侧写正文真相 |
| 边界防漂移检查 | `摄影参数` 不越权改 `焦距 / 光圈 / 景深 / 对焦点` 等静态光学字段 | 父级或 leaf 偷改 `1-分镜表现` 真源 |
| 下游交接检查 | `6-转场特效 / 6-视频` 可直接继承未处理项与下一入口 | 本层完成后仍需下游重新猜缺口 |
| 可见 / 隐藏分层检查 | 可见层只保留父级裁决快照，不混入长篇内部比较过程 | 正文开始罗列大量被放弃的废案 |
| 工具后反思检查 | 关键读取节点后，都发生过“继续 / 回退 / 重判”的二次判断 | 模板或 leaf 结果一回来就机械继续 |
