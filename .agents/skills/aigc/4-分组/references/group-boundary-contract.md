# Group Boundary Contract

本文件定义 `4-分组` 的分镜组边界、ID 和保真规则。

## Group ID Rule

分镜组 ID 固定为 `x-y-z`：

| segment | meaning | source |
| --- | --- | --- |
| `x` | 集号 | `第N集.md` 文件名或 frontmatter `集数` |
| `y` | 场景号 | 上游摄影稿中的真实 `### 场景N` |
| `z` | 场内分镜组序号 | 同一场景内从 1 连续递增，跨场景重新从 1 开始 |

不得为了顺手编号把跨场景内容拼进同组。若上游缺场景标题，应先报告阻塞或修复上游，而不是猜场号。

## Complete Group Word Limit

1980 字限制适用于完整分镜组构成中除组底 YAML 统计以外的全部内容，包括：

- `[分镜组ID]`
- north_star 三项风格纯内容组头
- `入场画面：` 及其内容（第 2 组起；每集首组省略）
- 从 `3-摄影` 划定的分镜剧本正文
- `出场画面：` 及其内容

组底 YAML 统计本身不计入 1980 字限制，也不计入 `字数统计`。

## Dialogue Constraint

| dialogue type | default ceiling | handling |
| --- | --- | --- |
| 长对话、信息密集问答 | 约 4 句对白 | 在不截断 atomic unit 的前提下拆到相邻完整声画单元 |
| 短对白、快速反应、群像短句 | 约 6 句对白 | 可稍微灵活，但不得让组内节奏变成整场对话堆叠 |
| 非对白或少对白动作组 | 无机械对白目标 | 以画面完整性、字数和视频生成节奏为主 |

对白计数以 `对白（...）：`、`独白（...）：`、`旁白（...）：` 等有明确说话内容的声音字段为主；`音效` 不计入对白句数。

## Atomic Unit Rule

以下内容必须作为不可拆单位移动：

- 一个画面字段及其紧随的 `镜头语言：` 块。
- `镜头语言：` 下连续的全部 `分镜N:`。
- 声音字段与紧随的对应画面字段，例如 `对白 -> 对白画面`、`音效 -> 音效画面`，若拆开会导致声画失配。
- `道具特写`、`规则显影`、`系统画面` 等信息揭示字段及其镜头语言。

不得把同一画面句子的多分镜截断到两个分镜组。若字数超限，移动整个 atomic unit 到前一组或后一组。

## Density Review

- 分镜组不是按情绪、话题或危险信息转折切出来的短节拍，而是后续生成可稳定消费的完整声画单元。
- 在不超过 1980 字、不突破对白密度、不截断 atomic unit 的前提下，应优先让分镜组保持足够信息密度。
- 完整组构成低于约 1000 字时必须执行回填复核：优先从前后相邻组移动完整 atomic unit，或与相邻组重组边界。
- 完整组构成低于 850 字通常视为边界过碎；除非上游单场景/单事件本身极短且无法与相邻组同属一个完整声画单元，否则不得保留。

## Boundary Preference

优先边界从高到低：

1. 场景边界。
2. 字数密度、对白密度和 1980 上限的交集。
3. atomic unit 完整边界。
4. 明确转场、入场、出场、门开、灯灭等可供桥接的空间/运动切口。
5. 字数上限附近的完整 atomic unit 边界。

不得仅因为情绪变化、话题变化、危险信息显影或所谓节奏转折而切出低密度短组。

## Preservation Rule

分镜剧本正文必须同步原换行。允许添加组头、入场/出场画面、组底 YAML 和 frontmatter；不得在正文中删改字段名、对白、镜头语言、场景标题或原有顺序。
