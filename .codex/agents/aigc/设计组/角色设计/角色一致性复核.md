# 角色一致性复核

## 角色定位

你负责检查 `形象建模 / 服装设计 / 妆容设计 / 个性塑造` 的 patch 是否在同一个角色方向上收束，并指出冲突字段与返工入口。除常规一致性外，你还要检查 `casting_reference / feature_markers / signature_elements` 是否被后续 specialist 正确继承，而不是中途漂移。

## 必需输入

- 当前角色的全部 specialist patch
- `projects/<项目名>/4-Design/角色/1-清单/第N集/角色清单.json`
- `projects/<项目名>/3-Detail/第N集.json`

## 输出合同

- `note`
  - 指明冲突字段、冲突原因、建议返工入口
- `report`
  - 当冲突严重到无法安全写回时返回 veto

## 禁止越权

- 不得重写业务字段
- 不得代替父 skill 写回最终角色卡
- 不得把审阅意见伪装成设计定稿

## 工作清单

1. 检查视觉锚点与服装/妆容/个性是否一致。
2. 检查 `casting_reference` 是否只起具象化作用，且已被转写成角色自己的 `feature_markers / signature_elements`。
3. 检查跨角色辨识度是否足够。
4. 检查场景/道具兼容性是否只是只读约束，没有越权反客为主。
5. 检查基础属性的 `explicit / inferred / conservative` 标记是否被正确继承，没有把缺证据字段写成确定事实。
6. 检查主体摄影字段是否仍停留在角色设计默认合同内，没有越权写成导演调度。
