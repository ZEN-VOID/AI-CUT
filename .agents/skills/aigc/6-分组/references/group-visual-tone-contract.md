# Group Visual Tone Contract

本文件定义每个分镜组的组级画面属性输出标准。画面属性从上游 5-摄影 的镜头设计中提炼，以自然语句写入每个分镜组。

## Core Rule

每个分镜组在组装时，必须从组内镜头设计中提炼组级画面属性，写成自然语句。画面属性覆盖该组的构图布局核心选择、构图方式关键子维度、光源效果、色彩基调和关键摄影技术参数。

画面属性不是 north_star.yaml 的风格投影——两者是不同层面：
- **north_star 风格行**（项目级）：全局风格提示词、类型元素提示词、画面风格——由 `north-star-projection-contract.md` 管理
- **画面属性**（组级）：从组内镜头设计提炼的视觉特征——由本合同管理

## Fields

### 构图布局 composition_layout

从组内镜头设计中提炼：主体位置与占比倾向、陪体/前景/背景的主要空间安排。

### 构图方式 composition_method

从 `scene-visual-constraint-contract.md` 的七个子维度中，选取当前组最关键的 2-3 个：
- 形状感：画面主导形状及其心理暗示
- 线条感：画面主导线条方向及视线引导
- 影调感：明暗对比倾向
- 虚实感：景深焦点倾向
- 节奏感：元素重复/渐变/突变倾向
- 纹理质感：画面质地效果
- 气势：画面整体情绪感染力

### 光源效果 lighting

主光源方向、质感、强度的可感知结果；辅助光和逆光如有重要作用也纳入。

### 色彩基调 color

色相主导、色温倾向、饱和度策略的可感知结果。

### 关键摄影技术参数 camera_params

影响画面质感的关键选择（焦距、光圈等），只写对该组画面影响最大的 1-2 项。

## Output Rules

1. 画面属性以自然中文语句呈现，覆盖上述五个分区中对该组画面最关键的维度。不要求五个分区全部覆盖，选取最重要的即可。
2. 不输出参数清单、标签列表或模板填空。
3. 画面属性出现在每个分镜组的 `定场镜头：` 之后、north_star 风格行之前。
4. 同一场景内镜头设计视觉特征不变时，相邻分镜组可使用相同或相似的画面属性语句；视觉特征发生变化时（如场景内从平静到恐慌），画面属性必须更新。
5. 画面属性从组内镜头设计中**提炼**，不是从 5-摄影 的内部 `scene_visual_constraint` 直接搬运。提炼时应综合组内各分镜的构图、光源、色彩和摄影参数，形成统一的视觉描述。

## Knowledge Base Binding

画面属性的裁决和落盘以 `knowledge-base/摄影构图/`（路径：`.agents/skills/aigc/5-摄影/knowledge-base/摄影构图/`）作为知识库参考。裁决时应参考其中的构图技法和影像创造理论，确保画面属性语句在构图布局、构图方式、光源、色彩等维度上有摄影构图理论支撑，而不是凭直觉生成。

## Output Example

```markdown
## 1-1-1

场景1：内景 教室 - 日

定场镜头：教室前排课桌区内，林寂坐在第二排靠窗座位，身体朝向讲台但脸偏向右侧的苏红叶；苏红叶站在林寂课桌右前方半步处，面向林寂。

画面属性：课桌行列形成水平线重复节奏，对称构图把空间压成矩形牢笼；冷白日光灯从正上方打硬光，高反差让白色校服过曝、黑窗帘完全吞光；50mm 标准焦段保持观察距离。

视频生成的画面风格... <全局风格>
<类型元素>
<细分风格>

环境描写：冷白日光灯压在课桌表面...
```
