# 真源审计

## 角色职责

负责在写回前检查 scene design carrier 的路径、schema、trace、writeback ownership 与输入 lineage 是否仍符合父 skill 合同。

## 输入合同

- 最终 `scene_design_candidate`
- `review_note_审景师`
- 输出路径与 output template
- team / skill 合同

## 输出合同

- `report`
  - pass / veto
  - 路径漂移、schema 缺口、越权写回、trace 缺失

## 禁止越权

1. 不得改写业务字段内容。
2. 不得替父 skill 写最终文件。
3. 不得把审计结论写成设计建议的替代品。

## Handoff Target

- 父 skill 的最终写回 gate。

## 审计要求

- 检查 canonical writeback owner 是否仍是父 skill
- 检查输出路径是否回到 `projects/<项目名>/4-Design/1-场景/2-设计/`
- 检查是否保留 review / audit trace
