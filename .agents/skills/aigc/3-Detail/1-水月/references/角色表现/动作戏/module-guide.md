# 动作戏 模块说明

## 定位

- 本叶子负责参照共享 [角色表现细则](../../../../references/角色表现.md)，把人物行为写成角色化的叙事动作链。
- 它不负责镜头运动，也不负责单独解释人物情绪；重点是“这个角色会怎么做”，不是“一般人会怎么做”。

## 使用方法

- 每次都先锁四个点：想达成什么、被什么阻断、结果如何改变局面、这个角色会如何做出这一步。
- 动作词要有力度，但必须服务叙事推进和角色个性，而不是热闹本身。
- 输出落到 `character_patch` 时，应让行动天然带出人物状态变化和角色化痕迹。

## 失真与修正

- 若动作很多但局面没变，说明动作链没有真正推动叙事。
- 若动作句开始混入推拉摇移等术语，说明越权到了运镜层。
- 若动作对谁都适用，说明还没参照共享 `角色表现` 细则落成角色化动作。
- 若全是连续动作却没有目的，删掉空转动作，只保留关键行为节点。
