# Scene List Heuristics

## Stable Heuristics

- 清单阶段宁可保守暴露待核风险，也不要把不同空间合并到一个名称下。
- `原文描述（关键词式）` 的价值是让后续设计者回忆来源，不是让读者看到完整场景氛围。
- 如果 YAML `场景` 字段使用泛称，优先用同组标题和正文关键词做命名消歧。
- 如果 YAML `场景` 字段使用多个地点，先判断是否是路线、镜头切换还是一个复合空间。
- 归并后的 canonical 名称应服务后续设计检索，尽量清楚、短、可复用。

## Common Pitfalls

- 把“教室内”和“教室门口”合并，导致门口视角资产缺失。
- 把“夜晚教室”和“白天教室”拆成两行，却没有资产差异理由。
- 把正文中提到的回忆地点加入清单，但该组 YAML 并未把它列为场景。
- 为了让表格好看而改写关键词，制造了上游没有的美术设定。
