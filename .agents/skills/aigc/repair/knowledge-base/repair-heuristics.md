# AIGC Repair Heuristics

## Source-First Repair

- 图像/视频层的失败经常是更早阶段的表达不够可执行；先回看 `4-摄影`、`10-分组` 和 `11-主体`，再决定是否重跑生成。
- `14-审片` finding 是证据入口，不是直接写回权。
- 多子技能包整体调整优先寻找共同 source owner，避免逐文件补丁互相冲突。

## Chinese Polish

- 好的中文润色应让目标更清楚、更自然、更可执行，而不是更文学化。
- 字段化阶段的中文优先保持字段纯度；不要把解释写进字段。
- prompt 或分镜组文字应减少抽象形容，增加可见动作、空间关系、光影锚点和镜头连续性。
- 对用户-facing repair 报告，豆包的目标不是更华丽，而是去掉翻译腔、工程腔和生硬模板句，让中文读起来像真实制作沟通。

## Creative Inspiration

- 创意候选应当是“可被某阶段吸收的修复方向”，不是独立点子库。
- 对每个候选标注可应用层：导演、表演、摄影、分组、设计、图像、视频或审片。
- 候选需要显式说明不改变哪些既有事实和禁区。
- 本土文化气息优先体现在人物关系、生活细节、情绪表达、类型片语感和短剧可消费性，不应变成堆砌流行词。

## Provider Evidence

- 豆包输出要保留 sidecar 和 report；没有 evidence 的 provider 口径会破坏审计链。
- provider 失败不等于任务失败；可以降级为 repair plan，但不能宣称已经执行。
