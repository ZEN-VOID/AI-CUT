# Inter-Group Head-Tail Connector Contract

本文件定义 `5-分组` 的组间首尾帧连接件。当前规则恢复组内 `出场画面：` 尾钩；连接件不替代尾钩，而是消费上一组 `出场画面：` 与下一组 `入场镜头：` 两端证据，额外生成 3-4 秒缝纫补丁。

## Purpose

当前 AIGC 视频工具存在单次生成最长约 15 秒的物理硬限制。单个分镜组在 15 秒内可以表现良好，但相邻 15 秒组之间容易出现结尾状态和下一组开头状态对不上，形成断裂感和孤岛感。

组间首尾帧连接件用于在既有分镜组划分完成后，基于第 N 组 `出场画面：` 尾钩、第 N 组原尾帧、第 N+1 组 `入场镜头：` 首态和第 N+1 组原首帧，额外设计 3-4 秒的“缝纫补丁”。连接件本身不占据剧本正文内容，不改写上游剧情，不进入分镜组 `时长估算` 或 1680 字正文目标；它服务首尾帧参照图生视频，使画面从 A 状态连续抵达 B 状态。

## Mandatory Shape

每一对相邻分镜组之间必须生成一个连接件；连接件是与上下分镜组同级的独立板块，必须物理落盘在上一个分镜组与下一个分镜组中间；每集首组之前和最后一组之后不生成连接件。

```markdown
## <上一个分镜组ID>~<下一个分镜组ID>

<同场景连接时：场景N：场景标题；跨场景连接时：场景标题A ➡️ 场景标题B>

视频生成的画面风格，光影和氛围与场景参照图保持一致。需要生成现场物理互动音效、氛围感音效、环境声、自然现象声、动作声，不要生成任何字幕，不要生成背景音乐。 <根据连接件首尾场景与运动过程整理出的当前连接件全局风格句，300字以内>
<直引 north_star.yaml 类型元素.类型元素提示词>
<直引 north_star.yaml 细分风格.画面风格>

连接类型：<同场景连接 | 跨场景连接>
连接方法：<一句简要具体方法描述；不要只写“依赖型/流动型/变形型/复合型/无连接”>
时长：3-4秒
变化过程：<只描述从实际尾帧参照图到实际首帧参照图之间的连续变化过程，不复述端点画面>
主体运动：<主体、锚定物、能量或环境元素的运动路径、速度、方向、遮挡和前中后景关系>
运镜设计：<镜头运动、景别变化、机位高度、焦段感、轴线方向、焦点转移和节奏>
透视适应：<景别、缩放、位置、角度如何随首尾帧自适应>
避免元素：<不允许出现的新剧情、新人物、新对白、字幕、背景音乐、端点复述、画面漂移和错误物件>
```

| field | requirement |
| --- | --- |
| `连接件标题 ID` | `## <上一个分镜组ID>~<下一个分镜组ID>` 是连接件唯一分镜 ID；块内不得再重复输出 `分镜ID：` 字段 |
| `场景标题行` | 位于连接件标题之后、三项风格行之前；同场景连接重复当前场景标题，例如 `场景1：外景 扶桑战船外舷与黑礁 - 夜`；跨场景连接必须写成 `场景标题A ➡️ 场景标题B`，例如 `场景1：外景 扶桑战船外舷与黑礁 - 夜 ➡️ 场景2：内景 船舱底层 - 夜` |
| `三项风格行` | 位于场景标题行之后、`连接类型` 之前；以隐藏标题字段的三行纯内容写入风格整理结果。第 1 行必须以固定前置词 `视频生成的画面风格，光影和氛围与场景参照图保持一致。需要生成现场物理互动音效、氛围感音效、环境声、自然现象声、动作声，不要生成任何字幕，不要生成背景音乐。` 开头，后接根据连接件首尾场景、场景关系、运动过程和 `north_star.yaml` 全局风格母稿整理出的当前连接件风格句，300 字以内；第 2、3 行分别直引 `类型元素.类型元素提示词` 与 `细分风格.画面风格` |
| `连接类型` | 场景关系维度，只能是 `同场景连接` 或 `跨场景连接` |
| `连接方法` | 画面缝合方法维度，必须写成简要具体方法描述，说明用什么可见锚定物、能量痕迹、主体变形、背景保持或硬切理由完成 A 到 B；不得只填 `依赖型`、`流动型`、`变形型`、`复合型` 或 `无连接` |
| `时长` | 默认 `3-4秒`；用户明确指定时可局部改写 |
| `变化过程` | 只写中间变化过程，不复述 `起点尾帧` 或 `目标首帧` 的画面状态，避免与实际首尾参照图偏移 |
| `主体运动` | 写清主体、锚定物、能量、光影或环境元素的起势、路径、方向、速度、遮挡、前中后景层次和终止趋势 |
| `运镜设计` | 写清推拉摇移跟环绕、景别变化、机位高度、焦段感、镜头轴线、screen direction、焦点转移和节奏，不只写“镜头推进” |
| `透视适应` | 说明锚定物、能量、主体或背景在首尾帧中因景别、位置、角度变化产生的缩放和透视调整 |
| `避免元素` | 负面约束字段，写清不允许出现的新剧情、新人物、新对白、字幕、背景音乐、端点复述、画面漂移、错误物件或风格偏移；不再重复整合前面的正向过程描述 |
| `truth boundary` | 不新增关键剧情事实、不新增未出场角色、不新增关键道具、不改写剧本因果 |
| `count boundary` | 不计入分镜组 `时长估算`、1680 字常规目标、1980 字硬上限或 YAML `字数统计` |
| `section boundary` | 连接件不属于上一个分镜组，也不属于下一个分镜组；它必须是位于二者之间的独立 `##` 板块 |

## Execution Order

1. 先完成第 N 个分镜组正文划分。
2. 再自然完成第 N+1 个分镜组正文划分。
3. 回看第 N 组 `出场画面：`、第 N 组最后原尾帧、第 N+1 组 `入场镜头：` 与第 N+1 组第一个原首帧，形成 `connector_pair_profile`。
4. 判断 `连接类型` 是同场景连接还是跨场景连接。
5. 按 `连接方法决策树` 在内部选择依赖型、流动型、变形型、复合型或特殊无连接，但落盘到 `连接方法` 字段时必须改写为一句具体可执行的画面办法；若确需场景硬切，必须写明硬切理由和首尾帧断点。
6. 只提炼首尾参照图之间的变化过程，不在字段中复述第 N 组 `出场画面：`、尾帧或第 N+1 组 `入场镜头：`、首帧的静态画面状态。
7. 按接近 `7-图像/A-分镜画面` 的细节密度补足主体运动和运镜设计：主体路径、遮挡、前中后景、镜头轴线、景别变化、机位高度、焦点转移和节奏。
8. 按 `透视自适应原则` 检查锚定物、能量、主体或背景在首尾帧中的景别、缩放、位置和角度变化。
9. 写入场景标题行、三项风格行、过程字段与避免元素，并按 `## 上一组`、`## 上一组~下一组`、`## 下一组` 的顺序强制落盘为独立板块。

不得在只看到第 N 组时预写下一组连接件；连接件的核心依据必须包含第 N+1 组开始画面。

不得输出 `起点尾帧：` 或 `目标首帧：` 字段。实际端点应由首尾帧参照图承担，文本只描述两张参照图之间的连续变化、主体运动和运镜。不得输出 `连接件提示：` 字段；正向提示已经由变化过程、主体运动、运镜设计和透视适应承担，末尾只保留 `避免元素：`。

## Core Methodology

组间连接件不是普通转场注释，而是分镜故事板之间的画面连续性设计。以下四类是内部思考方法论，帮助选择画面缝合逻辑；它们不是给 AIGC 生成工具直接消费的字段值。落盘时，`连接方法` 必须改写成一句具体方法描述。

示例：

- 不写：`连接方法：依赖型`
- 应写：`连接方法：用红纸飞机作为跨场景锚定物，从尾帧右下角飞过镜头，进入首帧中央后背景切换为操场。`
- 不写：`连接方法：流动型`
- 应写：`连接方法：让日光灯的冷白闪烁沿走廊方向延伸成首帧门缝里的同向白光。`

### Anchor-Based Connection

中文名：依赖型。

通过一致的形状、颜色、物体等具象实物的不变锚定物，在背景场景改变的情况下，锁定锚定物于尾出帧与首入帧中的对应性展示。

适用场景：

- 场景切换。
- 时空跳跃。
- 蒙太奇过渡。

典型锚定物：

- 生物类：一只猫、一只鸟、一条鱼。
- 物件类：一个纸飞机、一个气球、一封信、一杯酒。
- 武器或动态物：射出去的子弹、箭矢、飞刃。
- 建筑或宏观物：一个建筑物，用于沧海桑田、斗转星移。
- 符号类：月亮、太阳、时钟、眼睛。

执行要点：

- 锚定物在尾出帧与首入帧中必须清晰可识别。
- 锚定物周围的环境可以完全改变。
- 必须考虑锚定物的运动轨迹连续性。
- 锚定物可按透视自适应原则缩放、偏移或旋转，不要求等比例复制。

### Energy Flow Connection

中文名：流动型。

不锁定具象实物，而是关注一种能量显现或抽象感觉在尾出帧与首入帧之间的流动与传递。

适用场景：

- 动作延续。
- 情感传递。
- 氛围延展。

能量矢量逻辑：

- 力学类：力、冲击波、爆炸余波。
- 光热类：光线、热浪、火焰。
- 电磁类：电流、闪电、魔法能量。
- 武学类：剑气、内力、气场。
- 自然类：风、烟雾、水流、尘埃。

感官逻辑：

- 视觉延伸：目光所及、视线追踪。
- 听觉延伸：侧耳倾听、声波传递。
- 嗅觉延伸：千里飘香、气味追踪。
- 触觉延伸：压迫感、温度变化。

执行要点：

- 能量的方向矢量必须在尾出帧到首入帧之间保持一致。
- 能量强度可以渐变，包括衰减或增强。
- 能量痕迹必须有可见或可感知的画面表现。
- 流动方向应服务下一组首帧，而不是只做抽象气氛堆叠。

### Transformation Connection

中文名：变形型。

此类型不要求首尾帧参照物一致，反而允许主体不一致；但周围环境应尽量保持一致，以形成衬托对比。

适用场景：

- 变身。
- 变装。
- 形态转换。
- 时间流逝。

典型变形：

- 人物变形：美少女变装、德鲁伊变身、法天象地、狼人变身。
- 物体变形：蝴蝶破茧、花开花落、冰化成水。
- 状态变形：青年到老年、完整到破碎、光明到黑暗。
- 表情变形：微笑到痛苦、平静到暴怒。

执行要点：

- 环境或背景必须保持高度一致，作为变形的锚定背景。
- 变形前后的主体应保持相同或可追踪的画面位置。
- 变形应给出中间态暗示，避免直接跳成无因果硬切。
- 若背景无法稳定，应改用依赖型、流动型或复合型。

### Composite Connection

中文名：复合型。

将依赖型、流动型、变形型中的多种方法组合混搭，用于复杂叙事过渡。

适用场景：

- 多层次叙事。
- 平行剪辑。
- 复杂蒙太奇。

组合示例：

- 依赖型 + 流动型：子弹飞行作为锚定物，同时保留子弹轨迹的能量线。
- 流动型 + 变形型：魔法能量流动导致角色变身。
- 依赖型 + 变形型：同一物体作为锚定物，在不同时代中发生变化。

执行要点：

- 必须声明主方法和辅方法，避免把所有连接逻辑平均堆叠。
- 主方法负责首尾帧抵达；辅方法只增强节奏、氛围或信息层。
- 复合型仍必须保持 3-4 秒信息量，不写成完整分镜组。

## Connection Method Decision Tree

决策流程：

1. 场景是否切换？
   - 是：优先考虑 `依赖型`，寻找跨场景锚定物。
   - 否：继续判断。
2. 角色是否变换或变形？
   - 是：优先考虑 `变形型`，确认环境是否稳定。
   - 否：继续判断。
3. 是否存在能量或动作延续？
   - 是：优先考虑 `流动型`，确定能量矢量方向。
   - 否：继续判断。
4. 多条件同时满足？
   - 是：考虑 `复合型`，组合主次连接逻辑。
   - 否：选择最契合叙事需求的类型。

特殊情况：

- 首个分镜组：前序连接为无，不输出连接件。
- 末尾分镜组：后序连接为无，不输出连接件。
- 场景硬切：可选择 `无连接`，但必须在 `连接方法` 中说明为什么强连接会破坏节奏、逻辑或首尾帧真实性。

字段落盘规则：

- `连接方法` 不输出抽象分类名本身。
- 可以在内部使用依赖型、流动型、变形型、复合型判断，但最终必须写成具体可生成的画面办法。
- 若确需无连接，`连接方法` 应写成 `无连接，理由是...`，并说明为什么硬切比强行缝合更符合剧情、节奏或首尾帧真实性。

## Perspective Adaptation Principle

核心规则：不管哪一种连接方法的锚定，在首尾帧中都不必等比例复制；必须遵循当前分镜帧设计中的景别、角度和运动透视关系，按近大远小进行自适应同步调整。

| 要素 | 说明 |
| --- | --- |
| 景别变化 | 远景到特写，或特写到远景 |
| 锚定物缩放 | 根据景别变化调整锚定物在画面中的占比 |
| 位置偏移 | 锚定物或能量痕迹在画面中的位置变化，例如中心到边缘 |
| 角度旋转 | 视角变化带来的锚定物朝向调整 |

示例：

- 前帧远景中锚定物占画面约 5%，后帧特写中同一锚定物可占画面约 40%。
- 前帧锚定物位于画面右下，后帧可位于画面中央，只要运动、构图和视角变化可被解释。
- 流动型连接中，能量痕迹可因镜头推进从细线变成铺满画面的光带，但方向矢量必须一致。

## Same-Scene Connector

同场景连接用于两个相邻组仍处于同一场景、同一空间或同一连续事件内。

设计重点：

- 充分理解实际首尾参照图的主体状态、位置、动作相位、视线、光线、声音和构图关系，但不把端点状态复述成输出字段。
- 优先使用自然连续动作：视线未落定、手指继续移动、灯光轻颤、角色呼吸、群像停顿、焦点转移、镜头继续推拉摇移。
- 连接件可以短暂补足空间关系，但不得加入新事件或新信息。
- 最终画面必须能自然抵达下一组原首帧的构图、主体和动作状态。

## Cross-Scene Connector

跨场景连接用于相邻组跨越场景、空间、时间段或明显叙事段落。

设计重点：

- 这部分承担转场连接，转场主创归属在 `5-分组`，不再由 `4-摄影` 分镜明细穿插落盘。
- 先保护剧情逻辑和观看可读性，再选择专业标准转场或创意化转场。
- 可使用声画桥、动作匹配、形态匹配、光色桥、文字/符号显影、空间出口、遮挡切换、反应切、过曝/熄灯/光变等方式。
- 创意转场必须有清楚接口：A 画面的主体、动作、声音、光线、颜色、文字、道具轮廓或空间出口，能转化为 B 画面的进入点。
- 若接口不足，采用克制的建立入口或声画桥，不硬造高能转场。

## Connection Criteria

连接件必须同时满足：

- 能从第 N 组最后原尾帧状态出发。
- 能抵达第 N+1 组第一个原首帧状态。
- 时长和信息量匹配 3-4 秒，不写成完整分镜组。
- 画面可被首尾帧参照图生视频消费。
- 不占剧本正文，不承担新剧情推进。
- 同场景连接重在状态缝合；跨场景连接允许更高创意，但必须有可执行接口。
- 连接件开头必须显式标明场景关系：同场景重复该场景标题，跨场景使用 `场景标题A ➡️ 场景标题B`。

## Good Connector Sources

- 灯光：灯管轻颤、光斑滑动、过曝残影、阴影压下、熄灯后再亮起。
- 环境结构：门缝、窗帘、走廊线、课桌边缘、黑板中轴、楼梯转角。
- 道具：红珠反光、课本扉页、粉笔字、手机屏幕、教鞭末端；只在它出现在实际尾帧/首帧、承担关键剧情/规则/证据/危险源，或被人物明确互动时使用。
- 身体：未落定的视线、停住的手指、群像同步转头后的静止、肩膀轻微起伏。
- 声音：脚步声余波、广播底噪、日光灯嗡鸣、翻页声的尾音。
- 形态：白光到日光灯、红点到红色道具、黑板字到系统文字、门缝到走廊线。

## Anti-Patterns

- 在连接件块内输出旧版 `入场画面：`，或重复输出组内字段 `出场画面：`。
- 把连接件写进分镜组正文，导致组内 `时长估算` 或 1680 字目标被尾钩污染。
- 使用 `## 组间连接件：上一组~下一组` 这类带前缀标题，导致连接件不是以自身分镜 ID 作为独立板块标题。
- 把连接件集中放到文件尾部、文件头部、单独连接件区，或放在不相邻的两个分镜组之外。
- 使用 `上一组 -> 下一组`、`上一组-下一组` 或其他形态替代 `上一组~下一组` 作为连接件分镜 ID。
- 只看第 N 组结尾就预写连接，不回看第 N+1 组开始画面。
- 写成对白、旁白解释、心理独白或新剧情段落。
- 为了连接而加入上游没有的新人物、新关键道具、新规则或新因果。
- 为了连接而把普通无互动道具升级成锚定物，例如杯水涟漪、餐具轻响、纸角阴影、桌面反光；若它不是实际首尾帧可见接口、关键道具或人物互动对象，应改用人物动作、声音、光线或空间结构连接。
- 只写抽象气氛词，例如“气氛更加紧张”，没有 A 到 B 的可见变化。
- 缺少三项风格行，或没有把固定视频生成前置词放在连接件场景标题行后的第一条风格行最前。
- 缺少场景标题行，或跨场景连接没有使用 `场景标题A ➡️ 场景标题B`。
- 用 `连接件提示：` 再次整合前面字段，造成冗余；应改为 `避免元素：`。
- `连接方法` 只填 `依赖型`、`流动型`、`变形型`、`复合型` 或 `无连接`，没有转换成具体画面办法。
- 跨场景连接炫技但无法反推出实际首尾参照图之间的可执行变化路径。
