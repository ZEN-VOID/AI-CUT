# 负向经验

本文件只保存可复用失败模式、根因和修复打法，不承载 workflow 规范真源。

## NE-WF-01 Sidecar 字幕不等于视频里有字幕

- 症状：最终视频旁边有匹配 `.srt`，但 MP4 画面里没有出现字幕，用户验收时认为“没有字幕”。
- 根因：完成门只检查了语义校正 sidecar，没有把可见字幕渲染到最终 MP4 作为默认交付。
- 立即修复：用最终校正版 SRT 重新渲染长片和每条切片，把字幕以高对比 narration-caption 样式烧录/叠加进画面，并保留 sidecar。
- 系统预防：已晋升到 `SKILL.md` 的 `Visible Subtitle Rendering Contract`、review gate、模板和测试 prompt；后续默认完整渲染必须记录 `subtitle_render_mode` 和 sampled-frame caption QA。

## NE-WF-02 教学切片不能按平均时长切

- 症状：切片虽然存在，但开头/结尾没有“有始有终”，像按时长切出的片段而不是可独立学习的教学段落。
- 根因：切片完成门强调数量和覆盖，但对首尾内容边界、自然收束和平均切割反模式约束不够硬。
- 立即修复：回到 N3，以主题转换、操作闭环、因果解释和字幕语义重选每条切片的 start/end，必要时保留上下文。
- 系统预防：已晋升到 `Dual Output Contract`、N3/N6 gate、review contract、模板和测试 prompt；后续 slice plan 必须记录 content-boundary start/end rationale。

## NE-WF-03 字幕可见但样式不符/不在底部/自动换行仍然不合格

- 症状：最终 MP4 已有烧录字幕，但字号不符合当前源层默认或用户精确指定值、位置偏高，或同一段字幕自动换成两行，影响软件界面教学观看。
- 根因：可见字幕完成门只验证“出现”和“对比度”，没有硬性检查 narration caption 的底部位置、默认/覆盖字号和单行 cue 策略。
- 立即修复：重新生成 visible-render 输入，把长句按语义短语拆成连续 cue；默认用 `SKILL.md` 当前规定的 100 px@1080p，并按输出高度 `round(100 * output_height / 1080)` 等比计算实际字号，若用户明确指定其他字号则按精确覆盖值，底部对齐、禁用自动换行后重新烧录长片和切片。
- 系统预防：已晋升到 `SKILL.md` 的 `Subtitle Style Contract` / `Visible Subtitle Rendering Contract`、review gate、模板和测试 prompt；后续 sampled-frame QA 必须检查字号、底部安全区、单行策略和遮挡风险。

## NE-WF-04 有 SRT 时间戳不等于字幕显示已校订

- 症状：组合切片或短切片已有同名 `.srt` 和时间戳，但单条字幕显示时句子被切断、术语或动作短语不完整，用户追问是否又要重做短切片。
- 根因：流程把“最终时间轴字幕存在”误当成“画面显示 cue 可用”，缺少渲染前的显示句段校订和重渲染影响范围判定。
- 立即修复：先回到 N4 校订 SRT/ASS 的显示 cue，记录拆分、合并、时间微调和风险项，再判断 `rerender_scope`；只重渲染受影响或无法证明采用校订字幕的视频，除非系统性样式/时间轴问题影响整批。
- 系统预防：已晋升到 `SKILL.md` 的 `Subtitle Display Proofing Contract`、review gate、模板、README、type context 和测试 prompt；后续完整渲染和字幕修复必须先有 display-proofing evidence。

## NE-WF-05 线性课程不等于可以自动省略重组部分

- 症状：完整 workflow 只交付长片和顺序切片，没有 `final/slices/combinations/` 下的重组视频，用户追问“重组部分呢”。
- 根因：执行时把“单轮线性课程有前置顺序”扩大解释成组合切片豁免理由，跳过了保序重组方案。
- 立即修复：回到 N3/N4，用固定开头和收束、按学习路径选择中间模块的方式补产 order-preserving recombination，并更新组合计划、manifest、QA 和最终报告。
- 系统预防：已晋升到 `SKILL.md` 的 `Combination Slice Utilization Contract` 和 review/test prompt；后续除非用户明确豁免或证据显示重组阻塞，线性课程也必须先尝试保序重组。
