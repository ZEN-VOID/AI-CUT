# 2-Global 思行网络

本文件细化 `.agents/skills/aigc/2-Global/SKILL.md` 的思行节点与执行门。
若与主 `SKILL.md` 冲突，以主合同为准。

## Purpose

- 把 `2-Global` 的思考流与执行流收编为一张可落盘的节点网络。
- 让 `episode-bootstrap / incremental-patch / compatibility-projection` 三种模式共用同一主骨架，而不是各写一套平行步骤。

## Mode x Node Matrix

| mode | 必跑节点 | 可缩窄节点 | 禁止动作 |
| --- | --- | --- | --- |
| `episode-bootstrap` | `N1 -> N2 -> N3 -> N4 -> N5 -> N6 -> N7` | 无 | 跳过项目级 synthesis 直接写组级字段 |
| `incremental-patch` | `N1 -> N2 -> N5 -> N6 -> N7` | `N3 / N4` 仅对命中字段重跑 | 无 scope 地全量覆盖全部 groups |
| `compatibility-projection` | `N1 -> N2 -> N6 -> N7` | 只在 JSON 已确认时执行 | 在 canonical JSON 未确认前直接产出 Markdown |

## Detailed Node Table

| node_id | inputs | actions | write_target | fail_if |
| --- | --- | --- | --- | --- |
| `N1-MODE-LOCK` | 用户请求、现存 `episode_root.json`、旧下游需求 | 判断模式、命中集、命中组、命中字段与 compat 需求 | `mode_decision_note` | 同一轮出现两个主模式 |
| `N2-INPUT-LOCK` | 分组正文、`north_star`、`init_handoff`、`team.yaml`、旧 JSON | 锁定上游真源、字段边界、advisory 可用性与禁止项 | `input_lock_note`、`invariant_brief` | 漏掉必需输入，或容忍 shot-level 漂移 |
| `N3-PROJECT-GLOBAL` | `north_star`、`init_handoff`、必要 advisory | 提炼 `project_global.全局风格 / 全集类型元素` | `project_global_patch` | 把组级情境写成项目总则 |
| `N4-GROUP-GLOBAL` | 分组正文、`project_global`、必要 advisory | 逐组整理 `剧本正文 / 全局风格 / 类型元素 / 导演意图` | `group_global_patch_set` | 跨组混写，或把正文写成摘要 |
| `N5-WRITEBACK` | 模板、旧 JSON、`project_global_patch`、`group_global_patch_set` | 对 `episode_root.json` 做 patch-in-place | `episode_seed_patch`、`writeback_patch_set` | 发明新壳，或覆盖未命中 scope |
| `N6-CONVERGENCE` | 已写 JSON、compat 需求 | 做结构审计、derived-only 审计与 handoff 审计 | `convergence_report` | compat 先于 canonical，或 handoff 不可消费 |
| `N7-CLOSURE` | 审计结果、采纳结果、阻塞原因 | 写验收、思考过程、closure triad 与 `3-Detail` handoff note | `validation-report.md`、`handoff_note` | 没有验收或没有根因上溯 |

## Advisory Consumption Gate

`team.yaml` 仅在以下条件全部成立时才被消费：

1. `team.yaml.enabled == true`
2. 当前阶段命中 `roles.supervision`
3. advisory 能为 `全局风格 / 全集类型元素 / 类型元素 / 导演意图` 提供前置判断帮助

硬门：

- advisory 只能作为 `N2-N4` 的前置输入
- advisory 不得在 `N5` 之后接管 refine owner
- 未采纳的 advisory 也要在 `思考过程` 或 `validation-report.md` 中说明原因

## Convergence Gates

### Gate-A: JSON Structural Integrity

- `meta`
- `project_global`
- `groups[]`

### Gate-B: Group Alignment

- 每个 `groups[].分镜组ID` 与分组正文一一对应
- `global.剧本正文` 保留完整正文
- `global.类型元素 / 导演意图` 不跨组污染

### Gate-C: Derived-Only Compatibility

- compat 文件只有在 JSON 已确认后才能生成
- compat 文件只能复述已确认 JSON，不新增业务事实

### Gate-D: 3-Detail Handoff Readiness

- 下游能直接读到 `project_global.*`
- 下游能直接读到 `groups[].global.*`
- 当前阶段没有偷写 shot-level 字段
