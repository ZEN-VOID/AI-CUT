# 漫画剧本写作规格

本文件定义“适合继续做漫画生成，并可直接指导 `2-九刀流漫画提示词`”的漫画剧本规格。

相关细则回链：

- [visual-spectacle-engine.md](visual-spectacle-engine.md)
- [pacing-hook-thrill-engine.md](pacing-hook-thrill-engine.md)
- [chunibyo-intensity-engine.md](chunibyo-intensity-engine.md)

## 1. 目标文体

目标不是传统纯文学，而是“好追更、好画、好拆场景、好切九刀”的漫画剧本。

最低文体特征：

- 格式裁决明确
- 场景单元明确
- 场景先行
- 对话占比可观
- 动作与情绪变化清楚
- 章末必须悬停
- 氛围必须能落成镜头压力，而不是只停留在文字修辞
- 兼容解说漫时，叙述句必须兼具朗读顺滑度与画面承载力

## 1.0 漫画优先改编立场

对虚构原著或已有故事素材，本技能默认不是“越像原文越好”，而是“越像能爆成漫画越好”。

执行含义：

- 允许换序、并戏、删过桥解释、前置名场面、放大关系失衡与视觉奇观。
- 保真的对象优先是 `人物关系核 / 情绪核 / 卖点核 / 关键代价`。
- 当内容说明与页势能冲突时，优先让大格、页末急停、视觉停顿与爽感成立。
- 若题材适配，允许进一步叠加中二感：命名升级、誓言升级、宿命升级、绝对化对白与自我神话化登场。
- 对现实新闻/热搜输入，上述许可只能作用于叙事组织，不得作用于事实改写。

## 1.1 先裁格式，再写正文

默认只允许以下三种格式变体：

- `scene-script`
  - 场景推进优先，动作/对白/转场明确。
- `narration-script`
  - 旁白推进优先，适合解说漫或强叙述型素材，但仍必须保留页级动作锚点。
- `compare`
  - 内部双路比较后择一，不保留双 canonical 成品。

最终必须同步落出一组按组组织的漫剧剧本文件：

- `第1组.md`
- `第2组.md`
- `第3组.md`

分组硬规则：

- 默认以“原文约 `1000` 字”为一组。
- 不满 `1000` 字的内容也按一组处理。
- 一次长文切分后，最后一组若剩余 `300` 字以内，归并到上一组。
- 最后一组若剩余 `700` 字以上，可单独成组。
- 最后一组若剩余 `301-699` 字，默认并入上一组；只有在存在明确场景闭合、钩子闭合或 payoff 边界时才允许单独成组。
- 分组优先尊重 scene / action / hook / payoff，自然边界高于机械字数切分。
- 每个组文件必须显式写出 `【边界判定】`，说明当前起止边界为何成立；尾组还必须写明 `尾组决议`。
- 每个组文件必须显式写出最小类型包 handoff：`type_stack_active_packs`、`type_pack_projection_script_adaptation`、`type_pack_projection_nine_blade`。

## 1.2 对齐 `2-格式` 的基础格式化标题

漫画技能的基础格式化处理，需要对齐 `.agents/skills/aigc/1-Planning/references/script-format-contract.md` 的共享字段纪律。

正文与结构化包至少要能稳定表达以下标题：

- `对白（主体）`
- `对白画面`
- `内心独白（主体）`
- `内心独白画面`
- `旁白（主体）`
- `旁白画面`
- `动作画面`
- `镜头语言预设`

规则：

1. `主体` 不是装饰字段，而是单一真源。相同角色、讲述者、旁白主体必须在全文和 `speaker_registry[]` 中同名。
2. `对白（主体）` 默认尽量保真原台词；若来源并无稳定对白，可压缩整理，但不能把说明句偷渡成对白。
3. `旁白（主体）` 在解说漫兼容模式下默认统一为 `讲述者`，除非用户另行指定。
4. 每条文本条目必须就近补对应 `*画面`；动作推进用 `动作画面` 承载。
5. `镜头语言预设` 只写漫画拆页或视角理解真正需要的最小提示，不写过量导演说明。

## 1.3 漫画专属扩展字段标题

在完成基础格式化层后，再叠加漫画专属表现层。推荐标题与结构化字段如下：

| 中文标题 | 结构化字段 | 用途 |
| --- | --- | --- |
| `分格建议` | `panel_split_hints[]` | 把文本切成格级推进 |
| `视觉焦点` | `panel_focus_map[]` | 锁定当前格的主视觉 |
| `跨页/大格建议` | `spread_splash_hints[]` | 声明需要拉开版面的节点 |
| `气泡负载建议` | `balloon_load_plan[]` | 控制单格对白/旁白容量 |
| `SFX建议` | `sfx_cues[]` | 给字效层或画面音效以直接提示 |
| `页末钩子` | `hook_pack[] / page_turn_candidates[]` | 服务翻页悬停与追更驱动 |

## 2. 章节/场景最小骨架

每个主场景至少包含 4 个段：

1. `冷开场`
   - 直接把人物放进事件里，不先长解释。
2. `关系/目标显形`
   - 让读者知道谁要什么。
3. `冲突升级`
   - 信息差、阻力、代价或误会必须上来。
4. `场末钩子`
   - 在最贵的位置停住。

## 2.0 节奏优先于解释

正文默认遵循：

- 晚进入：能不从最早背景开始，就不从最早背景开始。
- 早退出：冲突一旦成立，就尽快停在最贵的位置。
- 解释后移：把必要说明拆进对白、动作反应、后续场景与钩子。
- 中段抬升：不要把全部强画面都押在结尾，中段也要有至少一次节奏暴涨或视觉拔高。
- 爽点可见：角色压制、关系翻转、异物显形、危险逼近要能落成具体页面。

## 2.1 冲击画面预演

在正式写正文前，先锁定以下内容：

- `高冲击画面候选`
  - 默认至少 3 处；短篇单章也不低于 2 处。
- `翻页停笔点`
  - 默认至少 1 处，最好与异常显形、真相半揭或危险逼近重合。
- `氛围母题`
  - 默认至少 1 条，需能反复转化为声场、光线、物件异常或空间压迫。

优先命中的冲击类型：

- 首次异常显形
- 人物关系骤然失衡
- 关键物件失常
- 真相只揭半寸
- 危险逼近但尚未落地

补充要求：

- 高冲击画面不能只停留在摘要句，必须能转写为 `动作画面 / 视觉焦点 / 跨页或大格建议`。
- 若某一炸点成立但解释过满，应优先删解释，不优先降炸点。
- 视觉奇观的细则优先参照 [visual-spectacle-engine.md](visual-spectacle-engine.md)。

## 2.2 解说漫兼容的声画双栈

当用户要求兼容解说漫时，正文需要同时满足：

- `画面栈`
  - 能拆镜头、能抓大格、能保留异常物件与动作停顿。
- `声音栈`
  - 能被讲述者直接朗读，句子不绕，情绪不泄，信息不重复。

写法原则：

- 一句优先只推进一个画面或一个判断。
- 段内优先使用“画面先到 -> 感受后压”的顺序。
- 恐怖/悬疑题材优先短句、停顿句、半揭句。
- 旁白友好不等于口水化；禁止“你知道吗”“接下来”“此时此刻”这类低密度填充语。
- 读出来别像说明书，也别像散文诗；要像镜头在黑里往前推。

## 3. 场景化写法

优先写：

- 人物动作
- 目光、距离、触碰、停顿
- 场景物件
- 可视化反应
- 能入镜的情绪表现
- 能被拆成大格的异常瞬间
- 光线、声场、湿度、空间逼仄感
- 朗读时能自然停顿的句间节拍
- 页末或格末能形成悬停的动作

少写：

- 纯理论解释
- 连续大段背景讲述
- 重复概括同一层情绪
- 旁白里先说画面又解释画面再总结情绪的三连重复

快节奏时额外要求：

- 单场尽量只服务一个主矛盾与一个主炸点。
- 一页内若文字太多，先砍解释，再保留冲突动作和最贵一句。
- 相邻两场不能都处于同一种能量级；必须有“抓停 -> 释压半秒 -> 再升压”的波形。

中二感题材额外要求：

- 关键角色最好拥有一句能立住人设的绝对化宣言。
- 关键能力、组织、武器、仪式或称号最好具备命名势能。
- 中二感要落在动作、关系、誓言和代价上，不能只靠空喊设定词。

## 3.1 基础格式化层写法

当正文需要落成可复用的格式化层时，优先采用“场景块 + 条目块”结构：

- `对白（主体）` 后紧跟 `对白画面`
- `内心独白（主体）` 后紧跟 `内心独白画面`
- `旁白（主体）` 后紧跟 `旁白画面`
- 无台词推进补 `动作画面`
- 必要时再补 `镜头语言预设`

禁止：

- 不写主体直接放裸对白
- 把动作描写塞进引号文本
- 同一角色在同一稿内混用多个主体名
- 把漫画专属建议直接替代基础文本条目

## 4. 漫画友好指标

| 指标 | 说明 |
| --- | --- |
| `角色可辨识` | 读完后能直接拆出角色外观与状态 |
| `场景可入镜` | 读完后能说出场景地点、光线、氛围与关键物件 |
| `动作可切帧` | 冲突节点能切成至少两个镜头动作 |
| `对白可放气泡` | 关键台词简洁、有冲突性、适合放进漫画对白框 |
| `章末可翻页` | 结尾能自然驱动下一页/下一话 |
| `大格可成立` | 至少能明确指出哪几处适合做大格/跨页/静默特写 |
| `氛围可承压` | 压迫感能靠环境信号成立，而不是靠说明句硬说 |
| `旁白可朗读` | 抽出关键描述直接朗读时不拗口、不啰嗦、不散劲 |
| `九刀可消费` | 下游无需重猜 story units 即可切页 |
| `刺激曲线可见` | 能指出哪里抓停、哪里抬升、哪里急停 |

## 4.1 组文件写法建议

推荐每个 `第N组.md` 至少具备以下信息密度：

| 组成 | 作用 |
| --- | --- |
| `本组跨度` | 让下游快速知道这组覆盖了哪段剧情推进 |
| `边界判定` | 让执行者和 reviewer 快速知道为什么在这里切组 |
| `类型包 handoff` | 让 2 号技能无需重猜也能继承 active packs 与阶段投影 |
| `场景化正文` | 直接提供可拆页、可切九刀的正文母稿 |
| `动作与反应` | 保证每组不退化成一段纯解释 |
| `冲击画面` | 让下游能读出至少一个高冲击画面候选 |
| `组末钩子` | 给 `2-九刀流` 一个明确的九页收尾牵引点 |

## 5. 长度建议

| length_target | 建议章节数 | 每章重点 |
| --- | --- | --- |
| `short_burst` | 3-5 | 快速起冲突、快速留钩子 |
| `mini_serial` | 6-12 | 每章一个推进点，隔章小反转 |
| `medium_arc` | 12+ | 主线推进 + 支线勾连 + 阶段性大反转 |

## 5.1 快感与刺激性调节

| pacing_drive | 写法 | 风险控制 |
| --- | --- | --- |
| `stable` | 允许保留更多情绪铺垫与关系回味 | 避免前 3 场无抓停点 |
| `punchy` | 默认推荐；每场都要有推进、转折或压迫信号 | 避免一场里装太多信息 |
| `relentless` | 高频抓停、快切、强钩子、强炸点 | 避免只剩刺激没有角色代价 |

## 6. 角色卡最小要求

每个核心角色至少锁定：

- 角色身份
- 外观识别点
- 当前欲望
- 隐藏伤口或秘密
- 与主角的关系张力

## 7. 视觉桥接层最小要求

为后续漫画生成准备时，至少补：

- `角色视觉锚点`
- `关键场景锚点`
- `动作节点`
- `关键道具`
- `本章推荐钩子类型`
- `高冲击画面候选`
- `推荐大格/翻页点`
- `可直接朗读的旁白候选段`
- `speaker_registry[]`
- `分格建议 / 视觉焦点 / 气泡负载建议 / SFX建议`
- `panel_text_budget`
- `character_locks / scene_locks / prop_locks`

## 7.1 旁白可读性最低要求

- 单句默认 10-28 字为舒适区；超过时必须存在自然断点。
- 相邻两句不要重复同一主语、同一情绪判断或同一信息结论。
- 每个关键段至少保留一个“能被念出来有压迫感”的短句。
- 情绪最好藏在画面后面，让旁白读出来像在带观众看，而不是替观众下结论。

## 8. 典型错误

### 错误一：文学感很强，但画不出来

修正：

- 把抽象情绪改成可见动作。
- 把概括性叙述改成具体场景。
- 把“气氛很恐怖/很压抑”改成可画的光线、声场、物件异常和人物停顿。

### 错误二：信息很多，但读者不想追

修正：

- 删掉解释型段落。
- 提前暴露冲突，延后揭露真相。
- 不要把所有强画面都押在结尾，正文中段也要有一到两次视觉抬升。

### 错误三：章末像总结，不像钩子

修正：

- 用未闭合动作、未回答问题、危险逼近或关系反转收尾。

### 错误四：组文件能讲清剧情，但正文本身没有炸点画面

修正：

- 回到写前预演，先重做 `高冲击画面候选 + 翻页停笔点`。
- 把氛围句改写成可入镜压力，而不是抽象文气。

### 错误五：文字能画出来，但念出来发绕发虚

修正：

- 删掉解释性重句和情绪总结句。
- 把长句拆成“画面句 + 压力句”。
- 保留能形成声场停顿的短句，让旁白读出来有气口和悬停。

### 错误六：说话者主体漂移，导致同一个角色在结构化包中出现多个名字

修正：

- 回到 `speaker_registry[]` 固定 canonical 主体名。
- 统一 `对白（主体）/内心独白（主体）/旁白（主体）` 与 `character_locks[]` 的命名。
- 禁止在同一稿内混用角色全名、昵称、代词作为结构化主体键。
