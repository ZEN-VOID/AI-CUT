## Repair Playbook

1. 先锁定 `source_episode_path`，确认输出对应的上游 `第N集.md`。
2. 抽取上游 `【剧本正文】` 后正文，按原顺序建立事实承接清单。
3. 检查所有中文双引号里的对白，逐字回对上游原句；没有上游依据的新增对白必须删除或改为非 canonical 候选说明。
3.1. 检查所有 `对白` 字段标题：必须是 `对白（真实角色名，语态/状态短语）`；`原文角色`、`角色名`、`某人`、`说话者` 等只属于模板占位，不能进入终稿。第二项可以是自然短语，不强制一词或"地"字尾。
4. 检查每个声音字段后是否就近出现对应画面字段；缺失时补画面承托，不把声音文本复述到画面行。
5. 检查 `动作画面`、`角色动作` 是否可拍；心理和判断转入 `独白`、可感知 `心理反应`、可执行 `表演提示`。`心理反应` 不能成为抽象内心解释容器，必须写演员能演、观众能从画面或声音 GET 到的反应。
5.0.a. 检查关键情绪 beat 是否只写情绪标签或模板表情；若只有"愤怒/难过/紧张/害羞/开心"或皱眉、瞪眼、流泪、大笑，编剧层先补可拍基础反应，细化触发点、情绪动机、眉眼嘴/咬肌/鼻翼、手/肩/呼吸/重心/声线、环境声和微动态限制交给下游 `4-表演`。
5.0.b. 检查关键面部 beat 是否有正式落点：如果变化主要发生在眉、眼、嘴角、咬肌、下颌、喉头或眨眼频率，优先使用 `表情特写`；如果需要手、肩、呼吸、声线、道具或空间联动，落入 `心理反应`、`对白画面` 或 `角色动作`。
5.0. 检查主角内心想法和主角视角判断：不得删掉；不得写成客观第三方概括；优先投影为 `内心独白（主角）` + `内心独白画面`。
5.0.0. 检查内心独白人称：`内心独白（主角）` 引号内若承接第三人称小说叙述，主角自指必须改为第一人称；只在第三人称指向其他角色、引用文本或刻意自我疏离并留证时保留。`内心独白画面` 不按此规则改成第一人称。
5.0.1. 检查动作客观性：`角色动作` / `动作画面` 中出现"试图、想要、打算、意图、为了掩饰、准备借此"时，删除主观意图词，改成可拍动作、神态、语气、生理反应或空间运动。
5.0.2. 检查直接情绪词：`感到恶心/难受/愤怒/害怕/崩溃` 不直接进终稿，改成微表情、肢体动作、生理反应、声线变化或主角内心独白。
5.1. 检查动作速度和力度：上游出现"一闪、骤然、缓缓、忽然、迟疑、停住、挣出、扑倒"等信号时，`角色动作` / `动作画面` 必须保留相应快慢、停顿、节奏或力度。
5.2. 检查 `道具特写`：不要把当前项目的具体物件清单写成通用规则；只写物件的可见信息、状态、痕迹、归属和变化，解释性推理转入 `心理反应` 或声画承托。
6. 建立 slugline 去重表，合并同一集内完全相同的场景标题。
7. 最后才做编剧质量 polish：强化场景目的、冲突压力、表演任务和可见承托，但不改变事实和对白。
8. 交付前专门执行一次"画面纯度 pass"：所有 `*画面`、`环境描写`、`道具特写`、`表演提示` 必须落到可见或演员可执行事项；抽象信息转入 `心理反应`、`独白`，或删除。
8.1. 交付前专门执行一次"占位泄露 pass"：删除所有内部规则句、模板句、任务说明和尖括号占位；这些内容不能作为剧本正文保留。
8.2. 交付前专门执行一次"环境纯度 pass"：`环境描写` 只保留地点、空间结构、自然条件、空气介质、光照状态、承载面、围护面、开口边界、静置物件等写景材料；咬、凑近、说、握住、别过头、收袖、拔刀、奔跑、灌酒等人物动作必须拆出。
8.3. 交付前专门执行一次"小说表述转译 pass"：抽查作者评论、主角视角判断、心理内视、文学比喻、抽象概括、往日常态句、背景说明、因果解释、关系结论是否已转成可拍声画、表演、空间、道具、群像、主角内心独白、短旁白或留白；不得改写引号内对白，不得把小说叙述变成新增台词，不得新增无关人物过往、物品来历或回忆性信息。
8.4. 交付前专门执行一次"声音本体 pass"：所有 `音效` 引号内必须是可听见的声音，不得是时间、事件或描述性概括。
9. 交付前把 review finding 当成修复输入，而不是报告附件；阻断项先在本阶段修复并复审，仍失败再阻断或回源层。

## Reusable Heuristics

- `2-编剧` 的"创作"主要发生在投影方式、画面承托和场景标题上，不发生在剧情事实重写上。
- 对白标签也是表演入口：角色名解决"谁说"，第二项解决"以什么状态说"；它应灵动、自然、鲜活，但只服务表演展示，不能改对白原文。
- 好的影视化不是把小说写短，而是把小说信息拆成可拍字段，让下游能解析、调度和表演。
- 声音字段要像录音部门能拿走的材料，画面字段要像导演和摄影能拍出来的材料。
- 角色内心可以保留，但要转成有主体的独白、心理反应或表演提示；不要假装内心解释是可拍动作。
- 主角内心尤其要保留：主角对他人行为的判断、怀疑、误读和确认，是主观视角资产；它应进入 `内心独白（主角）`，而不是被删掉或改成全知客观结论。
- 内心独白不是旁白。把第三人称小说叙述投进 `内心独白（主角）` 时，主角自指要变成"我/我的/自己"；画面承托再用第三人称写可拍动作。
- 动作字段是镜头事实，不是动机说明书；"试图/想要/打算/意图"通常说明写作者还没有找到可拍外显，应回到手、眼、呼吸、停顿、声线、重心和道具。
- 直接情绪词是草稿信号，不是终稿画面；恶心可以是喉头、嘴角和手背，愤怒可以是指节、下颌和停顿，难受可以是呼吸和重心。
- `表情特写` 是下游链路的面部锚点，不是每句台词的默认伴随字段。只有当情绪变化主要集中在眉眼嘴、咬肌、下颌、喉头、眨眼频率或脸部细节时才开；否则让 `心理反应`、`对白画面`、`角色动作` 承接更自然。
- 往日常态和人物前史容易把剧本拖回小说解释；除非当前主线必须承接，否则删掉，只保留当下能拍到的物、声、人和反应。
- slugline 是下游分组的锚点，不是剧情 beat 标题；同一个空间和时间段内只需要一个场景编号。
- 质量提升的顺序是：保真 -> 可拍 -> 声画清楚 -> 表演可执行 -> 下游可分组。
- "画面"不是解释画面的意义，而是把意义放进可见物：手、眼、门、纸、黑板、光线、座位、空位、屏幕字、身体距离。
- 小说二次画面化的关键不是把句子改得更美，而是把小说句子的功能转译出去：作者判断给空间和群体，心理给演员身体，比喻给物象和材质，概括给 beat，背景给画面证据与短旁白。
- 台词是冻结真源；二次加工只能发生在台词外的说话状态、对白画面、动作、停顿、空间距离、道具反应和环境余波里。
- `环境描写` 不是万能承接字段；它写景，人物做事进 `角色动作`，对白附近反应进 `对白画面`，群体状态进 `群像画面`。
- `角色动作` / `动作画面` 不是动作清单；关键动作要有速度和力度，让演员知道是骤发、迟疑、缓慢压下、猛地扑出，还是停半拍后才动。
- `道具特写` 不是"某几个固定道具"的字段；它按功能承接信息载体、规则显影、线索痕迹、归属压力和状态变化。
- 占位句是比空字段更严重的失败：宁可少写一条字段，也不要把"本场按上游..."或"说话者的视线..."这类规则句当正文。
- "音效"不是声音的说明书；把"八点四十分"这类信息放到画面上下文，把引号内留给真正能听见的声音。
- 对白角色名必须真实：`原文角色`、`角色名`、`某人`、`说话者` 是模板占位，不能进入终稿；对白标题要同时承载谁说和以什么状态说。
