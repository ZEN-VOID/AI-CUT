# Frame Image Heuristics

- `分镜画面` 当前默认拓扑是组级多图：一个普通 `## x-y-z` 分镜组形成一个 `.agents/skills/cli/imagegen` 组级任务包，每个 `shot_id` 对应一个 imagegen task spec。
- 组内普通 `分镜N` 的数量决定 `image_count / n`，并映射为 `x-y-z-N` 输出图片。
- 多图任务不是故事板拼图；输出必须是多张单独 bitmap。
- 组级 prompt 必须直接消费完整组稿，不能只喂单镜摘要。
- 默认画面比例为 16:9；用户显式要求时才切换到 9:16 或其他比例。
- 角色和场景一致性优先通过同一调用、共享主体图、共享场景图和共享一致性约束实现。
- 缺少主体图时记录 missing；不要用相似名称、泛词或 JSON 文件冒充图片。
- 若 provider 单次多图上限不足，阻断或请求用户授权拆组，不能静默改变任务拓扑。
