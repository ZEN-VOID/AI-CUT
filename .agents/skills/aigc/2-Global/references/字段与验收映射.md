# 2-Global 字段与验收映射

本文件细化 `2-Global` 的字段槽位、pass 责任与验收钩子。
若与主 `SKILL.md`、`templates/episode-root.template.json` 或 `references/io-contract.md` 冲突，以这些真源为准。

## Field Master

| field_id | target_path | owner_node | requirement |
| --- | --- | --- | --- |
| `FIELD-GLOBAL-01` | `meta.剧名 / 集数 / 组数 / 总时长` | `N5-WRITEBACK` | 必须与当前项目和当前集严格一致 |
| `FIELD-GLOBAL-02` | `project_global.全局风格` | `N3-PROJECT-GLOBAL` | 形成项目级统一风格前缀；真人古装影视项目默认遵循 `references/全局风格词最佳实践.md` |
| `FIELD-GLOBAL-03` | `project_global.全集类型元素` | `N3-PROJECT-GLOBAL` | 形成项目级类型总则 |
| `FIELD-GLOBAL-04` | `groups[].分镜组ID / global.剧本正文` | `N4-GROUP-GLOBAL` | 保留命中组原始正文与组标识 |
| `FIELD-GLOBAL-05` | `groups[].global.全局风格` | `N4-GROUP-GLOBAL` | 默认继承项目级风格前缀，并允许组级微调 |
| `FIELD-GLOBAL-06` | `groups[].global.类型元素` | `N4-GROUP-GLOBAL` | 对齐当前组的类型信号 |
| `FIELD-GLOBAL-07` | `groups[].global.导演意图` | `N4-GROUP-GLOBAL` | 对齐当前组的导演执行导向；至少包含观看策略、执行抓手、禁用方向三层，禁止写成剧本正文摘句 |
| `FIELD-GLOBAL-08` | `validation-report.md` | `N7-CLOSURE` | 写回验收、阻塞与根因上溯 |

## Pass-to-Field Matrix

| pass | direct_write_target | hard_gate |
| --- | --- | --- |
| `1-项目级风格` | `project_global.全局风格`、`groups[].global.全局风格` | 不得写成具体镜头操作或工具参数；真人古装影视项目必须保持写实摄影、物理照明、真实材质、克制镜头与反动画化禁区 |
| `2-项目级类型` | `project_global.全集类型元素` | 不得混入某一组的临场打法 |
| `3-组级类型` | `groups[].global.类型元素` | 必须逐组对齐，不得跨组混写 |
| `4-导演意图` | `groups[].global.导演意图` | 必须可被 `3-Detail` 直接消费，且不得只是剧情复述或正文截句 |
| `5-组正文入壳` | `meta`、`groups[].分镜组ID`、`groups[].global.剧本正文` | 必须完整保留命中组正文 |
| `6-验收` | `validation-report.md` | 必须记录阻塞、根因上溯与下一阶段回接 |

## Direct JSON Writeback Contract

运行时 `第N集.json` 最低必须满足；字段结构同构于 `templates/episode-root.template.json`：

- `meta`
  - `剧名`
  - `集数`
  - `组数`
  - `总时长`
- `project_global`
  - `全局风格`
  - `全集类型元素`
- `groups[]`
  - `分镜组ID`
  - `global.剧本正文`
  - `global.全局风格`
  - `global.类型元素`
  - `global.导演意图`

字段规则：

1. `project_global.全局风格` 与 `groups[].global.全局风格` 默认保持同值，便于 `3-Detail` 与旧下游直接读取。
   - 真人古装影视项目的默认核心为：高清电影级真实摄影语法、真实人物皮肤与服饰纹理、自然毛发细节、可感材质反射、物理照明、高光不过曝与暗部可读、电影化纵深与权力关系调度、克制稳定镜头、真实物理动作反馈、4K HDR、影院级调色、轻胶片颗粒。
   - 禁止把 `全局风格` 写成动画描线、赛璐璐分层、夸张残影、过度粒子拖尾、非物理高光或工具参数堆叠。
2. `project_global.全集类型元素` 只承载项目级类型总则。
3. `groups[].global.类型元素` 与 `groups[].global.导演意图` 必须严格对齐当前 `分镜组ID`。
4. `groups[].global.导演意图` 必须能直接指导 `3-Detail` 的镜头、表演、调度、节奏或空间处理；推荐定稿形态为“观看策略：...；执行抓手：...；禁用...”，不得只截取 `global.剧本正文` 中的动作或对白。
5. `groups[].global.剧本正文` 必须完整整理命中组全文。
6. 当前阶段不得新增 `detail`、`分镜列表`、`分镜数` 或任何镜级字段。

## Acceptance Hooks

执行结束前至少检查：

1. `第N集.json` 的 `meta + project_global + groups[].global` 同构于 `templates/episode-root.template.json`
2. `groups[].global.剧本正文` 未被摘要化
3. `groups[].global.导演意图` 通过 `scripts/validate_director_intent.py` 或等价人工复核，不像正文摘句且具备三层执行导向
4. compat 文件若存在，都能回链到 JSON 对应字段
5. `validation-report.md` 记录了本轮模式、命中 scope、阻塞与 handoff
