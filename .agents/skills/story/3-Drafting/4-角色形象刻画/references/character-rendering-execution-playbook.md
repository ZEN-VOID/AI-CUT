# Character Rendering Execution Playbook

本细则服务 `3-Drafting / 4-角色形象刻画`，用于把角色卡与当前场面压力，稳定翻译成“可见的人物存在感”。

## Four-Slot Character Pass

对每个本章焦点角色，至少先回答四个槽位，再动正文：

| slot | 要回答的问题 | 允许落到正文的典型形式 |
| --- | --- | --- |
| `surface_drive` | 他此刻最想要什么？ | 抢先开口、绕开风险、占位、回避、逼近 |
| `defense_or_wound` | 他最怕暴露什么？ | 停顿、嘴硬、先发制人、装轻松、转移话题 |
| `relationship_posture` | 他对当前对手是谁上谁下、谁欠谁、谁离不开谁的判断是什么？ | 称呼变化、站位、看与不看、命令/商量/试探 |
| `visible_arc_shift` | 本章最小可见的偏移是什么？ | 立场松动、底线暴露、第一次承认、第一次失手、第一次退让 |

## Testimony And Justification Pass

当当前故事命中口供、证词冲突、互相指认、真假不明、道德困境或人物急于自证清白时，再额外回答四个槽位：

| slot | 要回答的问题 | 允许落到正文的典型形式 |
| --- | --- | --- |
| `face_to_protect` | 他最想保住什么体面、利益或自尊？ | 清白、勇敢、忠诚、体面、爱、专业性 |
| `fact_to_trim` | 他最想删掉或改轻哪一段事实？ | 省略开头、改口顺序、弱化责任、转移主动性 |
| `justification_style` | 他如何让自己的版本听起来还能成立？ | 先发制人、诉苦、装笨、强调不得已、把责任推给情势 |
| `exposed_by_action` | 哪个小动作会出卖他真正的裂缝？ | 抢着解释、摸物件、避开目光、改称呼、先骂对方 |

## System Pressure Pass

当当前故事命中科技设定、灾难系统、组织机器、文明危机或其他强世界规则变化时，再额外回答四个槽位：

| slot | 要回答的问题 | 允许落到正文的典型形式 |
| --- | --- | --- |
| `rule_change` | 哪条新规则已经压到这个人身上？ | 倒计时、装置限制、命令链、资源短缺、生存协议 |
| `system_role` | 他在系统里是什么位置？ | 执行者、维护者、反对者、替补者、被牺牲者 |
| `collective_cost_view` | 他如何理解集体代价？ | 主动扛责、冷处理、算账、迟疑、逃避、反抗 |
| `scale_response` | 同样的大压力，他的独特反应是什么？ | 把任务当信仰、把生存当算术、把命令当羞辱、把牺牲当职责 |

## Rewrite Priorities

1. 先改选择，再改动作，再改形容词。
2. 先让角色在冲突里露出“怎么做”，再补“像什么样的人”。
3. 若命中证词/道德困境，先改“他如何为自己辩护”和“什么动作出卖了他”，再补情绪解释。
4. 若命中宏观设定，先改“规则怎样压到他”和“他怎样承担系统后果”，再补个人情绪。
5. 若角色仍然扁平，优先改他处理关系、体面和责任的方式，不要先补更多背景说明。
6. 若角色的“个人化”只体现在外貌、服饰、职业名词，而不是行动方式，视为仍未达标。

## Arc Minimum Unit

`角色弧光` 在 Step 4 不要求完成整季逆转，但至少要留下一个可被 validation / review 指认的偏移：

- 做了平时不会轻易做的选择
- 没守住一条平时很稳的防线
- 对某人态度升温或降温
- 第一次付出可见代价
- 第一次让真实欲望或羞耻露头

## Differentiation Guard

若两个角色互换名字后，段落依然基本成立，通常说明仍失败。优先补以下差异，而不是补更多形容词：

- 节奏差异：快、拖、压、停
- 风险偏好：顶上去、绕过去、等一等
- 关系姿态：压人、让人、躲人、试人
- 身体习惯：摸、扣、站、坐、看、避
- 代价承受方式：硬扛、转嫁、嘴硬、沉默

如果当前故事存在巨型装置、灾难系统、战争机器或文明任务，还要补：

- 责任姿态：服从、计算、承担、拆解、逃离
- 集体尺度感：只顾眼前、能看全局、愿为后代忍耐、把众人当数字
- 系统语言的处理方式：照本宣科、技术化消解、拒绝承认、转成人话、故意沉默

如果当前故事存在证词冲突或道德裂缝，还要补：

- 体面保护点：清白、尊严、勇气、忠诚、被爱、专业权威
- 说法偏差：先说结果、漏说起因、换动词、偷换责任
- 暴露方式：物件、停顿、目光、次序、动作先于解释

## Common Failure Patterns

| symptom | direct cause | rewrite move |
| --- | --- | --- |
| 角色很忙，但不鲜活 | 只写任务动作，没写个人处理方式 | 给相同事件补“这个人会怎么做”的偏差 |
| 有设定，但没弧光 | 只把角色卡翻成静态标签 | 补一处选择偏移或关系位移 |
| 角色各不相同，但写出来像同一作者 | 个性化只落在背景，不落在动作节奏 | 回到 `surface_drive / defense / relationship_posture` 重写 |
| 主角成长只出现在总结里 | 没把成长翻成可见行为证据 | 把成长改写成选择、失手、忍住或承担 |
| 设定很大，但人物像旁观者 | 没把世界规则变化压到角色职责和风险上 | 回到 `rule_change / system_role / collective_cost_view` 重写 |
| 巨型危机下人人都一个腔调 | 没区分不同角色的责任姿态和尺度判断 | 在同一系统压力下重写每个人的承担方式 |
| 证词戏很热闹，但人物没有锋利度 | 每个人都只在给信息，没有保护自己的版本 | 回到 `face_to_protect / fact_to_trim / justification_style` 重写 |
| 道德困境有了，但读者看不见真正代价 | 没把困境压成可验证动作或物件证据 | 让一个小动作先于解释出现，再让对白追上来 |
