# 导演意图思维链与字段主表

本文件承载 `导演意图` 的字段主表、可见思维快照、工具后反思与返工入口真源。

## 模式与对象

| 项目 | 当前合同 |
| --- | --- |
| 默认模式 | `思维链优化设计`；若叶子技能被整体重建，可临时降级为 `原生思维链设计`，但最终仍回到本文件接口 |
| 目标对象 | 按集承载的逐组导演设计真源，服务 `projects/<项目名>/编导/第N集.json` 中 `分镜组列表[].组间设计.导演意图` 字段 |
| 默认模型策略 | 若调用模型具备 reasoning / thinking 能力，优先使用高层判断 + 可见快照 + Gate；仅在普通模型下使用少量显式步骤支架 |
| 可见性边界 | 只保留可审计、可回写、可返工的导演判断快照；不要求外显完整内部推理 |
| 工具 / 证据依赖 | 强依赖完整 `第N集.json` 中已稳定的 `全局风格 / 类型元素` 字段区块，以及 `1-规划/3-分组/第N集.md` |
| 受保护接口 | 不破坏 `FIELD-DI-01` 到 `FIELD-DI-05` 与五区块主输出；不越权写成 shot list 或分镜页 |

## Think-Think Design Snapshot

### 启发式工作链

- `不可删性`: 删掉这轮判断后，文档会失去哪一种“这集为什么必须这样拍”的集级收束能力？
- `最先推动`: 默认先推动 `FIELD-DI-01` 与 `FIELD-DI-02`；若本集命题和观看契约没锁住，禁止提前美化 `FIELD-DI-03`。
- `先砍什么`: 先砍项目口号复读、类型词堆砌、各组可互换文案、只抒情不落地的导演腔。
- `真正比较尺`: 比较的不是谁更有腔调，而是谁最能同时锁住观看契约、拉开组级层次、降低 `3-明细` 的二次猜谜成本。
- `立场纠偏`: 当前判断默认站在 `3-明细` 与部门执行者的消费立场，而不是文档作者自我表演的立场。
- `工具后反思`: 读完上游真源后，先问容器是否足够、冲突来自哪一层、是否要回上游补件，再继续写组级构思与桥段锚点。
- `落盘门`: 每条保留判断都必须落到 `FIELD-DI-01` 到 `FIELD-DI-05`、`Gate Summary`、`Fail Code` 或 `Rework Entry` 之一；落不到就删。

### 三轴三重裁决

| 裁决角色 | 在 `导演意图` 的具体问题 | 主要落点 |
| --- | --- | --- |
| `方向轴` | 这一集到底该如何被观看，观众贴谁、离多近、带着什么情绪距离进入 | `FIELD-DI-01`、`FIELD-DI-02` |
| `成立轴` | 这些导演判断是否被 `全局风格 / 类型元素 / 分组容器` 共同支撑，并已翻译成部门可执行后果 | `FIELD-DI-03`、`FIELD-DI-05` |
| `优选轴` | 在成立解里，哪种组级放大与情绪推进最能让本集收束、相邻组拉开差异、下游最易继承 | `FIELD-DI-03`、`FIELD-DI-04`、`FIELD-DI-05` |
| `硬门禁轴` | 是否出现缺容器、组间可替换、无部门后果、越权滑成 shot list、下游不可消费等强制否决条件 | `FAIL-DI-01` 到 `FAIL-DI-05` |

| 收敛层 | 本层先裁什么 | 服务 `field_id` | 本层收窄了什么 | 正确产物 |
| --- | --- | --- | --- | --- |
| `粗裁决` | 本集先以哪条导演命题与观看契约收束全局 | `FIELD-DI-01`、`FIELD-DI-02` | 收窄掉项目总纲复读、泛风格口号、仅复述剧情的伪命题 | 唯一 `本集总导演命题 + 本集观看方式` |
| `细裁决` | 每个组在总命题中承担什么职责、适合借哪类桥段机制，并要被怎样看见 | `FIELD-DI-03` | 收窄掉组间可互换文案、无部门后果的抽象判断、空泛片单联想、抢写镜头表的越界细化 | 可区分的 `分组导演构思`，且含桥段锚点与部门级执行后果 |
| `离散裁决` | 情绪如何跨组推进，并把哪些不可稀释项交给 `3-明细` | `FIELD-DI-04`、`FIELD-DI-05` | 收窄掉平铺直叙的情绪串联、空泛 handoff、越权 shot design | `情绪推进 + 下游加载提示` |

### 可见 / 隐藏分层

| 层级 | 保留内容 | 去向 |
| --- | --- | --- |
| `可见快照层` | `本集总导演命题`、`本集观看方式`、可区分的 `分组导演构思`、`情绪推进`、`下游加载提示` | 进入目标文档与字段表 |
| `Gate Summary 层` | 当前阻塞、主导失败码、返工入口、confidence、unknowns | 进入验收记录或 sidecar |
| `隐藏推理层` | 被判废的候选观看契约、备选组级放大方案、未采用的情绪编排比较 | 仅留在内部推理，不进可见合同 |

## 工具后反思与 Gate Summary

### 工具后反思

| 触发节点 | 必问反思 | 继续条件 | 不满足时回退 |
| --- | --- | --- | --- |
| 读取 `全局风格 / 类型元素 / 分组结果` 后 | 上游三者是否互相支撑；是否已经足够锁定本集命题与观看契约 | 能明确回答“这集为何这样拍、观众怎么进入” | 回上游补件或暂停写作 |
| 完成 `分组导演构思` 初稿后 | 每组是否都具备独立职责、最该看见什么、清楚的桥段锚点、至少 3 类部门后果 | 任意相邻组互换后会明显失配，且桥段锚点不会泛化到任意组都能套用 | 回 `FIELD-DI-03` 重写 |
| 准备写 `下游加载提示` 前 | handoff 是否真的把不可稀释项交给 `3-明细`，而不是重复本层或越权下钻 | `3-明细` 可直接继续放大，且不需要猜导演意图 | 回 `FIELD-DI-04` / `FIELD-DI-05` 收束 |

### Gate Summary Contract

| 项目 | 必须回答 |
| --- | --- |
| `status` | `green / yellow / red` 三选一，不能省略 |
| `dominant_fail_code` | 当前最主要的失败码或 `none` |
| `rework_entry` | 若失败，必须指回 `S1` 到 `S5` |
| `confidence` | `0-1` 浮点值，说明当前导演判断稳定度 |
| `unknowns` | 尚未被上游真源解决的缺口，最多 3 条 |

## Field Master

| field_id | 输出位置/字段 | 内容要求 | 默认责任 Step | 质量维度 | 失败码 |
| --- | --- | --- | --- | --- | --- |
| FIELD-DI-01 | `本集总导演命题` | 锁定本集主题拷问、表达核心与“为什么必须这样拍”的唯一收束句；不得退化成项目总纲或剧情摘要 | S1 | 集级收束度 | FAIL-DI-01 |
| FIELD-DI-02 | `本集观看方式` | 明确观众通过谁的眼睛进入、情绪距离、基调与叙事视点；要能排除至少一种不应采用的观看契约 | S2 | 观看准确性 | FAIL-DI-02 |
| FIELD-DI-03 | `分组导演构思` | 逐组翻译为可区分的导演执行语言，至少覆盖组级职责、最该看见什么、`参照桥段锚点` 与 3 类部门后果；相邻组互换后应明显失配 | S3 | 组级差异度 | FAIL-DI-03 |
| FIELD-DI-04 | `情绪推进` | 写清本集情绪线如何跨组升级、断裂、回收，并解释主客观切换与推进节拍依据 | S4 | 情绪连贯性 | FAIL-DI-04 |
| FIELD-DI-05 | `下游加载提示` | 给 `3-明细` 的加载说明，明确不可稀释项、继续放大方向、禁止越权项与下沉边界 | S5 | 交接可执行性 | FAIL-DI-05 |

## Thought Pass Map

| step_id | 聚焦字段 | 核心问题 | 生成动作 | 未达标信号 |
| --- | --- | --- | --- | --- |
| S1 | FIELD-DI-01 | 这集到底在逼问什么，为什么必须这样拍而不是只复述剧情 | 写唯一总命题，并排除至少一种伪命题 | 退化成项目总纲、人物小传或剧情摘要 |
| S2 | FIELD-DI-02 | 观众通过谁的眼睛进入，又被放在什么情绪距离上 | 锁观看契约、基调与视点，并写清不采用的观看方式 | 只有情节说明，没有观看设计 |
| S3 | FIELD-DI-03 | 每组在总命题中的职责是什么，适合借哪类桥段机制，抽象判断怎样翻译为部门级执行法则 | 写可区分的组级构思、桥段锚点与至少 3 类部门后果 | 各组同质化，或桥段锚点只是模糊片单联想 |
| S4 | FIELD-DI-04 | 情绪怎样跨组推进，何时主观化、何时抽离、何时反压 | 写情绪线、转折依据与切换门槛 | 组间割裂，或所有组都维持同一情绪平面 |
| S5 | FIELD-DI-05 | 如何交给 `3-明细` 继续放大而不被稀释，也不越权代写镜头表 | 写 handoff、继续放大方向与边界说明 | 后续不可消费，或直接滑成 shot list |

## Validation Matrix

| 检查项 | 通过标准 | 失败信号 |
| --- | --- | --- |
| 前置容器检查 | 已读取并对齐 `全局风格 / 类型元素 / 分组结果` 三类上游真源 | 缺任一上游仍继续写正文 |
| 遮轴名快检 | 遮掉“主题/情绪/风格”等抽象词后，仍能说清这集为何这样拍、观众如何进入、脚本应继承什么 | 只剩一组好听的大词 |
| 组级替换压测 | 将任一组的 ID 替换为相邻组后，文本会明显失配 | 替换组 ID 后文案几乎不变 |
| 参照桥段回译检查 | 每个桥段锚点都已回译为当前项目的冲突机制、情绪目标或关系翻面方式 | 只写片名、名场面昵称，或离开原作就失去解释力 |
| 部门落盘追踪 | 每组至少落下 `摄影/灯光`、`声音/音乐`、`表演` 三类中的任意三类执行后果 | 所有组都停在气氛形容词层 |
| 可见 / 隐藏分层检查 | 可见层只保留可审快照，备选方案与比较过程不混入正文 | 文档开始讲大量被放弃的思路历程 |
| 工具后反思检查 | 每个关键上游读取节点后，都发生过“是否继续 / 是否回退 / 是否补件”的二次判断 | 上游结果一回来就机械继续 |
| 越权扰动测试 | 若继续下钻，文档仍停在“部门级法则”，不会自然滑成逐镜头机位表 | 稍微细化就直接变成分镜或 shot list |
| 下游 handoff 测试 | `3-明细` 能直接抽取不可稀释项与继续放大方向 | handoff 只有空泛提醒，没有可执行继承法则 |
