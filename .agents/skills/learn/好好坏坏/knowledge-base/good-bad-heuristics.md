# Good/Bad Heuristics

本文件保存跨 skill 可复用的好/坏示例源层调优经验。它按需检索，不作为每次执行固定全量上下文。

## Heuristics

- 好示例不是模板真源，坏示例也不是反向模板真源；两者共同暴露的是源层规则缺口。
- 若好示例与坏示例都满足当前 `SKILL.md`，但用户明显偏好好示例，通常说明 review gate 或 `CONTEXT.md` 经验层缺少可执行偏好。
- 若坏示例违反了目标 Output Contract 却仍被交付，优先修 review gate，而不是只改模板。
- 若坏示例内容事实错误，先检查资料读取和证据保留步骤，再考虑输出风格。
- 若坏示例字段正确但判断低质，优先检查 steps 的思行节点是否只有动作没有判断。
- 若好/坏差异只在项目口味，不要污染跨项目 skill；写项目 `MEMORY.md` 或项目级 `CONTEXT/`。
- 若用户给出多组好/坏样例，先找交集机制；只出现一次的差异先列为低置信 follow-up。
