# Statistics YAML Contract

本文件定义每个分镜组底部 YAML 统计的字段、统计边界和抽取规则。

## YAML Shape

每个分镜组底部必须附加 fenced YAML：

```yaml
字数统计: 0字
时长估算: 约0秒
角色:
  - 角色名
场景:
  - 场景名
道具:
  - 道具名
```

无角色、场景或道具时使用空列表 `[]`，不得省略字段。

## Count Boundary

`字数统计` 统计该分镜组标题后重复输出的场景标题行，以及从 `5-摄影` 划定并同步原换行的分镜剧本正文，包括正文内场景标题、字段名、对白标签、原有 `分镜明细：` 和 `分镜N（约X秒）:` 内容。不统计分镜组标题、north_star 三项组头、组底 fenced YAML，也不统计任何 `## <上一个分镜组ID>~<下一个分镜组ID>` 块。

`时长估算` 只累计组内正文中所有显式 `分镜N（约X秒）:` 的 `X秒`。组标题、标题后重复输出的场景标题行、north_star 三项组头、组底 fenced YAML 和组间连接件均不计入。若上游缺少显式秒数，应回到 `5-摄影` 补齐；临时兼容旧稿时，执行报告必须说明缺秒数来源和估算风险。

以下结构不计入 `字数统计`、`时长估算`、1680 字目标上限或 1980 字硬上限：

- 分镜组标题与组头 north_star。
- 组底 fenced YAML。
- 组间首尾帧连接件。

## Role Extraction

角色字段优先来源：

1. `对白（角色名）`、`独白（角色名）`、`旁白（角色名）` 等声音主体。
2. `角色动作：`、`表演提示（角色名）：`、`表情特写：` 中明确出现的人名。
3. 场景正文中明确可见或可听见的角色名。

群像可写为 `学生群像`、`围观人群` 等稳定称谓。未实际出场但被提及的角色不列入，除非本组以影像、声音或道具显影方式让其成为生成主体。

## Scene Extraction

场景字段优先取当前 `### 场景N：...` 的场所名。一般一个分镜组只对应一个场景；若跨场景，优先重新拆组。确需多场景时，按出现顺序列多个场景名，并在执行报告说明原因。

## Prop Extraction

道具字段只列重要叙事道具、规则道具、视觉钩子或后续生成必须锁定的物件。普通环境物件只有在本组承担叙事功能时才列入。

道具抽取必须先做同物识别与归一合并，再写入 YAML：

- 同一物品只列一次，使用最稳定、最利于后续资产锁定的称谓；不要把颜色、位置、破损状态、持有人或镜头景别拆成多个道具。
- 形容词差异、状态差异、持有人差异、镜头差异、单复数差异和局部部件默认合并到同一 canonical 道具名，例如 `红色手机` / `手机屏幕` / `她的手机` 可合并为 `手机`，除非正文明确存在多部不同手机且会影响生成一致性。
- 若同一大件的部件本身承担独立叙事功能，才可单列部件；否则部件并入主体物件，例如 `木门` 与 `门把手` 默认合并为 `木门`，只有门把手被特写、触发机关或作为规则显影核心时才单列。
- 同类多个普通背景物不逐项铺开；需要保留群体功能时写成集合称谓，例如 `课桌椅`、`散落文件`、`监控屏幕组`。
- 道具名应短而稳定，避免写成动作句、镜头句或状态句；状态变化写在正文或执行报告中，不在 YAML 道具列表中重复展开。

优先来源：

1. `道具特写：`
2. `规则显影：`
3. `系统画面：`
4. 组间连接件中的关键视觉承接点仅可作为连接件证据，不应污染本组正文统计；若该物件也在本组正文中出现，按正文证据列入
5. 对白或动作中被明确使用的关键物件

建议流程：先从组内正文收集候选物件，再删除普通环境背景物，随后按“同一实物 / 同类集合 / 独立叙事部件”三类合并，最后只输出去重后的 canonical 道具名。无法确定是否同一物品时，优先少列；只有多物并存会影响角色动作、规则显影或后续生成一致性时才拆开。

## Review Notes

YAML 是统计证据，不是新的剧情真源。若 YAML 与正文冲突，以正文为准并修复 YAML。

每个 YAML 统计项都应能回指本组正文、场景标题、对白主体、角色动作、道具特写、规则显影或系统画面中的证据。`时长估算` 应能回指组内 `分镜N（约X秒）:` 的累计结果。组间连接件不是组内角色/场景/道具统计真源；若连接件使用某个承接物，但该物件未在本组正文出现，不应反向写入本组 YAML。机械 validator 只检查字段形状、计入场景标题行后的正文字数估算、显式时长累计和精确重复道具名 warning；角色、场景、道具是否准确、同物异名是否应合并以及边界时长是否合理必须由 `statistics_evidence_review` 与 `boundary_review` 做 LLM 或人工语义复核。
