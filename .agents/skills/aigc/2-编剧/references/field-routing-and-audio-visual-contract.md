# Field Routing And Audio-Visual Contract

> **契约地位**：本文件是 `2-编导` script layer 的**执行细则**，定义字段路由、声画配对和可感知化规范。
>
> **交叉引用**：本文件中引用的保真规则编号对应 `script-adaptation-contract.md`：
> - `FR-1` ~ `FR-7` 定义不可违背底线
> - `Concrete Visual Rule` 定义画面化要求
> - 其他执行问题以本文件为准

---

## 1. Audio-Visual Pairing（声画配对）

> **规则来源**：`FR-2` 信息量保真 + `FR-5` 声音字段纯度

声画一体性是**强制门禁**。文本与声音条目必须就近配对同命题画面字段。

### 1.1 配对矩阵（完整版）

| 声音字段 | 必须配对的画面字段 | 配对说明 |
|---------|------------------|----------|
| `对白（角色，语态）` | `对白画面` | 语态项描述说话方式，非强制一词 |
| `独白（角色）` | `独白画面` | 角色为独白发声者 |
| `内心独白（角色）` | `内心独白画面` | 角色为独白发出者 |
| `旁白（主体）` | `旁白画面` | 主体可以是系统、叙述者、画外音来源 |
| `音效（来源）` | `音效画面` | 来源可以是可见声源或 `音效（不可见来源）` |

### 1.2 字段纯度规则

| 字段类型 | 只写什么 | 禁止写什么 |
|---------|---------|-----------|
| 声音字段 | 可听见/可说出/可声音设计的声音本体 | 时间说明、事件说明、叙述概括 |
| `*画面` 字段 | 可见画面、表演、空间或承托 | 声音文本重复解释、抽象概念、心理结论 |
| `*画面` 字段 | 必须落到可见的身体、空间、道具、文字、光线、运动或表情 | 主题解释、比喻结论、任务说明 |

### 1.2A Same-Frame Continuity（同画面连续性）

声画配对不等于必须新增一个独立拍摄画面。若配对画面与上一条或下一条画面字段实际发生在同一时刻、同一空间、同一主体或同一动作链上，必须先判断是否应合并，避免下游分组把重复表述拍成两个画面。

**必须合并或压缩的情况**：

| 触发信号 | 风险 | 处理 |
| --- | --- | --- |
| 相邻两条字段描述同一主体的同一停顿、同一手部动作、同一视线或同一道具状态 | 下游误判为两个分镜/拍摄单位 | 保留信息更完整的一条，另一条并入同一字段 |
| `对白画面` 只是换词复述前一条 `角色动作` / `表情特写` | 拍摄阶段重复拍同一反应 | 将对白承托写成前一动作的补充细节，或删除重复句 |
| `音效画面` 与 `道具特写` 指向同一声源和同一可见状态 | 声源被拆成两个不必要画面 | 合并为一条声源画面，保留声音触发和人物反应 |
| `心理反应` 与 `表情特写` 描述同一脸部变化 | 表演任务被拆成重复提示 | 只保留最适合字段；另一项变成补充短语 |

**允许连续保留的分界条件**：

- 主体变化：说话者转为听者、个体转为群体、主角转为道具。
- 空间焦点变化：门内转门外、桌面转人群、近处道具转远处反应。
- 时间推进：动作前、动作中、动作结果形成清楚先后。
- 信息变化：新文字显影、新物件状态、新声音来源或新关系压力出现。
- 节奏功能变化：前一字段负责动作落点，后一字段负责沉默余波或反应扩散。

报告要求：正式写回时，`same_frame_continuity_map` 至少记录 `visual_cluster`、`time_space_relation`、`merge_or_keep_decision`、`kept_field` 和 `downstream_split_risk`。

### 1.3 禁止的声音字段输出

| 错误 | 原因 | 修正 |
|------|------|------|
| `音效（铃声）：“八点四十分铃声。”` | 时间说明，不是声音 | `音效（铃声）：“叮铃铃——”` |
| `音效（走廊外）：“一声短促的闷响。”` | 描述性句子，不是声音本体 | `音效（走廊深处）：“咚……咚。”` |
| `音效（苹果）：“尖叫声。”` | 声音类别，不是声音表现 | `音效（苹果）：“咿——！”` |
| `音效（教室）：“大家害怕得不敢说话。”` | 叙述误作声音 | 用 `群像画面` 或 `心理反应` 表达 |

### 1.4 对白字段格式

**标题格式**：`对白（真实角色名，语态/状态短语）`

**角色名要求**：`原文角色`、`角色名`、`某人` 只是规则占位，不得作为角色名进入终稿。

**语态/状态短语要求**：
- 可以是声音、呼吸、节奏、情绪、姿态或关系压力
- 例如：`压着笑意`、`声音发颤`、`故作轻松`、`眼里带笑`、`酒意未散`、`冷静得近乎无声`
- 不要求一词，不要求以"地"结尾
- **不得**借该项改变对白含义、人物态度或新增心理解释

## 2. Concrete Visual Rule（具体画面规则）

> **规则来源**：`script-adaptation-contract.md` §Concrete Visual Rule

画面字段必须回答"摄影机看见什么"，而不是"观众理解什么"。

### 2.1 禁止模式

| 模式类型 | 错误示例 | 修复方式 |
|---------|---------|---------|
| **抽象概念** | `规则测试开始`、`压抑气氛继续`、`现实身份被覆盖`、`人数压力`、`底牌` | 转成可见物：黑板文字、系统字、课桌、手指、目光、灯光、屏幕 |
| **解释性因果** | `因为没有人敢缺席`、`第一次意识到人数本身就是规则`、`确认队友候选` | 转成行为链、道具状态、群体反应 |
| **作者判断/主题句** | `这是一场小型审判`、`不做灾害奇观化`、`真正的规则` | 删除，或转为可见压力承托 |
| **心理/感受替代** | `不敢靠近`、`正面摊牌的爽快`、`像在等什么发生` | 转成身体执行：呼吸、眼神、停顿、手指、退半步 |
| **纯章节/任务说明** | `第二节课是课堂作业`、`规则测试开始`、`没有字数限制` | 删除，或改为可见动作/道具 |
| **内部规则泄露** | `本场按上游原文顺序承接`、`说话者的视线`、`不新增事件结果` | 删除，仅作内部约束 |

### 2.2 修复原则

| 类型 | 修复原则 | 示例 |
|------|---------|------|
| 抽象概念 | 转成可见物 | `人数压力` → 每张课桌上方浮着相同编号的黑点 |
| 心理判断 | 转成身体执行 | `不敢靠近` → 他退到门框边缘，手背贴住墙面 |
| 比喻 | 转成具体构图 | `像在等什么发生` → 他盯着屏幕，拇指悬在按钮上方 |
| 任务说明 | 删除或改写为可见动作 | `规则测试开始` → 黑板中央浮出冷白光点 |

## 3. Field Routing Table（字段路由表）

> **规则来源**：`FR-2` 信息量保真

### 3.1 原文内容类型 → 字段映射

| 原文内容类型 | 优先字段 | 处理要求 |
|------------|----------|----------|
| 角色可见动作、位移、手部动作、姿态变化、速度节奏、力度质感 | `角色动作` / `动作画面` | 改写成可拍摄的短句；只写客观动作、神态、语气、生理反应 |
| 地点、空间结构、光照状态、承载面、静置物件 | `环境描写` | 只写场景本身的写景材料；可重复出现于环境刷新 |
| 高跟鞋、铃声、机械提示、走廊异响 | `音效` + `音效画面` | `音效` 写声音来源、质感、节奏；`音效画面` 写声音触发时的可见源头和人物反应 |
| 信息载体、规则显影物、关键物件、线索痕迹 | `道具特写` / `规则显影` | 让物件承担可见信息、规则压力或状态变化 |
| 天道系统、好感度、死亡通报 | `系统画面` / `规则显影` / `旁白（系统提示）` | 文字信息由旁白承载，画面字段负责可视化呈现 |
| 群体反应、全班凝固、集体低头 | `群像画面` | 用可见行为表达群体心理 |
| 角色主观判断、恐惧、推理、记忆回闪 | `独白` / `内心独白` / `心理反应` | 优先转成可见身体、表情、道具停点或空间距离 |
| 主角视角下对他人行为的解读、判断 | `内心独白（主角）` + 主角观察证据 | 把"她在试探他"改成主角内心独白 |
| 非引号客观叙事、公共事实、群体共识、关系压力 | 优先可拍字段；必要时按 `narration-to-voice-adaptation-contract.md` 转 `对白` / `独白` / `内心独白` / `旁白` | 先通过 voice owner、知识依据、信息差安全和语音预算；不得改写上游已有对白 |
| 直接情绪感受：恶心、难受、愤怒、害怕、崩溃 | `表情特写` / `心理反应` / `角色动作` | 关键面部变化优先落 `表情特写`；非面部联动、生理反应和身体动作分别落 `心理反应` / `角色动作` |
| 往日常态、重复熟悉、过往背景 | 删除 / `内心独白（主角）` | 非当前主线必要信息删除；必要信息只转成当前可拍证据 |
| 角色目标、阻碍、试探、潜台词 | `对白画面` / `表演提示` | 转成带目的的行为策略：停顿、避视、道具动作 |
| 权力关系、信任变化、压迫位置 | `场面调度` + 具体位置字段 | 用站坐、高低、道具归属表达关系 |
| 对白后的沉默、迟疑、反应 | `表演提示` / `表情特写` / `群像画面` | 默认不新增对白；非引号客观叙事满足专项 gate 时才可派生语音，否则用呼吸、停顿、纸张声消失承托 |
| 转场、未出口对白、下一场压力浮现 | `音效画面` / `道具特写` / `群像画面` / `转场` | 从声音桥、物件状态、动作中断中选择 |
| 国运灾难、新闻碎片、现实世界同步后果 | `现实灾难画面` / `系统画面` | 作为短促压力插针，不挤占主叙事 |

### 3.2 字段路由决策树

```
输入：上游小说原文
│
├─ 是否为上游已有对白/独白/内心独白？
│   └─ 是 → 按 Dialogue Freeze (FR-3) 逐字保留，配对对应 *画面
│
├─ 是否为非引号客观叙事且需要语音化？
│   └─ 是 → 先按 narration-to-voice gate 判断；通过后转 `对白` / `独白` / `内心独白` / `旁白` 并配对 *画面
│
├─ 是否为可拍摄的身体动作？
│   └─ 是 → `角色动作` / `动作画面`
│       ├─ 带有明确速度/力度？→ 必须保留快慢、停顿、力度质感
│       └─ 没有速度信号？→ 只在表演需要时补轻量节奏
│
├─ 是否为心理/情感/判断？
│   └─ 是 → 优先拆分：
│       ├─ 核心可见变化集中在脸部？→ `表情特写`
│       ├─ 需要身体/呼吸/声线/空间/道具联动？→ `心理反应`
│       ├─ 需要角色自己说出口？→ `内心独白（角色）`
│       └─ 需要留给演员执行？→ `表演提示`
│
├─ 是否为主角视角判断他人？
│   └─ 是 → `内心独白（主角）` + 主角观察到的客观证据
│       └─ 禁止写成客观第三方概括
│
├─ 是否为群体行为/反应？
│   └─ 是 → `群像画面`（用可见行为表达）
│
├─ 是否为道具/系统/规则信息？
│   └─ 是 → `道具特写` / `系统画面` / `规则显影`
│
└─ 是否为环境/氛围/空间？
    └─ 是 → `环境描写`
        ├─ 场景开篇？→ 建立地点、光线、氛围
        └─ 环境焦点变化？→ 环境刷新（同一 slugline 内可重复）
```

## 4. Action Field Dynamics（动作字段动态）

> **规则来源**：`FR-6` 动作字段纯度

### 4.1 动作维度矩阵

| 维度 | 内容 | 示例 |
|------|------|------|
| **身体部位** | 手、眼、肩、背、脚步、呼吸、重心 | 手指收紧、后背僵住、脚步顿住 |
| **位移与空间** | 靠近、后退、转身、扑倒、掠过、贴墙、越过门槛 | 退到门框边缘、挪近半寸 |
| **姿态变化** | 站起、伏低、僵住、偏头、俯身、半跪 | 僵在原地、下颌绷住 |
| **速度节奏** | 骤然、猛地、飞快、迟半拍、慢慢、缓缓、停半拍 | 骤然转身、缓缓收进袖中 |
| **力度质感** | 压住、攥紧、松开、砸下、轻碰、拖拽、收住 | 指节发白、攥紧袖口 |

### 4.2 禁止词清单

| 禁止词 | 原因 | 处理方式 |
|--------|------|----------|
| `试图`、`想要`、`打算`、`意图`、`准备借此` | 主观预判/心理意图 | 改入 `内心独白（角色）` 或改成可见动作 |
| `为了让/为了掩饰`、`想借此` | 目的说明 | 若承载主角内心，应改入主角内心独白 |

### 4.3 上游速度信号处理

| 情况 | 处理规则 | 示例 |
|------|----------|------|
| 上游明确写出速度/力度 | **必须保留** | `"鞭影一闪"` → `角色动作：雪千寻腕口一抖，鞭影骤然掠过甲板` |
| 上游没有速度信号 | 只在表演/悬念/危险需要时补轻量节奏 | 普通动作不添加"飞快"等强化词 |
| 快慢必须服务人物状态或场景压力 | 快速=袭击/逃离/反应失控；慢速=迟疑/压迫/决定落下 | - |

### 4.4 对比示例

| 不合格 | 合格 |
|--------|------|
| `角色动作：林寂试图压住怒火，想要装作没听见。` | `角色动作：林寂把纸角抚平两次，第二次指节压得发白；他抬眼时下颌绷住，没有接话。` |
| `动作画面：雪千寻攻击忍者。` | `动作画面：雪千寻腕口一抖，鞭影骤然掠过甲板；忍者还未转身便扑倒在月光里。` |
| `角色动作：他把关键物件收进袖中。`（丢失"缓缓"的决定感） | `角色动作：他沉默良久，缓缓将关键物件折入袖中，指尖在袖口停了半拍。` |

## 5. Environment Field Purity（环境描写纯度）

> **规则来源**：`FR-2` 信息量保真

### 5.1 环境描写定义

`环境描写` 只写场景本身的可见写景材料：
- 地点、空间结构、自然景物
- 光照状态、空气介质、承载面
- 围护面、开口边界、静置物件
- 整体氛围

**服务对象**：美术、置景、情绪烘托、场景进入感

**禁止承载**：剧情推进、人物动作、对白引出、心理解释、规则说明

### 5.2 环境刷新规则

`环境描写` 不限定一场只出现一次。同一 slugline 内以下情况可刷新：

| 刷新触发 | 示例 |
|----------|------|
| **空间层次变化** | 从室内移动到门廊、窗边、院中、楼梯口 |
| **视野背景变化** | 画面背景从主厅转到墙角、案边、窗外 |
| **光线/空气变化** | 日光被云遮住、火光压低、雾气加重 |
| **材质/承载面变化** | 地面、桌面、墙面、甲板成为新的情绪承托面 |
| **情绪背景变化** | 自然景物、静物、空位、阴影随剧情压力显出新氛围层 |

**限制**：
- 不因每个动作或每句对白机械重复
- "环境刷新"不得替代 `角色动作`、`对白画面`、`群像画面`、`道具特写`
- 若地点或日夜已构成真实场景变化，应按 slugline 规则另开场景

### 5.3 允许与禁止

**允许写**：
- 自然景物：飘雪、落叶、朝露、风沙、雨丝、雾、烟、尘
- 光照状态：日光、月光、烛火、灯管、屏幕光、阴影边界
- 承载面：地面、甲板、桌面、墙面、门窗、舱门
- 静置物件：家具、光源、痕迹、纸张在空间中的状态

**禁止写**：
- 人物动作：咬、凑近、握住、说话、收袖、拔刀、奔跑
- 对白引出句：`他小声说：`、`她低声对他说：`
- 剧情结果、背景概要、任务说明
- 关系结论或心理判断：`两人关系变了`、`众人被震慑`

### 5.4 拆分规则

若一句同时包含写景和人物动作，必须拆分：

```md
环境描写：海雾贴着船舷，湿木板上浮着一层淡淡盐霜。
角色动作：令狐冲咬下一口番薯，被热气烫得偏过头，仍往任盈盈身边挪近半寸。
对白（令狐冲，声音压低）：“<上游原句>”
对白画面：任盈盈别过脸，没有接话；她的指尖在袖下收紧，扣住令狐冲的手腕。
```

## 6. Psychological Reaction Field（心理反应字段）

> **规则来源**：`FR-7` 心理反应字段锚定

### 6.1 核心定义

`心理反应` 不是内心解释字段。它只能写观众可通过画面、声音或演员表演接收到的反应。

### 6.2 可接收反应清单

| 反应类型 | 具体表现 |
|---------|----------|
| **身体微动作** | 喉结滚动、手指颤抖、嘴角抽动、身体后退半步 |
| **表情变化** | 眉心收紧、瞳孔放大、眼皮跳、嘴角抿紧 |
| **呼吸** | 吸气未满、屏住呼吸、呼吸变浅变急 |
| **停顿** | 动作中断、手停在半空、话卡在嘴边 |
| **声线变化** | 声音发紧、发颤、变轻、变冷、沉默 |
| **道具停点** | 手停在物件上、指尖压住纸角、眼睑低垂 |
| **空间距离** | 退后、不自觉靠近、被群体目光压住 |
| **对手不接话** | 沉默、回避、转移话题、用动作填补空白 |
| **短促可听独白** | 吞咽声、深吸气、压抑的叹息 |

### 6.3 禁止模式

| 禁止 | 错误示例 | 修复示例 |
|------|---------|---------|
| 抽象内视 | `心理反应：他意识到自己拿到了底牌。` | `心理反应：林寂的目光从教鞭移到第一排同学僵直的肩背，手指慢慢离开桌沿。` |
| 文学比喻 | `心理反应：她的内心像坠入无边黑暗。` | `心理反应：她吸了一口气没有吸满，手背贴住杯壁，杯中水面轻轻晃了一下。` |
| 直接主观情绪 | `心理反应：林寂感到恶心。` | `心理反应：林寂喉头轻轻一滚，嘴角压住，手背抵到唇边又放下。` |
| 无主体结论 | `心理反应：因为没有人敢在课堂时间缺席。` | 改入 `独白` 或 `群像画面` |

### 6.4 主体要求

- 必须写明确主体的可感知反应
- 不写无主体心理结论
- 主角视角下对他人行为的判断优先进入 `内心独白（主角）`

## 7. Facial Expression Close-Up Field（表情特写字段）

`表情特写` 是编剧阶段给下游表演、摄影和图像链路预留的正式面部表演落点。它只在关键情绪触发、压抑情绪泄露、认知翻转、未出口对白、羞耻/恐惧/心虚/强忍等需要面部可见变化承接时使用；不是每句对白、每次心理活动都必须输出。

### 7.1 字段定义

| 维度 | 写什么 | 不写什么 |
| --- | --- | --- |
| 触发来源 | 当前上游事件、对方一句话、系统信息、道具状态或沉默压力引发的面部变化 | 无来源的“突然难过/突然愤怒” |
| 面部分区 | 眉、眼、眼睑、眨眼频率、鼻翼、嘴角、唇线、咬肌、下颌、喉头、皮肤状态 | 泛泛“表情悲伤/表情愤怒/表情复杂” |
| 瞬间形态 | 情绪刚裂开、刚压回去、笑意停住、眼神闪开半拍、嘴角收住 | 长时间维持的模板脸或表情轮播 |
| 边界 | 只写脸部及紧邻面部的生理细节；必要时一句点明触发 | 摄影机位、景别、推拉摇移、灯光方案、剧情解释 |

### 7.2 与相邻字段的分工

| 情况 | 优先字段 |
| --- | --- |
| 台词说法、气口、断句、对手反应、手部和身体距离更重要 | `对白画面` |
| 情绪需要身体、呼吸、声线、道具停点或空间距离共同承托 | `心理反应` |
| 关键变化集中在眉眼嘴、咬肌、下颌、喉头或眨眼频率 | `表情特写` |
| 角色正在做可见动作或位移，表情只是动作的一部分 | `角色动作` / `动作画面` |

### 7.3 使用限制

- 不新增剧情事实、人物动机、对白含义或未授权情绪；必须能回指上游文本或当前声画上下文。
- 不把 `表情特写` 写成摄影指令；`特写` 是字段标题，不授权提前写机位、景别、焦段或分镜。
- 不用作万能心理解释字段；如果正文离开字段标题后仍只是在说“他很痛苦/她不信任”，视为失败。
- 多人场景不让每个人都输出同强度 `表情特写`；优先给 action driver 或 reaction receiver。

### 7.4 对比示例

| 不合格 | 合格 |
| --- | --- |
| `表情特写：她很悲伤。` | `表情特写：她的嘴角先抬起一点，又在听到最后两个字时收住；下眼睑泛红，眨眼比刚才慢半拍。` |
| `表情特写：他露出愤怒的表情。` | `表情特写：他的咬肌缓慢绷起，唇线压平，视线没有移开，只在对方停顿时眨了一下。` |
| `表情特写：镜头推近他的脸。` | `表情特写：他吸气只到一半，鼻翼轻轻动了一下，眼神从对方脸上错开半寸。` |

## 8. Performance Field（表演字段）

> **规则来源**：`script-adaptation-contract.md` §Dramatic And Performance Projection

### 8.1 表演提示要求

`表演提示` 在规划层必须可执行：
- 写演员可做的身体任务、目光任务、呼吸、停顿、动作力度和对手关系
- 终稿应优先拆入具体 `对白画面`、`角色动作`、`表情特写`、`群像画面` 或 `心理反应`
- **不得**在场景末尾总结式列出

### 8.2 禁止模式

| 不合格 | 原因 | 合格 |
|--------|------|------|
| `表演提示：角色想要...外显为...` | 总结式列条 | 拆入具体动作/画面字段 |
| `表演提示：他似乎拿到一张别人没有的底牌...` | 小说式抽象 | `表演提示：林寂不抬头，指腹压住课本边角，眼神保持余光警戒。` |
| `表演提示：林寂不信任陈大雷。` | 心理标签，演员不可执行 | `表演提示：林寂接过纸条前停半拍，先看陈大雷的手，再看纸条背面。` |

### 8.3 微表情与身体联动

| 元素 | 进入字段 |
|------|----------|
| 微表情 | `表情特写`、`对白画面`、`心理反应` |
| 身体联动 | `角色动作`、`动作画面`、`对白画面`、`心理反应` |
| 环境声音 | `音效`、`音效画面`、`环境描写`、`群像画面` |
| 情绪动机 | 作为规划证据、主角内心独白或可见反应进入终稿；**不得**写成动作字段主观意图 |
| 微动态限制 | 只能写停顿、压住、收回、变轻、变慢或局部变化；**不得**写摄影慢镜、推镜 |

## 9. Scene Title Contract（场景标题契约）

> **规则来源**：`FR-4` 场景标题规范

### 9.1 格式要求

```md
### 场景1：内景 永夜私立中学二年级A班教室 - 日
### 场景2：外景 学校操场 - 夜
```

### 9.2 判断规则

| 情况 | 新场景判断 | 说明 |
|------|-----------|------|
| 日/夜不同 | ✅ 新场景 | 同一地点，白天和夜晚是不同的场景 |
| 真实地点/空间范围变化 | ✅ 新场景 | 从教室到走廊、从室内到室外 |
| 规则宣告 | ❌ 同场景 | 只改变叙事状态，不改变物理空间 |
| 角色入场 | ❌ 同场景 | 同 slugline 连续 |
| 公告/能力觉醒 | ❌ 同场景 | 叙事 beat 变化，不构成新场景 |
| 连续 slugline | 只首次写标题 | 后续 beat 直接接正文 |

### 9.3 转场规则

转场必须服务因果、压力或信息推进。**视线动作不得成为转场的默认承托**。

| 转场方式 | 说明 | 示例 |
|----------|------|------|
| 声音桥 | 上一个场景尾声音与下一个场景声音衔接 | `音效画面：铃声渐弱...` + `音效（走廊外）：“...”` |
| 物件状态 | 物件动作串联两个场景 | `道具特写：名单被折进袖口...` + 新场景 `角色动作：他把名单展开` |
| 动作中断 | 在一个场景中被打断，切入下一个场景 | `角色动作：他正要开口，广播突然响起...` |
| 空间阻隔 | 门关上、窗帘拉上、阴影吞没等 | `环境描写：门在身后关上，走廊灯一盏盏亮起。` |
| 环境刷新 | 不推荐视线望向远方 | 改用 `环境描写：窗外的欢呼声一点点散开` |

## 10. Dialogue Freeze（对话冻结）

> **规则来源**：`FR-3` 对话冻结

### 10.1 基本规则

- 上游已有对白必须逐字保真
- 不做润色、删改、同义替换、语序重排
- 不把小说叙述句、作者判断、概括说明随意改写成新对白；非引号客观叙事只有通过 `narration-to-voice-adaptation-contract.md` 的 source-grounded gate，才可形成派生语音

### 10.2 语态/状态短语处理

**来源**：上游已有或上下文可确认的说话方式与角色状态

**示例**：
- `对白（林寂，声音发颤）：“我看见他了。”`
- `对白（阿真，眼里带笑）：“别闹。”`
- `对白（陈大雷，故作轻松）：“这没什么大不了的。”`

**限制**：
- 不要求一词
- 不要求以"地"结尾
- **不得**借语态改写对白含义或新增人物心理

### 10.3 Long Dialogue Beat Segmentation（长对白节拍拆分）

当上游小说原文中同一说话者存在单段长对白时，`2-编导` script layer 必须在对白冻结前提下建立 `long_dialogue_beat_map`。这不是改写台词，而是把同一段原文对白拆成可演、可拍、可被下游摄影承托的连续节拍。

**触发建议**：
- 30 字以内且只有一个明确动作的对白可保留为单句。
- 30-80 字若含转折、条件、追问、威胁或情绪变向，拆成 2-3 个节拍。
- 80-160 字通常拆成 3-5 个节拍。
- 160 字以上必须输出 `long_dialogue_beat_map`，并检查是否有非说话者反应、动作停点或环境声承托。

**硬规则**：
- 每个节拍的 `exact_text_segment` 必须是上游对白的连续原文片段；所有节拍按顺序拼接后必须逐字等于上游对白。
- 不在专名、数字、固定称谓、成语、未完成因果短语或依存子句中间断开。
- 优先在语义动作变化处断开：试探转施压、解释转请求、铺陈转揭示、伪装转泄露、命令转威胁。
- 优先在表演气口处断开：呼吸换气、短暂停顿、吞咽、笑意收住、对手不接话、手部动作停点。
- 若上游明确要求一口气说完，仍可建立节拍图，但标记 `breath_or_pause_hint: continuous/no_pause`，由 `2-编导` performance layer 处理连续气口。
- 每个节拍必须有就近 `对白画面` 或相邻 `表情特写`、`心理反应`、`群像画面`、`道具特写` 承托；不得让整段长对白共用一个泛化画面。
- 每 2-3 个长对白节拍至少出现一次非说话者信息：听者反应、群像压力、手部动作、空间距离、道具状态、环境声或沉默余波。

**禁止模式**：
- 将长对白整体压成一行，后面只接一条“说话者看着对方说完”的 `对白画面`。
- 为了断句而润色、删词、换词、补充台词或重排语序。
- 把“长对白需要多机位”等摄影指令写入 `2-编导` script layer 正文；编剧阶段只提供节拍、气口、反应和可见承托，镜头方案交给 `4-摄影`。

## 11. Field Examples（字段示例）

### 11.1 Good Patterns

```md
音效（高跟鞋）：“嗒、嗒、嗒。”
音效画面：教室前门外，红色细高跟鞋的影子压过门缝；三十多名学生同时停住动作，看向门口。

场面调度：苏红叶站在讲台内侧，教鞭横在讲桌边缘；林寂坐在第三排靠窗，前排同学的肩膀把他半遮住。

对白（苏红叶，微笑保持不动）：“各位同学，早上好。”
对白画面：苏红叶站在讲台前，教鞭末端的红珠停在桌沿，嘴角保持完美到不自然的微笑。

内心独白（林寂）：“她发现了。”
内心独白画面：林寂低着头，后背的校服布料被冷汗洇深；苏红叶的视线从他身上移开。

心理反应：林寂盯住黑板上最后一行规则，喉结轻轻动了一下，压在课本边缘的手指慢慢收紧。

表情特写：林寂的眼睑停住半拍，嘴角压平，视线从黑板最后一行规则移开时没有立刻落回课本。
```

### 11.2 Bad Patterns 修复对照

| 不合格 | 问题 | 合格 |
|--------|------|------|
| `系统画面：规则测试开始。` | 任务说明，不是画面 | `系统画面：黑板中央浮出"课堂纪律条例"标题前的冷白光点。` |
| `环境描写：第二节课是课堂作业。` | 课程说明，不是环境 | `动作画面：苏红叶把白纸发到每张课桌，纸面中央印着题目。` |
| `对白画面：陈大雷语气压低，却有一种正面摊牌的爽快。` | 感受性判断 | `对白画面：陈大雷身体前倾，手掌摊开压在桌面，眼睛直视林寂。` |
| `对白画面：她在试探林寂。` | 潜台词结论 | `对白画面：她把问题说到一半停住，视线落在他握紧的手指上，等他先补完后半句。` |
| `环境描写：苏红叶掌控全场。` | 权力关系结论 | `场面调度：苏红叶站在讲台内侧，教鞭横在讲桌边缘；第一排学生坐得笔直，没人越过桌线。` |
| `环境描写：这种压抑的课每天都重复，所有人早已熟悉。` | 无法实拍的往日常态 | `群像画面：前排学生没有抬头，笔尖停在同一行空格上。` |
| `道具特写：这把刀来自他多年前的旧事。` | 背景说明混入道具 | 删除或 `内心独白（主角）：“这把刀...我记得...”` |
| `对白（苏红叶，微笑保持不动）：“欢迎来到这里。”她微笑着看向全班。` | 引号内混入动作 | 拆分：对话 + `对白画面：苏红叶...微笑着看向全班。` |
| `对白画面：说话者的视线、手部动作、身体距离和对手反应就近承托对白；引号内不加入动作。` | 规则说明泄露 | 删除，改写为具体可见承托 |
| `表情特写：她露出难过的表情。` | 情绪标签，不是可见面部变化 | `表情特写：她嘴角抬到一半停住，下眼睑微红，吸气后没有立刻开口。` |
