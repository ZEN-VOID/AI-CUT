# Volume Rhythm Framework

## Purpose

给 `2-章节规划` 用的整书节奏骨架参考。

这里不把 `Save the Cat` 当成死板百分比尺子，而是把它转成适合长篇/连载的：

- `beat checkpoints`
- `volume waves`
- `episode rhythm roles`

目标是让 Step 2 在“锁容器”时顺手锁住整部书的大呼吸，避免 Step 3 以后才发现中段塌、尾段赶、B Story 漂。

## Core Rule

`Save the Cat` 在本技能里只负责回答五件事：

1. 读者最先被什么 promise 勾住。
2. 主角何时真正进入“新游戏规则”。
3. 中段靠什么改向，而不是只加码。
4. 何处必须让局势见底。
5. 尾段如何加速兑现，而不是一路疲软到结尾。

## Beat Corridor Map

| beat corridor | serialized planning question | Step 2 需要锁什么 |
| --- | --- | --- |
| `Opening Image + Setup` | 作品一开始承诺的世界、人物缺口和读感是什么 | 开篇卷的 promise、主角缺口、第一卷不能缺席的场域与关系 |
| `Catalyst` | 哪个事件让故事不可回头 | 不可逆启动事件落在哪个卷段/章节走廊 |
| `Debate -> Break Into 2` | 主角何时停止观望，正式入局 | 从“被动卷入”到“主动推进”的切换点 |
| `B Story` | 谁来承载主题与镜像变化 | 关系线、副线、导师线或镜像角色线 |
| `Fun and Games` | 前半部 promise 如何真正交付 | 哪几卷负责“兑现卖点”而不是光铺设定 |
| `Midpoint` | 中盘怎样改写游戏规则 | `false victory / false defeat`、目标升级、代价放大 |
| `Bad Guys Close In` | 压力如何系统性回收 | 外部反压、内部裂缝、时间债、资源债 |
| `All Is Lost -> Dark Night` | 整书最低点怎么形成 | 最低点走廊、主角失去什么、旧自我如何破裂 |
| `Break Into 3 -> Finale` | 尾段靠什么加速而不赶工 | 新方案、集结、反打、兑现顺序、回收清单 |
| `Final Image` | 读者最终看到怎样的“after” | 终局镜像、人物/世界变化的可视落点 |

## Step 2 Output Hints

### 1. `macro_rhythm_scaffold`

至少应含：

- `theme_carrier`
- `b_story_carrier`
- `beat_checkpoints`
- `volume_wave_distribution`
- `midpoint_shift`
- `all_is_lost_corridor`
- `finale_acceleration_rules`

### 2. `volume_wave_distribution`

每卷至少回答：

- 这一卷主要是 `promise / expansion / reversal / squeeze / payoff` 哪一种 duty
- 卷内 1-3 个主动作
- 卷尾留下什么 `exit_hook`

### 3. `episode_rhythm_roles`

不是给每一集写详细事件，而是给关键集一个职责标签，例如：

- `hook-entry`
- `promise-delivery`
- `midpoint-turn`
- `pressure-squeeze`
- `false-victory`
- `false-defeat`
- `all-is-lost`
- `rebuild`
- `final-run`
- `payoff-close`

## Anti-Flatness Checks

- 若前 20% 还没有真正把主角推入新局，说明 `Catalyst / Break Into 2` 太弱。
- 若中段只有“更大、更强、更多敌人”，但玩法没变，说明 `Midpoint` 不成立。
- 若尾段只是把前面开的坑排队回收，没有“新方案”或“主角已变”，说明 `Break Into 3` 不成立。
- 若 B Story 只在局部漂亮，和主题、转变、最低点无关，说明副线未承担真正节奏职责。
- 若每卷都在做同一种 duty，说明 `volume wave` 失效，整部书会显得平。
