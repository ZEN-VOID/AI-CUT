# 节奏优化与钩子刺激引擎

本文件服务 `漫画剧本改编` 的钩子、爽感、刺激性和快节奏推进，不与主 `SKILL.md` 竞争真源。

## 1. 核心原则

快节奏不等于快进摘要，而是：

- 更晚进入场景
- 更早切断场景
- 更明确地把信息分层释放
- 更稳定地让每场承担一个“抓停 / 抬升 / 急停”职责

## 2. 三种驱动模式

| `pacing_drive` | 用法 | 典型风险 |
| --- | --- | --- |
| `stable` | 适合成长、温压、慢悬疑 | 容易前段发平 |
| `punchy` | 默认推荐；抓停频率高，兼顾角色和刺激 | 场景过多时容易拥挤 |
| `relentless` | 高频急停、强钩子、强炸点 | 容易失去情绪回收和代价感 |

## 3. 重排工具箱

| 工具 | 作用 | 什么时候用 |
| --- | --- | --- |
| `前置炸点` | 先用名场面抓人，再倒回解释 | 原作前段铺垫过长 |
| `并戏` | 合并功能重复的桥段 | 多场都只在解释同一件事 |
| `后移解释` | 把背景说明拆散到后文 | 当前页最需要的是刺激而不是说明 |
| `压缩过桥` | 用一句话或一个动作跨掉无效连接 | 场景之间只有流程、没有戏 |
| `中段抬升` | 在中段插入一波关系或危险升级 | 全部高潮都挤到结尾 |
| `章末急停` | 在最贵的动作或半揭真相处收束 | 收尾像总结陈词 |

## 4. 爽感触发器

最稳的“爽感”通常来自以下几类，而不是单纯大喊大叫：

- 压制被打破
- 权力关系翻转
- 隐藏信息突然显影
- 危险离脸更近一寸
- 角色做出了本该不敢做的选择
- 系统/规则/力量突然兑现，但带着代价

## 5. 单场节奏模板

### 模板 A：抓停型

`直接进入异常或冲突 -> 用 1-2 格确认关系和代价 -> 停在未完成动作`

### 模板 B：抬升型

`延续上一场压力 -> 给一个短暂错误安全感 -> 立刻把代价抬高`

### 模板 C：急停型

`蓄势 -> 半揭 -> 不解释完 -> 断掉`

## 6. 章末钩子与页末钩子的区别

- `页末钩子`
  - 服务翻页。
  - 可以是一个动作没完成、一个眼神没解释、一个门还没开完。
- `章末钩子`
  - 服务追更。
  - 更适合身份错位、真相半揭、代价升级、危险逼近。

## 7. 避坑

以下症状说明“快节奏”已经写坏：

- 所有场都很急，但没有峰谷差。
- 全文一直在喊，却没有一个真正让人屏住呼吸的静默格。
- 信息都在旁白里一次性说完，导致动作只剩装饰。
- 每章都用同一种钩子，读者很快会预测。

## 8. 自检

1. 前 3 场里是否至少有一个明确抓停点？
2. 中段是否至少有一次关系或危险的明显抬升？
3. 章末是否停在“还没完成”的瞬间，而不是“已经总结完”的瞬间？
4. 如果删掉一段解释，这一章会不会反而更顺？
