# Reader Experience Simulation Contract

本文件是 `story2026` 的读者体验模拟合同。它在传统写作技术维度之外，从读者视角建立代入感、期待管理和信息饥渴度三个评估维度，供验收时参考。

本文件不拥有正文写权，不是独立阶段。评估维度以诊断参考方式嵌入 `3-初稿` 和 `4-润色` 验收包。

## Core Principle

好的小说不只是"写得好"，更是"让读者读得进去、不愿放下、惦记着下一章"。读者体验模拟不是替代写作技术验收，而是补充"读者视角"的盲区——技术验收通过但读者读不进去的作品，质量不算真正合格。

## Three Reader Experience Dimensions

### 1. 代入感评分（Reader Immersion Index）

评估读者能否代入主角视角、身临其境。

| 分数 | 锚点描述 |
|------|----------|
| 0-2 | 第三人称叙事距离过远，读者始终是"旁观者"；主角的感知被省略/概括化 |
| 3-4 | 主角的视角存在但断断续续；关键场景能感知但日常场景跳回客观叙事 |
| 5-6 | 主角视角基本连贯；能感知到主角的情绪和身体反应；但心理活动有时过于抽象 |
| 7-8 | 主角视角强且自然；读者能"通过主角的眼睛看世界"；感知颗粒精细，细节选择有意义 |
| 9-10 | 读后很难意识到这是"一本书"——像是直接经历了主角的体验；五感+情绪+认知无缝融合 |

**检查要点**：
- 关键场景是否通过主角的感官（视觉/听觉/触觉/嗅觉/体感）呈现
- 心理活动是否由具体刺激触发而非凭空展开
- 叙事距离是否在可接受的POV范围内（不跳出主角视角做上帝评论）
- 读者是否知道主角"此刻"的身体状态（冷/热/饿/痛/紧张）

### 2. 期待管理评分（Anticipation Management Index）

评估章末钩子是否自然牵引下一章，读者是否"惦记着下一章"。

| 分数 | 锚点描述 |
|------|----------|
| 0-2 | 章末无钩子或钩子生硬（突兀断章、为悬念而悬念） |
| 3-4 | 有钩子但力道不足，读者可以放下不继续读 |
| 5-6 | 钩子自然有效，读者会好奇接下来发生什么；但力道中等 |
| 7-8 | 钩子从本章压力中自然生长，读者放下后会惦记："后来怎么样了？" |
| 9-10 | 章末钩子 + 角色命运关心 + 世界观疑问 — 三重牵引让读者迫切想继续 |

**检查要点**：
- 章末钩子是否从本章真实压力/悬念/关系张力中生长（非外力强加）
- 是否存在多项未解疑问（不限于"下一步会怎样"，还包括"XX关系会怎样/XX秘密是什么"）
- 钩子是否对人物/关系/世界观有真实影响潜力

### 3. 信息饥渴度评分（Information Hunger Index）

评估读者对"想知道更多"的渴望程度。

| 分数 | 锚点描述 |
|------|----------|
| 0-2 | 读者对任何未解信息都没有好奇心；故事没有建立认知落差 |
| 3-4 | 有基本的"接下来会怎样"的好奇，但缺少"真相是什么"的深层渴求 |
| 5-6 | 读者对主线谜题/角色秘密/世界观有中等好奇；信息释放节奏合理 |
| 7-8 | 多层信息饥渴：即时悬念（短期）+ 角色谜题（中期）+ 世界观真相（长期） |
| 9-10 | "停不下来"的信息饥渴——每个答案带来两个新问题；读者在章节间隙还在猜测/推理 |

**检查要点**：
- 是否有至少一个"我想知道"的问题在读者脑中悬而未决
- 信息释放是否"给一点、藏两点"（不是全揭、也不是全藏）
- 是否有让读者推理/猜测的空间（不要所有信息都由叙事者直接告知）

## Integration

### 在验收包中

```json
{
  "reader_experience_simulation": {
    "immersion_index": 0-10,
    "anticipation_index": 0-10,
    "information_hunger_index": 0-10,
    "composite_score": 0-10,
    "key_findings": [],
    "warnings": []
  }
}
```

### 评估时机
- `3-初稿/N6-AUTO-ACCEPTANCE`：初稿验收时作为参考维度
- `4-润色/P5-AUTO-ACCEPTANCE`：终稿验收时作为参考维度
- 三个维度为 **诊断参考**，不计入 PASS/FAIL 判定；score < 3 时发出 WARN

### Token Budget
- 评估时不需要额外加载文件
- 评估基于当前验收已有的正文和 planning 信息
- 输出控制在 300 tokens 以内
