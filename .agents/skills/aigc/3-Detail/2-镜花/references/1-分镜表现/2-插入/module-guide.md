# 2-插入 模块说明

## 定位

- 本叶子不只是“把标签塞进句子里”，而是把 `1-切换` 已经锁定的镜数预算、秒位窗口和节拍判断，压回 `1-水月` prose，形成 **可被父级 `3-Detail` 聚合、也可被下游技能包继续抽镜** 的 shot-local prose spine。
- 它的直接职责是：把每个 `[分镜N 第X秒-第Y秒]` 标记插到最自然、最可读、最可继续结构化的位置。
- 它不负责重新决定镜头数量，不负责提前做 `3-构图` 的 descriptor 选择，也不负责直接输出 `5-Image / 6-Video` 请求。

## 技能包级对齐

`2-插入` 的判断不能只夹在 `1-切换` 与 `3-构图` 两个模块之间看，还必须同时看上下游技能包。

### 上游技能包

- `1-水月`
  - 给当前组的动作链、情绪链、空间关系和关系张力。
  - `2-插入` 的每个标记都必须能回指到 `1-水月` 已有的一句、一拍或一段动作。
- `2-Global`
  - 只提供风格温度、类型打法、设计倾向。
  - 它可以影响“哪里更值得显化”，不能影响“这里发生了什么”。

### 当前父技能包

- `3-Detail`
  - 父技能后续要把 `1-水月 + 2-镜花` sidecar 汇回 `projects/aigc/<项目名>/3-Detail/第N集.json`。
  - 所以 `2-插入` 不能只求文气顺，还要让每个标记段将来足够支持父层补 `分镜明细[]`。

### 下游技能包

- `4-Design`
  - 它最终消费的是 `3-Detail/第N集.json` 的镜头事实，而不是这里的 prose。
  - 因此插入后的段落要让人物、空间、动作主轴清楚，不能把镜头窗口写成只剩文气。
- `5-Image / 1-提示词蒸馏`
  - `分镜故事板` 需要稳定的组级镜头顺序与镜头窗口。
  - `分镜帧` 需要能清楚回链到单一分镜的动作焦点。
- `6-Video / 1-提示词蒸馏`
  - `全能参照` 需要按整组镜头顺序保留镜头控制项。
  - `首帧参照` 需要某个分镜在 prose 层有清楚的可见剧情桥段与时间锚点感。

结论：

- `2-插入` 真正要做的是把 `1-水月` prose 变成 **既能顺读、又能抽镜、还能回写 JSON 的中间骨架**。

## 当前真正任务细化

本叶子至少要同时解决四件事：

1. 让 `1-切换` 给出的每个镜头窗口，在 `1-水月` 中找到唯一、自然、可解释的落点。
2. 让每个标记都承接一个清楚的 `动作/视线/停顿/揭示` 任务，而不是只承接一个语法停顿。
3. 让插入后的句段仍保留 `1-水月` 的主行动线、主情绪和主空间关系。
4. 让后续父层和下游技能包能据此继续提取：
   - 当前镜的动作焦点
   - 当前镜承接的关系或视线
   - 当前镜的大致视觉任务
   - 当前镜与前后镜的顺序关系

## 具体创作方法

### 1. 先把“镜头预算”翻译成“可插入窗口”

不要直接从句号、逗号或换行处找插入点。先做一层翻译：

- `1-切换` 给的是：
  - `preferred_shot_count`
  - `shot_time_ranges`
  - `candidate_beat_segments`
  - `switching_rationale`
- `2-插入` 要把这些预算翻成：
  - 这镜承接哪一个动作阶段
  - 这镜落在谁的视线或反应上
  - 这镜是动作推进、反应着陆、信息揭示，还是节奏停顿

换句话说，`2-插入` 找的不是“句子切口”，而是“镜头职责切口”。

### 2. 逐镜锁“插入类型”

每个插入点至少先判一种主类型：

- `动作转折`
- `视线切点`
- `反应着陆`
- `揭示落点`
- `节奏停顿`
- `尾钩预映`

只有先知道这镜为什么独立，插入点才不会变成机械断句。

### 3. 先保水月承接，再谈句面顺滑

每个标记都要先回答：

- 它承接 `1-水月` 的哪一句或哪段动作
- 这句里真正应该被镜头显化的是什么
- 如果轻改 prose，是为了让镜头更清楚，还是为了把原句改得更漂亮

若后者占主导，说明已经越权。

### 4. 把插入点做成“可抽镜块”

合格的插入结果，必须让每个分镜标记后的句段具备最小 shot-local 可抽取性。至少应让后续读取者看得出：

- 当前镜主要拍谁或拍什么
- 当前镜里发生的关键动作或关键停顿是什么
- 当前镜与前后镜相比，观看重心为什么变化

这一步是给父层 `3-Detail` 汇回 `第N集.json`，以及给 `5-Image / 6-Video` 继续做 prompt 蒸馏留抓手。

### 5. 轻改 prose 只做三类动作

- 把一个过长复句拆成更适合镜头落点的自然承接
- 把本来隐含的动作先后顺序写得更清楚
- 把本来被修辞包住的视线、反应或揭示点轻轻显化

禁止：

- 把原本没有的动作补出来
- 把原本没有的关系转折写出来
- 为了让镜头更“好看”而重写一整段情节

### 6. 末尾做一次下游可消费性检查

在交给 `3-构图` 之前，至少回问三件事：

- 父级 `3-Detail` 之后还能不能把这些标记段稳定转成 `分镜明细[]`
- `5-Image / 分镜故事板` 读到这一组时，是否还能看出镜头顺序与每镜职责
- `6-Video / 全能参照` 或 `首帧参照` 之后抽取时，是否还能看出某镜的时间锚点感、动作焦点和桥段边界

若三者至少有一项说不清，说明插入点还只是“读着顺”，没有成为真正的 handoff 骨架。

## 失真与修正

- 若读起来像机械断句，说明插入点没有服从动作和节奏。
- 若标记脱离 `1-水月` 原句承接，说明没有真正回指原始信息。
- 若一镜只是对应一小段修辞，没有对应到动作焦点、视线切点或反应落点，说明插入点只解决了句法，没有解决镜头职责。
- 若插入后 prose 虽然顺，但父级很难看出这一镜到底要补什么 `分镜明细[]`，说明还没形成可聚合骨架。
- 若 `5-Image / 6-Video` 之后难以看出某镜的桥段边界，说明镜头窗口写得太虚，应回到 `INSERT-N2/N3`。
- 若为了插入而重写了一大段剧情，说明越权了。
