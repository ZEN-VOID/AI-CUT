# 3-构图 模块说明

## 定位

- 本叶子负责把已插入的各分镜补成 `主陪背景 -> 布局 -> 构图方式 -> 焦点/观看路径 -> 镜头描述子槽 -> 空间锚点` 的画面组织骨架。
- 它不负责改镜头数和插入点，也不负责直接替后续摄影或运镜做决定。
- 默认要吸收上游 `3-Detail/references/分镜表现.md` 中 `SB-5 ~ SB-11 / SB-14` 对构图链路的要求，而不是只给几个抽象美学词。

## 分镜表现维度细则

本叶子最少要同时锁住五类镜头描述子槽：

- `景别`
- `镜头属性`
- `镜头框架`
- `镜头类型`
- `镜头视角`

它们不是后续随手补的标签，而是当前镜头为什么这样看的直接证据。构图阶段如果没有把这五槽位锁住，摄影、运镜和转场就会在下游重新猜一次镜头任务。

## 着手方面

先检查这一镜是否已经回答下面这些问题：

- 每个分镜的主体、陪体、背景三层关系是否清楚
- 构图布局是否把叙事重心、负空间与观看路径放在正确位置
- 构图方式是否真的服务当前镜头任务，而不是只追求形式感
- 五个镜头描述子槽是否已经锁定
- 每个分镜最该突出的焦点是什么
- 空间锚点、镜头轴线、主体几何关系是否成立
- 当前段落的 `叙事内核 / Previous End / Current Mission / Next Start / 情绪引导` 如何落成景别与视点节奏
- 当前段落更适合哪种 `景别节奏模板` 与 `POV 策略`
- 命中镜头窗口内，实际景别曲线、心理距离与情感目标是否一致

## 具体创作方法

### 1. 先锁主体、陪体、背景三层关系

每镜第一步不是选美学招式，而是先判断：

- 谁是主体
- 谁是陪体
- 背景承担的是信息、压力还是氛围

如果这三层关系不清，后面的景别、视角、构图方式和观看路径都会漂。

### 2. 再锁叙事重心、负空间和观看路径

构图不是摆得好看，而是安排观众先看哪里、再看哪里、最后停在哪里。

最低要求：

- 主重心落点
- 第二焦点落点
- 入口与出口
- 负空间承担的是压迫、悬置还是释放
- 观看路径是否支持当前镜头任务

### 3. 再把构图方式和镜头描述子槽对齐

构图方式可以用：

- `居中`
- `对角`
- `框中框`
- `三角`
- `压迫遮挡`
- `对称失衡`

但构图方式永远只是服务镜头任务，不是目的本身。

每次选择后，都要立刻回问五个镜头描述子槽是否与之协调：

- 当前景别会不会拉错心理距离
- 当前镜头属性会不会把本应稳定的关系拍得过于主观
- 当前镜头框架是否承接了既定观看路径
- 当前镜头类型是否真的帮助信息组织
- 当前镜头视角是否站在正确的道德与情感立场上

### 4. 用景别节奏模板校准命中镜头窗口

`SB-4` 预判的模板到了 `SB-14` 不能停在抽象层，必须落到命中窗口的逐镜复核。

| 模板 | 节奏曲线 | 适用场景 |
| --- | --- | --- |
| `渐进式` | `远 -> 全 -> 中 -> 近 -> 特 -> 极特` | 逐步揭示，悬疑探索 |
| `冲击式` | `特 -> 远 -> 中 -> 特 -> 全 -> 特` | 情感爆发，高潮时刻 |
| `呼吸式` | `中 -> 远 -> 中 -> 近 -> 中 -> 远` | 常规叙事，平稳推进 |
| `对峙式` | `中 -> 特 -> 中 -> 特 -> 中 -> 全` | 对话交锋，张力对抗 |
| `包围式` | `远 -> 中 -> 近 -> 近 -> 中 -> 远` | 情绪聚焦后释放 |
| `自定义` | `[___]` | 根据叙事需求设计 |

至少要回答：

- 选定模板是什么
- 为什么选它
- 它怎样服务 `叙事内核 / Current Mission / 情绪引导`

### 5. 同时锁 POV 策略

`SB-8` 最少要锁三件事：

- 选定视点
- 道德立场
- 切换策略

可选视点类型：

| 视点类型 | 观众位置 | 情感效果 |
| --- | --- | --- |
| `主角 POV` | 与主角同在 | 代入感、共情、紧张 |
| `对手 POV` | 站在对立面 | 威胁感、理解反派 |
| `旁观者 POV` | 冷眼旁观 | 客观、疏离、审视 |
| `全知 POV` | 上帝视角 | 宏观、命运感、讽刺 |
| `受害者 POV` | 弱者立场 | 同情、恐惧、压迫感 |
| `混合 POV` | 多视点切换 | 复杂、多层次、张力 |

### 6. 最后做心理距离与反直觉检验

`SB-14` 不能只说“整体没问题”，必须按命中窗口逐镜复核：

| 2 | `[___]` | `[亲近/中立/疏离]` | `[___]` | `[是/否]` |
| 3 | `[___]` | `[亲近/中立/疏离]` | `[___]` | `[是/否]` |
| 4 | `[___]` | `[亲近/中立/疏离]` | `[___]` | `[是/否]` |
| 5 | `[___]` | `[亲近/中立/疏离]` | `[___]` | `[是/否]` |

反直觉检验最少包括：

- 特写是否反而造成疏离
- 远景是否反而造成亲近
- 中景是否沦为“安全但平庸”的默认解
- 景别选择是否服务情感目标，而不只是信息传递

## 失真与修正

- 若写成“高级、对称、电影感”等抽象词，说明没有真正给出画面组织。
- 若只写主看点、不写观看路径，说明焦点链没有锁稳。
- 若五个镜头描述子槽迟迟不锁，只写“到摄影阶段再定”，说明 Stage-1 越权外包了核心判断。
- 若构图判断改变了镜头数或插入位置，说明越权。
- 若构图开始发明新空间关系或新轴线，回退到 `1-水月` 的既有空间事实。
- 若近景反而疏离、远景反而亲近，却说不清情绪收益，说明景别和构图没有真正对齐心理距离。
- 若 POV 在组内频繁切换，却说不清切换策略和道德立场，说明视点组织还停在“看上去有变化”的层面。
- 若组级模板预判是 `对峙式`，实际窗口却落成平铺中景，说明 `SB-14` 没有真正复核预判与落地之间的偏差。
