# 系统流题材模板 (System Flow)

> **核心卖点**: 系统面板 + 数据成长 + 规则互动。数字化的成长路径，可视化的爽感体验。

## 1. 核心流派细分

### 任务发布流
- **特点**: 系统发布任务，完成获得奖励——任务=剧情推动器
- **核心爽点**: 任务完成、奖励到账、能力提升
- **创作要点**: 任务要有"不可拒绝"的理由——不是系统强迫，是"拒绝的代价比接受更大"

### 签到打卡流
- **特点**: 每日签到获得奖励，日积月累
- **核心爽点**: 累计奖励、稀有掉落、量变质变
- **创作要点**: 签到不只是"点一下"——每次签到的环境风险不同

### 抽卡/盲盒流
- **特点**: 随机抽取卡牌/能力/道具——不确定性 = 惊喜
- **核心爽点**: 欧皇时刻、稀有品质、搭配组合
- **创作要点**: 抽卡有情绪曲线——非酋累积→欧皇爆发

### 升级加点流
- **特点**: 经验值升级，自由加点——数值成长可视化
- **核心爽点**: 升级快感、属性暴涨、解锁新能力

### 商城兑换流
- **特点**: 积分/金币兑换物品——资源管理
- **核心爽点**: 积少成多、关键兑换、资源博弈

## 2. 系统设计铁律

### 数值膨胀控制阀
- **主角同级无敌但不跨阶**: 一阶极限是秒杀同级，但面对二阶只能保命
- **稀有道具限制数量**: 神话级道具全书不超过3个
- **经验曲线递减**: 越往后升级越慢——从3章一级到30章一级

### 系统与宿主关系三阶段
1. **依赖期**(1-100章): 没有系统活不下去
2. **对等期**(101-300章): 系统和宿主互相成就
3. **超越期**(301章+): 宿主超越系统——最终揭示系统的来源和目的

## 3. 追读心理

| 驱动力 | 执行手段 |
| --- | --- |
| 数据增长 | 每5章至少更新一个关键数据（等级/技能/属性） |
| 抽卡期待 | 抽卡节奏——每10-15章一次；但每次抽卡环境要不同 |
| 任务好奇 | 系统任务=剧情预告——读者通过任务猜到接下来发生什么 |

## 4. AI 写作专项指南

### 常见陷阱
1. **系统水文**: 大段系统面板刷字数 → 面板每次只显示变化的部分
2. **奖励无后果**: 拿到神器后面就不提了 → 每个重要奖励必须至少3次后续回响
