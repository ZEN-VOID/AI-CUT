# Source Fidelity Contrast

用于分析好/坏示例对任务资料、事实来源、输入证据和引用边界的忠实度差异。

## Fixed Checks

- 好示例是否正确使用了任务资料中的关键事实、名称、数量、顺序、范围或限制。
- 坏示例是否遗漏、错引、混淆、过度推断或引入未授权信息。
- 目标 skill 是否明确要求先读资料、保留证据、区分事实与推断。
- review gate 是否能拦截无来源事实、证据漂移或不确定判断。

## Patch Bias

- 资料使用规则优先落 `references/` 或 `steps/`。
- 事实验收优先落 `review/`。
- 若资料来源缺失，输出 `residual_risks[]`，不得把推断写成硬规则。
