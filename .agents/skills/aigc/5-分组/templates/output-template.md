# Output Template

## Output Contract Alignment

### Required output

- `OUTPUT-GROUP-EPISODE`: `projects/aigc/<项目名>/5-分组/第N集.md`
- `OUTPUT-GROUP-REPORT`: `projects/aigc/<项目名>/5-分组/执行报告.md`

### Output format

- 逐集文件：Markdown 分镜组稿。
- 执行报告：Markdown 报告。

### Output path

```text
projects/aigc/<项目名>/5-分组/
├── 第1集.md
├── 第2集.md
└── 执行报告.md
```

### Naming convention

- 逐集分组稿：`第N集.md`
- 报告：`执行报告.md`
- 分镜组 ID：`x-y-z`

### Completion gate

- 输入摄影稿与 `north_star.yaml` 可回指。
- 每组标题后先写当前场景标题行，例如 `场景1：外景 扶桑战船外舷与黑礁 - 夜`；即便未切换场景，新的分镜组也必须重复同一个场景标题；随后写 `入场镜头：`，用自然描述说明入口源、入场触发、镜头/观看位置、主体入画状态和动态关系变化。首组从本组第一个原始分镜可见状态进入；第二组及之后优先从上一组 `出场画面：` 尾钩进入，并结合上一组末帧、上一组尾帧、上一组最后一帧或连接件抵达首态校正，具体复现上一组尾钩末态。场景身份、环境身份、氛围、光影、色彩和声音底色只作为入场路径必要上下文，不得写成旧式静态定场段落；再写 `画面构图：`，并按固定顺序完整输出 `左侧：`、`中间：`、`右侧：`、`前景：`、`中景：`、`背景：` 六个构图分区，把全部可见或需要生成角色、群体、关键道具和场景锚点布置成带形容属性、状态属性、站坐蹲躺靠等姿态、空间方位关系、朝向和关系变化的主体账本；每个分区都必须展示具体可见主体、环境锚点、遮挡、光影材质或空场压力，并写出该分区如何引入、压住、牵引、遮挡、靠近、拉开或改变主体关系；不得使用泛化占位句或静态站位清单，不得在组头额外输出 `角色：` / `场景：` / `道具：` 清单；不得把入场镜头写成单纯人物动作摘要；再写画面属性、三项 north_star 风格整理、正文；正文后、YAML 前必须写 `出场画面：`，记录本组最后一个原始分镜/本组末帧/尾帧的主体末态、空间关系、动作余势、镜头/观看位置和承接锚点，不得写成下一剧情预告或连接件正向提示词；其中第 1 行风格行必须以固定前置词 `视频生成的画面风格，光影和氛围与场景参照图保持一致。需要生成现场物理互动音效、氛围感音效、环境声、自然现象声、动作声，不要生成任何字幕，不要生成背景音乐。` 开头，后接根据当前分镜组从全局风格母稿抽取的匹配风格句，300 字以内。
- YAML `字数统计` 必须计入分镜组标题后的场景标题行、入场镜头、画面构图、构图分区、画面属性、出场画面和分镜剧本正文；这些组头/组尾描述不计入 `时长估算`；YAML fenced block 本身不计入 `字数统计`。
- 每组以 `分镜N（约X秒）` 显式时长累计为主，优先接近 15 秒，通常约 12-18 秒可接受；单组显式时长累计不得超过 18 秒，超过 18 秒必须拆分、重组或回退 `4-摄影` 修复，不能写完整性例外理由放行；字数和对白只作辅助风险复核。
- 每对相邻组之间物理夹放 `## <上一个分镜组ID>~<下一个分镜组ID>`，标题本身作为连接件唯一分镜 ID；标题后先写场景标题行，同场景连接重复同一个场景标题，跨场景连接写成 `场景标题A ➡️ 场景标题B`；随后在 `连接类型` 前写三项 north_star 风格行，且第 1 行以固定全局风格前置词开头并接 300 字以内连接件风格整理句，字段含连接类型、具体连接方法描述、3-4 秒时长、变化过程、主体运动、运镜设计、透视适应和避免元素；连接件消费上一组 `出场画面：` 和下一组 `入场镜头：`，但块内不重复输出组内 `出场画面：` 字段；连接方法不得只写抽象分类名，避免元素只写负面约束，不复述起点/目标端点，不输出 `分镜ID：`、`连接件提示：` 或旧版 `入场画面：`。
- 正文同步 `4-摄影` 原换行且未改写。
- 执行报告同时记录每组 `时长估算`、validator error/warning 与 `boundary_review`、`entry_shot_review`、`exit_picture_review`、`bridge_review`、`faithfulness_review`、`statistics_evidence_review` 语义结论。
