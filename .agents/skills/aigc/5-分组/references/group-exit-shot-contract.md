# Group Exit Picture Tail-Hook Contract

本文件定义每个分镜组结尾的 `出场画面：` 输出标准。`出场画面：` 是恢复的尾钩字段，用来记录本组最后一帧的可见末态，并为下一组 `入场镜头：` 与独立组间连接件提供可回指端点。

## Core Rule

每个分镜组的原正文之后、组底 YAML 之前必须写 `出场画面：`。它不是独立转场块，也不替代 `## <上一个分镜组ID>~<下一个分镜组ID>` 连接件；它只负责把本组末帧状态钉牢，让后续承接有真实画面依据。

`出场画面：` 必须回答四个问题：

1. **本组停在哪里**：明确回到本组最后一个原始分镜、本组末帧、本组尾帧或本组最后一帧。
2. **谁或什么留下来**：记录末帧可见主体、群体、关键道具、场景锚点、残留光线/烟雾/水纹/碎片/衣袍/发丝/声音余波等。
3. **以什么状态离开本组**：说明主体位置、姿态、朝向、视线、距离、遮挡、身体接触、道具关系、动作相位和镜头/观看位置的末态。
4. **下一段怎样接得上**：只写可供下一组入场或连接件消费的视觉、运动、声音、光线或空间惯性；不得写下一组尚未发生的剧情。

`出场画面：` 必须来自本组正文、上游 `4-摄影` 分镜、组内镜头设计、角色动作、分镜明细和可确认的场景事实；不得为了连接方便虚构新的出口、道具、人物位置、视线方向或动作结果。

## Required Content

`出场画面：` 使用自然中文描述，必须覆盖：

- 末帧来源：本组最后一个原始分镜、本组末帧、本组尾帧或本组最后一帧。
- 末态主体：最后一帧仍可见、可听见或必须作为下游生成主体保留的角色、群体、关键道具和场景锚点。
- 空间关系：左右、前后、远近、内外、高低、遮挡、身体接触、视线、朝向和距离在末帧如何定格。
- 动作余势：未完全结束的动作、声音余波、光线流动、烟雾、水纹、碎片、衣袍、发丝或焦点漂移等可被连接件接住的惯性。
- 镜头/观看位置：本组以什么景别、机位、焦点、前景遮挡或观看距离离开。
- 承接可用性：说明下一组 `入场镜头：` 或组间连接件可承接的尾帧锚点，但不替连接件写完整 `变化过程`、`主体运动` 或 `运镜设计`。

## Evidence-To-Exit Workflow

生成 `出场画面：` 时必须先做内部取证，再落盘：

1. 锁定本组最后一个 atomic unit 和最后一个 `分镜N（约X秒）`。
2. 抽取末帧主体、姿态、朝向、视线、距离、遮挡、道具关系、光影、声音余波和镜头设计。
3. 标注仍在延续的动作或环境惯性；若动作已经结束，也写清最后稳定状态，而不是制造无证据悬念。
4. 检查是否有无证据的下一组信息、未来动作、解释性剧情或新增道具进入文本。
5. 写入 `出场画面：`，先写末帧来源，再写主体末态，最后写可供下一组入场和连接件消费的惯性或锚点。

每个分镜组固定顺序：

```text

入场镜头：...

左侧：...
中间：...
右侧：...
前景：...
中景：...
背景：...

画面属性：...

视频生成的画面风格...
<类型元素>
<画面风格>

<从 4-摄影 划定的分镜剧本正文，同步原换行>

出场画面：<本组最后一个原始分镜/本组末帧/本组尾帧 + 主体末态 + 空间关系 + 动作余势 + 镜头/观看位置 + 可供承接的锚点>

```yaml
...
```
```

`出场画面：` 位于原正文之后、组底 YAML 之前。它不进入组间连接件块，不替代独立 `## <上一个分镜组ID>~<下一个分镜组ID>`。

## Count Boundary

`出场画面：` 计入 YAML `字数统计`，并进入 1680 字目标上限 / 1980 字硬上限的辅助风险口径；但不计入 `时长估算`，也不改变组内 `分镜N（约X秒）:` 的累计结果。

## Do Not

- 不把 `出场画面：` 写成下一剧情预告、剧情解释、连接件正向提示词或无证据悬念。
- 不复述整组剧情，只记录最后一帧仍成立的画面状态。
- 不新增本组正文没有的人物、道具、空间出口、道具互动、视线方向、动作结果或环境事件。
- 不替连接件写完整转场过程；连接件仍由 `references/bridge-shot-contract.md` 管理。
- 不把 `出场画面：` 计入组内时长，也不把它写入组间连接件。
