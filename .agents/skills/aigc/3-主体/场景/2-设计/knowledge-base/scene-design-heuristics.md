# Scene Design Heuristics

本文件保存 `$aigc-scene-design` 的可复用经验。稳定到必须执行的规则应晋升到 `SKILL.md` 或对应规范分区。

## Heuristics

- 先找空间规则，再找美术形容词；没有规则的场景很难生成一致资产。
- 现实场景的可信度来自材料、尺度、使用痕迹和光源，而不是风格标签数量。
- 超现实场景需要一个现实锚点，否则提示词会漂成抽象概念图。
- `MEMORY.md` 中的项目初始化记忆应变成可执行的空间、材质、光线和风格倾向，不应写成直接仿某位作者的口号或新增顾问身份。
- 同一项目批量设计时，保留全局风格一致，但每个场景要有独立的空间锚点或空间冲突；只有建筑/室内/街区场景才需要建筑锚点。
- 英文 prompt 不要重复解释故事背景；图像生成更需要可见对象、材质、光线和构图。
- 当时间、地域、建筑风格或空间风格无法从资料中直接得出时，用 `scene_inference` 或保守英文锚点标注推断，不要伪装成事实；自然、超现实、交通或抽象空间不要强行套建筑风格。

## Common Failures

| symptom | likely cause | repair |
| --- | --- | --- |
| 所有场景都像同一个空间 | 过度依赖全局风格 | 为每个场景补独特空间锚点 |
| prompt 很长但画面不可控 | 抽象词太多、可见细节太少 | 压缩情绪词，增加结构、材料和光线 |
| 摄影段与空间无关 | 先写电影感，后补场景 | 从空间边界和动线反推镜头 |
| 研究段像百科粘贴 | 考据没有服务设计 | 只保留影响形制、材料、色彩、仪式或动线的信息 |
| 物语替代剧本 | 没区分场景功能和剧情事件 | 改写为空间如何改变人物关系与观看方式 |
