## Repair Playbook

1. 先确认失败属于来源不足、画面基调继承缺失、角色卡化、剧情动作污染、场景镜头越权、参考人物复制、维度缺失、字数不合规还是报告证据缺失。
2. 若来源不足，回到 `N1-INTAKE`，不要用热门角色审美或随机参考补空。
3. 若画面基调缺失，标记候选状态；除非用户明确授权，不写回正式项目协议。
4. 若剧本证据带出具体人物，回到 `N2-SCRIPT-CHARACTER-EVIDENCE`，提取人群结构、时代质感、身体性和外观压力，不保留姓名或剧情。
5. 若参考图/视频带来具体脸、服装套装或姿势，回到 `N3-TONE-REFERENCE-EVIDENCE`，只保留角色层风格事实。
6. 若 prompt 出现姓名、角色卡字段、剧情动作、场景、镜头或单人造型，回到 `N6-PROMPT-DISTILL`，用角色层风格语言替换。
7. 若输出只剩空泛形容词，回到 `N4-CHARACTER-STYLE-GENES`，补 8 个解析维度。
8. 用户指出“脚本化/偷懒/未思考/未差异化”时，直接废弃候选协议回源层重做，不接受替换几个姓名、身份或造型词。
9. 修复后必须重新执行 `GATE-CS-01` 到 `GATE-CS-10`，并把失败码、返工点和复审结果写入 `Repair Log`。

## Reusable Heuristics

- 角色风格不是角色卡；它给角色卡阶段提供“应该怎么长、怎么有质感、哪些不要碰”的风格协议。
- 角色层 prompt 的稳定结构：整体气质 -> 轮廓语言 -> 妆发纪律 -> 服装结构倾向 -> 体态/年龄质感 -> 负面边界。
- 服装可以写结构和纪律，不写单件衣服；妆发可以写处理原则，不写某个具体角色的定妆。
- 年龄质感应写成“跨度、纹理、姿态、骨相、疲惫度或生命阶段感”，不要写具体岁数。
- 表演外观边界只描述看起来如何承载表演，不写具体动作、台词或剧情任务。
- 参考图/视频分析先做“去人物化”：遮掉具体脸、姓名、姿势和场景后，剩下的轮廓、妆发、体态、结构和质感才适合进入协议。
- 如果一句话只能服务某个具体人物，它不属于角色风格；如果它能约束全片角色群体的生成倾向，它才有资格进入本技能输出。
- `约 100 字` 的目标不是机械压缩，而是保证角色卡阶段仍有展开空间。
