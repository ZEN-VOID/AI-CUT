# Shot Continuity Contract

本文件定义 `5-摄影` 的镜头连续性规则。分镜明细不能是孤立的单一画面自顾自表现；它必须回看整集中临近画面，形成连贯、统一、可剪辑的镜头链。

## Core Rule

为每个画面句子写 `分镜明细：` 前，必须在内部判断中回顾临近至少前 3 个画面单位的分镜明细表现。若当前是本集前三个画面单位，则回顾已有全部画面单位，并补足场景建立逻辑。

连续性回看默认不在 `分镜明细：` 输出中显式展示，避免浪费字数。只有在跨轴线、反向运动、光色突变、景别断崖、场景变化或强断裂时，才用一句短动机写出当前画面如何进入或如何交出，其余情况直接集中描述当前画面的镜头运动。

连续性不等于段落吞并。若需要统筹相邻 3-6 个画面单位的视觉母题、注意力接力或运动家族，必须读取 `visual-sequence-alignment-contract.md` 并形成内部 `sequence_profile`；但当前 `分镜明细` 仍只服务正上方画面句子，不能把后文主体动作、对白反应、记忆段或道具揭示提前写入当前块。

连续性首先是人物动作链连续，而不是物件或气氛连续。每次切镜、焦点转移或插入细节前，先按 `../../_shared/action-first-continuity-contract.md` 检查人物姿态、位置、朝向、身体接触、动作方向、可达对象和退出状态；未通过准入的环境/道具细节不得成为切点、焦点终点或时值理由。

多人场面的连续性还必须消费 `../../_shared/lived-in-character-behavior-contract.md`。镜头先判断谁是当前 `action_driver`，谁是 `reaction_receiver`，哪些角色只是 `ambient_participants`。不要为了让每个人都同时有表演镜头而牺牲轴线、动作链、视线关系或主体焦点。

回顾对象包括：

| continuity_axis | must check |
| --- | --- |
| 空间轴线 | 人物、讲台、黑板、门、窗、课桌之间的左右/前后关系是否延续 |
| 运动方向 | 推、拉、摇、移、跟、甩镜方向是否顺接，是否无动机反向 |
| 景别梯度 | 是否从远到近、近到远或远近跳变有清楚动机 |
| 景深策略 | 深景深、浅景深、焦点拉移是否延续注意力逻辑 |
| 镜头视角 | 平视、低角度、高角度、主观、过肩是否服务同一权力关系 |
| 镜头类型 | 广角、长焦、微距、手持、稳定器等质感是否突然断裂 |
| 运镜速度 | 静止、极慢、慢速、中速、快速、急停是否有速度曲线承接 |
| 光影色彩 | 冷白、黑、红、金色等视觉母题是否连贯，是否无动机跳色 |
| 声画桥 | 声音、光变、文字显影、动作匹配是否为切换提供接口 |
| 场景切换 | 上一场景交出点和下一场景进入点是否清楚，是否需要建立镜头或软桥接 |

## Continuity Profile

每个 `visual_unit` 在注入前形成一个隐式或显式 `continuity_profile`：

| field | question |
| --- | --- |
| `previous_3_units` | 前 3 个画面单位分别在看什么、怎么动、落在哪里 |
| `current_entry_point` | 当前镜头应从上一个注意力落点、声音或动作接口进入哪里 |
| `axis_policy` | 继续同轴、通过反打换轴、用建立镜头重置，还是用主观视角合理跳轴 |
| `movement_policy` | 顺接上一运动方向、减速停住、反向但给动机，还是用静止切断 |
| `palette_policy` | 延续光色、局部强调新颜色，还是用光变/显影解释跳色 |
| `shot_scale_policy` | 延续景别梯度，还是用特写/远景作为有动机的压力变化 |
| `handoff` | 当前分镜最终把注意力交给下一画面哪个入口 |
| `unit_ownership` | 当前镜头承接段落连续性时，仍归属于哪条上游画面句子；哪些相邻信息不得外溢 |
| `action_first_continuity_check` | 当前分镜是否先承接人物动作链；对象是否空间可达；道具/环境是否通过准入 |
| `action_reaction_focus` | 多人 beat 中谁推动动作、谁接住反应、谁只维持背景状态；镜头是否保留因果焦点 |

## Continuity Decision Rules

- 默认延续上一镜头的空间轴线；跨轴线必须通过反应镜头、主观视角、建立镜头、声画桥或明显角色转身完成。
- 默认让运动方向顺接；若从左移突然变右移、从推进突然后撤，必须写出原因，例如危险逼近、角色退缩、规则反噬。
- 景别变化要像呼吸：远景建立，中景调度，近景加压，特写揭示；不要无动机从大远景跳大特写。
- 景深变化要服务注意力转移：深景深看关系，浅景深锁主体，焦点拉移把信息从 A 交到 B。
- 光色变化要有来源：日光灯闪动、黑板波纹、红色道具入画、金色规则显影都可作为动机。
- 边界交出不是任意跳跃；必须找到形态、动作、声音、光线、颜色或文字等可见接口。
- 场景变化本身只要求本阶段记录交出锚点和进入提示；连接强度、匹配方式和创意转场由 `6-分组` 连接件裁决。
- 当前画面如果需要强断裂，也必须让断裂“有意图”：惊吓、规则打断、记忆闪回、系统公告或感知异常。
- 段落级运动方向可以延续，但当前画面必须完成自己的主体、动作、信息或对白承托；如果连续性让当前块只剩“延续上一镜气氛”，应回到 `shot_design_plan` 重写。
- 道具、反射、静物、环境声或光影只能在通过准入时参与连续性。若删掉该物件细节后人物动作更顺，必须删除或降级为背景，不得把它保留为“高级感”切点。
- 触碰、取用、查看或避开对象的分镜，必须在同镜或上一镜可读出方向、距离和遮挡关系；否则先补人物行动路径，而不是切到对象。
- 多人 beat 默认不是人人同等强度表演。优先使用行动者镜头、反应者镜头或能同时读出动作-反应因果的关系镜头；背景参与者只保留必要站位、朝向和低强度状态。
- 若一个角度必须同时照顾行动者、反应者、道具和背景，导致人物动作或视线关系别扭，应拆回行动-反应两步，或删除非必要物件/背景焦点。

## Continuity Sentence Patterns

连续性通常存在于内部 `continuity_profile`，不必逐条外露。仅当需要解释明显变化或转场动机时，可在 `分镜N` 中短句呈现：

- `承接上一镜落在 <上一落点> 的注意力，从 <当前入口> 进入...`
- `沿用前两镜的 <运动方向/冷白顶光/同轴构图>，镜头继续...`
- `为避免跳轴，先用 <反应镜头/过肩/主观视角> 重置空间，再...`
- `把 <声音/动作/颜色/文字形态> 留作当前画面的交出锚点...`
- `故意打断前三镜的稳定节奏，以 <急停/光变/甩镜> 制造规则入侵。`

## Action-Reaction Focus

多人 beat 的镜头连续性必须保留动作-反应因果。内部先判断 `action_driver / reaction_receiver / ambient_participants`，再决定关系镜头、反应镜头或切分顺序。

执行原则：

- 行动者镜头回答“谁推动当前局面”，反应者镜头回答“这个动作如何被接住或躲开”。
- 可以用同一关系镜头同时容纳行动和反应，但前提是轴线、视线、动作方向和主体焦点清楚。
- 背景参与者只承担站位、朝向和低强度状态，不为每个人安排同等强度表情特写。
- 如果一个镜头为了同时纳入多人、物件和环境而让动作关系变别扭，优先拆成行动者 -> 反应者，或删除非必要物件焦点。

## Action Anchor Inheritance

本节定义跨分镜组、跨镜头切换时的动作锚点继承规则。这是 AI 视频生成器最常见的穿帮原因：上一组分镜结尾两人坐着，下一分镜组开头两人站着，中间没有任何动作变化描写。

### Core Rule

当人物动作在多个分镜或分镜组之间延续时（如拥抱、牵手、坐着、靠墙、搀扶），必须在每个分镜的描述中携带当前动作锚点，确保 AI 视频生成器知道人物的身体姿态、位置和交互状态。

### 要求

- 每个 `分镜N` 开头必须在视觉上能读出当前人物的站/坐/蹲/跪/躺状态
- 跨分镜组时，下一组第一个 `分镜1` 必须描述人物的物理姿态（坐在哪里/站在哪里/靠在什么上）
- 如果上一镜结尾动作是持续性的（如拥抱、牵手、搀扶），下一镜必须延续该动作锚点，直到分镜明确交代动作变化
- 动作锚点包括：人物朝向、身体接触、手部位置、坐着/站着、坐在什么上（凳子/床/台阶/地板）、与另一个人的距离和接触

### Continuity Profile 新增字段

| field | question |
| --- | --- |
| `action_anchor_inheritance` | 当前人物的动作姿态是否已在上一镜/上一组中被建立；下一镜/下一组是否需要继承并携带该锚点 |
| `anchor_break_point` | 是否存在动作变化的明确分界点（如起身、放手、松开、转身）；没有分界点则不得改变姿态 |

### Continuity Decision Rules 补充

- 默认延续上一镜的动作锚点：上一镜结尾人物坐着，下一镜也必须坐着，直到分镜文字明确描述了"起身""站起""转头"等动作变化
- 跨分镜组时强制携带：分镜组边界是动作锚点检查的强制节点，下一组首镜必须描述当前人物姿态
- 动作锚点检查优先于摄影炫技：镜头再好看，如果人物姿态断了，下游 AI 视频生成器会穿帮
- 若需要在分镜组边界改变姿态，必须在该分镜组最后一镜末尾明确描述动作变化（如"他慢慢松开手，从床边站起身"），让下一组首镜的姿态变化有视觉分界

### Anti-Patterns

- 上一组分镜结尾两人在沙发上坐着，下一组首镜直接写"他站在窗前"（没有起身动作描写）
- 上一镜结尾两人面对面站着，手搭在对方肩上，下一镜直接切到他们走路（没有手放下来的动作）
- 上一镜结尾他扶着她站在门口，下一镜直接切到她坐在床上（没有搀扶→放手的动作）

### 示例

```
✅ 分镜1（约2秒）: 他双手扶住她的腰，两人面对面站着；镜头从他的肩线慢慢贴近，落在她眼角还没落下的泪痕。
（下一组分镜组首镜）分镜1（约2秒）: 他扶着她在床边坐下——她的手还搭在他肩上没松开，镜头继续贴着她的侧脸。
❌ 分镜1（约2秒）: 他扶她坐下，镜头贴近她的侧脸。（没有说明他们是否站着，下一分镜组如果直接写两人在走动，AI 视频生成器会穿帮）

✅ 分镜2（约3秒）: 两人依然坐在床沿，他的右手从她腰侧慢慢松开——镜头从她的泪光慢慢下摇，停在他松开的手背上。
❌ 分镜2（约3秒）: 镜头从泪光下摇到手背。（缺少"她的手还搭在他肩上没松开"这一持续性动作锚点）
```

## Shot Scale & Camera Angle Rules

本节定义景别切换与镜头角度的配合规则，是镜头连续性的核心子维度。

### Core Rule

**同一画面主体，不同景别时，必须使用不同镜头角度。** 景别变角度不变会导致画面呆板、缺乏立体感；角度变景别不变会产生跳变感。

### Camera Angle Types

| 角度类型 | 视觉特征 | 适用场景 |
| --- | --- | --- |
| 正面 | 主体面对镜头，表情清晰 | 对话、情感表达 |
| 正侧 | 主体侧面90度，轮廓分明 | 动作展示、方向感 |
| 正反打 | 过肩或越过肩的视线方向 | 关系镜头、空间交代 |
| 俯角 | 摄影机高于主体，头大脚小 | 弱势化、压迫感 |
| 仰角 | 摄影机低于主体，脚大头小 | 强势化、崇高感 |
| 顶角 | 摄影机正上方垂直向下 | 紧张感、图案化 |
| 耳侧角 | 摄影机从耳后约30-45度 | 主观听觉暗示 |
| 倾斜角 | 摄影机倾斜，地平线不正 | 失衡、混乱、醉酒 |

### Shot Scale & Angle Combination Matrix

| 景别变化 | 建议角度变化 | 规则说明 |
| --- | --- | --- |
| 远景→中景 | 俯角→平视或耳侧角 | 建立空间后切入关系 |
| 中景→近景 | 平视→正侧或微俯 | 加压同时保持可辨识性 |
| 近景→特写 | 正侧→正面或微仰 | 从局部转向完整表情 |
| 特写→中景 | 正面→正反打或俯角 | 从孤立细节回归人物关系 |
| 大远景→特写 | 俯角→仰角 | 极端跳变制造视觉冲击 |

### Genre-Specific Guidelines

#### 快节奏武侠片（Wuxia Action / 高能战斗）

**规则：景别快剪辑 + 角度剧烈跳变 = 冲击力**

- 大远景（establishing shot）→ 近景/特写：允许跳轴，用角度跳变承托景别跳变
- 特写（局部细节，如剑尖、水滴）→ 大远景（战场全貌）：用俯角到仰角的跳变连接
- 连续战斗中：每2-3秒切一次景别，配合角度从正面→正侧→俯角→仰角的循环
- 允许极端跳变：特写切大远景用硬切（flash cut），角度从正面直接跳到俯角
- **禁止**：连续同景别同角度超过3镜（会导致节奏拖沓）

```
✅ 示例：令狐冲剑刺
分镜1（约1秒）：大远景俯角看黑蓑人钉在梁下三尺处，火光把他的轮廓压成一个暗块。
分镜2（约0.5秒）：极短闪切正面近景，令狐冲剑尖已经贴上他胸口，正面角度直接跳入。
分镜3（约0.3秒）：闪切特写，剑线横过胸口，白光像火苗跳了一下。

❌ 反例：景别变但角度不变
分镜1：正面中景，令狐冲举剑。
分镜2：正面特写，剑尖。（同一正面角度，缺乏立体感）
```

#### 慢节奏情感片（Emotional Drama / 温情叙事）

**规则：景别缓变 + 角度微调 = 呼吸感**

- 景别变化幅度小：不用远景→特写的跳变，而是远景→中远景→中景→近景的渐进
- 角度变化幅度小：从正面微调到正侧，或从平视微俯到微仰
- 同一角度可以延续多镜：让情感呼吸，不急于切换
- 允许"不切"：关键情感瞬间用 held 镜头（4-6秒），景别和角度都不变
- 景别切换信号：角色动作（转身、起身）、视线方向变化、或明确的情绪转折

```
✅ 示例：温情告别
分镜1（约4秒）：正面中景，阿袖愣住的脸，火光在她眼里晃一下；平视不动。
分镜2（约3秒）：极慢下移正面近景，她的眼眶慢慢泛红；角度不变，景别缓变，给眼泪时间。
分镜3（约2秒）：她用力点头，侧脸从正面滑入正侧；角度从正面过渡到正侧，呼应"执行"的动作。

❌ 反例：情感戏用武侠节奏
分镜1：中景平视。分镜2：近景俯角。分镜3：特写仰角。（景别跳变太急，情感戏失去了呼吸）
```

### Shot Scale Decision Flow

```
当前镜头需要确定景别时，按以下顺序判断：

1. 叙事优先级：这段要表达什么（关系/动作/细节/情绪）
2. 上游景别：上一个镜头是什么景别
3. 景别梯度：应该趋近（远→近）还是拉远（近→远）
4. 景别跳变幅度：同主体时，景别变化越大，角度变化也应越大
5. 类型节奏：武侠片允许大跳变，情感片要求缓变
6. 镜头数量预算：景别跳变次数受总镜头数约束
```

### Continuity Profile 补充字段

| field | question |
| --- | --- |
| `shot_scale_transition` | 从上一镜到当前镜，景别是趋近、拉远、还是跳变 |
| `camera_angle_policy` | 当前镜头的角度选择（正面/正侧/俯/仰/耳侧/顶角/倾斜） |
| `angle_motivation` | 角度跳变的动机（叙事需要、类型节奏、动作配合） |
| `scale_angle_coherence` | 景别和角度是否配合（景别变角度是否也变了） |

### Anti-Patterns

#### 景别变角度不变（呆板）
- 上一镜正面中景，下一镜正面近景：角度没变，景别跳变显得单调
- 修复：加入角度微调（正面→微侧）或镜头运动（缓慢推进改为下移）

#### 角度变景别不变（跳变感）
- 上一镜正面近景，下一镜俯角中景：角度剧变但景别接近，产生不稳定感
- 修复：让景别同步变化（俯角远景建立新空间），或让角度渐变过渡

#### 武侠片角度过于保守
- 战斗戏一直用正面或正侧，景别变化也慢
- 修复：引入俯角/仰角跳变，用大远景→特写的 flash cut 增强冲击力

#### 情感片角度过于跳脱
- 温情对话戏每镜换一个角度（正面→俯角→仰角→侧），景别也频繁跳变
- 修复：固定主角度（正面），只在关键情绪转折时微调角度

## Anti-Patterns

### 景别与角度失调类（新增）

- 上一镜正面中景，下一镜正面近景（景别变角度不变，画面呆板）
- 上一镜正面近景，下一镜俯角中景（角度剧变但景别接近，产生跳变感）
- 快节奏战斗戏一直用正面正侧，不引入俯角/仰角跳变（缺乏视觉冲击）
- 慢节奏情感戏每镜换角度，景别也频繁跳变（失去呼吸感）

### 动作锚点断裂类

- 上一组分镜结尾两人在沙发上坐着，下一组首镜直接写"他站在窗前"（没有起身动作描写）
- 上一镜结尾两人面对面站着，手搭在对方肩上，下一镜直接切到他们走路（没有手放下来的动作）
- 上一镜结尾他扶着她站在门口，下一镜直接切到她坐在床上（没有搀扶→放手的动作）

### 动作-反应焦点丢失类

- 多人对话中每个人都被安排同等强度近景，导致观众不知道谁推动当前局面、谁在反应。
- 为了同时拍到行动者、反应者和桌面物件，镜头选择一个不服务任何一方动作链的奇怪角度。
- 反应者的停顿、呼吸或距离变化被无关背景人物小动作抢走焦点。

### 运动与轴线断裂类（原有）

- 上一镜是沿课桌从左向右横移，下一镜无说明突然从右向左横移。
- 上一镜在讲台低角度仰拍苏红叶，下一镜无反应或建立直接跳到教室后排俯拍。
- 前三镜保持冷白日光灯和黑窗帘，下一镜突然大面积暖橙色却没有光源或情绪动机。
- 前一镜特写红色道具，下一镜直接写全班远景，没有反应镜头、声画桥或焦点回收。
- 每个画面句子都重新设计一套完全不同的镜头风格，整集像不同影片拼接。
- 为了让整段横移或跟拍不断，把多个画面点揉成一个镜头，导致当前块无法回指正上方原文。

## Output Requirement

`分镜N（约X秒）` 不得机械显式写“前 3 个画面单位”。成稿应集中于当前画面的动态镜头，连续性只作为内部检查；若确需说明承接关系，压缩为一句短动机。

多分镜块内部也必须连续：`分镜1` 的终点应成为 `分镜2` 的入口、反应、动作、声音、光色或转场接口；最后一镜必须把注意力交给下一画面可消费的入口。若相邻分镜无法说明“上一镜如何进入下一镜”，应回到 `shot_design_plan` 重排或合并分镜。

```text
分镜N（约3秒）: 从黑板冷白字缓慢下摇到学生们停住的嘴唇；景深从深景深收窄到中景深，讲台红色教鞭留在画面右侧作为危险锚点。
```
