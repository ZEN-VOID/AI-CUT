# Source Detail Incremental Fusion Contract

本文件定义 `4-摄影` 如何把上游画面性字段正文升级为原字段标题下的连续分镜时间段：摄影语言必须增量叠加到原画面事实上，而不是把原画面描述摘要、压缩、改写成更短的镜头说明。

## Core Rule

`分镜画面` 的基本写法是：

```text
原画面可见事实完整保留 -> 摄影机选择观看位置 -> 运镜/焦点/光线/构图有机融入 -> 在准确秒点交出下一注意力
```

保真和非复述必须同时成立：

- **保真**：上游画面句子中的人物、动作、姿态、服装/造型、空间、道具、文字、光线、色块、表演微动态、身体距离和异常可见变化，凡未被判定为非画面信息、重复信息或明确越权信息，必须进入 `source_visible_fact_map`，并在终稿某个 `[起始秒-结束秒]` 段中自然可见。
- **非复述**：在保留这些事实的基础上，每个时间段还必须新增观看方式：机位、景别、焦点、路径、速度、停点、光线结果、构图关系、注意力接力或交出接口。不得为了通过源句复述扣除测试而删掉原画面事实。

## Incremental Fusion Modes

| mode | 用法 | 成稿特征 |
| --- | --- | --- |
| `source-led` | 上游画面描述已经具体、密度高 | 先保留原有主体/动作/细节，再把镜头位置、焦点和光线嵌入句中 |
| `camera-led` | 上游画面较简短，但动作或信息明确 | 先写摄影机入口和观看路径，再在路径中完整显影原事实 |
| `beat-synced` | 原句含多个动作相位、微表情、文字读秒、声画撞点或信息揭示 | 每个关键事实绑定一个秒点或焦点变化，形成精准卡点 |
| `continuity-anchored` | 当前画面承接前后动作链或空间轴线 | 原事实必须携带 entry/action_anchor/exit/handoff，摄影语言只补清楚如何接住和交出 |
| `density-curved` | 连续 3-8 个画面形成段落级密度曲线 | 当前块仍保留自身事实，但密度、时值和交出点服从段落曲线 |

## Source Visible Fact Map

每个 `visual_unit` 在进入 `N6.5-SHOT-PLAN` 前必须内部形成 `source_visible_fact_map`：

| field | required content |
| --- | --- |
| `source_visual_unit` | 上游画面句子原文或对应字段 |
| `must_preserve_facts` | 必须进入成稿的可见事实：人物、动作、姿态、表情、手部、服装、空间、道具、文字、光线、异常变化 |
| `compressible_facts` | 可合并表达但不可消失的重复/低价值可见事实 |
| `excluded_facts` | 不进入成稿的理由：非画面、抽象心理、重复、越权、与道具准入冲突 |
| `fact_to_segment_binding` | 每个必须保留事实绑定到哪个 `[起始秒-结束秒]`、哪一次焦点/构图/运镜/光线变化 |
| `loss_risk` | 若该事实被删，会损失人物状态、空间身份、动作因果、信息显影、情绪读秒、下游视频稳定性中的哪一项 |

`compressible_facts` 只能合并表达，不能整体删除。例如“她穿着湿透的白衬衫，袖口沾着灰”可以融合成“白衬衫贴在肩背，镜头停到袖口灰痕”，不能缩成“她衣着狼狈”。

## Precision Timing Fusion

精准卡点不是机械多分段，而是把原画面事实与摄影变化绑定到正确的观看时刻：

1. 动作起点进入镜头入口：手伸入、门开、脚步落地、视线抬起。
2. 动作过程对应运镜路径：推、拉、横移、跟拍、静止等待、焦点拉移。
3. 动作结果对应停点：手停住、字显影、脸被光切开、物件落地、反应出现。
4. 情绪或信息读秒对应时值：微表情、规则文字、道具证据、沉默反应需要停留；普通过场不拖长。
5. 交出点对应下一注意力：视线、声音尾巴、光色、空间边缘、动作余势。

若一个原句中有多个必须保留事实，`fact_to_segment_binding` 必须说明它们是同段内通过框内运动/焦点拉移完成，还是拆成多个时间段。不能把多个事实压缩成一句泛化形容词。

## Organic Fusion Writing

终稿应把原事实和摄影语言写成同一个画面动作，而不是“原句 + 摄影说明”的拼接：

| 弱写法 | 问题 | 强写法方向 |
| --- | --- | --- |
| `镜头推近，少女很狼狈。` | 原服装/姿态/细节被摘要 | 写出湿衣、灰痕、姿态，再让镜头停到最关键细节 |
| `特写黑板上的规则。` | 原文字/空间/角色反应可能丢失 | 先确定黑板在画面中的位置，再绑定角色视线或焦点拉移 |
| `镜头跟随他走过去。` | 动作被泛化，缺少方向和卡点 | 写清从画面哪侧进入、经过什么前景、停在什么位置 |
| `压迫感加强。` | 抽象效果替代可见事实 | 写出谁被构图压住、哪条阴影/线条/距离造成压迫 |

## Rewrite Guard

以下情况直接判定为融合失败：

- 上游画面句子有 3 个以上具体可见事实，终稿只留下 1 个泛化事实。
- 上游明确写到服装、姿态、微表情、手部动作、身体距离、场景文字或异常物理变化，终稿没有对应可见落点。
- 为了“自然”“克制”“非复述”把原事实改成抽象标签，如“狼狈”“紧张”“诡异”“高级”“压迫”。
- 摄影术语替代原事实：只剩“近景/慢推/虚化/侧光”，看不出原画面到底发生了什么。
- 精准卡点失败：时间段有秒数，但原动作起点、撞点、结果或信息显影没有绑定到具体秒点。
