## Repair Playbook

1. 先确认失败属于来源不足、摄影语法缺维度、运动连招缺结构、连续性缺失、内容污染、字数不合规还是报告证据缺失。
2. 若来源不足，回到 `N1-INTAKE`，不要用热门运镜或随机摄影师补空。
3. 若参考图/视频带来具象污染，回到 `N3-REFERENCE-CAMERA-EVIDENCE`，只保留构图、景别、机位、运动、衔接、稳定性和节奏事实。
4. 若剧本分析只剩情绪摘要，回到 `N2-SCRIPT-CAMERA-EVIDENCE`，从节奏、空间压力、动作密度、视角距离和段落转换需求抽取摄影证据。
5. 若 prompt 出现剧情、角色、道具、场景或单镜头正文，回到 `N7-PROMPT-DISTILL`，用摄影层级表达替换。
6. 若运动连招只是术语列表，回到 `N5-MOVEMENT-COMBO`，补起手、过渡、落点和连续性理由。
7. 用户指出“脚本化/偷懒/未思考/未差异化”时，直接废弃候选协议回源层重做，不接受替换几个摄影术语或润色句子。
8. 修复后必须重新执行 `GATE-CS-01` 到 `GATE-CS-10`，并把失败码、返工点和复审结果写入 `Repair Log`。

## Reusable Heuristics

- 摄影风格不是“镜头清单”，而是下游分镜和视频生成可继承的观看协议。
- 构图规则要写成秩序和关系，不写具体画面内容。例如“以主体间距离组织张力”优于“主角站在门口”。
- 运镜规则要写动势和衔接，不要堆“推拉摇移升降环绕”。每个运动都要说明为什么接得上。
- 参考视频分析先做去内容化：遮掉人物和场景后还剩下的景别递进、机位关系/视角距离、透视/景深纪律、焦点行为、稳定性和剪接动势，才适合进入摄影协议。
- 画面基调提供媒介和氛围边界，摄影风格负责把边界转成观看距离、运动纪律和连续性，不反向修改全局视觉 prompt。
- 约 100 字 prompt 的稳定结构：构图秩序 -> 景别体系 -> 机位关系/视角距离 -> 透视/景深纪律 -> 焦点行为 -> 运动方式 -> 连贯性规则 -> 负面摄影禁区。
- 如果一句话只能服务某个具体镜头，它不属于摄影风格协议；如果它能约束一整集或整部作品的观看方式，它才有资格进入 prompt。
- AIGC 视频生成常见失败是运动太复杂或焦点/透视规则不稳导致漂移。协议中应优先强调“可连续生成”的运动组合、焦点纪律、视角关系和稳定性，而不是展示技术名词。
- 单集摄影风格可以承接本集观看距离、运动密度和连续性压力；不要覆盖整季摄影风格基线。

## Boundary Scan Checklist

1. 删除剧情事件、台词、角色动作和段落结局。
2. 删除角色身份、服装、道具、建筑、场景地点、材质和色彩方案。
3. 删除镜头编号、单镜头正文、具体镜头对象和逐镜头指令。
4. 检查每个运镜词是否有连续性理由；没有理由则降级为负面或删除。
5. 保留构图秩序、景别体系、机位关系、视角距离、透视/景深纪律、焦点行为、运动方式、速度纪律、稳定性、节奏密度和连续性规则。

## Positive Pattern

好的摄影风格应该让下游读到后能判断：

- 这个项目整体如何安排观看距离和视角关系。
- 镜头运动靠什么连续，而不是靠随机运镜炫技。
- 景别、机位和运动速度如何承托叙事节奏。
- 哪些摄影行为明确禁止，以减少 AIGC 视频漂移。
- 分镜阶段仍保有具体镜头对象、顺序和剧情表达空间。
