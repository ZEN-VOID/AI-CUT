# Statistics YAML Contract

本文件定义每个分镜组底部 YAML 统计的字段、统计边界和抽取规则。

## YAML Shape

每个分镜组底部必须附加 fenced YAML：

```yaml
字数统计: 0字
角色:
  - 角色名
场景:
  - 场景名
道具:
  - 道具名
```

无角色、场景或道具时使用空列表 `[]`，不得省略字段。

## Count Boundary

`字数统计` 统计范围为该分镜组从 `## x-y-z` 分镜组标题到 `出场画面` 内容结束的全部中文、英文、数字和标点字符估算；不包含 fenced YAML 统计块本身。

统计 YAML 本身不计入：

- 1680 字目标上限与 1980 字硬上限。
- `字数统计`。

## Role Extraction

角色字段优先来源：

1. `对白（角色名）`、`独白（角色名）`、`旁白（角色名）` 等声音主体。
2. `角色动作：`、`表演提示（角色名）：`、`表情特写：` 中明确出现的人名。
3. 场景正文中明确可见或可听见的角色名。

群像可写为 `学生群像`、`围观人群` 等稳定称谓。未实际出场但被提及的角色不列入，除非本组以影像、声音或道具显影方式让其成为生成主体。

## Scene Extraction

场景字段优先取当前 `### 场景N：...` 的场所名。一般一个分镜组只对应一个场景；若跨场景，优先重新拆组。确需多场景时，按出现顺序列多个场景名，并在执行报告说明原因。

## Prop Extraction

道具字段只列重要叙事道具、规则道具、视觉钩子或后续生成必须锁定的物件。普通环境物件只有在本组承担叙事功能时才列入。

优先来源：

1. `道具特写：`
2. `规则显影：`
3. `系统画面：`
4. 补位画面中的关键视觉承接点
5. 对白或动作中被明确使用的关键物件

## Review Notes

YAML 是统计证据，不是新的剧情真源。若 YAML 与正文冲突，以正文为准并修复 YAML。

每个 YAML 统计项都应能回指本组正文、场景标题、对白主体、角色动作、道具特写、规则显影、系统画面或补位画面中的证据。机械 validator 只检查字段形状和字数估算；角色、场景、道具是否准确必须由 `statistics_evidence_review` 做 LLM 或人工语义复核。
