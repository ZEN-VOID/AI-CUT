# 制作组共享创作质量手册

本文件是 `.codex/agents/aigc/制作组/` 的共享创作方法与质量门禁真源。

它不取代父 skill 的阶段路由、field-slot 与 canonical writeback 合同，只回答一件事：制作组各角色怎样稳定地产出高质量、可聚合、可下游消费的 `agents_plan + patch / note / report`。

提示工程层的共享身份、输入解释、决策顺序、回退协议与最小评测，统一由 `./.codex/agents/aigc/制作组/_shared/PROMPT_STABILITY_CONTRACT.md` 承担；本手册只保留创作方法与质量门禁，不再平行复制那套稳定性合同。

## 加载合同

- 父 skill、team 与命中的角色 agent 都应显式回指本文件。
- 父 skill、team 与命中的角色 agent 也必须显式回指 `PROMPT_STABILITY_CONTRACT.md`。
- 冲突优先级：用户显式请求 > 根 `AGENTS.md` > `.agents/skills/aigc/3-Detail/SKILL.md` > `team.md` > 本手册 > 单角色局部补充。
- 本手册是共享方法真源；单角色文件只写本角色 delta，不应平行复制整套方法。

## 共享目标

每个制作组角色都必须同时满足以下目标：

1. 先服务叙事任务，再追求风格表达。
2. 把抽象意图翻译成可见、可拍、可合成、可 merge 的字段 patch。
3. 保持与 `2-Global`、shot skeleton、既有 draft 和 shared schema 一致。
4. 在证据不足时保守退化，而不是用漂亮空话补齐。

## 通用工作法

1. 先锁本镜的叙事任务：
   - 这一镜要让观众知道什么、感到什么、期待什么。
2. 再锁可见锚点：
   - 主体是谁、空间锚点是什么、动作因果在哪里、视线先后顺序如何。
3. 再锁本角色真正负责的字段：
   - 只补自己拥有的字段，不把判断扩写成整镜总论。
4. 再做风格化：
   - 一切风格表达都必须回链到 `2-Global/全局风格.md`、`类型指导.md`、`导演意图.md`。
5. 最后收束为 merge-safe 交付：
   - `patch` 只给命中字段。
   - `note` 解释为什么这样做。
   - `report` 只在证据不足、锁轴冲突、风格冲突或无法稳定时触发。

## 共享质量门禁

每个角色输出前都要自检以下六项：

1. `服务任务`
   - 是否先服务本镜叙事任务，而不是先炫技或先堆形容词。
2. `可见可拍`
   - 是否能被演员、摄影、场景、后续画面/视频阶段直接想象和消费。
3. `具体不空泛`
   - 是否包含可观察对象、方向、层次、节奏、力度、关系或视觉锚点，而不是只写气质口号。
4. `节制`
   - 是否只给最小必要表达，没有为了“高级感”额外加戏。
5. `兼容`
   - 是否与 shot skeleton、上游导演意图和同轮其他 patch 不打架。
6. `可回退`
   - 若证据不足，是否明确说明缺口与保守方案，而不是硬编。

## `agents_plan + patch / note / report` 最小合同

### `agents_plan`

- 只承载当前角色的思考计划、候选路径、裁决摘要与 handoff 方案。
- 不替代业务字段 patch，不冒充 canonical JSON 或最终镜头结论。
- 重点说明为什么本轮应该这样补、哪些冲突仍需父 skill 裁决。

### `patch`

- 只包含当前角色拥有的命中字段。
- 语言必须可见、可拍、可聚合。
- 不得把分析过程、审美口号或跨字段总论塞进业务字段。

### `note`

至少包含以下四类信息中的三类：

- 选择了什么方向
- 依据了哪些上游锚点
- 放弃了什么备选方向及原因
- 这一选择的风险或边界

### `report`

至少包含：

- `verdict`: `BLOCKED` / `REWORK`
- 缺失或冲突的证据点
- 建议回到哪个角色或哪个 rework entry
- 保守处理建议

## 角色族群方法

### `分镜规划`

- 先切任务，再切时间，再切 coverage。
- 一镜只承担一个主任务，必要时才兼容次任务。
- 宁可多一个简洁稳定的镜，也不要把互相冲突的信息硬塞进同一镜。

### `分镜构图 / 景观设计 / 氛围设计`

- 先给观众稳定的观看路径，再给风格。
- 空间关系要能支撑后续动作、角色和运镜，不要只追求静态美图。
- 氛围要来自空间、时间、空气、声音感、压迫感等可感知来源，而不是单纯形容词。

### `角色表现` 族群

- 一切情绪、动作、关系都要写成镜头可见的表演信息。
- 少写“很痛苦 / 很愤怒 / 很紧张”，多写停顿、换气、视线、手势、攻守、退让、失衡、发力、迟滞。
- 多角色时先锁关系轴，再写表演细节。

### `运镜手法`

- 先回答“为什么要动”，再回答“怎么动”。
- 默认优先选择最能传达信息的镜头运动。
- 挑战方案只能提升表达上限，不能破坏叙事清晰度。

### `摄影美学`

- 摄影总协调先统一视觉控制线，再吸收光影与色彩子补丁。
- 光影与色彩都要说明其戏剧功能，不能只给漂亮标签。
- 不允许把摄影判断写成与运镜、场景或角色职责冲突的第二总线。

### `转场设计 / 特效设计`

- 默认克制，只有确有叙事收益时才进入。
- 转场先服务时间、空间、段落连接；特效先服务世界观与必要视觉桥接。
- 若直切更强，就明确保留直切，不为“做了点什么”而做。

### `连续性复核 / 真源审计`

- reviewer 先看“读起来是否连、拍起来是否顺、字段之间是否互相支持”。
- auditor 先看“是否仍服从单一真源、shared schema 和写回边界”。
- 两者都必须给出可执行返工入口，不能只说“建议优化”。

## 常见失败样式

- 用抽象情绪词代替可见表演。
- 用摄影或运镜炫技掩盖 shot task 不清。
- 用漂亮景观词堆砌没有叙事用途的环境细节。
- 用光影、色彩、氛围互相越权。
- 明明证据不足却不触发 `report`，而是硬补字段。
- `note` 只有赞美性解释，没有备选方案、风险和证据锚点。

## 保守回退策略

遇到以下情况时，优先触发保守方案：

- 上游导演意图不够明确：
  - 返回 `report`，指出缺的是剧情任务、情绪方向还是空间锁轴。
- 已锁定 shot skeleton 与本角色想法冲突：
  - 服从已锁 skeleton，只在 `note` 中标出局限与返工入口。
- 有多个等价方案但证据不足以做强判断：
  - 选择更节制、更稳定、更容易被下游消费的方案。
- 风格表达会压过信息清晰度：
  - 降级到默认叙事优先路线。
