# Actor Performance Control Contract

## Purpose

本细则把“角色演技”从情绪标签升级为可执行的表演控制系统。它服务 `4-表演` 的演员任务、心理反应、对白画面、角色动作、表情特写、环境声音和受控增强，不授权新增剧情事实、对白、事件结果、人物动机或摄影方案。

核心原则：不要只写“开心、愤怒、难过、害羞、紧张”等情绪结论；必须把情绪拆成上游触发点、人物动机、面部肌肉、身体联动、声音/环境承托和视频微动态限制。否则 AI 视频容易得到模板化表情，而不是有演技的角色。

## Ownership

- 本文件拥有 `actor_performance_control_pass` 的表演层细则：微表情、情绪动机、身体联动、环境声音、非对称瑕疵和微动态限制。
- `performance-and-scene-craft-contract.md` 负责场景状态差、演员任务、台词表演、潜台词行为和场面调度，本文件只细化”演员如何把情绪演出来、把台词怎么说出来”。
- `psychological-reaction-contract.md` 负责 `心理反应` 字段语义和 GETability；本文件为 `心理反应`、`表情特写`、`角色动作`、`对白（角色，状态）` 和 `对白画面` 提供表演变量。
- `../2-编剧/references/field-routing-and-audio-visual-contract.md` 负责字段纯度；本文件不改变字段边界。
- `../3-导演/references/visual-aesthetic-contract.md` 负责画面美学；本文件不写机位、景别、镜头运动或分镜编号。
- `stanislavski-method-reference.md` 提供三层情绪执行模板和情感瞬间与微动态控制的技术理论来源；本文件负责把该方法落实为微表情分区、身体联动描写和环境声承托的具体投影变量。调用路径：stanislavski_method_evidence 技术层级 → 本文件 Five-Part Expression Control Formula → 既有字段内嵌。
- `../../_shared/lived-in-character-behavior-contract.md` 提供角色活人感小事、下意识反应、情绪落点和多人动作-反应分工；本文件负责把这些变量纳入演员身体控制，而不是只强化脸部细节。

## Trigger Scope

必须执行本 pass 的信号：

- 上游出现强情绪、压抑情绪、情绪切换、沉默反应、未出口对白、强忍、试探、羞惭、紧张、愤怒、离别、恐惧、心虚、关系变化或权力压迫。
- 编导稿中出现“很愤怒、很难过、很害怕、很开心、很紧张、崩溃、压抑、复杂”等情绪标签，但缺少面部、身体、呼吸、声线、道具或空间承托。
- `心理反应` 只能靠字段标题理解，正文没有演员可演、观众可见/可听的反应。
- `对白（角色，状态）` 的第二项或 `对白画面` 只写抽象情绪，没有说话时的语气、情绪、气口、断句、节奏、呼吸、停顿、声线、视线、手部或对手反应。
- 用户要求“演技更好”“人物更真实”“表情更精准”“不要模板化表演”“AI 人物不僵硬”。
- 角色看起来只在展示脸、五官、光影或发丝，缺少当前正在做的小事、下意识反应和情绪落点。
- 多人场面中每个角色都在同等强度表演，缺少“一个人行动、另一个人反应”的因果焦点。

## Actor Performance System

每个关键情绪 beat 应尽量形成以下五层，不要求每个字段逐项全写，但执行报告证据必须说明取舍。

| layer | question | preferred projection |
| --- | --- | --- |
| `emotion_trigger` | 情绪因上游哪个动作、信息、声音、规则或关系变化发生 | `source_anchor`、`trigger`、`scene_dramatic_map` |
| `inner_motive` | 角色想达成什么，又为什么不能直接说或直接做 | `performance_task_map`、`内心独白（主角）`、`心理反应` |
| `micro_expression` | 情绪出现在眉、眼、嘴、鼻翼、咬肌、下颌、皮肤状态的哪里 | `表情特写`、`对白画面`、`心理反应` |
| `body_linkage` | 情绪如何影响肩、手、呼吸、重心、姿态、道具动作和声线 | `角色动作`、`动作画面`、`对白画面`、`音效画面` |
| `ambient_support` | 环境声、余声、沉默、自然声或空间声音如何放大情绪 | `音效` / `音效画面`、`环境描写`、`群像画面` |

关键对白 beat 还必须形成 `dialogue_delivery` 证据：语气/情绪、气口、断句、重音、声线变化和尾音处理。`dialogue_delivery` 不改写对白正文，只决定这句话如何被说出，并与微表情、身体联动和对手反应配套。

## Lived-In Micro Activity Pass

关键人物 beat 除五层表演控制外，还要检查角色是否像“正在生活中被事情推着走”的人。内部形成 `lived_in_behavior_evidence`：

| variable | question | projection |
| --- | --- | --- |
| `micro_activity` | 角色此刻正在做哪件场景内成立的小事 | `角色动作`、`对白画面`、`内心独白画面` 中的低风险动作或动作中断 |
| `subconscious_response` | 压力进入后，身体哪里不由自主地先反应 | 呼吸、手指、肩颈、重心、声线、停顿、视线落点 |
| `emotional_landing` | 该 beat 最后停在哪里 | 姿态、距离、注意力、手部状态、沉默或尾音 |
| `action_reaction_pair` | 多人场面谁行动，谁反应 | 行动者/反应者清楚，其他人不抢戏 |

约束：

- `micro_activity` 必须有上游锚点、场景任务、角色目标/阻碍或空间状态支撑；不得为了真实感随机塞“忙动作”。
- 多人场面不让所有人同时演满。优先让一个人推动动作，另一个人用反应承托状态差。
- 小事不能绕过道具准入。若行为必须依赖无互动普通物件、反射、涟漪或孤立静物才能成立，优先删除。
- 情绪落点必须能交给下一 beat 或下游摄影：人物站坐、朝向、距离、手部、身体接触或注意力落点不能断。

## Five-Part Expression Control Formula

### 1. Muscle Zone Control

把情绪落到面部区域，而不是只写情绪名。

可选变量：

- 眉：眉头内收、眉心竖纹、眉峰轻挑、眉尾压低。
- 眼：半眯、睁大、湿润、失焦、闪躲、盯住某一点、眨眼频率变化。
- 嘴：抿嘴、嘴角上扬又压下、嘴角不对称、牙关咬紧、嘴唇发干。
- 面部肌肉：苹果肌隆起、咬肌收紧、鼻翼轻张、下颌锁住。
- 声线关联：尾音收住、气声变轻、字与字之间停顿变长。

弱写法：

```md
表情特写：她很愤怒。
```

强写法：

```md
表情特写：她的眉心压出一道竖纹，眼睛半眯，视线死死钉在对方脸上；咬肌缓慢收紧，嘴唇抿成一条线。
```

### 2. Three-Layer Emotion Motive

关键情绪不要只写单一情绪。优先判断：

- `surface_emotion`：表层正在展示什么，例如平静、笑、沉默、冷淡。
- `suppressed_emotion`：被压住的真实情绪，例如愤怒、害怕、难过、不甘。
- `hidden_motive`：角色为什么压住它，例如不能暴露底牌、不能让对方担心、不能激怒对手。

示例：

```md
心理反应：阿真听见门外脚步声后没有立刻回头，只把杯沿转了半圈；她嘴角还维持着笑，指腹却在杯壁上停住。
```

这类写法让观众看到“表面平静”和“底层紧张”的冲突。

### 3. Non-Facial Physiological Linkage

真实情绪会影响身体。关键情绪 beat 必须优先寻找脸以外的反应。

| emotion pattern | body variables |
| --- | --- |
| 紧张 | 眨眼变快、肩膀内收、手指摩擦、呼吸变浅、脚尖停住 |
| 强忍难过 | 眼眶发红但不落泪、笑意压回去、吸气不完整、手指抓衣角 |
| 愤怒压制 | 指节发白、下颌绷住、身体反而静止、声线压低、动作变慢 |
| 害羞 | 视线下落、身体后缩、手指抓袖口、说话变轻、停顿变多 |
| 心虚/遮掩 | 整理无关物、先低头再抬眼、话到半句停住、错开距离 |

弱写法：

```md
角色动作：他很紧张。
```

强写法：

```md
角色动作：他把杯子换到另一只手，指腹在杯沿上来回摩擦；说话前吸了一口气，却只吸到一半。
```

### 4. Gait And Movement Pattern Control

步态是可见的行动语言。写清楚人物的走路方式能让观众仅从步态就读出状态、性格和当下情绪。

| gait component | writing requirement | example |
| --- | --- | --- |
| 起脚方式 | 脚跟先离地还是脚尖先起，重心在哪只脚 | `他左脚拖在地上，迟迟不肯抬起来` |
| 步幅节奏 | 大步还是碎步，步间距是否均匀，是否忽快忽慢 | `她一步一步挨过去，步子越来越小` |
| 身体重心 | 前倾、后仰居中、下沉 | `重心压在脚跟上，身子往后仰` |
| 手臂摆动 | 自然摆动、僵硬贴体、一只不动 | `右手插在口袋里，左手垂着不动` |
| 头部位置 | 抬着、低着、偏向一侧 | `头压得很低，只看着自己脚尖` |
| 行走目的 | 去哪里、见谁、为什么停 | `他在门口停住，脚尖抵住门槛` |

步态必须与情绪/状态联动：

```yaml
# 紧张时的步态
角色动作：林寂从座位起身，绕过前排椅背时脚步放轻，步子比平时碎；在第三排空位前停住，脚尖朝前但身体还没转过来。

# 愤怒压制时的步态
角色动作：他的步子比平时沉，重心压在后脚跟，手臂贴体不动，走到对方面前才停住。

# 犹豫时的步态
角色动作：他往门口走了两步，脚尖在门槛边磨了磨，重心换来换去，又退回半步。
```

### 5. Asymmetry And Imperfection Control

真实表演不应完美对称，也不应像模板表情。关键表演可轻量加入非对称、瑕疵和不完全控制，但不得堆成外貌描写。

允许：

- 一侧咬肌比另一侧更明显。
- 嘴角只抬起一边，又很快压下去。
- 眼角湿润但没有落泪。
- 嘴唇略干、脸色发白、皮肤保留细小疲惫感。
- 右手比左手更用力，袖口被抓出皱褶。

禁止：

- 为追求“真实”新增伤疤、疾病、妆容、外貌设定或与上游不符的身体状态。
- 把瑕疵写成剧情线索、伤势结果或新人物设定。

### 6. Micro-Dynamics And Video Restraint

视频表演最怕表情乱跳、动作过大。关键情绪要写变化顺序和幅度限制。

可用结构：

```text
前段静止/假装平静 -> 触发点进入 -> 局部肌肉或身体变量变化 -> 收住或泄露一点 -> 余波停顿
```

示例：

```md
对白画面：她前两个字说得很稳，第三个字前停了一下；嘴角本来要抬起，随即压回去，手指在袖口里慢慢收紧。
```

约束：

- 不用”大喊、崩溃、泪流满面”替代强情绪，除非上游明确支持。
- 不把”微动态”写成摄影慢动作；只写演员动作的快慢、停顿和幅度。
- 不让角色持续做夸张表情；真实情绪常出现在切换瞬间。
- 不把“开心/愤怒/难过/害怕”写成固定脸谱。真正有演技的瞬间通常是情绪刚要出现、被压回去、从表层情绪切到真实情绪，或身体先于意识泄露半拍。

### Emotion Transition Moment Rule

关键情绪 beat 优先抓“切换瞬间”，而不是让角色从头到尾挂着同一表情。

| transition type | usable signal | projection |
| --- | --- | --- |
| 表面情绪裂开 | 笑意还在，尾音突然变轻 / 嘴角抬起后压下 | `对白画面`、`心理反应` |
| 压制情绪泄露 | 眼神闪开半拍 / 指腹忽然停住 / 吸气只到一半 | `角色动作`、`表情特写` |
| 强装平静失败 | 前两个字平稳，第三个字前停顿 | `对白（角色，状态）`、`对白画面` |
| 情绪刚被触发 | 眨眼频率变快 / 肩膀僵住 / 下颌锁住 | `心理反应`、`角色动作` |

执行限制：

- 情绪切换必须有上游触发点，不能凭空让角色“忽然崩溃”。
- 切换瞬间应短、准、可演；不要扩写成连续多种表情轮播。
- 多人场面中，行动者负责触发，反应者负责接住切换；背景角色不需要同时发生同级表演。

### 微动态时序控制

微动态控制演员在情绪临界点的动作时序和幅度，必须有明确的触发信号和退出边界。

#### 时序序列模式

```yaml
# 标准情绪时序
emotion_sequence:
  前段:
    description: “情绪积累期，身体已有变化但意识在压制”
    pattern: “呼吸变浅/动作变慢/手部有细微动作/视线绕开”
    duration: “持续到触发点进入”

  触发点:
    description: “上游哪个信号打破了平衡”
    requirement: “必须有明确上游触发点，不是凭空产生”
    timing: “单个动作或单个词后进入”

  临界点:
    description: “情绪即将泄露但被压住，或终于泄露一点”
    pattern: “嘴角本来要抬起又压下去/眼眶湿润但没有落泪/话到嘴边停住”
    core_principle: “不是爆发，是临界态”
    duration: “0.5-1个动作单位”

  余波:
    description: “情绪通过身体某处泄露后，其余部分的状态”
    pattern: “呼吸停顿一拍后恢复/手指慢慢松开/沉默延长半拍”
    resolution: “身体回到压制态，或维持轻微泄露”
```

#### 幅度限制指引

| 强度等级 | 幅度上限 | 退出边界 |
|---------|---------|---------|
| **1级·微痕** | 单个部位轻微变化，幅度 < 0.5cm | 变化完成即退出到静止 |
| **2级·轻度** | 单一部位明显变化，其他部位保持克制 | 变化完成 + 余波停顿（0.5-1秒） |
| **3级·中度** | 多个部位协同变化，需有收住动作 | 临界点后强制收住，余波1-2秒 |
| **4级·强度** | 全身参与，但必须有压制失败和重新压制 | 爆发后必须有身体重新控制动作 |
| **5级·爆发** | 仅在上游明确授权时使用 | 必须有恢复/余波/后续压制 |

#### 微动态变量库

| 动态类型 | 典型写法 | 时序位置 |
|---------|---------|---------|
| 启动-停顿 | “他本要上前，脚尖却停在门槛边” | 触发点 |
| 启动-收回 | “嘴角本来要抬起，随即压回去” | 临界点 |
| 持续-增强 | “压在桌沿的手指慢慢收紧，指节发白” | 前段→临界点 |
| 持续-减弱 | “笑意还挂在嘴角，但眼底的亮光一点点暗下去” | 余波 |
| 切换-突然 | “她忽然停住，喉结动了一下，把后半句咽回去” | 触发点→临界点 |
| 切换-渐变 | “视线从他脸上滑开，落在窗外，声音也跟着变轻” | 临界点→余波 |
| 余波-呼吸 | “话音落下后，她吸了一口气却没有吸满，手背抵到唇边又放下” | 余波 |
| 余波-手部 | “她的手指在袖下松开，指尖却还在微微发颤” | 余波 |

详细强度梯度定义见 [performance-appendix.md](../appendix/performance-appendix.md#附录c情绪强度梯度指引)。

#### 执行边界

- 时序序列的每个阶段必须有上游信号支撑，不得凭空插入阶段。
- 前段压制和临界点泄露之间的比例反映角色控制力；控制力越强，压制越长，泄露越少。
- 余波是可选阶段，仅在需要延展情绪张力或为下一 beat 做铺垫时使用。
- 连续使用同一微动态变量（启动-停顿、启动-收回等）超过两次时，必须更换变量类型。

## Dialogue Delivery Control

台词表演是演员控制系统的一部分。每段对白都必须明确“怎么说”，不能只保留台词字面或抽象状态。对白保真不等于表演空白；引号内不改，字段标题和画面承托必须补足表演。

### Required Variables

| variable | question | projection |
| --- | --- | --- |
| `tone_state` | 这句话的语气是什么，角色表面呈现什么状态 | `对白（角色，语气/状态短语）` 第二项 |
| `emotional_pressure` | 语气下面压着什么情绪或动机 | `对白画面`、`心理反应`、`actor_performance_control_evidence` |
| `breath_point` | 开口前、句中或收尾时气息在哪里断、收或漏 | `对白画面`、`音效画面` |
| `pause_pattern` | 断句停在哪里，哪一段被压住或让出去 | `对白画面` |
| `stress_or_release` | 重音落在哪个词/信息点，尾音如何收束 | `对白（角色，状态）`、`对白画面` |
| `paired_body` | 说话时身体哪里泄露真实情绪 | `表情特写`、`角色动作`、`对白画面` |

### Projection Rules

- `对白（角色，状态）` 第二项必须包含可执行台词状态，不得省略为 `对白（角色）`。
- 状态短语优先写成“语气 + 情绪/控制 + 节奏”组合，例如 `压低声音，怒意收在齿间`、`强作轻快，尾音发虚`、`先冷后软，中段停顿`。
- `对白画面` 必须就近承托关键对白的气口、断句、声线、呼吸、身体联动或对手反应；不得只复述“他说得很生气/很悲伤”。
- 若一句对白内有明显转折，必须写出前后段处理：前半句稳住、某个词前停顿、后半句放轻、尾音断掉或收住。
- 台词表演不得把潜台词写进引号内；未说出口的信息只能通过停顿、声线、气息、动作或反应外显。

弱写法：

```md
对白（阿真，紧张）：“没事。”
```

强写法：

```md
对白（阿真，强作平稳，尾音发紧）：“没事。”
对白画面：她开口前先压下一口气，“没”字落得很轻，尾音却收得过快；手指还停在杯沿，没有松开。
```

## Environment Sound Support

环境声是表演承托，不只是背景。关键沉默、离别、压迫或紧张场必须考虑是否需要声音层帮助观众 GET 到情绪。

### 通用场景声音矩阵

| scene emotion | sound support candidates |
| --- | --- |
| 离别 | 远处车声、脚步声渐远、风吹树叶、门轴轻响 |
| 紧张 | 钟表、低频嗡鸣、短促吸气、衣料摩擦、纸张声消失 |
| 孤独 | 空房间回声、雨声、冰箱/电流低频、窗框轻响 |
| 希望 | 远处孩童笑声、清晨鸟鸣、风声变轻 |
| 焦躁 | 键盘/打字声、电话铃、急促脚步、杂声叠加 |
| 压迫 | 群体忽然安静、杯底磕桌、金属轻响、呼吸被压低 |

### 朝代/空间多版本声音表

详细分场景声音表见 [performance-appendix.md](../appendix/performance-appendix.md#附录a场景情绪声音表按朝代环境分类)。以下为快速检索入口：

| 场景类型 | 适用附录 | 压力来源声音 |
|---------|---------|-------------|
| 古代室内（厅/堂/房） | A1古代·室内 | 更漏声、烛芯爆炭、衣料摩擦 |
| 古代室外（街/路/野） | A1古代·室外 | 远处更鼓、风声忽变、脚步忽远忽近 |
| 现代室内（教室/办公室/家） | A2现代·室内 | 钟表滴答、键盘敲击、纸张声消失 |
| 现代街道/公共空间 | A2现代·街道 | 车流声忽变、远处警笛、脚步忽快忽慢 |
| 架空/奇幻室内 | A3架空·室内 | 魔法屏障颤动、符文闪烁不定 |
| 架空/奇幻室外 | A3架空·室外 | 异象浮现、天空颜色异常 |
| 系统/规则压迫场 | A3架空·系统 | 系统警告闪烁、数据流加速 |

### 声音使用决策规则

使用限制：

- `音效` 写声音本体，`音效画面` 写声音和人物反应/空间承托。
- 环境声不得新增事件、人物、线索或外部因果。
- 能用环境声表达的沉默，不要新增对白解释。
- 选择声音时必须回指当前场景的具体情绪压力源，不得泛化使用。
- 同一情绪在不同朝代/空间应选择对应版本的声音变量。
- 环境声不是 BGM 标签。它应来自当前场景身份和空间材质，例如年代明确的压迫空间可有电流声，当代私人房间可有门外闷响，旧式居住空间可有管线声；不能为了情绪随意贴“悲伤音乐/紧张音乐”。

### 7. Role-Specific Writing For Multi-Character Scenes

多人场景不能只有主角的戏。写多人场景时，先分配每个角色的权重，然后为每个角色设计动作。

| 角色类型 | 戏份权重 | 写法要求 |
| --- | --- | --- |
| 主角 | 核心驱动 | 完整动作链、心理反应、对白、内心独白 |
| 对手/关键配角 | 次级驱动 | 动作反应、对白、对主角的判断 |
| 群体 | 环境承托 | 群像反应、集体动作、差异化层次 |

多人场景检查清单：

- 每个关键角色在这个 beat 里是否有明确动作或反应？
- 群体的反应是否有 2-3 个差异化层次，而不是全员同频？
- 主角的对手是否在观察/回应主角的行为？
- 动作链是否在多角色间形成因果，而不是并列？

弱写法（只有主角）：

```md
角色动作：林寂握紧拳头。
```

强写法（主角+对手+群体）：

```md
角色动作：林寂的手慢慢握紧，指节发白。
群像画面：前排有人转过头，又迅速低回去；后排几个人的脚往课桌下缩了半寸。
对白画面：苏红叶的视线在林寂手上停了一拍，嘴角微微一动，又收回去。
```

### 8. Action Chain Breakdown Method

动作链把一个复杂行为拆成可执行的最小单元。每个关键动作写进终稿前，必须能回答：

1. **起点状态**：人物从什么姿态、站/坐/蹲/躺开始
2. **动作向量**：正在向哪里移动、转身、伸手、停住、退开
3. **可达性**：若触碰对象，对象是否在空间可达范围内
4. **互动方式**：如何接触、取用、避开或被阻碍
5. **终点状态**：动作结束后姿态、位置、朝向、身体接触

弱写法：

```md
角色动作：林寂走上前。
```

强写法：

```md
角色动作：林寂从座位起身，绕过前排同学的椅背，在第三排空位前停住脚步；他的脚尖朝前，但身体还没有完全转过来。
```

## Projection Targets

表演控制结果必须进入既有字段，不新增 `演技提示词`、`表情控制公式`、`演员调度公式` 等正文第二体系。

| performance material | target fields |
| --- | --- |
| 微表情 | `表情特写`、`对白画面`、`心理反应` |
| 身体联动 | `角色动作`、`动作画面`、`对白画面`、`心理反应` |
| 台词语气/情绪 | `对白（角色，状态）` 第二项、`对白画面` |
| 气口/断句/重音/尾音 | `对白（角色，状态）` 第二项、`对白画面`、`音效画面` |
| 声线/呼吸 | `对白（角色，状态）` 第二项、`对白画面`、`音效画面` |
| 道具动作 | `角色动作`、`道具特写`、`对白画面` |
| 环境声 | `音效`、`音效画面`、`环境描写` |
| 情绪动机 | 规划证据、`内心独白（主角）`、可见反应；不得写成动作字段主观意图 |
| 微动态限制 | `角色动作`、`对白画面`、`表情特写` 中的停顿、压住、收回、变轻 |

## Review Checklist

- 情绪是否有上游触发点，而不是凭空贴标签。
- 是否至少给出一个微表情变量和一个非面部身体变量；关键情绪场建议两类以上。
- 是否能看出表层情绪、压制情绪或隐藏动机中的至少两层。
- 是否用呼吸、手部、肩颈、重心、声线、道具或空间距离参与表演。
- 是否抓住情绪切换瞬间，而不是持续挂同一张模板表情。
- 是否用环境声或沉默承托关键情绪，而不是只依赖 BGM 或解释性文字。
- 是否避免模板化表情：只写“皱眉、瞪眼、流泪、大笑”但缺少动机和微动态时不合格。
- 是否避免过演：没有上游依据时，不写崩溃、大喊、泪流满面、夸张扭曲。
- 是否避免摄影越权：不写镜头推进、特写、慢镜、快切等摄影方案；只写演员和声画材料。
- 是否全部内嵌到既有字段，没有新增第二套正文体系。
