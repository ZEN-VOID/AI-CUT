# Episode Split Heuristics

- P1 集标是硬边界；没有 P1 时，章节/回目是默认集边界。`第N章`、`第N回`、chapter、回目和 story 章节文件名默认一章/一回一集。
- 默认字数窗只用于连续正文或用户要求重组，不应凌驾于章回完整性之上。
- 多文件小说源常见风险是排序错误，尤其是 `1.md`、`10.md`、`2.md`；执行前要自然数字排序。
- 标题、frontmatter 和报告可以规范化；正文内容不应被规范化成另一种文体。
- 若用户指定外部小说原文路径但没有项目名，应先确认或从当前项目上下文推断项目根；无法推断时只给出切分计划，不写项目产物。
