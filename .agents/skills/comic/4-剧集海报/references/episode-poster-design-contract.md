# 剧集海报设计合同

本文件补充 `.agents/skills/comic/4-剧集海报/SKILL.md` 的细则，不构成第二真源。

## 0. 上游真源回读规则

进入 4 号剧集海报技能时，不得把海报当成脱离项目的独立平面设计任务。

至少要回读：

- `1-漫画剧本改编/第*.组.md`
- `2-九刀流漫画提示词/page-group-*.json`
  - 多集项目优先升级为 `2-九刀流漫画提示词/第N集-page-group-*.json`
  - 仅 legacy 项目才回退读取 `nine_blade_comic_prompts.json` / `第N集-nine_blade_comic_prompts.json`

推荐同时回读：

- `2-九刀流漫画提示词/思考过程摘要.md`
  - 多集项目优先升级为 `2-九刀流漫画提示词/第N集-思考过程摘要.md`
- `3-漫画生成/` 已存在输出

回读后至少要沉淀三类信息：

- 角色阶段锚点：当前集人物处在什么阶段，不能偷渡后续形态
- 场景连续锚点：当前集哪个场景或哪类空间最能代表这一集
- 风格锚点：上游已经锁定的漫画风格、线条气质、光影语法和世界观味道
- 类型包锚点：当前集活跃 pack 组合，以及 `episode_poster` 阶段偏好的标题机制、画面气质和传播钩子方向

## 1. 构成元素清单

每份 `comic_episode_poster_design.v1` JSON 至少覆盖：

- 创意：一句话创意核心，说明这张海报卖什么。
- 画面：与剧情强相关的代表性画面，而不是无剧情的 pose。
- 主体：仅使用当前剧集中实际出场、且足以承担传播重心的角色。
- 前景：制造钩子、压力或层次的物件/动作/遮挡。
- 背景：交代空间、世界、威胁、关系或后果。
- 文字：
  - 默认水平+垂直双居中。
  - `第N集` 字号略小。
  - 大标题更醒目，默认中文，强调钩子意义。
- 氛围：观众第一眼感受到的情绪方向。
- 配色：主色、强调色、对比关系、光线方式。
- Prompt：能继续交给 `.agents/skills/cli/imagegen` 或设计执行的完整描述。
- Imagegen handoff：记录后续生图工具、prompt 来源、建议输出目录与持久化策略。

## 2. 主体选择规则

- 优先选择本集“行动最贵”的角色，而不是全剧最热门角色。
- 若为双主角对抗/拉扯集，可使用 2 名主体，但仍需有明确主次。
- 若是群像集，也必须有一个主导关系，而不是平均排排站。
- 若分组剧本、九刀流 JSON 与海报主体冲突，以“本集实际出场 + 本集冲突核心”优先。

## 3. 代表性画面规则

代表性画面应满足至少 2 项：

- 能让人立刻感到“本集出事了”
- 能对应本集最大冲突、秘密、代价或欲望
- 有明确的空间/前景/动作关系
- 能支撑标题钩子，而不是靠标题单独发力

不建议：

- 纯角色站姿
- 无空间信息的脸部特写
- 与当前集无关的结局级剧透画面
- 只有情绪没有事件的抽象背景

## 3.1 剧情高光点发现机制

海报不得直接拍脑袋选“最漂亮的一幕”。必须先列出 `3-5` 个高光候选，再筛选。

候选来源优先级：

1. `第*.组.md` 中已经写明的高冲击画面、组末钩子和冲突高潮
2. `九刀流 JSON` 中的 `story_beat_map / page_role / main_character_lock / scene_continuity_bible`
3. `主稿` 和 `思考过程摘要` 中已经被确认的命题转折

筛选维度：

- 剧情命题价值：这一下是不是本集真正改变人物命运或关系的地方
- 视觉冲击力：能不能用单张海报成立
- 传播钩子：标题和画面能不能互相咬住
- 风格承接：这张海报是不是还像这个项目已经长出来的作品

最终应写明：

- `candidate_highlights`
- `selected_highlight`
- `selection_reason`

## 4. 标题与文字系统

推荐层级：

1. 钩子大标题：醒目、可传播、与本集事实相关
2. `第N集`：双轴居中，字号较小，作为剧集索引

定位硬规则：

- `text_system.horizontal_alignment = center`
- `text_system.vertical_alignment = center`
- `episode_label.position.horizontal = center`
- `episode_label.position.vertical = center`
- `hook_title.position.horizontal = center`
- `hook_title.position.vertical = center`

说明：

- 这里的 `center` 不是“只在横向中线”，而是标题栈整体压在画布几何中心附近。
- 若需要细微上下错位，也应围绕中心组排，而不是把标题整体顶到上三分之一。
- 若用户显式提供标题文字，优先直接作为 `hook_title.text` 使用；只允许为适配海报排版做最小必要的裁切、断句或压缩，不得改写其核心语义。

标题公式可从以下方向择一：

- 秘密型：`她听见门后有人`
- 代价型：`这一刀要有人偿命`
- 关系型：`他终于不装了`
- 威胁型：`今晚谁也别想出去`
- 反差型：`笑着的人先崩了`

避免：

- 纯摘要句
- 与剧情无关的热词堆砌
- 直接泄露结局反转

## 5. Prompt 写法建议

正向 prompt 至少要包含：

- 海报类型与比例
- 主体人物及其关系
- 代表性画面动作/瞬间
- 前景/背景与空间层次
- 文字系统与双轴居中规则
- 氛围与配色
- 风格限制与上游风格连续性

后续生图工具固定交接到 `.agents/skills/cli/imagegen`。本技能只负责输出海报设计 JSON 与可生图 prompt；若用户要求生成海报图，必须先通过本技能 validator，再加载 `.agents/skills/cli/imagegen/SKILL.md + CONTEXT.md` 执行。

负向 prompt 优先限制：

- 未出场角色
- 平面拼贴式海报
- 与剧情无关的泛用 pose
- 过多小字
- 标题偏离双轴居中层级
- 风格突然跳成另一套作品语言

## 6. 路径与批量规则

- 单集默认路径：`projects/comic/[项目名]/4-剧集海报/第N集-剧集海报.json`
- 多集场景：一集一个 JSON，不把多集海报揉成一个大文件。
- 若批量设计多集，可额外生成索引，但单集 JSON 仍是 canonical output。
- 海报生图结果默认由 `.agents/skills/cli/imagegen` 持久化到 `projects/comic/[项目名]/4-剧集海报/imagegen/` 或用户指定的项目目录。
