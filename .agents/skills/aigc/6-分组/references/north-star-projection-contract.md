# North Star Projection Contract

本文件定义从 `projects/aigc/<项目名>/0-初始化/north_star.yaml` 向每个分镜组组头投影风格字段的规则。

## Required Fields

每个分镜组组头必须投影三项，但输出时不显示字段标题：

| output order | north_star.yaml key | output rule |
| --- | --- | --- |
| 第 1 行 | `全局风格.全局风格提示词` | 在行首前置固定词：`视频生成的画面风格，光影和氛围与场景参照图保持一致。需要生成现场物理互动音效、氛围感音效、环境声、自然现象声、动作声，不要生成任何字幕，不要生成背景音乐。`，再接原文 |
| 第 2 行 | `类型元素.类型元素提示词` | 直引原文 |
| 第 3 行 | `细分风格.画面风格` | 直引原文 |

直引优先级高于摘要。除第 1 行固定前置词外，不得把三项字段改写成概括句，也不得把项目 `MEMORY.md` 或 `CONTEXT/` 的风格偏好替代 north_star 三项。

## Output Lines

每个分镜组标题后先写场景标题行，再写 `定场镜头：` 和 `画面属性：`，随后组头固定写成三行：

```text
视频生成的画面风格，光影和氛围与场景参照图保持一致。需要生成现场物理互动音效、氛围感音效、环境声、自然现象声、动作声，不要生成任何字幕，不要生成背景音乐。 <全局风格.全局风格提示词原文>
<类型元素.类型元素提示词原文>
<细分风格.画面风格原文>
```

不得输出 `[全局风格]`、`[类型元素]`、`[画面风格]` 等可见标题字段；不得使用中文括号包裹字段内容；不得在三项之间添加逗号、顿号、连接词或其他装饰性拼接符。第 1 行固定前置词必须出现在全局风格原文之前，不得单独成第 4 行，也不得加到第 2 或第 3 行。若原字段是 YAML 列表，应保持原列表语义，可用分号连接列表项，但不得删改内容。若字段是多行字符串，应保留主要换行或用空格拼接成可读单行，并在执行报告说明投影方式。

## Missing Field Handling

- 三项任一缺失时，不得猜测补写。
- 可读取相近字段时，只能在执行报告中列为候选，不得静默替代。
- 需要继续执行时，应先修复 `north_star.yaml` 或取得用户明确授权。

## Word Count Handling

north_star 三项与第 1 行固定前置词不计入组内 `时长估算`、1680 字目标上限、1980 字硬上限或 YAML `字数统计`。每组标题后的场景标题行、定场镜头、画面构图、构图分区和画面属性计入 YAML `字数统计`，但不计入 `时长估算`。组内时长只累计从 `5-摄影` 划定正文里的 `分镜N（约X秒）`；1680/1980 只作为场景标题行、定场镜头、画面构图、构图分区、画面属性加纯分镜剧本正文的辅助风险线。即便 north_star 三项较长，也不得摘要 north_star 或删除固定前置词；边界裁决只能回到分镜剧本正文的完整 atomic unit 与显式时长累计。
