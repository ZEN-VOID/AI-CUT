# Grouped-Files 输出模式

## Match Signals

- 用户给出项目名、目标目录、要求落盘，或 comic 父级进入完整 pipeline。

## Fixed Context

- 默认落点：`projects/comic/<项目名>/1-漫画剧本改编/`。
- 文件命名：`第1组.md`、`第2组.md`、`第3组.md`，连续递增。
- 默认以原文约 `1000` 字为一组；不满 `1000` 字仍一组。
- 尾组规则：
  - `<=300` 字默认并入上一组。
  - `>=700` 字可单独成组。
  - `301-699` 字默认并入上一组；只有明确场景闭合、钩子闭合或强节奏边界时才允许独立。
- 每组必须写明 `估算原文字数`、`尾组决议` 与 `【边界判定】`。

## Review Gate

- 文件级和目录级 validator 都通过。
- 下游 `2-九刀流漫画提示词` 可逐组消费，不需要回到整篇原文重新切组。
