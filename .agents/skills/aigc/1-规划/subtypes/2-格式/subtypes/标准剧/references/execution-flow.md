# 标准剧 / Execution Flow

本文件是 `标准剧` 的局部执行流程真源。

## Phase Flow

```mermaid
flowchart TD
    A["Phase 1: 读取父级裁决"] --> B["Phase 2: 锁定表演优先边界"]
    B --> C["Phase 3: 转写场景标题与字段骨架"]
    C --> D["Phase 4: 生成 2-格式/第N集.md"]
    D --> E["Phase 5: 按需 sidecar"]
```

## Atomic Steps

1. 读取父级裁决与上游种子。
2. 确认本轮不需要解说剧式旁白主导。
3. 先依据 `story-source-manifest.yaml` 或 `metadata.source_profile` 锁定场景编号粒度；若命中混合源/分镜源且锁定了 `scene_boundary`，则场景编号只按连续时空递增。
4. 若上游可解析出原场景标题或镜头块标题，优先复用这些结构证据，不得先清洗成泛化标题。
5. 若上游明确写出运镜/镜头提示，优先整理为紧跟相关 `*画面` 的 `镜头语言预设：...`；未明确写出时，不得脑补新增。
6. 把原文按标准剧格式转写为 `2-格式/第N集.md`，并显式写出 `镜号范围 / 锚点继承`。
7. 对分镜源，画面处理重点是规范化整理既有结构，而不是普通叙事源式补齐。
8. 确保场景标题、动作画面、对白、对白画面、镜头语言预设等字段可直接阅读。
9. 仅在调试/复核时保留 sidecar。

## Fallback

- 发现高旁白密度要求：回到父级重新判模。
- 样例只能靠旁白解释：回到 `Pass 2` 重做字段承载分配。
- 字段缺少 `*画面` 配对：回到 `Pass 3` 补齐。
- 分镜源镜头语言被删除、挂错位置或脑补新增：回到 `Pass 5` 重做镜头语言保留。
