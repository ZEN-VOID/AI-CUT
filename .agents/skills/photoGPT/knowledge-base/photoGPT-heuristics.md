# photoGPT Heuristics

## Stable Lessons

- 图像编辑 prompt 的第一优先级是“类型定位”：先落到五大类十四子类，再标注图片角色。
- `元素替换` 要显式说“仅替换 X”，同时列出“Y/Z/W 严禁改变”。
- `修图` 默认轻量自然；过度审美词容易诱发重绘，尤其会误入 `风格化`。
- `多视图` 必须把 layout 当成生产规范，而不是普通风格描述。
- `多图融合` 必须指定每张图的职责；商品/主体/场景/构图/风格不能混成一团。
- `风格化/滤镜` 与 `风格化/风格迁移` 的区别要写清：前者偏后期调色，后者偏视觉语言迁移。
- `photoGPT` 不拥有 provider 参数真源；调用、落盘、透明背景和 fallback 都回到 imagegen。

## Failure Patterns

| pattern | prevention |
| --- | --- |
| 参考图主体被复制到输出 | 写清参考图只提供对应子类需要的背景/服装/身份/风格/构图，不复制无关主体 |
| 输出像拼贴 | 增加光线、阴影、边缘融合和透视一致性 |
| 多视图每格形态不一致 | 增加同一身份/同一道具/同一场景锚点跨面板一致 |
| 电商广告商品被改包装 | 将商品图列为 `product_reference`，锁定 logo、包装结构和关键文字 |
| 分镜构图变成多格漫画 | 明确输出是单一画面而非多格漫画，构图参考只提供机位和画面重心 |
| 滤镜变成重绘 | 将 change_scope 限制为色彩、影调、颗粒、对比度和后期质感 |
| prompt 过长但无约束 | 使用 prompt plan 字段检查，不靠字数判断质量 |
