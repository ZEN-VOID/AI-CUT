# Usage Protocol

## 1. Agent 平时默认从哪些章节取材

- 默认先取 `core-concepts.md`，获取改良沪语、闲谈推进、不响留白和块状群像的骨架
- 其次取 `source-map.md` 中的两篇中国作家网官方访谈
- 当任务明确涉及对白压力或场景关系时，再调仓内 `一流对话场景.md` 与 `电影导演方法.md`
- 只有需要做基础身份确认时，才读取奖项页

## 2. 什么任务必须先读知识库再发言

- 需要判断“这段为什么不像《繁花》原著”
- 需要重写对白、闲谈、群像场景
- 需要决定是否该补心理描写，还是该让上下文与不响承担
- 需要判断地名、器物、称呼和时代细部该如何入场
- 需要区分原著小说方法和电视剧改编语法

## 3. 哪些内容可以成为公开表达依据

- 改良沪语的原则是保留韵味、避免阅读障碍
- 原著重视文本样式、块状结构与传统叙事资源
- 不宜用全知视角把人物内心全部说透
- 城市细部、器物、称呼和生活杂质是叙事引擎，不是装饰
- 小说与影视的语法不同，原著判断不能被改编效果反向吞掉

## 4. 哪些内容只能作为潜台词、风格支撑或黑盒背景

- 对“假老上海腔”的尖刻判断
- 对用户文学感觉和细部控制力的私人评估
- 证据尚不足以支撑的逐段考据式断言
- 对电视剧《繁花》的好恶，不直接拿来替代原著判断

## 5. 何时必须克制，何时禁止硬套

- 如果用户任务是电视剧镜头设计或美术风格，而不是小说创作，应降低本知识库的优先级
- 如果当前段落并不依赖上海语境，就不要硬挂地名和沪语词
- 如果材料不能支撑精确句法、版本差异或作者手稿动机，不准装作见过原文依据
- 如果项目已有更强的风格合同，金宇澄知识库只能做增强层，不能越权覆盖主合同
- 如果加入方言会制造阅读障碍，应退回“改良沪语”而非生硬模仿

## 6. 输出纪律

- 先给小说判断，再给可执行动作
- 先说“哪里没有生活”，再说“如何补回生活”
- 不把引用写成炫耀，而是转成这次任务真正有用的判断
- 一旦涉及《繁花》原著与电视剧边界，必须显式标明当前依据站在哪一侧
