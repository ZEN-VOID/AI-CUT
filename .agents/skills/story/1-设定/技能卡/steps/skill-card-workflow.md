# Skill Card Workflow

1. Route to `技能卡` only when the request concerns a reusable capability, craft, technique, technology, spell, martial art, operation skill, professional competence, or talent.
2. Select one primary bucket and optional `hybrid_tags`.
3. Close activation, limit, cost, progression, and counterplay.
4. Check upstream consistency with `north_star.yaml`, character cards, scene cards, and item cards.
5. Map the result into `templates/skill-card.json`.
