# 正剧 Presentation Type

本类型卡定义 `正剧` 呈现方式在 `4-编剧` 中的策略偏置。它不是子技能，不拥有输出路径、节点、gate 或最终审查权。

## Identification

| field | value |
| --- | --- |
| `type_id` | `presentation-zhengju` |
| `axis` | `presentation_axis` |
| `screenplay_mode` | `zhengju` |
| `aliases` | 正剧、影视化正剧、当前编剧方式、默认剧本 |
| `default_rule` | 无显式 `解说剧` 信号时默认选择 |

## Strategy

| strategy_slot | rule |
| --- | --- |
| `voice_policy` | 陈述性 source 可转为动作、对白、独白、内心独白、旁白、音效、道具证据或场面反应，但新增语音必须有 source anchor、合法 voice owner 和画面承托。 |
| `source_unit_policy` | 不强制逐单元覆盖全 source，但关键事实、既有对白、事件顺序和关系变化必须进入 `source_to_script_map`。 |
| `field_variety_policy` | 优先用正式画面字段和声音字段共同推进戏剧，不用旁白兜底解释。 |
| `dramatization_policy` | 可以执行父级允许的受控影视化补写，必须回接 `dramatic_intent_map`、`dramatization_gap_map`、`controlled_adaptation_plan` 和 `rewrite_scope_check`。 |
| `report_evidence` | `Screenplay Mode Decision`、`Narration To Voice Adaptation Map`；无派生旁白或派生语音时写 N/A reason。 |

## Prohibited Drift

- 不得把“正剧”理解成可以自由新增对白、动机、反转或人物选择。
- 不得把小说陈述直接堆成旁白清单。
- 不得因为题材是武侠、玄幻、科幻或魔幻而新增 source 不支持的能力、规则、道具功能或线索。

## Combination Notes

`正剧 x 题材类型` 的组合重点是：用题材材料增强可见冲突、节奏承托、高潮和尾钩，但仍由父级保真合同控制改写边界。
