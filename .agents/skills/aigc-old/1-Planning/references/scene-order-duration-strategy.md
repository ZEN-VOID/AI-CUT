# 场景顺序与时长策略投影

## 角色定位

- 本文件是 `3-分组` 对参照源 `/Volumes/AIGC/AIGC-ZEN-VOID/.agents/skills/aigc2026/2-导演/3-拍摄段落/SKILL.md` 中“场景顺序与时长策略规则”的规划层投影。
- 真正的导演阶段镜头化、场景标题、空间主键、主体抽取、分镜字段仍由 `3-拍摄段落` 或其后继技能承担。
- `3-分组` 在当前阶段只继承“如何量化判断该不该拆、能不能并、是否过载”的机制，不继承导演字段本体。
- 本文件是完整方法论真源；`SKILL.md` 负责 executable digest，`scripts/grouping_quantizer.py` 与 `validate_grouping_output.py` 负责计算与校验落地。

## 术语边界

- `上游场景单位`：`2-剧本` 主稿（runtime: `2-格式/第N集.md`）中的 `### 场景N：...` 标题和其正文，是排序与引用锚点；`3-分组` 只能原样继承。
- `分镜组`：`3-分组` 产物中的 `## 【episode-scene-group】 <分组名>`，是规划阶段的节拍/字窗/组界单位；分组名不得冒充场景标题。
- `物理场景链`：同一空间或连续空间关系的分析证据，只用于判断能否拆并组；它不改写上游场景单位标题，也不替代后继导演阶段的空间主键。
- `锁定分镜组`：已被用户、`2-Global`、`3-Detail` 或四段式 `分镜ID` 消费的 `分镜组ID`；它是 downstream anchor，不得因 `warn_low` 过轻而自动并入相邻组。

硬区分：

- `### 场景N` 不是天然分镜组。它只保证顺序和回指，不保证 15 秒字窗、节奏密度或下游可拍负载。
- 若“一场景一组”的输出中存在任何组低于 `warn_low` 或高于 `warn_high`，必须回到拆并组检查；不得用“场景完整 / 场景匹配 / 原样继承”作为通过依据。
- 只有当每个一场景组都处于 `warn_window` 内，或存在下游锁定锚点/用户显式豁免时，一场景一组才可作为最终分组形态。

## 继承原则

1. 先上游场景单位顺序，后组内切分：
   - 若输入里已经存在场次、空间切换或可恢复的场景链，必须先锁定场景顺序，再讨论分组。
   - 场景顺序必须与上游文本出现顺序一致，不跳段、不倒序；但若 `2-剧本` 因相同 slugline 复用同一场景号，分组中允许该场景号在文本后段再次出现，不能把编号回返误判为文本倒序。
   - 同一空间连续叙事，不因单纯事件变化而机械切新组。
   - 上游 `### 场景N：...` 标题必须逐项保真；若分组横跨多个上游场景单位，正文中仍保留每个原始场景标题。
   - 若分镜组已经被下游 `分镜ID` 锁定，禁止把“同一空间连续叙事”反向解释成必须并组。
2. 先时长策略，后负载均衡：
   - 先求生效时长与节奏字窗，再看结构断点与依赖。
   - `group_load_score` 只能做规划摘要，不能推翻时长与字窗硬门槛。
3. 先上游 preset/style，再用默认值：
   - 用户自然语言 > 局部策略 > 全局策略 > 项目预设 > 默认 `15 秒`

## 场景类型字典（继承）

- 仅允许三类主类型：
  - `室内`
  - `室外`
  - `野外`
- 归并规则：
  - 历史 `水域` / `空域` 统一并入 `野外`
  - 历史 `过渡连接空间` 默认并入 `室外`，若明确命中室内锚点则并入 `室内`
- 判定优先级：
  - 明确空间锚点 > 方位词（内/外/上/下）> 动作语义

## 时长策略优先级（继承）

1. 用户自然语言局部策略
2. 用户自然语言场景级策略
3. 用户自然语言全局策略
4. 项目 preset / `director.segment` 生产约束
5. 默认 `15 秒`

若自然语言策略含糊且无法安全标准化，回退到默认 `15 秒`，并在输出中标明回退原因。

## 节奏与字窗公式（继承）

- 节奏档位：
  - `慢节奏 = 0.7`
  - `中节奏 = 1.0`
  - `快节奏 = 1.3`
- 基准字窗：
  - `base_text_window = duration_seconds * 10 * pace_coefficient`
- 容差带：
  - `warn_low = round(base_text_window * 0.8)`
  - `warn_high = round(base_text_window * 1.0)`
  - `hard_text_window = round(base_text_window * 1.1)`
- 15 秒参考：
  - `慢节奏: 105 / 84-105 / 116`
  - `中节奏: 150 / 120-150 / 165`
  - `快节奏: 195 / 156-195 / 215`

## 有效字数（继承）

- 若上游文本已具备字段化正文，则按以下系数计算 `effective_text_chars`：

| field_type | 匹配规则 | 系数 |
| --- | --- | --- |
| `voice_text` | `对白(*)` / `独白(*)` / `内心独白(*)` / `旁白(*)` / `【旁白】` | `1.2` |
| `voice_visual` | `对白画面` / `独白画面` / `内心独白画面` / `旁白画面` | `0.0` |
| `sound_visual` | `音效画面` | `0.3` |
| `action_visual` | `动作画面` 及未命中上述标题的段落 | `慢 0.7 / 中 0.5 / 快 0.3` |

- 计算公式：
  - `effective_text_chars = Σ(各段去空白字数 × 对应字段系数)`
- 若当前阶段还没有字段化正文：
  - 允许使用“原文可见字数近似值 + 场景单位数 + 转折点 + 强依赖”做规划级估算。
  - 但必须在报告中标注“当前为规划估算，不是导演阶段精算”。

### 混合源 / 分镜源下的强制回算

当 `primary_story_source.source_type in {storyboard_script, hybrid_story_text}` 且当前组 `source_span` 能解析出镜号范围时，`effective_text_chars` 不再允许手填估算，必须按故事主源正文回算。

硬规则：

1. `source_span` 必须保留可机读镜号范围，例如 `镜1-5`、`镜11`、`镜12-13`。
2. validator 应优先从 `projects/aigc/<项目名>/0-Init/story-source-manifest.yaml -> primary_story_source.path` 读取主源，再按镜号范围复算 `effective_text_chars`。
3. 若回算值与产物中填写值不一致，不得以“只是规划阶段估算”为由继续落盘。
4. 只有在主故事源不是 `storyboard_script / hybrid_story_text`，或当前组确实无法映射到镜号范围时，才允许退回“规划级估算”路径。

## 分组时间切分交接协议

本节承接的是“下游可消费的时间切分真源”，而不是要求 `3-分组` 在当前阶段直接产出帧级分镜表。

### 组总时长真源

1. 每个分镜组默认总时长为 `15秒`，只有命中显式上游时长证据时才允许偏离。
2. 当项目存在非均匀组时长时，必须在 episode 主文件 frontmatter 中显式补写：
   - `默认组时长`
   - `分镜组时长映射`
   - `时长偏离证据`
3. `分镜组时长映射` 只登记偏离默认值的组；无偏离时写 `{}`。
4. 若 `分镜组时长映射` 非空，`时长偏离证据` 必须同时登记非空证据；无证据不得偏离默认值。
5. `3-分组` 当前阶段的 `estimated_duration_seconds` 是组总时长的规划层投影，必须与 `分镜组时长映射 -> 默认组时长` 解析结果一致。
6. 下游 `5-分镜构图` 与其 validator 的组总时长读取链固定为：
   - `分镜组时长映射 -> 默认组时长 -> 切分时长策略`
7. 禁止把“整集 100 秒”误判为“每组 100 秒”；组总时长必须始终在组级基线上解析。

### 帧级时间字段格式

- 格式固定为：`X-Y秒`
- 第一个分镜必须从 `0` 开始，例如 `0-3秒`
- 最后一个分镜必须覆盖到组总时长结束，例如 `12-15秒`
- 相邻分镜必须连续无间隙：前一帧结束秒 = 后一帧起始秒

### 场景类型化单帧时长策略

| 场景类型 | 单帧时长范围 | 分配策略 |
| --- | --- | --- |
| `dialogue` | `2-5秒` | 说话帧偏长（`3-5秒`），反应帧偏短（`1-2秒`） |
| `action` | `1-3秒` | 高密度短帧为主，关键动作帧可拉长至 `3秒` |
| `crowd` | `1-4秒` | 全景建立帧偏长（`3-4秒`），局部快切偏短（`1-2秒`） |
| `inner` | `3-6秒` | 意象帧偏长，允许单帧超 `5秒` 营造沉浸 |
| `emotion` | `2-5秒` | 情绪蓄力帧渐长，爆发帧骤短 |

### 节奏联动

- `超慢 / 慢节奏`：单帧均值偏向区间上限
- `中节奏`：均匀分配，允许 `±1秒` 波动
- `快 / 超快节奏`：单帧均值偏向区间下限，允许出现 `1秒` 极短帧

### 硬约束与失败码

1. 单帧最短 `1秒`，普通组单帧最长 `8秒`。
2. 若 `panel_count=1` 且命中 `single_panel_long_take` 特例，则允许忽略“单帧最长 `8秒`”建议上限；该单帧必须完整覆盖组总时长，且需在裁决理由中显式说明。
3. 所有帧时长之和必须等于组总时长。
4. 下游失败码：
   - `FAIL-TIME-CONTINUITY`：时间不连续
   - `FAIL-TIME-OVERFLOW`：总时长溢出

## 规划阶段判定规则

1. 若某候选组 `effective_text_chars > hard_text_window`：
   - 默认视为过载，不得以“结构正好到这里”为由保留不拆。
2. 若某候选组 `effective_text_chars` 落在 `warn_high` 以上但未超 `hard_text_window`：
   - 必须优先检查最近可见结构断点或语义转折点能否拆开。
   - 若同一上游场景单位内部已出现明确的二段或三段 beat 链，必须先执行同场景内拆组试算，再决定是否保留为单组；不得直接用“连续峰值”跳过拆组。
   - 若首轮同场景拆组后，某个子组内部仍可辨识出新的二段或三段 beat 链，必须继续递归复查该子组；不得因为已经“拆过一次”就停止。
3. 若某候选组 `effective_text_chars < warn_low`：
   - 必须优先检查能否与同一物理场景链或同一上游场景单位内的下一语义单元合并。
   - 若该候选组已有固定 `分镜组ID`、固定镜数、beat 或四段式 `分镜ID`，则保留组界，并在 `judgement_basis` 登记锁定依据。
   - 若该低字数组是同场景 beat 拆分后的结果，只要它本身承载完整且不可压缩的子拍点，可保留；但 `judgement_basis` 必须说明“已先做同场景拆组检查”，而不是只写“独立信息落点”。
   - 若保留结果来自递归拆组后的子组，`judgement_basis` 还应明确写出这是“复查子组后保留”的结果，避免把首轮粗拆误写成最终边界。
4. `warn-low / warn-high / error` 只允许存在于候选分析与返工判断，不允许直接落入正式 `第N集.md` 作为通过结果。
5. 若 `hard_lock / soft_lock` 与严格量化 gate 冲突，当前轮应判为失败并上溯请求显式豁免；不得再以“先保锚点，再解释偏差”方式直接落盘。
6. 默认禁止跨物理场景链凑时长：
   - 相邻上游场景单位若仍处于同一连续物理场景链，可在保留原 `### 场景N` 标题的前提下并入同一分镜组。
   - 仅当命中 `快切 / 闪回 / 记忆 / 蒙太奇微场景 / 极短过场` 时，允许跨物理场景链的 `A→B(→C)` 链式场景组。
7. 尾组剩余时长 `< 5 秒` 且存在前组时：
   - 默认并入前组，除非尾组承担明确信息落点。

### 场景匹配退化门禁

若检测到 `group_count == scene_unit_count`，且每个 `分镜组ID` 都是对应上游场景的第一组（例如 `1-1-1 / 1-2-1 / 1-3-1`），该结果视为“场景匹配候选”，不得直接交付。必须追加一次量化门检：

1. 全部组位于 `warn_window` 内：可保留，但 `judgement_basis` 必须说明“量化通过后保留一场景一组”。
2. 任一组低于 `warn_low`：必须优先尝试与相邻组并组；只有独立信息落点、锁定锚点或显式豁免可保留。
3. 任一组高于 `warn_high`：必须优先尝试按同场景 beat 拆组；只有未超 hard 且无可拆 beat 时可保留。
4. 任一组超过 `hard_text_window`：必须失败，不得交付。

## 组间预映机制：尾钩借焰

`尾钩借焰` 用于增强组与组之间的牵引力。它不是重新切组，也不是把下一组并回当前组，而是在当前组尾部“借”下一组开端的首个叙事拍点做预映。

定义规则：

1. 借入对象只允许是下一组开端的首个叙事拍点：
   - 若下一组开端是 `动作画面`，则借入该单行
   - 若下一组开端是 `对白 / 独白 / 内心独白 / 旁白`，则借入文本行，并优先连带其紧随的 `*画面` 行
2. 每个非末组最多一个 `尾钩借焰` 区块；末组不得借入。
3. `尾钩借焰` 只允许位于当前组尾部，不得插入组中段。
4. `尾钩借焰` 不改写：
   - 本组 `source_span`
   - 本组 `estimated_duration_seconds`
   - 本组 `effective_text_chars`
   - 下一组自身的开场保有权
5. `尾钩借焰` 只在 canonical 分组结果落定后追加，不得回流修改当前组的字窗裁决。
6. 若借入拍点属于下一场景，可在隐藏标记中保留最小元数据，但不得把该场景正式并入当前组的 canonical scene chain。

推荐落盘格式：

```md
<!-- tail-hook: from=<下一组ID> -->
<借入的首个叙事拍点>
```

## 与现有分组负载分的关系

- `episode_load_score` 与 `group_load_score` 保留为规划层摘要指标。
- 这两个分数只能帮助总结“结构复杂度 + 转折密度 + 依赖压力”，不能覆盖下列硬门槛：
  - 场景顺序
  - 生效时长策略
  - 基准字窗 / 硬上限
  - 默认不跨场景凑时长
  - 尾组 `< 5 秒` 并组规则

## 使用建议

- 统一多维量化裁决下：
  - 先用结构断点、连续时空和锚点约束形成候选边界。
  - 再用本 reference 检查各候选是否过载、过轻、越界或命中尾组规则。
  - 最后再用 `group_load_score` 作为摘要指标，不让它反向主导边界。
- 若上游预设或 `projected_group_ids` 与本 reference 的硬门槛冲突，必须在报告中标记“上游约束与量化硬门槛冲突”，不得直接通过。
