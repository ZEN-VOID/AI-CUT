# Reading Power Taxonomy

> 定位：`story` 当前活跃技能树的追读力共享真源。
>
> 用途：供 `3-Drafting/6-追读力强化`、context builder、query / status reporting、review 以及显式启用的 `type-pack` 投影共享读取。
>
> 原则：
>
> - 这是指导性分类，不是硬评分器。
> - 通用部分可被直接读取；题材 section 只有在项目显式启用对应类型化输入时才建议投影。
> - 章节追读力不是只有“张力”，还包括钩子、微兑现、爽点着陆和章末续读牵引。

## taxonomy-overview

### Hook Types

| hook_type | core_question | best_use | common_misuse |
| --- | --- | --- | --- |
| `危机钩` | 会不会立刻出事？ | 卷末、追击、危险逼近、即将暴露 | 只写“危险”而没具体风险 |
| `悬念钩` | 真相到底是什么？ | 信息缺口、规则怪异、身份异常、证据反常 | 故意藏信息但没有后续兑现 |
| `渴望钩` | 能不能马上得到想要的？ | 升级、收获、复仇、关系推进、真相将揭 | 期待一直拖着不兑现 |
| `情绪钩` | 读者心里能不能被刺一下？ | 冤屈、心疼、羞耻、心动、愤怒、不公 | 情绪没有铺垫、强行煽情 |
| `选择钩` | 他到底会怎么选？ | 两难、时间压力、原则冲突、生死二选一 | 选项没有代价，成了假两难 |

### Hook Strength

| strength | typical_slot | expectation |
| --- | --- | --- |
| `strong` | 卷末、关键转折、重大冲突前 | 读者强烈想立刻看下一章 |
| `medium` | 普通剧情章 | 读者明显愿意继续读 |
| `weak` | 过渡章、铺垫章 | 维持惯性，不强求爆点 |

### In-Chapter vs End-Chapter

| position | preferred_hook | primary_function |
| --- | --- | --- |
| `章内` | 悬念钩、情绪钩、轻渴望钩 | 保持沉浸和持续推进 |
| `章末` | 危机钩、选择钩、渴望钩 | 拉向下一章 |

### Cool-Point Patterns

| pattern | core_shape | fit | caution |
| --- | --- | --- | --- |
| `装逼打脸` | 嘲讽 -> 反转 -> 震惊 | 爽文、都市、升级 | 铺垫不够会像自说自话 |
| `扮猪吃虎` | 示弱 -> 暴露 -> 碾压 | 升级、都市、群像 | 前置隐忍必须合理 |
| `越级反杀` | 差距 -> 策略 -> 逆转 | 玄幻、战斗、悬生 | 不能全靠主角光环 |
| `打脸权威` | 权威压制 -> 公开挑战 -> 成功 | 爽文、校园、职场 | 权威不能写成纸糊反派 |
| `反派翻车` | 得意 -> 失算 -> 倒台 | 复仇、悬疑、都市 | 需要先让反派站上高位 |
| `甜蜜超预期` | 期待 -> 超预期 -> 升温 | 言情、古言、甜宠 | 不能为了发糖忽略人物逻辑 |
| `迪化误解` | 主角随手为之 -> 配角脑补升格 | 爽文、轻松日常 | 不能让配角全员降智 |
| `身份掉马` | 长期伪装 -> 关键揭露 -> 集体震惊 | 豪门、言情、升级 | 没有铺垫就会像强扭设定 |

### Micro-Payoff

| payoff_type | chapter_value | example_shape |
| --- | --- | --- |
| `信息兑现` | 读者获得一个新线索 | 真相推进一格 |
| `关系兑现` | 关系状态有实质变化 | 主动靠近、误会松动 |
| `能力兑现` | 主角真的更进一步 | 学会招式、稳住局面 |
| `资源兑现` | 取得物、位、援助 | 丹药、钥匙、证据、盟友 |
| `认可兑现` | 外界评价发生变化 | 被看见、被承认、被敬畏 |
| `情绪兑现` | 压着的感受被释放 | 说破、回击、告别 |
| `线索兑现` | 伏笔回收一小段 | “原来这就是上一章那句话的用处” |

### Assembly Heuristics

1. 同一章优先组合：
   - 一个主推进问题
   - 一个局部兑现
   - 一个章末主牵引
2. 爽点不是必须每章都有，但微兑现最好每章都有。
3. 章末主钩宜单一；副线只做陪衬，不抢主问题。
4. 高强度钩子出现时，更需要给读者一个能握住的局部满足，否则只会产生疲劳。

## shuangwen

- 主偏好：`渴望钩 + 危机钩`
- 常见爽点：`装逼打脸 / 扮猪吃虎 / 反派翻车 / 身份掉马`
- 微兑现优先：`能力兑现 / 资源兑现 / 认可兑现`
- 适配建议：
  - 每章最好给一个看得见的阶段收获。
  - 章末可拉向“更大更快的下一次反压制”。
  - 谨防“连续三章都靠同一种打脸”导致同构疲劳。

## xianxia / xuanhuan

- 主偏好：`悬念钩 + 渴望钩 + 危机钩`
- 常见爽点：`越级反杀 / 扮猪吃虎 / 身份掉马`
- 微兑现优先：`能力兑现 / 资源兑现 / 线索兑现`
- 适配建议：
  - 境界、资源、势力三轴可轮流承担推进。
  - 升级前最好先显影代价和阻力，避免白给突破。
  - 若大谜题尚远，章内至少回收一格局部谜底。

## urban-power

- 主偏好：`渴望钩 + 情绪钩 + 选择钩`
- 常见爽点：`装逼打脸 / 打脸权威 / 反派翻车`
- 微兑现优先：`认可兑现 / 资源兑现 / 情绪兑现`
- 适配建议：
  - 社会关系和现实压制感必须具体可见。
  - 章末牵引可落在“下一次公开翻盘”或“更高层级的对手”。

## romance

- 主偏好：`情绪钩 + 渴望钩 + 选择钩`
- 常见爽点：`甜蜜超预期 / 身份掉马`
- 微兑现优先：`关系兑现 / 情绪兑现 / 信息兑现`
- 适配建议：
  - 小推进比大宣言更重要。
  - 章末常见有效主钩不是“又来事故”，而是“关系是否会进一步 / 真相是否会说破”。
  - 羞耻、误会、克制和心动都能成为持续 reader pull。

## mystery

- 主偏好：`悬念钩 + 选择钩 + 危机钩`
- 常见爽点：`身份掉马 / 反派翻车`
- 微兑现优先：`信息兑现 / 线索兑现 / 情绪兑现`
- 适配建议：
  - 信息链、误导链、揭晓链最好分离，不要全靠最后一句反转。
  - 每章至少推进一格证据链，否则会像原地打转。
  - 章末主钩通常围绕“一个新异常”或“一个更危险的选择”。

## rules-mystery

- 主偏好：`悬念钩 + 危机钩 + 选择钩`
- 常见爽点：`反派翻车` 或“利用规则反制”
- 微兑现优先：`信息兑现 / 线索兑现 / 能力兑现`
- 适配建议：
  - 规则必须具体，不可只当恐怖气氛背景板。
  - 好的 reader pull 来自“新规则暴露 + 旧规则被错误理解”。
  - 章末主钩可落在“门要不要开”“这条规则到底真不真”。

## zhihu-short

- 主偏好：`情绪钩 + 选择钩 + 悬念钩`
- 常见爽点：`身份掉马 / 反派翻车 / 甜蜜超预期`
- 微兑现优先：`情绪兑现 / 信息兑现 / 关系兑现`
- 适配建议：
  - 每章密度要高，少铺长缓冲。
  - 追读力更依赖情绪刺点与快速反转，而不是长线升级。
  - 章末主钩应干净直接，一句话就能成立。

## realistic

- 主偏好：`情绪钩 + 选择钩`
- 微兑现优先：`关系兑现 / 情绪兑现 / 认可兑现`
- 适配建议：
  - 现实题材 reader pull 更来自处境压力，而不是戏剧奇观。
  - 章末牵引常是“关系会不会破”“处境会不会更坏”“人会不会说出口”。

## checker-usage-hints

| checker_focus | read_from_taxonomy |
| --- | --- |
| `reader-pull` | 主钩是否明确、章末是否存在有效续读问题 |
| `high-point` | 爽点模式是否有铺垫、是否出现局部兑现 |
| `context-builder` | 当前题材 section 的 hook / payoff 偏好 |
| `step-6 rewrite` | 本章缺的是压力、钩子、微兑现还是章末 pull |
