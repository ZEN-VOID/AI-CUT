# 1-分镜表现思维链设计报告

- 调用来源：`think-think`

- 目标模型策略：优先对齐 `reasoning / thinking` 模型的高层判断 + 可见快照 + Gate 结构

## 2. 基线与触发原因

- 基线事实：
  - 旧版目标文件仍停留在老式 `Field Master + Thought Pass Map + Pass Table` 最小结构。
  - 现有内容能覆盖“子路径顺序、内联格式、句段锚点、主文件回写”四个判断点，但缺少最新 `think-think` 要求的 `运行模式`、`启发式工作链`、`可见/隐藏分层`、`工具后反思` 与 `Gate Summary`。
  - 父技能当前真正的高杠杆问题不是“分镜怎么写得更漂亮”，而是“怎样稳定完成 `panel_count -> 锚点 -> 单行插回` 的判断链”。
- 触发原因：
  - 用户明确要求“按照最新的思维链设计规范”优化该文件。
  - 当前问题不是“没有思维链”，而是“已有字段台账，但还没升级成 reasoning 友好的判断合同”。

## 3. 启发式工作链

- 先锁 `FIELD-SBP-INPUT-02` 与 `FIELD-SBP-ANCHOR-03`；没有前置锚点时，禁止提前美化 `[分镜N]` 行。
- 先砍无锚点的镜头句、偷写 sibling 字段的越权信息，以及只顾术语感却解释不了插位的伪分镜。
- 真正的比较尺不是“哪句更像镜头语言”，而是“哪种插法最能解释这镜为什么存在、该挂到哪句之前，并且不改坏原文”。
- 站在主文件消费者和后续 sibling 层的立场判断，而不是站在文档作者的表演立场判断。
- 关键中间结果回来后，先判断是否要回退到 `锚点裁决` 或 `单行合同`，再决定是否继续回写。
- 每条保留判断都必须落到 `FIELD-SBP-*`、主文件写位、`validation-report.md`、`Fail Code` 或 `Rework Entry`。

## 4. 三轴三重设计

### 三轴

- `方向轴`：锁定当前任务是否真的属于 `1-分镜表现`，并坚持 `分镜密度 -> 分镜构图 -> 分镜插入` 的唯一主路径。
- `成立轴`：确认 `panel_count`、句段锚点与静态字段被 grouped source、密度结果和构图结果共同支撑。
- `优选轴`：在成立候选里，优选最稳、最省返工、最利于后续 sibling 继续发酵的插法。
- `硬门禁轴`：若命中原文改写、编号断裂、锚点漂移、越权字段或无返工入口，直接否决。

### 三重

- `粗裁决`：收窄出唯一的父级职责、输入范围与主路径。
- `细裁决`：收窄出每组 `panel_count + 锚点计划 + 单行字段合同`。
- `离散裁决`：收窄出最终前插结果、`validation-report` 结论与下一入口。

## 5. 快照与字段落盘

- 新增 `模块定位` 与 `运行模式判定`，显式锁定默认模式、可见性边界与禁止误用。
- 在 `Think-Think Design Snapshot` 中补入 `启发式工作链`、`三向三重裁决轴`、`可见快照与隐藏推理分层`。
- 将字段接口升级为六字段：`ROOT / INPUT / ANCHOR / INLINE / LAND / HANDOFF`，让每个判断都有清晰落点与返工入口。
- 新增 `字段落盘快照`，把锚点裁决、单行合同、主文件回写与 `Gate Summary` 分别挂到具体载体。

## 6. 工具后反思与评测

- 新增 `工具后反思`，覆盖三个关键节点：
  - `分镜密度` 结果返回后
  - `分镜构图` 结果返回后
  - 主文件回写与 `validation-report` 验收后
- 新增 `Gate Summary 最低合同`，要求固定回答 `PASS 信号 / 失败码 / 默认返工入口`。
- 评测上收紧为：
  - 边界与越权检查
  - 输入链完整性检查
  - 锚点成立性检查
  - 内联合同稳定性检查
  - 主文件回写守恒检查
  - 闭环与返工入口检查
