# aigc 3-明细 / 3-运镜手法 / Chain Of Thought

本文件承载 `aigc 3-明细 / 3-运镜手法` 的新版思维链合同、字段主表、可见快照与返工入口真源。

## 设计立场（Mandatory）

- 本模块默认按 `原生 reasoning / adaptive thinking 友好` 的高层引导执行：先锁终局、证据、波形与落盘，再让模型完成内部推理。
- 对外只保留 `Think-Think Design Snapshot`、`工具后反思`、`Gate Summary` 这些可审计快照，不外露完整内部推理全文。
- 若运行在普通模型上，允许把启发式工作链收束成少量显式步骤，但不得反向膨胀成流水账式脚本。
- `3-运镜手法` 的真问题不是“镜头术语够不够多”，而是“是否形成可执行的观看路径波形，并稳定落到逐镜 `运镜手法` 字段”。

## Think-Think Design Snapshot

### 启发式工作链（运镜版，Mandatory）

- `不可删性`：删掉本层，当前段落会失去哪条关键观看路径，导致镜头只剩静态信息而没有推进、压迫、回收或失衡感。
- `最先推动`：先推动 `FIELD-CAM-WAVE-03`，先判组级/段级主波形，再决定哪些命中分镜节点真的需要落成 `运镜手法` 字段。
- `先砍什么`：优先删掉模板化的推拉摇移套话、未被空间证据支撑的炫技、以及越权到光影/摄影/转场的句子。
- `真正比较尺`：多个候选都成立时，只比较哪条路径最能传出权力关系、视线重心、叙事动势，并且返工成本最低。
- `立场纠偏`：始终站在“观众如何被镜头带着看”的立场，不站在“我会多少运镜术语”的立场。
- `工具后反思`：读取分镜骨架、正文动作、组间 handoff、角色增强或上游改稿后，先重判证据是否还支撑原波形，再决定继续、改波形或降级为静稳观察。
- `落盘门`：本轮判断必须落到 `FIELD-CAM-INLINE-04`、`FIELD-CAM-DSP-07`、`FIELD-CAM-GATE-09`；若落不到逐镜写位、可见快照或 Gate，就不算有效思维链。

### 三向三重自省流（Mandatory）

| 裁决轴 | 在 `3-运镜手法` 的核心问题 | 主要落点 |
| --- | --- | --- |
| `方向轴` | 当前段落最需要怎样的观看路径，才能把权力关系、视线重心与叙事动势送到观众眼里 | `FIELD-CAM-WAVE-03`、`FIELD-CAM-DSP-07` |
| `成立轴` | 这条运镜是否被命中分镜节点的静态字段、正文动作、空间关系、组间 handoff 与角色增强共同支撑 | `FIELD-CAM-INPUT-02`、`FIELD-CAM-WAVE-03` |
| `优选轴` | 在成立的候选里，哪条运镜最有镜头人格、最好拍、又不污染后续 sibling | `FIELD-CAM-INLINE-04`、`FIELD-CAM-HANDOFF-06` |
| `硬门禁轴` | 是否出现越权、轴线失真、空间不可拍、写位漂移、模板复读或与上游证据冲突 | `FIELD-CAM-GATE-09` |

| 收敛层 | 本层先裁什么 | 绝不带进下一层什么 | 正确产物 |
| --- | --- | --- | --- |
| `粗裁决 / Objective Framing` | 当前组或段到底要解决哪一种观看问题 | “每镜都来点动势”的扩张冲动 | 主观看任务、消费者、主要未知项 |

| `离散裁决 / Visible Landing` | 最终哪些内容进入可见快照与逐镜 `运镜手法` 字段 | 冗长解释、完整内部推理、不可审计的酷句 | `运镜手法` 字段、Snapshot、Gate Summary、返工入口 |

层内自省固定四问：

1. 为什么这条观看路径比其他候选更能服务当前戏核？
2. 如果不用这条路径，会不会出现更稳、更省返工的方案？
3. 删掉当前保留项，下游会失去什么关键判断能力？
4. 如果把它写回普通流程脚本，为什么会更差或更难审计？

## 发现-激发-增写工作台（Heuristic Pool）

本节是启发式素材池，不是要求逐条外显的流水脚本。执行时按需要抽取，最终仍回收到 `Snapshot -> 行内落笔 -> Gate Summary`。

### 1. 发现（Discover）

逐组或逐段抽取以下锚点：

- 叙事节拍锚点：揭示、逼近、压迫、迟疑、释放、反打、失衡、爆发后的余波
- 静态镜头锚点：景别、构图布局、视角、焦距、对焦点、前景遮挡、纵深
- 身体与视线锚点：角色走位、停顿、回头、抬眼、躲闪、逼近、失手、后撤
- 空间通道锚点：门、廊、桌角、楼梯、窗框、遮挡物、纵深通道、可切入前景
- 组间 handoff 锚点：导演意图中的节奏提示、类型元素里的 motion bias、上游角色层已强化的冲突中心
- 连续性锚点：前后镜方向、速度、视线承接、组内波峰波谷、是否需要刻意静止

### 2. 激发（Ignite）

先裁决主波形，再决定是否需要一个辅动作。常见主路径包括：

- 信息揭示：慢推前压，末端微调视线落点
- 压迫升级：贴身跟移，近距离逼视后压停
- 关系重置：反向环绕半弧，收在新的支配位
- 行动爆发：横切抢位或甩入，迅速抓住动作起爆点
- 主观失衡：轻晃、倾斜、短促逼近，再在关键眼神处钉住
- 静稳观察：保持近乎不动，只让角色、视线或遮挡自己制造张力

硬规则：

1. 主波形必须先服务叙事动势，再追求镜头人格。
2. 若空间、轴线、文本事实或上游骨架不支持，不得硬写高难炫技。
3. 当“高炫技”和“可拍摄性”冲突时，先重排路径与节拍，不直接退化成空泛摘要。
4. 当最佳选择其实是静止，必须说明“为什么静止比运动更成立”，而不是把静止视为失败。

### 3. 增写（Expand）

1. 先在侧车或内部判断里写清组级/段级波形，再拆到逐镜 `运镜手法` 字段。
2. 每个 `运镜手法` 字段都要能回答：起点、路径、变速、收束、目的。
3. 逐镜落笔必须与本镜静态字段、正文动作、前后镜连续性可解释地对齐。
4. 落笔后继续检查：所有 `运镜手法` 字段是否共同构成一条观看路径，而不是一堆互不相干的酷句子。

## 可见快照、工具后反思与 Gate Summary

### Think-Think Design Snapshot 可见项

- `dominant_wave`：本组/本段唯一主波形
- `evidence_anchors`：支撑主波形的关键锚点，不展开完整推理全文
- `shot_landings`：命中分镜节点的最小 `运镜手法` 字段落笔计划
- `rejected_candidates`：被排除的候选类型及一句理由
- `handoff_note`：需交给 `4-场景氛围 / 5-摄影美学 / 6-转场特效` 的未处理项

### 只留内部推理、不进入可见合同的内容

- 冗长候选比较过程
- 多轮自我试错与撤销记录
- 无法落到 `运镜手法` 字段或 Gate 的术语炫技
- 与本层无关的光影、摄影参数、后期包装细节

### 工具后反思（Mandatory）

当外部输入变化或工具结果返回后，先做以下二次判断：

1. 新证据是否改变当前段落的主观看任务。
2. 原波形是否仍被空间关系、动作节拍与视线承接支撑。
3. 若不再成立，应改波形、缩窄到局部补写，还是降级为静稳观察。
4. 只有在新证据足够支撑下一步时，才继续写逐镜 `运镜手法` 字段。

### Gate Summary（Mandatory）

| gate | 问题 | Pass Signal | Fail Code |
| --- | --- | --- | --- |
| `Boundary Gate` | 是否越权写入分镜骨架、摄影、光影、氛围或转场 | 只动本层允许的运镜写位与侧车 | `FAIL-CAM-ROOT-01` |
| `Evidence Gate` | 主波形是否有真实证据支撑 | 输入链完整且能回查到锚点 | `FAIL-CAM-INPUT-02` |
| `Wave Gate` | 组级/段级波形是否成立且有 dominant choice | 主波形唯一，候选排除理由清楚 | `FAIL-CAM-WAVE-03` |
| `Landing Gate` | 逐镜 `运镜手法` 字段是否可执行 | 每镜具备起点、路径、变速、收束、目的 | `FAIL-CAM-INLINE-04` |
| `Canonical Gate` | 是否仍回写同一份 `第N集.json` | 无平行稿、无偷改其他字段 | `FAIL-CAM-LAND-05` |
| `Handoff Gate` | 是否给后续层留下唯一下一入口 | Snapshot、报告、CHANGELOG、留口完整 | `FAIL-CAM-HANDOFF-06` |
