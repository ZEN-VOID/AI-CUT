## Repair Playbook

1. 先判断失败是漏提、噪声过多、评分无证据、弱候选混入主清单还是输出路径错误。
2. 若列表过长，优先剔除普通环境物和一次性无功能物件。
3. 若遗漏关键道具，优先回查信息载体、身份信物、危险源、规则装置和重复交互物。
4. 若某物件只因“出现多”被保留，必须补功能证据；补不了则降级。
5. 若下游需要造型设计，生成 JSON `design_spec`，但必须保留剧本中可见线索、叙事功能和推断边界。

## Reusable Heuristics

- 道具提取的关键不是“物件识别”，而是“叙事功能筛选”。
- 一个物件如果删掉后剧情、关系、信息、规则、危险或关键画面都不变，通常不应进入主清单。
- 频繁出现的环境物不一定是道具；一次出现但承载真相或身份的物件反而可能是核心道具。
- 道具设计规格必须先服务“这个物件如何被制造、持握、老化和辨认”，再进入英文提示词。
