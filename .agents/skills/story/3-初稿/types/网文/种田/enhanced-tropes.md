# 种田题材增强包

> 补充维度：套路拆解 / 情节模式 / 人物原型 / 反套路策略 / 质量过滤

## 一、核心套路与行规

### 1.1 种田文核心魅力
种田文的独特吸引力在于：
- **可预期的成长**：耕耘→收获，付出→回报的确定性
- **一步步建设的满足感**：从无到有、从小到大、从简到繁
- **庇护所幻想**：在一个安全可控的环境中慢慢变好
- **日常的治愈力**：平淡但不无聊，细水长流的幸福感

### 1.2 "种田"不只是种地
| 种田维度 | 内容 | 写作要点 |
|----------|------|----------|
| 空间建设 | 开荒/建房/规划/扩展 | 每次扩展要有明确的功能需求驱动 |
| 经济经营 | 种植/养殖/贸易/产业链 | 让经济数字有情感意义（养的羊终于下崽了） |
| 社区经营 | 与邻里/村民/雇佣者的关系 | 不是"全听我的"一言堂，关系有博弈和磨合 |
| 技能成长 | 学习新技能/改良技术/创新 | 技能进步与服务生活/社区紧密绑定 |
| 关系建设 | 家人/邻居/朋友/伴侣关系 | 日常互动中关系自然升温 |

### 1.3 常见套路陷阱
- 纯流水账——今天种了XX，明天收了XX（没有冲突驱动）
- 天降系统解决一切——失去了"耕耘→收获"的核心理念
- 经济数字游戏——通货膨胀、合理性崩塌
- 主角全知全能——什么都会、什么都做得比别人好

## 二、情节模式与节奏规则

### 2.1 种田文的三层冲突
```
微观层（每章）：日常中的小问题——天气、虫害、邻里纠纷、物资短缺
中观层（每卷）：建设中的中等障碍——竞争对手、政策变化、自然灾害、技术瓶颈
宏观层（每部）：与外部世界的大冲突——战争影响、经济危机、权力介入
```

### 2.2 节奏铁律
- 前3章必须建立主角的出发状态和第一个目标
- 每章至少有一个小收获（不限于物质——学到新东西、解决了一个问题、关系有进展）
- 平和场景中埋设小冲突（不是大起大落，而是小而真实的阻力）
- 每卷有一个中观挑战，既是障碍也是推动力

### 2.3 爽点设计
| 爽点类型 | 种田文实现 |
|----------|-----------|
| 建设成果 | 荒地上盖起了房子/田地丰收/作坊开工 |
| 经济突破 | 第一次卖出商品/打开新销路/利润翻倍 |
| 关系进步 | 与邻居从陌生到信任/与伴侣从相识到相知 |
| 技能成就感 | 学会了新手艺/解决了一个技术难题 |
| 社区认可 | 帮了别人被感谢/在社区中获得声望 |

## 三、反套路策略

### 3.1 反"天降系统"
- 没有万能系统——靠的是观察、学习、实践和失败
- 失败是常态——第一次种地收成很差，从失败中学习
- 技能有学习曲线——不是"叮"一下就学会了

### 3.2 反"乌托邦"
- 种田世界不完美——有不讲理的邻居、有欺压的权贵、有无法控制的自然
- 建设有代价——不是所有地都肥沃、不是所有人都善良
- 冲突真实而不夸张——不是灭门仇杀，而是"隔壁占了你的水源"

### 3.3 反"完美主角"
- 主角会犯错——判断失误导致减产、信任错人导致损失
- 焦虑和自我怀疑——"我这样做对吗？""我配得上现在的生活吗？"
- 倦怠——日复一日的劳作会有疲惫，需要找到持续的动力

## 四、质量过滤规则

### 4.1 日常描写的功能性检查
- [ ] 每段日常描写服务于：推动建设进度 / 展现人物性格 / 建立关系 / 埋设伏笔（至少其一）
- [ ] 没有连续3段以上的纯描述性日常（无冲突/无变化/无功能力）
- [ ] 经济数字/资源数量有逻辑自洽

### 4.2 失败码
| 失败码 | 症状 | 返工目标 |
|--------|------|----------|
| `FAIL-FARM-FLOW` | 流水账式日常 | 为关键日常段落增加冲突/意义/情感 |
| `FAIL-FARM-GOD` | 系统/外挂解决一切 | 减少系统作用，增加自身努力 |
| `FAIL-FARM-FLAT` | 连续多章无冲突/无变化 | 在中观层或微观层引入真实小障碍 |
