# Statistics YAML Contract

本文件定义每个分镜组底部 YAML 统计的字段、统计边界和抽取规则。

## YAML Shape

每个分镜组底部必须附加 fenced YAML：

```yaml
字数统计: 0字
时长估算: 约0秒
角色:
  - 角色名
场景:
  - 场景名
道具:
  - 道具名
```

无角色、场景或道具时使用空列表 `[]`，不得省略字段。

## Count Boundary

`字数统计` 统计该分镜组标题后重复输出的场景标题行，以及从 `9-光影` 或用户指定 source 划定并同步原换行的分镜正文，包括正文内场景标题、字段名、对白标签、分镜行标题和时间码内容。

以下结构不计入 `字数统计`、`时长估算` 或约 1680 字语义复核参考线：

- 分镜组标题。
- `全局风格：` 字段标题与字段内单行 `Global Style Prompt` 整理内容。
- 组底 fenced YAML。
- 独立 `## <上一个分镜组ID>~<下一个分镜组ID>` 连接件块禁止输出；若旧稿残留，必须删除，且不计入任何分镜组统计。

`时长估算` 取当前分镜组累计时间码中最后一个 `[N-N秒]` 的结束秒。落盘稿中的每个分镜行不再各自从 `[0-N秒]` 重启，而是按当前分镜组从 0 秒开始连续递增，后一个时间段起点必须等于前一个时间段终点；最终累计结束秒必须以 `.5` 结尾，若自然相加不是 `.5` 结尾则在组尾上调 0.5 秒，YAML 回指这个最终结束秒。组标题、标题后重复输出的场景标题行、`全局风格：` 和组底 fenced YAML 均不计入。若上游缺少 canonical 时间段，应回到 `9-光影` 补齐；临时兼容 legacy 旧稿时，执行报告必须说明缺少时间段来源和估算风险。

## Role Extraction

角色字段优先来源：

1. `对白（角色名）`、`独白（角色名）`、`旁白（角色名）` 等声音主体。
2. `角色动作：`、`表演提示（角色名）：`、`表情特写：` 中明确出现的人名。
3. 第一个普通分镜行或场景正文中明确可见或可听见的角色名。

群像可写为 `学生群像`、`围观人群` 等稳定称谓。未实际出场但被提及的角色不列入，除非本组以影像、声音或道具显影方式让其成为生成主体。

## Scene Extraction

场景字段优先取当前 `### 场景N：...` 的场所名。一般一个分镜组只对应一个场景；若跨场景，优先重新拆组。确需多场景时，按出现顺序列多个场景名，并在执行报告说明原因。

## Prop Extraction

道具字段只列重要叙事道具、规则道具、视觉钩子或后续生成必须锁定的物件。普通环境物件只有在本组承担叙事功能时才列入。

道具抽取必须先做同物识别与归一合并，再写入 YAML：

- 同一物品只列一次，使用最稳定、最利于后续资产锁定的称谓；不要把颜色、位置、破损状态、持有人或镜头景别拆成多个道具。
- 形容词差异、状态差异、持有人差异、镜头差异、单复数差异和局部部件默认合并到同一 canonical 道具名，例如 `红色手机` / `手机屏幕` / `她的手机` 可合并为 `手机`，除非正文明确存在多部不同手机且会影响生成一致性。
- 若同一大件的部件本身承担独立叙事功能，才可单列部件；否则部件并入主体物件。
- 同类多个普通背景物不逐项铺开；需要保留群体功能时写成集合称谓，例如 `课桌椅`、`散落文件`、`监控屏幕组`。
- 道具名应短而稳定，避免写成动作句、镜头句或状态句；状态变化写在正文或执行报告中，不在 YAML 道具列表中重复展开。

优先来源：

1. `道具特写：`
2. `规则显影：`
3. `系统画面：`
4. 第一个普通分镜行中明确可见或被人物互动的关键物件
5. 对白或动作中被明确使用的关键物件

旧连接件中的关键视觉承接点不得污染本组正文统计；若该物件也在本组正文或第一个普通分镜行中出现，按正文证据列入。

## Review Notes

YAML 是统计证据和机器可读索引，不是新的剧情真源，也不是分镜组开头的主体信息展示层。角色、场景和道具必须能回指本组正文或第一个普通分镜行，YAML 只保留短而稳定的 canonical 名称。

不得为了补充主体信息，在组头或 YAML 前额外输出 `角色：` / `场景：` / `道具：` 结构化清单。若需要更准确地传达主体属性，应回到上游 `9-光影` 正文或第一个普通分镜行中补足可回指信息。

每个 YAML 统计项都应能回指本组正文、场景标题、对白主体、分镜行、道具特写、规则显影或系统画面中的证据。`时长估算` 应能回指组内累计时间码的最后结束秒，且该结束秒必须以 `.5` 结尾。
