# Directing Heuristics

## Practical Heuristics

- 先不要急着“写得像剧本”，先给上游每个信息点找落点；落点稳定后，剧本质感自然会出来。
- 对白周围最容易混入动作，写完后专门扫一遍中文引号，凡是动作都移到 `对白画面` 或 `角色动作`。
- 系统提示适合用 `旁白（系统提示）` 承载文字，用 `系统画面` 做字体、空间、人物反应和规则显影。
- 诡异校园类场景的可拍核心通常在空间压迫、集体静止、规则文字、声音节奏和局部特写之间，不必靠新增剧情制造恐惧。
- 当多个 beat 都发生在同一教室或走廊，场景标题只出现一次，后续用字段推进节奏。
- “他意识到”通常不是动作；可转为眼神停顿、手指停住、呼吸变化、内心独白或表演提示。
- “他不信任/她在试探/他们被压住”不是表演提示；先问角色想达成什么、不能暴露什么，再写停顿、避视、道具动作、身体距离或声线变化。
- 场面调度优先写权力关系：谁站着、谁坐着、谁占据讲台/门口/黑板，谁被群体视线包围，关键道具停在谁手边。
- 沉默不是空白；用纸张声消失、呼吸停住、手指松开、对手不接话、群像迟半拍反应来承托。
- `表演提示` 和 `场面调度` 的规划句不要直接贴到场景末尾；把“想控制风险”拆成低声、停敲、压住密报、众人甲胄声低下去等细节。
- B 路线新增的判断口径：删掉新增细节后，剧情事实、对白、因果、规则和人物决定必须完全不变。
- 能进入 B 路线的新增通常是“质感和承托”，例如环境微动、群体迟疑、手指压纸、道具位置、声音余波；不能是新信息。

## Repair Patterns

| symptom | repair |
| --- | --- |
| 画面字段复述对白 | 改为说话时的表情、对手反应、空间压迫或画面承托 |
| 音效没有画面 | 补声音源头、人物反应或不可见来源的空间处理 |
| 场景标题像摘要 | 改成 `内景/外景 场所 - 日/夜` |
| 独白像旁白 | 判断主体是否在场并承受该经验；若是角色主观经验，用独白或内心独白 |
| 现实灾难插针过长 | 保留为短促画面，不吞掉主场景 |
| 表演提示像心理标签 | 改成目标、阻碍、策略、外显动作；至少写清其中两项 |
| 潜台词靠新增对白解释 | 删除新增对白，改用停顿、视线、道具、身体距离和沉默余波 |
| 场面调度写成镜头方案 | 删除机位、景别、运动和分镜编号，回到人物、空间、道具和视线关系 |
| 场景末尾堆表演/调度总结 | 拆回对应 beat：环境写空间权力，动作写手和视线，对白画面写声音压低和众人反应，道具特写写焦点物 |
| 受控增强变成剧情新增 | 删除新增剧情项，或转为需用户另行授权的候选稿；canonical 只保留有上游锚点的承托项 |

## Known Pitfalls

- 把“好莱坞级”理解为更自由的二创，会直接破坏 `2-编导` 的 canonical 职责。
- 把 `动作画面` 当作万能字段，会把心理、规则和背景说明都塞进去，导致下游无法拍。
- 同一 slugline 重复打印场景标题，会让分组阶段误判为多个场景。
- 把“高段位表演”写成理论词会让演员无事可做；它必须表现为可执行任务和外显行为。
- 把权力关系写成“掌控全场”“关系改变”会失去画面证据；应转为空间站位、视线压力和道具归属。
- “更影视化”不是默认可以加戏；没有 `controlled_enrichment_ledger` 的新增细节很容易滑向自由改编。
- 末尾总结块会让编导稿像审稿批注而不是可拍剧本；终稿应让表演和调度在对应动作时发生。
