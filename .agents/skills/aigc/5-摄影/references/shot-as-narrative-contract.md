# Shot as Narrative Contract

本文件定义镜头的叙事功能裁决规则。每个镜头选择必须回答"观众此刻应该知道/感受/期待什么"，而不是"什么景别好看"。镜头不是记录工具，而是叙事工具——景别、运动、焦点、构图都是为了完成特定的叙事功能。

## Core Rule

每个 `分镜N` 在完成技术参数选择前，必须先裁决其 **叙事功能**。叙事功能回答的是：这个镜头存在，是为了对观众做什么？

技术参数是手段，叙事功能是目的。没有叙事功能的镜头是废镜头——无论它的构图多么精美、运镜多么流畅。

## Narrative Function Taxonomy

### 叙事功能总表

| function_id | 叙事功能 | description | typical_technique | audience_question_answered |
| --- | --- | --- | --- | --- |
| `NF-01` | reveal（揭示） | 逐步向观众展示此前不可见或未注意到的信息 | 慢推向主体 / 焦点从前景拉到背景 / 前景遮挡物移开 / 从局部到全貌的摇镜 | "原来这里还有一个东西" |
| `NF-02` | conceal（隐藏） | 刻意不让观众看到某个信息，制造悬念或不安 | 景深虚化关键区域 / 画框裁切掉关键部分 / 只拍背影不给正面 / 关键位置处于暗部 | "它就在那里，但我看不见" |
| `NF-03` | emphasize（强调） | 强化观众对某个信息或情绪的感知强度 | 特写停顿 / 回切（重复出现） / 景别突变（中景切大特写） / 长时间不切 | "这个很重要，我必须注意到" |
| `NF-04` | compare（对比） | 通过并置或交替呈现两个信息，让观众自行建立关联 | 正反打 / 匹配剪辑（相似构图切换） / 分屏或分割构图 / 动作-反应交替 | "这两件事之间有关系" |
| `NF-05` | misdirect（误导） | 引导观众的注意力到错误方向，为后续揭示制造反转 | 焦点引导到次要物体 / 运动方向暗示错误的注意力焦点 / 声画分离（看这边听那边） / 用特写建立错误重要性 | "我以为重要的其实是这个" |
| `NF-06` | connect（连接） | 建立两个看似独立的信息之间的视觉或情绪联系 | 同框（将两个元素放进同一画面） / 运动连接（一个动作的终点成为另一个的起点） / 视觉匹配（形状、颜色、方向相似） / 光影/声音桥接 | "原来这两件事是一回事" |

### 叙事功能展开规则

每种叙事功能有其适用场景和禁用场景：

| function_id | 适用场景 | 禁用场景 |
| --- | --- | --- |
| `NF-01` reveal | 新空间建立、关键道具首次出现、规则显影、信息层递 | 无需悬念的日常动作、已被充分交代的已知信息 |
| `NF-02` conceal | 恐惧建立、悬念维持、身份隐藏、危险暗示 | 需要清晰交代的规则说明、群像混乱中需要空间定位的时刻 |
| `NF-03` emphasize | 高潮节拍、关键转折、情绪峰值、规则突破瞬间 | 日常过渡、低信息过场、已经通过其他手段充分强调的信息 |
| `NF-04` compare | 正反打对峙、前后对比（过去/现在）、角色内心矛盾外化 | 信息没有关联的随机画面、过于直白的相似对比 |
| `NF-05` misdirect | 悬疑反转铺垫、注意力陷阱、信息层级设计 | 不需要反转的直线叙事、已经揭示结局的场景 |
| `NF-06` connect | 跨场景情绪延续、主题母题呼应、因果关系建立 | 没有叙事关联的强行拼接、破坏独立场景完整性的连接 |

## Integration with Shot Design Plan

每个 `shot_design_plan` 必须新增 `shot_narrative_function` 字段：

```
shot_narrative_function: NF-01 reveal
```

当一个镜头同时承担多种叙事功能时，按优先级排列：

```
shot_narrative_function: NF-01 reveal > NF-03 emphasize
```

### field specification

| field | type | required | description |
| --- | --- | --- | --- |
| `shot_narrative_function` | enum / enum[] | 是 | 当前镜头的核心叙事功能，使用 NF-01 至 NF-06 编码 |
| `narrative_function_reason` | string | 是 | 一句话说明为什么此处需要这个叙事功能（不得只写"揭示信息"，必须说明揭示的是什么、给谁看、为什么此刻需要） |
| `narrative_risk` | string | 否 | 当前叙事功能可能失败的方式（如"揭示过早导致悬念丧失"） |

## Narrative Function Diversity Requirement

### 画面句子级约束

同一 `visual_unit` 内的连续分镜，叙事功能不得全部相同。至少每 3 个连续分镜要出现 2 种以上叙事功能。

| 合法组合（3 镜） | 不合法组合（3 镜） |
| --- | --- |
| reveal → emphasize → conceal | reveal → reveal → reveal |
| conceal → conceal → reveal | conceal → conceal → conceal |
| misdirect → reveal → emphasize | misdirect → misdirect → misdirect |
| compare → emphasize → connect | compare → compare → compare |

### 场景级约束

同一个场景中，如果所有画面句子的镜头都只使用 `reveal`，说明裁决者只是在"逐步展示"，而没有主动操控观众的注意力、情绪和期待。必须至少出现 2 种以上叙事功能。

### 集级约束

同一集中，叙事功能的分布应当反映叙事曲线：

| 叙事阶段 | 典型功能分布 |
| --- | --- |
| 开篇建立 | reveal 为主（约 60%），conceal 为辅（约 20%），connect 补充 |
| 发展中段 | compare + emphasize 为主（各约 25%），misdirect 开始出现 |
| 高潮段 | emphasize 为主（约 40%），reveal + misdirect 配合（各约 20%） |
| 收尾段 | connect 为主（约 40%），conceal（为续集留悬念）为辅 |

## Narrative Function and Shot Parameter Alignment

叙事功能决定技术参数，不是反过来：

| narrative_function | 倾向的景别 | 倾向的运镜 | 倾向的景深 | 倾向的时长 |
| --- | --- | --- | --- | --- |
| reveal | 从宽到窄（全景→中景→特写） | 推轨、摇镜、焦点拉移 | 从深到浅（逐步聚焦） | 中等（2-4 秒），渐进 |
| conceal | 中景或更宽，避免过近 | 固定或微动 | 浅景深虚化关键区域 | 较长（维持隐藏感） |
| emphasize | 特写或大特写 | 静止或极慢推 | 极浅景深隔离主体 | 较长（2-5 秒），让观众"吸收" |
| compare | 对称景别（两个元素同等大小） | 固定或平移 | 深景深让两者同时可读 | 交替出现，每侧 1-3 秒 |
| misdirect | 在误导目标上用强调景别 | 运动引导到错误方向 | 焦点落在误导目标上 | 中等，不给时间反思 |
| connect | 中景或全景（容纳两个元素） | 运动连接两者 | 根据连接方式调整 | 由运动速度决定 |

## Failure Modes

### FM-01: 镜头只是记录而非叙事

**症状**：镜头描述只有技术参数（"中景，f/2.8，慢推"），没有叙事功能说明。

**诊断**：问——"这个镜头存在，是因为观众必须在此刻看到/感受/期待某个特定东西，还是只是因为'该拍了'？"

**修复**：为每个镜头补充 `shot_narrative_function` 和 `narrative_function_reason`。如果找不到合理的叙事功能，这个镜头可能是多余的。

### FM-02: 叙事功能单一化

**症状**：整集或整场的镜头叙事功能几乎都是 `reveal`——每个镜头都在"逐步展示"。

**诊断**：统计当前场景的叙事功能分布，如果 `reveal` 占比超过 70%，即为功能单一化。

**修复**：回到每个镜头问——"观众此刻的情绪需要被强化（emphasize）？需要被误导（misdirect）？需要被隐藏（conceal）？需要被引导去比较（compare）？需要被建立联系（connect）？"

### FM-03: 叙事功能与技术参数矛盾

**症状**：裁决叙事功能为 `conceal`，但技术参数选了特写+深景深+慢推——完全暴露了要隐藏的东西。

**诊断**：检查 `shot_narrative_function` 与实际技术参数的一致性。

**修复**：参考上表"叙事功能与技术参数对齐"，调整技术参数使其服务于叙事功能。

### FM-04: reveal 被滥用为万能功能

**症状**：每遇到"需要让观众知道"的场景就用 reveal，但很多场景其实不需要"逐步揭示"的仪式感——信息可以直接交代。

**诊断**：问——"这个信息是否需要'逐步揭示'的张力？还是直接交代更合适？"

**修复**：如果信息不需要悬念，就不要用 reveal 仪式。直接交代可以用连接（connect）或强调（emphasize）完成。

### FM-05: misdirect 没有回报

**症状**：使用了 misdirect 但后续没有对应的 reveal 或反转来"兑现"误导。

**诊断**：每个 misdirect 必须有对应的"真相揭示"镜头，且揭示必须在合理的叙事距离内（通常同一场景或紧邻场景）。

**修复**：如果找不到合理的揭示时机，不要使用 misdirect；误导没有回报会让观众觉得被欺骗。

## Reusable Heuristics

1. **先问功能再选镜头**：每个镜头的第一句裁决词不是"用什么景别"，而是"这个镜头对观众做了什么"。
2. **reveal 是默认而非最佳**：大多数初学者的默认叙事功能是 reveal（逐步展示），这通常是安全但平庸的选择；先想是否需要 conceal、misdirect 或 compare。
3. **conceal 比 reveal 更难但更有力**：让观众"看到但看不清"比"逐步展示"需要更多精确控制，但悬念效果远强于直接揭示。
4. **每个 misdirect 必须有兑现**：误导是一张借条，必须用后续的真相揭示来偿还。
5. **compare 需要对称性**：两个被比较的元素在画面中的位置、大小、时长需要对称，否则观众感受不到"对比"。
6. **connect 不是 match cut**：连接功能不要求技术上的匹配剪辑——同框、运动桥接、声音关联都可以完成连接。
7. **叙事功能多样性是强制指标**：连续 3 个以上相同叙事功能是结构性问题，不是风格选择。
8. **镜头数量由叙事功能决定**：如果一个画面句子只需要一个 reveal 就完成了叙事任务，不要为了"丰富"而拆成多个镜头。
