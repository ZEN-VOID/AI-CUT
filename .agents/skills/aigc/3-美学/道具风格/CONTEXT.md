## Repair Playbook

1. 先确认失败属于来源不足、画面基调继承断裂、道具清单污染、剧情用途污染、单件设计越权、文化证据不足、参考照搬、字数不合规还是报告证据缺失。
2. 若来源不足，回到 `N1-INTAKE`，不要用热门道具风格或随机文化符号补空。
3. 若剧本分析变成道具清单，回到 `N2-SCRIPT-EVIDENCE`，从时代、阶层、技术水平、生产方式、仪式秩序和使用环境抽取风格证据。
4. 若参考图/视频带来具体物件污染，回到 `N3-TONE-REFERENCE-EVIDENCE`，只保留形制、表面行为、工艺、磨损、尺度和符号节奏。
5. 若 prompt 出现具体道具、用途、持有人、尺寸部件或生成指令，回到 `N7-PROMPT-DISTILL`，用风格层级表达替换。
6. 若文化元素像拼贴，回到 `N5-CULTURE-CRAFT-MAPPING`，补来源语境、使用边界和禁用边界；无法验证则标为候选。
7. 若边界不能保护后续设计空间，回到 `N6-BOUNDARY-CALIBRATION`，补清单、用途、单件、文化和制作五类边界。
8. 用户指出“脚本化/偷懒/未思考/未差异化”时，直接废弃候选协议回源层重做，不接受替换几个物件名、文化名或质感词。
9. 修复后必须重新执行 `GATE-PS-01` 到 `GATE-PS-10`，并把失败码、返工点和复审结果写入 `Repair Log`。

## Reusable Heuristics

- 道具风格不是“本片会出现什么道具”，而是“所有道具应共享怎样的形制、表面、工艺和使用痕迹纪律”。
- 从剧本提取时，不要记录具体物件；记录物件背后的技术水平、生产方式、社会秩序、资源稀缺度、仪式系统和维护习惯。
- 材料可以作为内部判断线索，但最终 prompt 更稳妥的写法是材料倾向、表面行为、加工方式和磨损逻辑。
- 使用痕迹是道具风格的关键，但不能变成剧情用途。写“长期修补、边缘磨耗、临时加固”，不要写“用于追踪敌人”。
- 文化符号必须有边界。可以写“符号重复与几何秩序”，不要随意点名现实宗教、族群或特定品牌符号。
- 参考图/视频分析先做“去物件化”：遮掉道具名字后仍然可见的比例、工艺、磨损、表面处理和符号节奏，才适合进入风格协议。
- `画面基调` 是道具风格的底层限制，不是直接复制文本。道具风格要把全局媒介和渲染纪律转成道具层面的形制、表面和工艺规则。
- 约 100 字 prompt 的稳定结构：文化气质 -> 形制趋势 -> 表面行为 -> 工艺/使用痕迹 -> 尺度/符号语言 -> 制作边界和避免项。
- 如果一句话只能指导一个具体物件，它不属于道具风格；如果它能约束整组道具的视觉共性，它才适合进入 `Prop Style Prompt`。
