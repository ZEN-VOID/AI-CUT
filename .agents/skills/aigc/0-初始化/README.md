## Quick Entry

Use `$aigc-init` when a project must enter or re-enter initialization.

The entry contract is in `SKILL.md`. Load `CONTEXT.md` with it, then read only the referenced partition for the current scaffold node:

- runtime paths: `references/scope-and-runtime.md`
- workflow: `steps/init-workflow.md`
- review: `review/init-review-gate.md`
- memory template: `templates/project-memory.template.md`
- project context template: `templates/project-context-readme.template.md`

Active initialization writeback is limited to the current `1-分集` through `10-画布` active runtime stage-root scaffold plus project root `MEMORY.md` and project root `CONTEXT/README.md`; it does not create a project-level `0-初始化/` folder. It does not recursively mirror `.agents/skills/aigc` leaf, domain, satellite, backup, workflow, or shared skill packages into the project directory. User-specified initialization information, including team configuration, reference material summaries, production constraints, and downstream context guidance, is written into `MEMORY.md`. Former outputs such as `north_star.yaml`, `init_handoff.yaml`, `story-source-manifest.yaml`, `team.yaml`, `STATE.json`, `CHANGELOG.md`, and `源/` are no longer generated by initialization. Downstream aesthetic context must come from `2-美学/类型风格.md`, `2-美学/画面基调/全局风格协议.md`, and current-episode or baseline style protocol outputs.
