# 2-设计输出契约

## 固定交付件

每轮 `2-设计` 至少应稳定交付：

1. 各命中主体的人读设计卡
2. 各主体域的机读设计 JSON
3. 每个命中主体的 thinking sidecar
4. `validation-report.md`
5. 指向 `3-审计 / 4-面板 / 下游阶段` 的唯一下一入口

## 硬规则

1. 必须优先消费 `1-清单` bridge，不得跳过解释层直接重判主体。
2. 每个命中主体至少同时产出人读设计卡与机读 JSON。
3. `thinking sidecar` 是固定交付，不是可有可无的附录。
4. 若与 bridge 冲突，必须说明覆盖理由与证据来源。
