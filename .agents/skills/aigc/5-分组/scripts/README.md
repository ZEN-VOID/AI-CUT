# Scripts

`5-分组/scripts` 只承载机械辅助：

- 读取 Markdown 分组稿。
- 估算每组计入场景标题行、`画面风格：` 和分镜剧本正文后的字数，并以当前分镜组累计时间码的最后结束秒作为组内画面块总时长；`全局风格：` 字段和三行 north_star 内容、组底 fenced YAML 本身不计入字数。
- 检查分镜组 ID、场景标题行、`全局风格：` 字段位置、固定前置词、组头第 1 行全局风格整理句不超过 300 字、`画面风格：` 位于第一个 `分镜画面：` 上方、禁用 `增补首帧：`、组内 `[N-N秒]` 是否按当前分镜组基准连续递增且没有后续画面块重启、组头不得另列 `角色：` / `场景：` / `道具：` 主体清单、组底 YAML 字段、精确重复道具名、组内画面块总时长，以及 `## A~B` 连接件块和连接件字段是否残留。
- 输出校验报告；结构错误、legacy 旧式时长、旧字段残留和单组画面块总时长超过 18 秒均为 error，低于 850 字或低于约 10 秒的组为 warning，必须交给语义 review 判断是否存在短场景例外或是否需要回填。

脚本不得生成分组边界、`全局风格` 整理句、`画面风格`、首帧衔接、角色/场景/道具创作判断、道具同物异名合并结论或 canonical 分组正文。遇到缺失内容、单组超过 18 秒或连接件残留时，应报错并要求 LLM 或人工回到主创环节修复。validator 只能拦截格式、缺项、场景标题行缺失、全局风格字段缺失/位置错误、固定前置词缺失、画面风格缺失/位置错误、组内时间码未按当前分镜组基准累计、旧 `增补首帧：` 字段、组头主体清单残留、全局风格整理句超长、旧连接件块或字段残留、旧 `入场镜头：` / `出场画面：` / `画面属性：` / `入场画面：` / `画面构图：` / 六类位置细节字段、单组超时、精确重复道具名和明显结构错误，不能替代分组边界、首帧衔接变体质量、当前组风格抽取准确性或道具语义归一判断。

## Commands

```bash
python3 .agents/skills/aigc/5-分组/scripts/validate_storyboard_groups.py projects/aigc/<项目名>/5-分组/第N集.md

```
