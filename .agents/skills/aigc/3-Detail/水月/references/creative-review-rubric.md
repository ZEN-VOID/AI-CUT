# 水月 创作评审 Rubric

## 评审维度

| 维度 | 核心问题 | 过线标准 |
| --- | --- | --- |
| 人物成立度 | 人物压力、目的和关系是否可见、可演 | 至少有一处可拍信号，且不是纯情绪标签 |
| 动作与空间清晰度 | 谁往哪里动、为什么现在动是否清楚 | 读取者能在一次阅读内形成动作和方位图 |
| 场感承载力 | 氛围是否由空间、物象和感官条件托住 | 情绪温度不悬空，空间参与叙事 |
| 字段可消费性 | `角色背景面 / 角色站位走位 / 道具及状态 / 镜头消费提示` 是否都可直接 merge | 每条都能回指固定 `剧本正文` 的组内锚点 |
| 收束质量 | 四链是否汇成同一组 factual patch | 输出像可写回 JSON 的字段块，而不是模块拼贴 |

## 一票否决

- 只剩抽象情绪词，没有可拍动作或神态
- 动作路径和方位关系读不清
- 氛围完全脱离空间承载
- 为了“好看”牺牲了主冲突或主行动线
- 输出像第二份正文，而不是 factual patch

## 评分锚点

- `可交付`：人物、动作、氛围都基本成立，四类字段都可直接被父层 merge。
- `有明显收益`：父层能明显感到比 grouped script 更可拍、更可演、更可感受。
- `出彩`：主行动线、人物压力、空间气候和视觉抓手被压进同一组 factual patch，写回 `分镜明细[]` 后仍然清晰有力。
