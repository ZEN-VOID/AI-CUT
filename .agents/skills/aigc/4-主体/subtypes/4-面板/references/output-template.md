# 4-面板输出契约

## 固定交付件

每轮 `4-面板` 至少应稳定交付：

1. 各命中主体域的面板主文件
2. 对应的 layout sidecar
3. `validation-report.md`
4. 面向 `5-画面 / 6-视频` 的参照交接说明
5. 唯一下一入口

## 硬规则

1. `4-面板` 只能消费已经稳定的 `2-设计` 或审计通过版本，不得反向发明设计。
2. 每个面板至少同时输出主文件与 layout sidecar。
3. 若 `3-审计` 存在未关闭的关键失败项，优先回到审计或设计，不直接做面板。
4. 交接说明必须清楚写明是供 `5-画面`、`6-视频` 还是两者联合消费。
