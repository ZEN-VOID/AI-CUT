# nano-banana-face-swap 经验层

face-swap 子技能操作经验与故障修复知识库。

## Type Map

| 类型 | 症状 | 根因 | 修复 | 预防 | 验证 |
|------|------|------|------|------|------|
| `TM-FS-IDENTITY-LEAK` | 换脸后原角色面部特征残留（五官/发色未完全替换） | 提示词中身份锁定描述不够强，或图二质量不足导致特征提取弱 | 强化提示词中"完整还原""严禁保留原角色"措辞；确保图二为清晰正面照 | 检查输出中面部是否完全匹配图B；图二优先使用高清定妆照 | 人工比对图B与输出的面部特征 |
| `TM-FS-COSTUME-DRIFT` | 换脸后服装细节发生变化（颜色偏移/纹理丢失/剪裁改变） | 提示词中服装锁定不够明确，或比例适配偏差导致重新构图 | 追加服装细节描述到 `--desc`；确认比例适配正确 | 检查输出中服装是否与图A一致；比例从原图动态计算 | 人工比对图A与输出的服装细节 |
| `TM-FS-IMAGE-ORDER` | 输出结果服装和面貌都来自同一张图 | `--image-url` 传入顺序颠倒，或提示词中图一图二指代错误 | 严格遵循：第一个 `--image-url` = 图A（服装源），第二个 = 图B（面貌源） | Step 1 参数解析时打印确认日志 | 检查输出同时保留图A服装和图B面貌 |

## Repair Playbook

换脸输出不符预期时，按以下顺序排查：

1. 检查两张图是否正确传入（路径存在、顺序正确）
2. 检查比例适配是否从原图动态计算（`sips` 输出 → 最近合法值）
3. 检查提示词是否完整（基础模板 + 可选 desc）
4. 若身份泄漏（原角色残留）：加强 `--desc` 中的身份替换描述，或更换更清晰的图B
5. 若服装漂移：在 `--desc` 中追加服装细节锚定描述
6. 若 API 调用失败：回退到父级 `../SKILL.md` 的失败排查流程（§10）

## Reusable Heuristics

- 图序严格：第一张 = 服装源（图A），第二张 = 面貌源（图B），颠倒必错
- 比例从原图动态适配，不硬编码；合法值仅 5 种（16:9, 9:16, 4:3, 3:4, 1:1）
- 换脸场景通常不需要 `--no-report`，保留报告便于比对排查
- 图B 优先使用正面、高清、背景干净的定妆照，侧面照或复杂背景会降低面貌提取质量
- 永不覆盖原文件：输出命名自动递增，避免误覆盖
