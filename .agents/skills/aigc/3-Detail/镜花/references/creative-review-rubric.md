# 镜花 创作评审 Rubric

## 评审维度

| 维度 | 核心问题 | 过线标准 |
| --- | --- | --- |
| Shot Spine 稳定度 | 分镜数、slot 映射、构图骨架是否先被锁稳 | 每组都能看见清楚的镜头脊柱和观看顺序 |
| 水月承接度 | 镜头语言是否回指 `水月` 已给信息 | 不发明关键动作、关系、空间或剧情结果 |
| 执行可感度 | 光影、色彩、质感、运镜是否可被摄影/剪辑想象 | 至少能回答光怎么来、镜头为什么动、节奏怎么走 |
| 视觉收益与克制度 | 画面是否有明显收益但不过满 | 有一到两项关键视觉增益，转场/特效不抢戏 |
| 字段可消费性 | 最终 `shot_patches[]` 是否像同一位导演在统一表达 | 不像四个模块轮流发言，也不退回文学扩写或正文改写 |

## 一票否决

- 未形成稳定 shot spine 就直接进入摄影/运镜层
- 镜头语言脱离 `水月` 发明新事实
- 摄影或运镜只剩抽象赞美词，没有执行线索
- 转场或特效抢走角色、动作和画面主轴
- 输出仍像 `水月` 的文学扩写或正文改写，没有导演/摄影组织力

## 评分锚点

- `可交付`：镜头脊柱已成立，`shot_patches[]` 至少有一层真正带来执行收益。
- `有明显收益`：读取者能感到这是可以继续拆镜、摄影、美术、剪辑共同消费的导演字段包。
- `出彩`：镜头组织、摄影气候、节奏推进和克制的视觉强化在同一组内形成统一语法，写回 `分镜明细[]` 后依然既好看又不脱轨。
