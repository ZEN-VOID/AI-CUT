# Character Design Heuristics

本文件保存 `角色/2-设计` 可复用经验。强制入口、路径和门禁以 `SKILL.md` 为准。

## Stable Heuristics

- 角色设计先从故事压力出发，再落到身体、服装和镜头；反过来容易得到漂亮但无叙事功能的设定。
- `team.yaml.init_synthesis` 中的冻结综合要转译为设计判断，例如“控制色彩”“强调材质状态”“使用长焦压迫感”，不要写成名单或角色扮演；磨损必须先过服装状态判断。
- 服装风格必须同时说明审美和功能；只写“复古”“未来感”“高级”会导致后续生成漂移。
- 主角的设计需要可持续变化空间；配角的设计需要快速辨识和资产复用。
- 角色设计要先确定来源匹配的审美路线和镜头辨识度，再做清单关键词转译；主角、核心情感线角色、长期复用角色、大反派、主要对抗者、长线威胁和终局 Boss 必须有 `charisma_floor=high` 的可见证据，但不按女性美丽、男性英俊机械默认。
- 真实人物灵感默认不用或泛化处理；只有用户/项目允许且有必要时，才提取骨相、眼神、妆发和镜头魅力等设计因子。最终必须是原创组合，不要生成可识别现实人物。
- 英文 prompt 不要把全部中文设计稿直译进去；应压缩为主体、服装、姿态、镜头、光线、全局风格和约束。
- 若角色证据薄，设计可以更克制，但必须把推演边界写清楚。

## Common Pitfalls

- 把 `原文描述复述` 写成新的剧情设定。
- 为每个角色重复同一组全局风格形容词，造成视觉同质化。
- 把角色设计成“准确但没有镜头魅力”的关键词复刻，尤其是主角或大反派缺少 `charisma_floor=high` 的来源匹配审美路线、气质、压迫性、服装 signature 或脸部/眼神/姿态钩子。
- 把大反派简单丑化、脏乱化或恐怖化，或让普通反派/配角只靠职业标签区分，没有个性化魅力或清晰可识别度。
- 把真实人物灵感写成现实人物精确复刻、换脸或同款肖像，或在未获允许时默认使用真实人物参考。
- 在 `Cinematography` 中写“他的故事很悲伤”，却没有分镜明细。
- 搜索资料后把真实世界考据凌驾于项目 north star。
- 初始化综合、研究证据和本地 review patch 互相矛盾时并列保留，导致最终稿没有唯一设计判断。
