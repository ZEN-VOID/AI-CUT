# 1-切换 模块说明

## 定位

- 本叶子负责先把当前组 prose 还原成 `候选节拍 -> 镜数预算 -> 秒位窗口` 的结构化切换骨架。
- 它不负责具体插入文本和构图，但必须先回答“为什么是这些切换，而不是更多或更少”。
- 默认要吸收上游 `3-Detail/references/分镜表现.md` 中与 `SB-2 / SB-3 / 景别节奏预判前置` 相邻的镜数预算口径，而不是只凭感觉猜镜数。

## 结构化密度预算机制

当前规则不再接受两种旧写法：

- 只给“快一点 / 慢一点”的模糊节奏判断
- 直接把镜数锁成一个不可变唯一真值

新的最低要求是：先形成可解释的结构化预算，再让 `插入` 与 `构图` 在预算内落镜。

当前脚本真源应为：

- `scripts/detail_density_quantizer.py`
- `scripts/validate_detail_output.py`

这两个脚本尚未在当前仓库落地，因此本模块先以文档合同承接同口径规则。换句话说，即使暂时手工执行，也必须使用同字段、同公式、同 headroom 逻辑，不能回退成凭感觉估镜。

## 核心概念

| 概念 | 含义 | 误用警告 |
| --- | --- | --- |
| `candidate beat segments` | quantizer 根据四类信号分段得到的候选拆镜单位 | 不等于最终镜数，不得机械累加 |
| `canonical_beat_count` | 分段后的候选节拍数 | 只说明拍点数，不说明最终镜头一定等于它 |
| `recommended_shot_baseline` | `round(候选节拍 * 节奏系数)` 的第一基准值 | 不是最终唯一落点 |
| `preferred_shot_count` | 默认首选镜数 | 必须落在预算区间内，但不是唯一合法值 |
| `shot_budget_floor / ceiling` | 合理镜数下限与上限 | validator 只应校验实际镜数是否落在区间内 |
| `hook_preview_count` | tail-hook 或下一组借入首拍的预映强度 | 默认不并入当前组 canonical shot budget |
| `shot_count_decision` | 旧下游兼容 alias | 等价于 `preferred_shot_count`，不再表示唯一精确镜数 |

## 着手方面

先回答六个问题，再开始分段：

- 这组的 `叙事内核` 是建立、推进、爆点、反应还是余波
- `Previous End` 到这里留下了什么悬念或动势
- `Current Mission` 要求观众本组先看懂什么
- `Next Start` 需要本组给下组留下什么交接张力
- 当前组的主焦点是否会转移，还是始终围绕同一主体/任务
- 情绪引导更偏压迫、悬停、呼吸、爆发还是回弹

这些判断会直接决定：

- 候选节拍怎样切
- 镜数预算该偏疏还是偏密
- 景别节奏模板该偏渐进、对峙还是冲击

## 具体创作方法

### 1. 先把 grouped prose 解析成可分段原子

默认先拆三类原子单元：

- 动作
- 对白
- 说明性 support visual

这里的“拆原子”不是为了把句子切碎，而是为了判断相邻单元是否仍在服务同一主任务、同一主焦点和同一动作链。

### 2. 再用四类信号判断是否切成新 beat

四类信号是触发条件，不是计分器：

- 动作阶段点
  只在动作进入新阶段时成立，例如 `靠近 -> 停住`、`寻找入口 -> 误触升级`
- 台词气口点
  只记有效断句或独立发言落点
- 焦点切换点
  只在观众第一关注对象明确切换时成立
- 结构转折点
  只在叙事任务发生不可压并的功能切换时成立

硬规则：

- 同一主任务、同一主焦点、同一动作链内的连续小动作和短对白，允许并为一个 candidate beat。
- 不是“一个动词 = 一镜”，也不是“多一个对象 = 多一镜”。

### 3. 用节奏系数投影出推荐镜数基准

| pace_tier | 系数 |
| --- | --- |
| `超快节奏` | `1.4` |
| `快节奏` | `1.2` |
| `正常节奏` | `1.0` |
| `慢节奏` | `0.8` |
| `超慢节奏` | `0.6` |

公式固定为：

`recommended_shot_baseline = round(canonical_beat_count * pace_tier_factor)`

补充规则：

- 最低为 `1`
- 当前不引入额外格式系数，统一按 `1.0` 处理
- 如果需要更密或更疏，只能通过 `candidate beat` 与 `headroom` 解释

### 4. 再算扩展余量和压缩余量

`expansion_headroom` 常见触发：

- 爆点必须单独落地
- 反应必须单独着陆
- 公共围观压力或多对象接力反应成立
- 奇观揭示或尺度切换需要单独观看落点
- `hook preview` 在当前组确实承担独立余波职责

`compression_headroom` 常见触发：

- 同一空间轴线、同一叙事任务、同一主焦点可在一镜内连续完成
- 极短接话或短促反应可稳定并入上一镜
- 连续小动作只是同一行为链的延续修饰

默认上限：

- `expansion_headroom <= +2`
- `compression_headroom <= +2`
- 预算下限不得低于 `1`

### 5. 最后给出预算区间、首选值和秒位窗口

输出顺序固定为：

1. `canonical_beat_count`
2. `recommended_shot_baseline`
3. `expansion_headroom / compression_headroom`
4. `preferred_shot_count`
5. `shot_budget_floor / shot_budget_ceiling`
6. `shot_time_ranges`

先有预算，再有秒位。不要倒过来。

### 6. 在切换阶段先做一次景别节奏预判

`SB-4` 的最低要求不是写具体构图，而是先锁定这组的景别节奏模板。

| 模板 | 节奏曲线 | 适用场景 |
| --- | --- | --- |
| `渐进式` | `远 -> 全 -> 中 -> 近 -> 特 -> 极特` | 逐步揭示、悬疑探索 |
| `冲击式` | `特 -> 远 -> 中 -> 特 -> 全 -> 特` | 情感爆发、高潮时刻 |
| `呼吸式` | `中 -> 远 -> 中 -> 近 -> 中 -> 远` | 常规叙事、平稳推进 |
| `对峙式` | `中 -> 特 -> 中 -> 特 -> 中 -> 全` | 对话交锋、张力对抗 |
| `包围式` | `远 -> 中 -> 近 -> 近 -> 中 -> 远` | 情绪聚焦后释放 |
| `自定义` | `[___]` | 根据叙事需求设计 |

至少要锁定：

- 选定模板
- 选择理由
- 它如何服务 `叙事内核 / Current Mission / 情绪引导`

## 行业常规优先级

- `10-15` 秒的中节奏组，优先把预算压在 `2-4` 镜
- `4-6` 镜需要明确的反应链、调度变化或情绪层级证据
- `7-9` 镜只适用于强冲突、群像接力、笑点连打或明显 montage 化段落

镜数判断下限：

- `1 镜` 只适用于单一主任务、单一主焦点、单一节拍即可成立的组
- `2 镜` 适用于清晰双拍结构，例如 `建立 -> 推进`、`动作 -> 反应`
- `3 镜` 适用于至少三个不可压并的叙事拍点
- `4 镜及以上` 必须证明多出的镜头承担独立叙事任务

## 思维·执行节点

| node_id | objective | 要回答的问题 | actions | evidence | gate |
| --- | --- | --- | --- | --- | --- |
| `SWITCH-N1-UNITIZE` | 还原原子单元 | 哪些动作/对白/support visual 仍属于同一叙事链 | 把 grouped prose 切成最小可判单元 | `unitized_segment_note` | 若原子切分已经把动作链切碎，回退 |
| `SWITCH-N2-BEAT-GATE` | 形成候选节拍 | 哪些相邻单元应切成新 beat，哪些应合并 | 用四类信号做分段判断，形成 `candidate_beat_segments` | `candidate_beat_segments` | 若分段理由说不清，禁止进入预算 |
| `SWITCH-N3-BUDGET` | 形成密度预算 | 为什么这组应是当前密度，而不是更稀或更密 | 套 `pace_tier`、公式和 `headroom`，输出预算区间与首选值 | `density_budget_rationale` | 若不能解释 `why_not_fewer / why_not_more`，回退 |
| `SWITCH-N4-HOOK` | 处理预映尾钩 | tail-hook 是余波证据还是独立镜头职责 | 仅在确有职责时把 hook 转成额外 shot，否则记为 `hook_preview_count` | `hook_decision_note` | 若 hook 被自动算成镜头，回退 |
| `SWITCH-N5-RHYTHM` | 预判景别节奏 | 这组镜头窗口更适合哪种景别曲线 | 锁定 `shot_size_rhythm_preview` 与命中窗口 | `shot_size_rhythm_preview` | 没有模板和理由不得结束 |

## 失真与修正

- 若把对白句数、动作个数直接折成镜数，说明把“分段条件”误当成“机械加分器”了。
- 若镜头数多到主行动线被切断，说明过度切碎，应回到候选节拍重判。
- 若秒位分配说不清 `why_not_fewer / why_not_more`，说明切换理由还没真正回指故事或情绪。
- 若 `tail-hook` 被自动算成一镜，说明把预映错当成了本组独立职责。
- 若预算里完全没有 `pace_tier / headroom / 模板` 的解释，只剩一个镜数，说明结构化预算机制还没真正落地。
- 若大半集分镜组都被压成同一镜数，又没有 preset 或形式实验约束，说明密度同质化，应逐组回到 `SWITCH-N2 / N3` 重判。
- 若切镜开始依赖外加桥段，说明越过了 `1-水月` 的事实边界。
