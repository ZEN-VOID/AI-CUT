# Group Visual Style Contract

本文件定义每个分镜组的组级 `画面风格：` 输出标准。`画面风格：` 从上游 `4-摄影` 的镜头设计中提炼，以自然语句写入每个分镜组，并且必须位于 `全局风格：` 三行内容之后、第一个原画面性字段标题之前。

## Core Rule

每个分镜组在组装时，必须从组内镜头设计中提炼组级 `画面风格：`，写成自然语句。它覆盖该组的构图布局核心选择、构图方式关键子维度、光源效果、色彩基调、关键摄影技术参数和必要的观看位置。

`画面风格：` 不是 north_star.yaml 的项目级风格投影，两者层级不同：

- **`全局风格：`**（项目级）：全局风格提示词、类型元素提示词、画面风格原文，由 `north-star-projection-contract.md` 管理。
- **`画面风格：`**（组级）：从组内镜头设计提炼的视觉特征，由本合同管理。

## Fields

### 构图布局 composition_layout

从组内镜头设计中提炼：主体位置与占比倾向、陪体/前景/背景的主要空间安排。

### 构图方式 composition_method

从摄影构图知识中选取当前组最关键的 2-3 个：形状感、线条感、影调感、虚实感、节奏感、纹理质感、气势。

### 光源效果 lighting

主光源方向、质感、强度的可感知结果；辅助光和逆光如有重要作用也纳入。

### 色彩基调 color

色相主导、色温倾向、饱和度策略的可感知结果。

### 关键摄影技术参数 camera_params

影响画面质感的关键选择（焦距、光圈、快门感、运动稳定度等），只写对该组画面影响最大的 1-2 项。

### 观看位置与空间感 viewer_position

从组内镜头设计中提炼：本组是否主要使用正面平视、低机位、贴地前景、遮挡视角、手持微晃、浅景深前景或逐步揭示。若组内包含人物行走、压迫、群像或空间建立，`画面风格：` 应说明观众如何被带入空间，而不是只写“画面清晰”或“正面展示”。

## Output Rules

1. `画面风格：` 以自然中文语句呈现，覆盖上述维度中对该组画面最关键的内容，不要求所有维度平均铺开。
2. `画面风格：` 必须位于 `全局风格：` 三行内容之后、第一个原画面性字段标题之前；不得位于原正文或 YAML 之后。
3. 不输出参数清单、标签列表或模板填空。
4. 同一场景内镜头设计视觉特征不变时，相邻分镜组可使用相同或相似的 `画面风格：` 语句；视觉特征发生变化时必须更新。
5. `画面风格：` 从组内镜头设计中提炼，不是从 north_star 投影，也不是从 `4-摄影` 内部约束直接搬运。
6. 若组内镜头主要是正面平视全信息展示，必须判断这是有意的中立观察/制度压迫/固定框线，还是扁平摆拍风险；后者应回到 `4-摄影` 修复，不在 `5-分组` 用 `画面风格：` 补救。

## Knowledge Base Binding

`画面风格：` 的裁决和落盘以 `.agents/skills/aigc/4-摄影/knowledge-base/摄影构图/` 作为知识库参考。裁决时应参考其中的构图技法和影像创造理论，确保语句在构图布局、构图方式、光源、色彩等维度上有摄影构图理论支撑，而不是凭直觉生成。

## Output Example

```markdown
## 1-1-1

场景1：内景 教室 - 日

全局风格：
视频生成的画面风格，光影和氛围与场景参照图保持一致。需要生成现场物理互动音效、氛围感音效、环境声、自然现象声、动作声，不要生成任何字幕，不要生成背景音乐。 <全局风格>
<类型元素>
<细分风格>

画面风格：课桌行列形成水平线重复节奏，对称构图把空间压成矩形牢笼；冷白日光灯从正上方打硬光，高反差让白色校服过曝、黑窗帘完全吞光；50mm 标准焦段保持观察距离。

角色动作：
[0-N秒] <普通分镜画面>
```
