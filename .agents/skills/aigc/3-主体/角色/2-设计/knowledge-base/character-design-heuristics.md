# Character Design Heuristics

本文件保存 `角色/2-设计` 可复用经验。强制入口、路径和门禁以 `SKILL.md` 为准。

## Stable Heuristics

- 角色设计先从故事压力出发，再落到身体、服装和镜头；反过来容易得到漂亮但无叙事功能的设定。
- `project_memory_init_context` 中的初始化记忆要转译为设计判断，例如“控制色彩”“强调材质状态”“使用长焦压迫感”，不要写成名单或角色扮演；磨损必须先过服装状态判断。
- 服装风格必须同时说明审美和功能；只写“复古”“未来感”“高级”会导致后续生成漂移。
- 主角的设计需要可持续变化空间；配角的设计需要快速辨识和资产复用。
- 多状态角色需要“可持续变化空间”的显式版本管理：默认稿锁定身份不变量，服装、战斗、战损、受伤、少年、老年等变体只改变状态 delta。
- 角色设计要先确定来源匹配的审美路线和镜头辨识度，再做清单关键词转译；主角、核心情感线角色和长期复用角色必须有 `lead_beauty_handsomeness_floor=required` 的帅哥/美女/主角级好看证据，也必须有 `lead_presence_temperament_floor=required` 的整体气质、主角感、精神状态和镜头存在感证据；主角、核心情感线角色、长期复用角色、大反派、主要对抗者、长线威胁和终局 Boss 必须有 `charisma_floor=high` 的可见证据。非主角不按女性美丽、男性英俊机械默认。
- 真实人物灵感默认不用或泛化处理；只有用户/项目允许且有必要时，才提取骨相、眼神、妆发和镜头魅力等设计因子。最终必须是原创组合，不要生成可识别现实人物。
- 英文 prompt 不要把全部中文设计稿直译进去；应压缩为主体、服装、姿态、镜头、光线、全局风格和约束。
- 面部阴影是气质修饰，不是角色脸部设计的替代物。定妆照 prompt 应先保留清晰眉眼、鼻梁、嘴部、骨相、肤色层次和表情意图，再用受控侧光、轮廓光或局部眼尾压暗表达危险感。
- 若角色证据薄，设计可以更克制，但必须把推演边界写清楚。

## Common Pitfalls

- 把 `原文描述复述` 写成新的剧情设定。
- 把少年期、老年期、战斗态、战损态、受伤态或礼服态写成新的角色，导致脸、身形和气质不连续。
- 为每个角色重复同一组全局风格形容词，造成视觉同质化。
- 把角色设计成“准确但没有镜头魅力”的关键词复刻，尤其是主角缺少 `lead_beauty_handsomeness_floor=required` 的帅哥/美女/主角级好看证据，缺少 `lead_presence_temperament_floor=required` 的整体气质、主角感、精神状态和镜头存在感证据，或主角/大反派缺少 `charisma_floor=high` 的来源匹配审美路线、气质、压迫性、服装 signature 或脸部/眼神/姿态钩子。
- 只把主角写成漂亮脸、好身材、华丽服装，但没有眼神意图、姿态重心、动作节奏、精神状态和身份压力支撑，仍然属于审美吸引力不足。
- 把大反派简单丑化、脏乱化或恐怖化，或让普通反派/配角只靠职业标签区分，没有个性化魅力或清晰可识别度。
- 把真实人物灵感写成现实人物精确复刻、换脸或同款肖像，或在未获允许时默认使用真实人物参考。
- 用 `shadowed face`、`deep facial shadow`、`low-key silhouette`、`dark face` 等词制造阴郁感，结果遮住角色五官、骨相和表情，导致主角帅/美证据、大反派魅力证据和图像一致性下降。
- 在 `Cinematography` 中写“他的故事很悲伤”，却没有分镜明细。
- 搜索资料后把真实世界考据凌驾于 `2-美学` 输出、项目记忆或用户禁区。
- 初始化综合、研究证据和本地 review patch 互相矛盾时并列保留，导致最终稿没有唯一设计判断。
