# 画面冲击力与视觉奇观引擎

本文件服务 `漫画剧本改编` 的视觉奇观设计，不与主 `SKILL.md` 竞争真源。

## 1. 核心原则

视觉冲击力不是“多写夸张形容词”，而是让读者在某一页、某一格、某一个翻页瞬间被强制停住。

优先级：

`页势能 > 解释完整度 > 细节复刻`

前提：

- 只对虚构来源默认放开高烈度奇观化。
- 现实新闻/热搜仍受 `truth_boundary` 约束。
- 奇观必须服务角色关系、代价升级或悬念暴露，不能只做空洞特效。

## 2. 常用奇观类型

| 类型 | 适用场景 | 最稳的表现 |
| --- | --- | --- |
| `首次显形` | 恐怖、悬疑、怪谈 | 先给异常迹象，再半揭，再给正脸或全貌 |
| `关系失衡` | 言情、虐恋、复仇 | 距离骤变、身体动作越界、身份地位翻转 |
| `力量爆发` | 玄幻、系统、科幻 | 先蓄压，再瞬间释放，再留下代价 |
| `环境异变` | 奇幻、灾变、悬疑 | 光线、空间、建筑、天气、物件集体反常 |
| `静默大格` | 高压情绪、真相半揭 | 少字甚至无字，用构图和表情撑页 |
| `速度断裂` | 动作、追逐、惊悚 | 前一格蓄势，后一格直接跃迁或撞击 |
| `反常细节特写` | 心理惊悚、悬念 | 小物件失常、手势卡顿、眼神异常、血迹/裂痕/倒影 |

## 3. 视觉奇观五步法

1. 锁定“这场最贵的一秒”。
2. 决定它该做 `大格 / 跨页 / 静默格 / 连续爆发格` 哪一种。
3. 决定读者在前 1-3 格里先看到什么，再看到什么。
4. 决定哪些解释可以后移，给这一下让路。
5. 把该炸点同时写进：
   - `impact_beats[]`
   - `panel_focus_map[]`
   - `spread_splash_hints[]`
   - 必要时的 `sfx_cues[]`

## 4. 奇观与保核边界

允许夸张：

- 画面规模
- 光线反差
- 动作幅度
- 关系瞬时张力
- 节奏停顿与急停

优先保核：

- 角色为什么这么做
- 这一炸点会带来什么代价
- 这一页炸完之后剧情如何继续

若出现以下情况，说明奇观过头：

- 画面很猛，但角色动机不成立。
- 每一场都想做大格，导致没有真正的峰值。
- 全靠旁白说明“很震撼”，画面本身却没有真正的停顿物和主视觉。

## 5. 页级落位建议

| 页位/段位 | 更适合的冲击 |
| --- | --- |
| 冷开场前 1-3 页 | 异常显形、危险逼近、关系错位 |
| 中段第一次抬升 | 力量展示、地位翻转、环境异变 |
| 章末急停 | 真相半揭、门即将打开、身份揭露一半、代价突然出现 |

## 6. 与格式化字段的对齐

- `动作画面`：写清楚动作起点、动作方向、动作结果。
- `视觉焦点`：写当前格最该被看到的物、人、关系线。
- `跨页/大格建议`：只给真正值得拉开的节点，不滥用。
- `SFX建议`：只写能增强画面冲击的声音，不写纯文学拟声。

## 7. 自检

写完一个主场景后，检查：

1. 我能不能指出这一场最贵的一格？
2. 这一格为什么值得比周围更大、更静或更炸？
3. 如果删掉两句解释，这一炸点会不会反而更强？
4. 这一炸点炸完之后，是否留下了下一格或下一页必须继续看的理由？
